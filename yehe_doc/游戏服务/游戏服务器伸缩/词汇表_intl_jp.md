### エイリアス

エイリアス（alias）は、フリート名を抽象化するために使用されます。特定のフリートIDの代わりにエイリアスを使用することで、エイリアスに関連付けられたサーバーフリートを変更して、プレイヤーのトラフィックをあるフリートから別のフリートに簡便かつシームレスに切り替え、サーバーのダウンタイムをゼロで更新できます。



### キュー

- GSEにおいてキューは、ゲームアセットの1つが実行されているエリア内のサーバーフリートを意味します。キューを使用して複数のエリアにまたがるフリートを作成し、キュー内の任意のフリートにゲームサーバーセッションを配置できるようにすることで、遅延を最小限に抑え、プレイヤーのエクスペリエンスを向上させることができます。より多くのフリート容量をより効率的に使用し、新しいゲームのために大容量をより迅速に提供し、ゲームの可用性をより柔軟なものにします。
- Cloud Message Queue（CMQ）において、キュー（Queue）はメッセージをストレージする宛先であり、ユーザーは自主的にキューからメッセージを取得します。MessageIdまたはReceiptHandleを使用してキュー内のメッセージを一意に識別します。



### サーバーフリート

サーバーフリート（fleet）は、CVMインスタンス形式のホスティングリソースです。このCVMインスタンスは、自動スケーリングやヘルスチェック等の機能を持ち、ホスティング用のサーバーフリートは、ゲームサーバーのデプロイに使用できます。



### アセット

アセット（asset）とは、CVMインスタンスにデプロイされたzipパッケージのことを意味します。そのディレクトリには、実行可能ファイル、依存パッケージ、インストールスクリプトを含む、ゲームサーバーの実行に必要なすべてのコンポーネントが含まれている必要があります。

### インスタンス

- GSEにおいて、インスタンスはCVMインスタンスに対応し、これはGSEが提供するクラウドサーバーのことを意味します。具体的なタイプについては、[GSE](https://console.cloud.tencent.com/gse/asset)コンソールでクエリーできます。
- TI-ONEにおいては、ワークフローが実行されるたびにインスタンスが生成されます。インスタンスタイプには、履歴インスタンス、パラメータインスタンス、再実行インスタンス、タイマーインスタンスがあります。
- Tencent Kubernetes Engine（TKE）において、インスタンスは、1つまたは複数の関連付けられたコンテナで構成され、これらのコンテナは同一のストレージとネットワークスペースを共有します。
- Batch Computeにおいて、インスタンスはCVMインスタンスに対応し、各ジョブに1つまたは複数のインスタンス（Instance）を指定して同一のインスタンスを実行できます。インスタンスはバッチのスケジューリングおよび実行ができる最小単位です。
- Data Security Governance Center（DSGC）においては、インスタンスは通常、Tencent Cloud上のデータアセットのユニットを意味し、サービスによってデータアセットユニットも異なり、データベースの場合はライブラリ、COSの場合はバケットとなります。
- Tencent Container Registry（TCR）において、指定のリージョンでインスタンスを購入して専用のコンテナイメージホスティングサービスを提供します。インスタンスのバックエンドサービスとコアデータのストレージでは、他のユーザーのインスタンスから独立しています。Dockerのネイティブコンセプトにおいて、インスタンスは独立したDocker Registryサービスと同じようなものです。つまり、クラウドでプライベートなDocker Registryサービスを購入してデプロイします。
