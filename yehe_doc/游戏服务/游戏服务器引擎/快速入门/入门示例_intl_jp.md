## 操作シナリオ

このドキュメントで、「ビギナー向け例」を通じたGSEメインフローと、ビギナー向けゲームサーバーホスティングサービスを適用する方法についてガイドします。

## 前提条件

[Game Server Elastic-scaling申請表](https://intl.cloud.tencent.com/apply/p/k0b6pvbhs6) に記入しており、使用資格を取得していること。

## 操作手順

### 手順1：デモパッケージのアップロード
 1. [Game Server Elastic-scalingコンソール](https://console.cloud.tencent.com/gse) にログインし、左側のメニュー【ビギナー向け例】をクリックします。
 2. 左上のサービスエリアを選択し、【デモパッケージのアップロード】をクリックし、デモパッケージをアップロードして、作成完了が表示されたら、【次へ】をクリックします。
    ![](https://main.qcloudimg.com/raw/aba9f81e1e145dd9233c962a71f9938e.png)
 >?
 >- GSEが提供するデモパッケージは、gRPCフレームと統合されており、ゲームプロセスとGSEがgRPCを通じて通信します。
 >- ご自身で作成する場合は、[生成パッケージの作成](https://intl.cloud.tencent.com/document/product/1055/36674) 操作ガイドを参照してください。

### 手順2：サーバーフリートの作成
【ワンクリックでサーバーフリートを作成】をクリックすると、「新規作成（実行中）」、「ダウンロード中（実行中）」の順で次々と表示されます。「サーバーフリートの作成成功」が表示されたら、【次へ】をクリックします。
 - サーバーフリートの作成で必要な6つの手順：新規作成（完了）、ダウンロード中（完了）、認証中（完了）、生成中（完了）、アクティブ化中（完了）、アクティブ（完了）。
 -現在の状態：作成中、サーバーフリートの作成成功。
![](https://main.qcloudimg.com/raw/8854d79ca8ce4f1edbb4f93529766bdb.png)
![](https://main.qcloudimg.com/raw/013f88008a3833369c8ab2ca1ae8a0a9.png)
![](https://main.qcloudimg.com/raw/d395303b0afa601cd1e7bd29713816e2.png)
 >?
 >- サーバーフリートを作成し、デモパッケージをサーバーフリート上でデプロイします。
 >- サーバーフリートは1セットのサーバーであり、Auto Scalingの能力を備えているため、デモパッケージは楽々世界中にデプロイできます。
 >- ご自身で作成する場合は、[サーバーフリートの生成](https://intl.cloud.tencent.com/document/product/1055/36675) 操作ガイドを参照してください。

### 手順3：ゲームサーバーセッションとプレイヤーセッションの作成
- 【ゲームサーバーセッションの作成】をクリックすると、ゲームサーバーセッションの作成成功と表示されます。 
![](https://main.qcloudimg.com/raw/00e1ad7d8190f60cf021f5a1ec82d437.png)
 >?
 >- この操作で「ゲームサーバーセッションの作成」TencentCloud APIを呼び出します。GSEは、ゲームサーバーセッションを作成し、それにサービスプロセスをアサインします。
 >- ご自身で作成する場合は、[ゲームサーバーセッションの作成](https://intl.cloud.tencent.com/document/product/1055/37139) APIドキュメンテーションを参照してください。 
- 【プレイヤーセッションの作成】をクリックすると、プレイヤーセッションの作成成功と表示されます。
![](https://main.qcloudimg.com/raw/3ac88d02e60888dafce042625927cba4.png)
>?
>- この操作で「ゲームサーバーセッションに参加」TencentCloud APIを呼び出しを行います。GSEはプレーヤーセッションを作成し、プレイヤーをゲームサーバーセッションに加えます。
>- ご自身で作成する場合は、[ゲームサーバーセッションに参加](https://intl.cloud.tencent.com/document/product/1055/37132) APIドキュメンテーションを参照してください。

### 手順4：クライアントのゲームサーバー接続
   【クライアントWebページにリダイレクト】をクリックしてクライアントのゲームサーバー接続ページに入り、【接続】をクリックすると、サーバー接続成功と表示されます。 


>?
>
>- プレイヤーセッションの作成が完了してから、プレイヤー（クライアント）は1分以内にサーバーに接続する必要があります。1分を過ぎると無効になります。
>- このデモパッケージはチャットサービスです。複数のプレイヤーがサーバーに接続していると、チャットができます。

 以上の4つの手順はGSEのすべての接続プロセスをシミュレートしたものです。詳細を知りたい場合は、[開発ガイド](https://intl.cloud.tencent.com/document/product/1055/36683)ドキュメントを参照してください。
