O SNAT não é executado para nenhuma instância do CLB de rede privada baseada em VPC (ou seja, instâncias configuradas com VPC durante a criação) adquirida após 15 de dezembro de 2016, o que significa que o IP de acesso obtido do servidor é o IP do cliente real. Para garantir o funcionamento normal do seu negócio, observe:
1. Para instâncias do CLB de rede privada adquiridas após 15 de dezembro de 2016, após uma política de grupo de segurança ter sido habilitada, você deve permitir as regras de entrada de todos os IPs de cliente para garantir o acesso normal.
2. Para as instâncias do CLB de rede privada adquiridas antes de 15 de dezembro de 2016, se você quiser convertê-las para o tipo não SNAT, [envie um tíquete] (https://console.cloud.tencent.com/workorder/category) para obter assistência. Após a conversão, o IP de acesso obtido do servidor será o IP do cliente e o negócio não será interrompido durante a conversão.
