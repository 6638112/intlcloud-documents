### HTTP/2란 무엇입니까?
- HTTP/2(하이퍼텍스트 전송 프로토콜 버전 2)는 World Wide Web에서 사용되는 HTTP 네트워크 프로토콜의 주요 개정판입니다.
- HTTP/2는 HTTP1.X의 성능 문제를 해결하여 네트워크 리소스를 더 잘 사용하고 네트워크 애플리케이션 대기 시간을 줄이도록 설계되었습니다.
- HTTP/2는 HTTP1.X와 역호환됩니다.

### 왜 HTTP/2를 사용해야 합니까?
HTTP1.X와 비교하여 HTTP/2는 응답을 더 빠르고 효율적으로 만들 수 있습니다. HTTP/2에는 다음과 같은 장점이 있습니다.
- 멀티플렉스: 동시 처리는 더 빠른 응답을 제공합니다.
- 서버 푸시: 서버가 클라이언트에 필요한 리소스를 사전에 푸시하여 요청 수를 줄입니다.
- 더 많은 기능에는 대역폭 제한, 요청 우선 순위, 헤더 압축 및 이진법 프레임이 포함됩니다.

<span id="5"></span>
### CLB에서 HTTP/2를 활성화하려면 어떻게 해야 합니까?
1. HTTPS 리스너에서 HTTP/2 활성화
 + CLB 인스턴스: CLB 인스턴스에서 HTTP/2 프로토콜을 활성화하거나 비활성화할 수 있습니다. 자세한 내용은 [Configuring HTTPS Listener](https://intl.cloud.tencent.com/document/product/214/32516)를 참고하십시오.
2. 클라이언트 액세스 시 프로토콜에 동의
클라이언트가 HTTP/2 지원 리스너에 액세스하면 HTTPS의 핸드셰이크 프로세스 중에 프로토콜 버전이 협상됩니다. 클라이언트는 ALPN(Application-Layer Protocol Negotiation)을 사용하여 지원되는 프로토콜 목록을 서버에 알립니다. 서버는 프로토콜 목록에 따라 HTTP/2 또는 HTTP1.X를 선택합니다. 클라이언트가 HTTP/2를 지원하지 않는 경우 서버는 추가로 구성할 필요 없이 자동으로 이전 버전과 호환됩니다.

>!
> 1. HTTP 리스너는 HTTP/2를 지원하지 않습니다. 주요 브라우저와 WebServer는 TLS 기반 HTTP/2 프로토콜만 지원합니다.
> 2. HTTP1.X 프로토콜은 CLB 인스턴스와 리얼 서버 간에 계속 사용됩니다.
