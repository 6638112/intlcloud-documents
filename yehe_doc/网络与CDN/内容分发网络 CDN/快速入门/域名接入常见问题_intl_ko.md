[](id:q1)
### 도메인 이름은 어떻게 연결합니까?
CDN 콘솔에서 도메인 이름을 연결할 수 있습니다. 자세한 정보는 [도메인 연결](https://intl.cloud.tencent.com/document/product/228/5734)을 참고하십시오.

[](id:q2)
### CDN에 연결하는 도메인에는 어떤 조건이 있습니까?
도메인을 CDN에 연결하여 가속을 진행하려면 다음과 같은 조건을 갖춰야 합니다.
1. 도메인 이름 길이 81자 이하.
2. 중국 본토 CDN을 사용하는 경우 공업정보화부 ICP비안이 완료되어야 함.
3. `a.test.com`, `a.b.test.com`과 같은 형식의 서브 도메인 또는 `*.test.com`, `*.a.test.com` 형식의 와일드카드 서브도메인.
4. 와일드카드 도메인 이름, 이미 다른 사용자에 의해 연결된 도메인 이름, 처음 연결하는 도메인 이름인 경우 소유권 인증이 완료되어야 함.

[](id:q3)
### CDN은 와일드카드 도메인 이름 연결을 지원합니까?
예, 하지만 소유권 인증 완료 후 연결 또는 검색할 수 있습니다.
기타 사항:
1. `*.test.com`과 같은 와일드카드 도메인 이름이 이미 Tencent Cloud에 연결되어 있는 경우 해당 하위 도메인 이름은 다른 계정으로 Tencent Cloud에 연결할 수 없습니다.
2. 와일드카드 도메인 이름 `*.test.com`이 이미 계정을 통해 Tencent Cloud에 연결된 경우 `*.path.test.com`과 같은 형식의 와일드카드 도메인 이름은 계정을 통해 Tencent Cloud에 연결할 수 없습니다.

[](id:q4)
### CDN 설정이 적용되는 데 얼마나 걸립니까?
일반적으로 CDN 설정이 적용되는 데 30분 미만이 소요되며, 30분 이내에 설정이 적용되지 않으면 [티켓 제출](https://console.cloud.tencent.com/workorder/category)을 통해 문의하십시오.

[](id:q5)
### 여러 개의 원본 서버 IP를 설정할 수 있습니까?
예, 여러 개의 IP를 설정할 수 있습니다. 설정 후, CDN은 원본 서버로 요청을 전달할 때 해당 IP 중 하나에 무작위로 접근합니다. 이 IP에 대한 origin-pull 실패 횟수가 임계값을 초과하면 해당 IP는 기본적으로 300초 동안 격리됩니다. 그 동안 어떤 origin-pull 요청도 IP로 전달되지 않습니다.

[](id:q6)
### 도메인 이름이 CDN에 연결된 후 어떻게 CNAME을 도메인 이름에 바인딩합니까?
[CNAME 설정](https://intl.cloud.tencent.com/document/product/228/3121) 문서를 참고하여 DNS 서비스 제공 업체측에서 CNAME을 바인딩할 수 있습니다.

[](id:q7)
### 도메인 이름을 비활성화할 수만 있고 삭제할 수는 없습니다.
사용자가 공동 작업자인지 확인하십시오. 공동 작업자는 작업에 대해 작성자로부터 관련 권한을 받아야 합니다. 공동 작업자에게 권한이 부여되었지만 여전히 작업을 수행할 수 없는 경우 지원을 위해 [티켓 제출](https://console.cloud.tencent.com/workorder/category) 하시기 바랍니다.

[](id:q8)
### 가속 서비스가 비활성화된 후에도 도메인 이름 설정이 유지됩니까?
비활성화된 후에도 도메인의 설정은 유지되지만, 가속 서비스는 중단되며 사용자의 요청에 404 상태 코드가 반환됩니다.

[](id:q9)
### 가속화된 도메인 이름을 삭제한 후 도메인 이름 설정은 저장됩니까?
도메인 이름이 삭제되면 해당 설정은 저장되지 않습니다.

[](id:q10)
### 가속 서비스를 종료하려면 어떻게 해야 합니까?
CDN 콘솔에서 가속 서비스를 비활성화할 수 있습니다. 자세한 작업 절차는 [가속 도메인 비활성화](https://intl.cloud.tencent.com/document/product/228/5736)를 참고하십시오.

[](id:q11)
### 가속 도메인 이름은 어떻게 삭제합니까?
CDN 콘솔에서 가속 도메인을 삭제할 수 있습니다. 자세한 작업 절차는 [가속 도메인 삭제](https://intl.cloud.tencent.com/document/product/228/5736)를 참고하십시오.

[](id:q12)
### 도메인 이름을 차단 해제하는 방법은 무엇입니까?
[티켓 제출](https://console.cloud.tencent.com/workorder/category)이 필요하며 전문적인 유지보수 담당자가 처리합니다.

[](id:q13)
### CDN은 어떤 서비스 유형을 지원하나요?
서비스 유형 선택에 따라 도메인을 스케줄링할 리소스 플랫폼이 결정됩니다. 각 리소스 플랫폼마다 가속 설정에 차이가 있습니다. 사용자의 서비스에 맞는 유형을 선택하시기 바랍니다.
- 정적 가속: 전자 상거래, 웹 사이트 및 게임 이미지 정적 리소스의 가속화 환경
- 다운로드 가속: 게임 설치 패키지, 멀티미디어 원본 파일 다운로드, 모바일 펌웨어 배포 및 기타 시나리오
- 스트림 미디어 VOD 가속: 음성, 비디오, 영상, 비디오 VOD 가속 등의 응용 시나리오

[](id:q14)
### CDN 도메인의 서브 항목은 어떻게 수정하나요?

[CDN 콘솔](https://console.cloud.tencent.com/cdn)에 로그인하여 왼쪽 사이드바에서 [도메인 관리]를 선택하고 도메인 이름 오른쪽의 [관리]를 클릭한 후, [기본 설정] 탭을 열어 도메인 이름 프로젝트를 수정합니다. 여러 도메인 이름의 프로젝트를 일괄 수정하려면 [도메인 관리] 탭에서 대상 도메인 이름을 선택하고 [추가 작업] 드롭다운 목록을 클릭한 다음 [프로젝트 편집]을 선택합니다. 한 번에 최대 50개의 도메인 이름을 선택할 수 있습니다.

>!CDN 권한 체계의 사용자는 이 작업으로 인해 서브 계정 권한이 변경될 수 있으니 주의하여 작업하시기 바랍니다.

### 공업정보화부의 ICP비안 등록이 완료된 도메인임에도 불구하고 CDN 가속 도메인 추가 시 ICP비안 미등록이라고 뜨는 이유는 무엇인가요?
ICP비안 등록 완료 후 공업정보부의 정보 동기화 및 Tencent Cloud CDN이 ICP비안 정보를 가져오기까지 일정 시간이 소요될 수 있습니다. 24시간 후에 다시 연결을 시도하시기 바랍니다.

[](id:q16)
### 가속 도메인 / 원본 서버에서 포트를 설정할 수 있나요?
- 가속 도메인 포트: 현재 CDN 가속 포트는 기본적으로 80, 443, 8080, 이 세 포트를 지원하며, 다른 포트는 현재 지원하지 않습니다.
- 원본 서버 포트: 원본 서버 주소 뒤에 포트(1 - 65535)를 설정할 수 있습니다.

[](id:q17)
### CDN 호스트 헤더 구성이란 무엇인가요?
CDN 노드의 Origin-pull 과정 중 원본 서버에서 액세스하는 사이트의 도메인입니다. 원본 서버와 호스트 헤더: 원본 서버에 설정한 IP/도메인은 CDN 노드가 Origin-pull 할 때 해당하는 원본 서버를 찾도록 가이드하며, 서버에 몇몇 Web 사이트가 남아 있더라도 호스트 헤더에서 리소스가 위치한 사이트를 명확히 알려줍니다.

[](id:q18)
### CDN이 적용되었는지 어떻게 확인하나요?

nslookup 명령어를 사용한 CDN 가속 도메인의 DNS 리졸브 쿼리: 도메인을 nslookup합니다. 쿼리한 도메인 리졸브에 위 이미지의 빨간 박스 부분과 같이 확장명이 dnsv1.com인 CNAME 리졸브 레코드가 있다면 연결한 도메인의 CDN 가속 서비스가 적용되었다는 의미입니다.

### CDN 파일을 다운로드할 수 없습니다.

CDN에서 파일을 다운로드할 수 없는 경우 다음 방법으로 문제를 해결하시기 바랍니다.
1. 원본 서버에서 정상적으로 파일을 다운로드할 수 있는지 확인합니다.
2. CDN 콘솔에서 도메인 이름이 올바르게 설정되었는지 확인합니다(기본 구성 탭의 원본 도메인 참조). 구성된 원본 도메인에 올바르게 액세스할 수 있는지 확인하십시오. 그렇지 않으면 Origin-pull 실패로 비즈니스에 영향을 미칠 수 있습니다.
3. 원본 서버의 보안 정책 확인: 원본 서버에 설정된 보안 정책으로 인해 Origin-pull 실패가 발생하는지 확인하고, 그렇다면 당사에 [문의하기](https://intl.cloud.tencent.com/support)를 통해 원본 서버 IP 대역을 확보하고 원본 서버를 화이트리스트에 추가할 수 있습니다.

[](id:q20)
### CDN 가속이 구성된 후 WordPress 백엔드에 로그인할 수 없습니다.
wordpress는 동적 요청을 포함합니다. 캐시 구성이 부적절하면 로그인 예외가 발생할 수 있습니다. 이 유형의 파일이 자동으로 캐시되지 않도록 해당 동적 파일 유형의 캐시 유효성을 0으로 설정하는 것이 좋습니다. 일반적인 동적 파일 유형에는 asp, .jsp, .php, .perl 및 .cgi가 포함됩니다. 자세한 내용은 [노드 캐시 설정](https://intl.cloud.tencent.com/document/product/228/35317)을 참고하십시오.
