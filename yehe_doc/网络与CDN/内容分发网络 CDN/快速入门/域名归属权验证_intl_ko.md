## 어떤 상황에서 도메인 이름 소유권 확인을 해야 합니까?
1. a.example.com과 같이 처음 도메인에 액세스하는 경우, 도메인에 액세스 후 b.example.com과 같은 동일한 수준의 도메인과 다음 수준의 도메인이 권한 있는 것으로 간주됩니다. 기본적으로 액세스할 수 있으며 확인이 필요하지 않습니다. 그러나 example.com과 같은 상위 수준의 도메인에 대한 액세스는 여전히 확인해야 합니다;
2. 다른 계정으로 서브 도메인에 액세스한 경우, 현재 도메인 이름 소유권 확인을 수행해야 하며, 확인이 통과되면 도메인 이름을 검색하여 현재 계정에 액세스할 수 있습니다;
3. 동일한 수준의 와일드카드 서브도메인에 액세스할 때 확인이 필요합니다. 예시: a.example.com에 액세스했고 *.example.com에 액세스할 때 여전히 확인이 필요합니다. *.a.example.com은 2차 와일드카드 서브도메인이며 확인 없이 액세스할 수 있습니다.

## 방법1: DNS 확인(권장)
1. 도메인 이름 추가 시, 해당 도메인 이름이 확인이 필요한 경우 도메인 이름 아래에 도메인 이름 소유권을 확인하라는 메시지가 표시됩니다. 확인 방법을 클릭합니다.
![](https://qcloudimg.tencent-cloud.cn/raw/1c6915b095d74f4358923be2ec7740dd.png)
2. 확인 방법에서 기본값은 DNS 확인입니다.
DNS 확인 방법을 사용하려면 DNS 업체로 이동하여 기본 도메인 이름 아래에 호스트 레코드 값이 _cdnauth인 TXT 레코드를 추가해야 합니다.
>!
추가해야 하는 도메인 이름이 c.b.a.example.com, *.example.com 또는 test.example.com이든 상관없이 멀티 레벨 도메인에서 호스트 레코드 값은 ​​기본 도메인 아래에 추가되어야 합니다. 예를 들어 추가한 도메인은 c.b.a.example.com이며, 새 DNS 레코드는 _cdnauth.example.com으로 추가해야 합니다.
![](https://qcloudimg.tencent-cloud.cn/raw/e1da0524ad0781544052fd65195d9798.png)
DNS 레코드를 추가한 후 TXT 레코드 값이 적용될 때까지 기다립니다. 적용 후, 아래 확인 버튼을 클릭하여 도메인 이름 소유권 확인을 완료할 수 있습니다. 만약 확인에 실패하면 현재 TXT 레코드 값이 DNS 업체에 적용되었는지 또는 올바른 TXT 레코드 값이 작성되어 있는지 확인하십시오.

## 방법2: 파일 확인
1. 도메인 이름 추가 시, 해당 도메인 이름이 확인이 필요한 경우 도메인 이름 아래에 도메인 이름 소유권을 확인하라는 메시지가 표시됩니다. 확인 방법을 클릭합니다.
![](https://qcloudimg.tencent-cloud.cn/raw/140a8b980bd0c30c0336dcd8f8e3607e.png)
2. 확인 방법에서 파일 확인 방법을 선택합니다.
![](https://qcloudimg.tencent-cloud.cn/raw/8883258277fef3bf2e42ceffb489324a.png)
3. verification.html 파일 다운로드를 클릭합니다.
4. 기본 도메인 이름(CVM, COS, Alibaba ECS, Alibaba OSS 등) 서버의 루트 디렉터리에 파일을 업로드합니다. 예시: 현재 추가된 도메인 이름이 test.example.com인 경우 example.com의 루트 디렉터리에 파일을 업로드해야 합니다.
5. http://example.com/verification.html을 통해 파일에 액세스할 수 있는지 확인한 후 확인 버튼을 클릭하여 확인할 수 있습니다. 파일 내의 레코드 값이 제공한 레코드 값과 일치할 경우 확인되며, 확인에 실패할 경우 위의 파일 링크에 액세스할 수 있는지, 업로드한 파일이 올바른 파일인지 확인하십시오. 다운로드한 파일과 일치 여부는 해당 파일의 링크를 방문하면 확인할 수 있습니다.

## FAQ
### 도메인 이름 소유권 확인의 TXT 레코드 값이 적용되는지 수동으로 점검하는 방법은 무엇입니까?
Windows 시스템 예시:
예를 들어 액세스 도메인이 cdnauth.example.com인 경우 시스템에서 cmd 명령 인터페이스를 열고 nslookup-qt=txt _test.example.com을 입력하면 현재 TXT 결과에 따라 DNS 레코드가 적용되었는지 또는 올바른지 확인할 수 있습니다.
![](https://qcloudimg.tencent-cloud.cn/raw/1728c7c311e4db18fc28287299ab2068.png)

Linux/Mac 시스템 예시:
예를 들어 액세스 도메인이 test.example.com인 경우 명령 인터페이스에 dig _cdnauth.example.com txt를 입력하면 현재 TXT 결과에 따라 DNS 레코드가 적용되었는지 또는 올바른지 확인할 수 있습니다.
![](https://qcloudimg.tencent-cloud.cn/raw/d07ca2df386f12185af00abe813d4b60.png)

### VOD 도메인에 직접 액세스할 수 없다는 메시지가 표시되나요?
현재 도메인 이름은 VOD의 사용자 지정 배포 가속 도메인에 이미 액세스되어 있습니다. 동일한 가속 도메인 이름의 중복 설정이 불가능하기 때문에 CDN 콘솔에서 가속 도메인 이름을 사용하려면 먼저 VOD 가속 도메인 이름을 삭제해야 합니다. (일단 도메인 이름 사용을 중지한 후 삭제하시기 바랍니다. 사용 중지만 해도 충돌이 있을 수 있습니다.) 삭제 후 약 1분 정도 기다린 후 CDN 콘솔에 액세스하거나 다른 서브 도메인 이름을 사용하여 CDN 콘솔에 액세스할 수 있습니다.
