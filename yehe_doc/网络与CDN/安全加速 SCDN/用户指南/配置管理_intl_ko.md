## 설정 수정

도메인 관리 페이지에서 비즈니스 니즈에 따라 도메인의 Web 공격 방어 설정, 사용자 정의 보호 정책 등을 수정할 수 있습니다. 

### Web 공격 방어

SCDN은 Tencent의 대규모 Web 공격 샘플 데이터베이스를 기반으로 액세스에 특징 매칭을 진행하여 SQL 인젝션, XSS 공격, 로컬 파일 포함 등 각종 Web 공격을 효과적으로 방어하고, 사용자 원본 서버를 실시간으로 보호합니다. ![2021-10-11 오후 7.48.27 스크린샷](https://tva1.sinaimg.cn/large/008i3skNgy1gvbm3td1w8j611g05s3yl02.jpg)![2021-10-11 오후 7.49.20 스크린샷](https://tva1.sinaimg.cn/large/008i3skNgy1gvbm4ehxhzj60h308haa802.jpg)

- Web 공격 방어 활성화를 선택하여 방어 등급을 조정할 수 있습니다. 
- 방어 모드: 차단을 선택하면 노드에서 Web 공격 요청을 감지할 경우, 해당 요청을 차단하고 차단 페이지 설정에 따라 기본 차단 페이지 및 403 상태 코드를 반환하거나 요청을 사용자 정의 차단 페이지로 리디렉션합니다. 
- 사용자 정의 페이지 주소: 차단 페이지를 ‘사용자 정의 페이지’로 설정할 경우, 노드는 모든 차단된 Web 공격 요청을 지정된 사용자 정의 차단 페이지로 리디렉션합니다. 
- 리디렉션 상태 코드: 상기 리디렉션 상태 반환 코드를 301 또는 302로 지정할 수 있습니다. 

### 사용자 정의 보호 정책

SCDN은 IP, URI, Referer, User-Agent, Params 등 필드의 복잡한 액세스 규칙 설정을 지원합니다. 비즈니스 시나리오에 맞춰 사용자 정의 보호 정책을 설정하고 다중 조건을 조합하여 요청을 필터링할 수 있습니다. ![2021-10-11 오후 7.35.22 스크린샷](https://tva1.sinaimg.cn/large/008i3skNgy1gvbm5wuk9cj60l90cqt9602.jpg)

1. 보호 규칙 추가 및 수정
   - 5개의 사용자 정의 규칙을 생성할 수 있으며 모든 규칙에 5개의 매칭 조건을 설정할 수 있습니다. 규칙, 매칭 조건은 **‘또는’**의 관계로, 실행 순서는 리스트 순서에 따라 위에서 아래로 진행되며 **어떤 규칙의 1개 조건이라도 만족하면 바로 차단합니다**.
   - 매칭 필드: Params, URL(path 부분. 예: `/test/1.jpg`. `?` 이후의 매개변수 미포함), IP(클라이언트 IP), Referer, User-Agent 지원.
   - 매칭 조건: 포함, 미포함, 같음, 같지 않음, 최대 길이, 길이, 최소 길이 지원.
   - 매칭 값은 **1개**의 매칭 항목만 입력할 수 있고, 현재 정규 매칭은 지원하지 않습니다. 입력하지 않으면 기본값은 공란입니다. 
2. 규칙 사용 중지 및 삭제
   - [사용 중지]를 클릭하여 규칙 1개를 중지할 수 있으며 [활성화]를 클릭하면 규칙을 다시 활성화할 수 있습니다. 
   - [삭제]를 클릭하여 규칙 1개를 삭제할 수 있으며 삭제된 규칙은 복구할 수 없습니다. 
3. 우선순위 조정
   - [우선순위 조정]을 클릭하여 규칙 리스트의 각 규칙 순서를 조정할 수 있습니다. 현재 매칭 로직에서 규칙 순서는 최종 차단 결과에 영향을 미치지 않습니다. 

### CC 방어 / DDoS 방어

SCDN은 Tencent Cloud의 우수한 특징 인식 알고리즘을 기반으로 요청 트래픽에 클리닝을 진행하여 SYN Flood, TCP Flood, ICMP Flood 등 각종 DDoS 트래픽 공격을 방어하고, 자체 개발 스마트 CC 공격 판정, 차단 특허 기술을 바탕으로 플랫폼 추천 차단 정책에 따라, 다차원의 사용자 정의 규칙을 결합하여 악성 CC 공격을 분석하고 차단합니다. 

- 도메인 CC 공격 방어, DDoS 방어는 기본적으로 활성화되어 있습니다.
- 사용자 정의 빈도 제한 설정이 가능합니다.

![2021-10-11 오후 7.51.52 스크린샷](https://tva1.sinaimg.cn/large/008i3skNgy1gvbm788ny2j611o0br3za02.jpg)

- 구매한 패키지가 탄력적 방어를 지원하는 경우, 탄력적 방어는 기본적으로 활성화되고, 패키지가 탄력적 방어를 지원하지 않는 경우, 기본적으로 비활성화됩니다. 자세한 내용은 [과금 안내](https://intl.cloud.tencent.com/document/product/1032/42540)를 참고하십시오.
