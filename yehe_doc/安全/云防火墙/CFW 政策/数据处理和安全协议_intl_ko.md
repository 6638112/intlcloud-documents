## 1\. 배경

이 모듈은 Cloud Firewall(“**기능**”)을 사용하는 경우 적용됩니다. 이 모듈은 (“[DPSA](https://intl.cloud.tencent.com/document/product/301/17347)”)에 있는 데이터 처리 및 보안 계약에 포함됩니다. 본 모듈에서 사용되었지만 정의되지 않은 용어는 DPSA에서 정의된 것과 같은 의미를 갖습니다. DPSA와 본 모듈 간에 상충이 있는 경우, 해당 상충 범위까지 본 모듈이 적용됩니다.

## 2\. 처리

당사는 기능과 관련하여 다음과 같은 데이터를 처리합니다.

| **개인 정보**                                     | **사용**                                                      |
| ------------------------------------------------------------ | ------------------------------------------------------------ |
| **자산 데이터:** 자산 인스턴스 ID/이름, IP 주소, 자산 유형, 지역, 프라이빗 네트워크, 리소스 레이블, 리소스 태그, 인바운드 및 아웃바운드 피크 대역폭, 인바운드 및 아웃바운드 누적 트래픽, 사이버 공격, 노출 포트, 노출된 취약성, 호스트 보안 기능으로 보호되는 Tencent Cloud 자산(호스트, 퍼블릭 IP 주소, 웹 서비스, 게이트웨이, VPC, 서브넷 및 데이터베이스)에 대한 정보 | 당사는 귀하에게 기능을 제공하기 위한 목적으로만 이 데이터를 처리합니다.<br/>이 데이터는 당사의 다른 Tencent Cloud 기능(사용자의 구성 및 액세스 권한에 따름)에서 공유되고, MySQL(“**MySQL**”) 용 TencentDB 클라우드와 데이터 웨어하우스("**Clickhouse**")에 저장되며, Cloud Object Storage(“**COS**”)에 백업됩니다. |
| **CFW 비즈니스 데이터:** 인바운드 및 아웃바운드 대역폭, 인바운드 및 아웃바운드 누적 유동률, 취약점 이름, 노출 포트, 방화벽 구성 데이터(액세스 제어 규칙, 침입 차단 보호 규칙, 침입 차단 보호 모델, 차단 목록, 릴리스 목록, 격리 목록, 허니팟 프로브 구성, 허니팟 네트워크 구성, 주소 템플릿), 기타 방화벽 구성 데이터(방화벽의 온오프 데이터, NATFW 인스턴스 구성 데이터) | 당사는 귀하에게 기능을 제공하기 위한 목적으로만 이 데이터를 처리합니다.<br/>이 데이터는 MySQL 및 Clickhouse 기능에 저장되며, COS 기능에 백업됩니다. |
| **공격자 데이터:** 식별 또는 차단할 공격자 정보의 로그 데이터(IP 주소, 지리적 위치, 페이로드 포함 공객 패킷, 공격 유형, 공격 이벤트 유형, 공격 추적 및 공격자 정보) | 당사는 귀하에게 기능을 제공하기 위한 목적으로만 이 데이터를 처리합니다.<br/>이 데이터는 MySQL 및 Clickhouse 기능에 저장되며, COS 기능에 백업됩니다. |
| **로그 데이터:** <br/><li>액세스 제어 로그(규칙 명령, 히트 시간, 액세스 소스, 소스 포트, 액세스 대상, 대상 포트, 계약(네트워크 통신 서비스 계약, TCP, UDP 및 HTTP), 유효한 정책 등의 데이터 포함) <br/><li>침입 보호 로그(공격 발생 유형, 위험 수준, 외부 액세스 소스, 소스 포트, 액세스 대상, 대상 포트, 계약(네트워크 통신 서비스 계약, TCP, UDP 및 HTTP), 발생 시간, 정책, 의사결정원 등의 데이터 포함) <br/><li>트래픽 로그(트래픽 명령, 시간, 외부 액세스 소스, 소스 포트, 액세스 대상, 대상 포트, 계약(네트워크 통신 서비스 계약, TCP, UDP 및 HTTP), 스트림 바이트, 지역, 운영자 등의 데이터 포함) <br/><li>작업 로그(시간, 운영 계정, 운영 유형, 운영 활동, 운영 세부 정보, 위험 수준 등의 데이터 포함) | 당사는 귀하에게 기능을 제공하기 위한 목적으로만 이 데이터를 처리하며, 여기에는 귀하의 자산 또는 자산에 대한 공격과 관련된 로그를 검토할 수 있도록 하는 것이 포함됩니다.<br/>이 데이터는 당사의 Elasticsearch Service 기능에 저장되며, COS 기능에 백업됩니다. |
