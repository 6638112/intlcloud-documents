## 1\. 背景

本モジュールは、お客様がCloud Firewall（「**機能**」）を使用する場合に適応されます。本モジュールは、（「[DPSA](https://intl.cloud.tencent.com/document/product/301/17347)」）に掲載されているData Processing and Security Agreement（データ処理およびセキュリティに関する契約）に組み込まれます。本モジュールにおける用語のうち本書内に定義のないものは、DPSA内で定義された意味を有するものとします。DPSAと本モジュールとの間に矛盾がある場合には、その不一致の範囲で本モジュールを優先するものとします。

## 2\. 処理

弊社は本機能に関連して、次のデータを処理します。

| **個人情報**                                     | **用途**                                                      |
| ------------------------------------------------------------ | ------------------------------------------------------------ |
| **アセットデータ：** この機能で保護されているTencent Cloudアセット（ホスト、パブリックIPアドレス、Webサービス、ゲートウェイ、VPC、サブネット、データベース）に関する情報：アセットインスタンスID/名前、IPアドレス、アセットタイプ、リージョン、プライベートネットワーク、リソースラベル、リソースタグ、インバウンドおよびアウトバウンドピーク帯域幅、インバウンドおよびアウトバウンド累積トラフィック、サイバー攻撃、公開ポート、公開された脆弱性、およびホストセキュリティ | 当社は、お客様に本機能を提供する目的でのみ、このデータを処理します。<br/>このデータは、Tencent Cloudの他の機能から（お客様の設定とアクセス許可に従って）共有され、TencentDB for MySQLに保存されることに注意してください（「**MySQL**」）およびクラウドデータウェアハウス（「**Clickhouse**」）機能、およびクラウドオブジェクトストレージ（「**COS**」）機能。 |
| **CFWビジネスデータ：** インバウンドおよびアウトバウンド帯域幅、インバウンドおよびアウトバウンド累積フロー、ループホールの名前、公開ポート、ファイアウォール設定データ（アクセス制御ルール、侵入防御防御ルール、侵入防御防御モデル、禁止リスト、リリースリスト、分離リスト、ハニーポットプローブ設定、ハニーポットネットワーク設定、アドレステンプレート）、その他のファイアウォール設定データ（ファイアウォールのオン/オフデータ、NATFWインスタンス設定データ） | 当社は、お客様に本機能を提供する目的でのみ、このデータを処理します。<br/>このデータは、MySQLおよびClickhouseの機能に保存され、COS機能にバックアップされます。 |
| **攻撃者データ：** 識別または傍受される攻撃者情報のログデータ：IPアドレス、地理的位置、攻撃パケット（ペイロードを含む）、攻撃タイプ、攻撃イベントタイプ、攻撃トレース、攻撃者情報 | 当社は、お客様に本機能を提供する目的でのみ、このデータを処理します。<br/>このデータは、MySQLおよびClickhouseの機能に保存され、COS機能にバックアップされます。 |
| **ログデータ：** <br/><li>アクセス制御ログ（ルールの方向、ヒット時間、アクセスソース、送信元ポート、アクセス先、宛先ポート、合意（ネットワーク通信サービス契約、TCP、UDP、HTTP）、有効なポリシーなどのデータを含む） <br/><li>侵入防御ログ（これには、攻撃イベントの種類、リスクレベル、外部アクセス元、送信元ポート、アクセス先、宛先ポート、合意（ネットワーク通信サービス契約、TCP、UDP、HTTP）、発生時刻、ポリシー、決定元などのデータを含む） <br/><li>トラフィックログ（これには、トラフィックの方向、時刻、外部アクセス元、送信元ポート、アクセス先、宛先ポート、合意（ネットワーク通信サービス契約、TCP、UDP、HTTP）、ストリームバイト、リージョン、演算子などのデータを含む） <br/><li>操作ログ（時間、操作アカウント、操作の種類、操作の動作、操作の詳細、リスクレベルなどのデータを含む） | 当社は、お客様に本機能を提供する目的でのみこのデータを処理します（お客様がお客様の資産および/またはお客様の資産に対する攻撃に関連するログを確認できるようにすることを含みます）。<br/>このデータは当社のElasticsearch Service機能に格納され、当社のCOS機能にバックアップされます。 |

## 5\. データ保持

当社は、本機能に関連して処理された個人データを、以下のとおり保管します（適用されるデータ保護法によって別途要求される場合を除きます）。

| **個人情報** | **保持ポリシー**                                         |
| ------------------------ | ------------------------------------------------------------ |
| アセットデータ               | 当社は、お客様が機能のサブスクリプションを終了するまでそのようなデータを保持し、その時点で当該データは14日以内に削除されます。 |
| CFWビジネスデータ        | デフォルトでは、当該データは7日間保持されます（ストレージ容量が50GBを超えると、当該データはそれ以前に削除されます）。本機能のログ分析サービス（メッセージキューCKafka機能と相互依存）を購入した場合、当該データは（お客様が決定したストレージ容量制限まで）6か月間保持されます。ストレージ容量の上限に達すると、データは都度置き換わります。 |
| 攻撃者データ            | デフォルトでは、当該データは7日間保持されます（ストレージ容量が50GBを超えると、当該データはそれ以前に削除されます）。本機能のログ分析サービス（メッセージキューCKafka機能と相互依存）を購入した場合、当該データは（お客様が決定したストレージ容量制限まで）6か月間保持されます。ストレージ容量の上限に達すると、データは都度置き換わります。 |
| ログデータ                 | デフォルトでは、当該データは7日間保持されます（ストレージ容量が50GBを超えると、当該データはそれ以前に削除されます）。本機能のログ分析サービス（メッセージキューCKafka機能と相互依存）を購入した場合、当該データは（お客様が決定したストレージ容量制限まで）6か月間保持されます。ストレージ容量の上限に達すると、データは都度置き換わります。 |

お客様は、DPSAに従って、当該個人情報の削除を要請することができます。
