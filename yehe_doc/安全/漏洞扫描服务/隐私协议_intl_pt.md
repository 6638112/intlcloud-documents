## 1\. INTRODUÇÃO

Este Módulo se aplica se você usar o Vulnerability Scan Service (“**Recurso**”). Este Módulo é incorporado à política de privacidade localizada na [Política de privacidade](https://intl.cloud.tencent.com/document/product/301/17345). Os termos usados, mas não definidos neste módulo devem ter o significado que lhes é atribuído na Política de Privacidade. Em caso de conflito entre o DPSA e este módulo, este Módulo se aplicará conforme a inconsistência.

## 2\. CONTROLADORIA

O controlador das informações pessoais descritas neste Módulo é o especificado na Política de Privacidade.

## 3\. DISPONIBILIDADE

Este Recurso está disponível globalmente para usuários, mas destina-se principalmente a usuários localizados no mesmo país/região que a região de serviço selecionada para um desempenho ideal.

## 4\. COMO USAMOS AS INFORMAÇÕES PESSOAIS

Usaremos as informações nos seguintes modos e de acordo com as seguintes bases jurídicas:

| **Informações Pessoais**                                     | **Uso**                                                      | **Base Jurídica**                                              |
| ------------------------------------------------------------ | ------------------------------------------------------------ | ------------------------------------------------------------ |
| **Dados operacionais do mecanismo de varredura de vulnerabilidade**: tempo de operação do mecanismo, contagem de operação do mecanismo, status de operação do mecanismo, dados de registro de tarefas do mecanismo | Usamos essas informações para garantir que o mecanismo funcione conforme necessário para fornecer este Recurso a você. <br/>Observe que esses dados são armazenados em nossos recursos TencentDB for Redis (Redis), TencentDB for MySQL (MySQL) e TencentDB for MongoDB (MongoDB) para este fim. | Processamos essas informações pois são necessárias para a execução de nosso contrato com você para o fornecimento do Recurso. |
| **Dados operacionais do mecanismo de mapeamento de ativos**: mapa de operação do mecanismo | Usamos essas informações para fornecer esse Recurso a você, inclusive para gerar varredura de segurança, riscos e relatórios relacionados, fornecer avisos de risco e notificações de detecção de vulnerabilidades, e fornecer sugestões de reparo.<br/> Observe que esses dados são armazenados em nossos recursos Redis, MySQL e MongoDB para este fim. | Processamos essas informações pois são necessárias para a execução de nosso contrato com você para o fornecimento do Recurso. |
| **Dados de cota do usuário:** data e hora de expiração, número de nomes de domínio, número de IPs, número de APIs | Usamos essas informações para calcular sua cota de usuário e fazer a cobrança, para fins de faturamento.<br/>Observe que esses dados são armazenados em nossos recursos Redis, MySQL e MongoDB para este fim. | Processamos essas informações pois são necessárias para a execução de nosso contrato com você para o fornecimento do Recurso. |
| **Dados de configuração da tarefa:** nome da tarefa, tipo de monitoramento, plano de varredura do ativo varrido, velocidade de varredura, duração da varredura, resultado da varredura | Usamos essas informações para fornecer este Recurso a você, e para solução de problemas. <br/>Observe que esses dados são armazenados em nossos recursos TencentDB for Redis, TencentDB for MySQL e TencentDB for MongoDB para este fim. | Processamos essas informações pois são necessárias para a execução de nosso contrato com você para o fornecimento do Recurso. |
| **Dados de correspondência de vulnerabilidades:** nome de vulnerabilidade, nível de ameaça, ativos afetados, tipo de vulnerabilidade, hora em que foi encontrada pela última vez | Usamos essas informações para fornecer este Recurso a você, e para solução de problemas. <br/>Observe que esses dados são armazenados em nossos recursos TencentDB for Redis, TencentDB for MySQL e TencentDB for MongoDB para este fim. | Processamos essas informações pois são necessárias para a execução de nosso contrato com você para o fornecimento do Recurso. |
| **Dados de correspondência da porta:** porta, sugestões de resposta à vulnerabilidade, protocolo, serviço da porta | Usamos essas informações para fornecer este Recurso a você, e para solução de problemas. <br/>Observe que esses dados são armazenados em nossos recursos TencentDB for Redis, TencentDB for MySQL e TencentDB for MongoDB para este fim. | Processamos essas informações pois são necessárias para a execução de nosso contrato com você para o fornecimento do Recurso. |



## 5\. COMO COMPARTILHAMOS E ARMAZENAMOS INFORMAÇÕES PESSOAIS

Como especificado na Política de Privacidade.

## 6\. RETENÇÃO DE DADOS

Reteremos informações pessoais de acordo com o seguinte:

| **Informações Pessoais**                     | **Política de Retenção**                                         |
| -------------------------------------------- | ------------------------------------------------------------ |
| Dados operacionais do mecanismo de varredura de vulnerabilidade | Armazenados por 7 dias.                                           |
| Dados operacionais do mecanismo de mapeamento de ativos          |  Armazenados por 7 dias.                                                            |
| Dados de cotas de usuários                              | Retemos esses dados enquanto você usar o Recurso. Quando seu uso do Recurso é encerrado, excluiremos esses dados após 7 dias. |
| Dados de registro de tarefas                                |    Retemos esses dados enquanto você usar o Recurso. Quando seu uso do Recurso é encerrado, excluiremos esses dados após 7 dias.                                                          |
| Dados de correspondência de vulnerabilidades                  |  Retemos esses dados enquanto você usar o Recurso. Quando seu uso do Recurso é encerrado, excluiremos esses dados após 7 dias.                                                            |
| Dados de correspondência de portas                           |   Retemos esses dados enquanto você usar o Recurso. Quando seu uso do Recurso é encerrado, excluiremos esses dados após 7 dias.                                                           |
