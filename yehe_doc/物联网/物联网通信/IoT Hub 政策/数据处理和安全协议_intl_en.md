In the event of any conflict between the DPSA and this Module, this Module shall apply to the extent of the inconsistency.

## 2.PROCESSING
We will process the following data in connection with the Feature:

<table>

<tr>
<th><b>Personal Information</b></th>
<th><b>Use</b></th>
</tr>
</thead>
<tbody>
<tr>
<td><b>Log Data</b>:device log (if you enable collection of this data) </td>
<td>We only process this data according to your configuration to facilitate debugging or troubleshooting by you.

</td>
</tr>
<tr>
<td><b>Configuration Data</b>: settings of products (product name, product ID), devices (device name, device key/certificate) and rules (SQL statement)</td>

Please note that this data is stored and backed up with our TencentDB for MySQL feature.
</td>
</tr>
<tr>
<td><b>Monitoring Data</b>: statistics of online devices, device messages, device shadows, rule engine and firmware</td>
<td>We only process this data to provide you with statistics of your use of the Feature.

</td>
</tr>
<tr>
<td><b>Resource Data</b>:resources related to platforms and devices uploaded by you (if you include personal data in such resources)</td>
<td>We only process this data to enable you to provide such data to your end users.
Please note that this data is stored and backed up with our Cloud Object Storage feature.
</td>
</tr>
<tr>
<td><b>Published Device Data</b>: type of device, state of device, rules and commands for controlling the device by SQL statement</td>
<td>We only process this data for device verification and message forwarding as instructed by you.

</td>
</tr>
<tr>
<td><b>Subscriber Device Data </b>:device command message </td>
<td>We only process this data to facilitate control of device(s) by your end users.
Please note that we have no access to this data. This data is transferred to other service(s) as instructed by you. 
</td>
</tbody></table>
