For enabled rules in Config, evaluation can be automatically triggered by the system or manually triggered by users, as described below. Disabled rules need to be re-enabled before the corresponding evaluation can be triggered.
1. Manual trigger

- Trigger evaluation for a single rule
Create or edit a rule and save it.
Enable a disabled rule.
Click **Evaluate** on the rule details page.
![](https://qcloudimg.tencent-cloud.cn/raw/d39ef4a7bbf1065f71a444796f020f6b.png)
- Trigger evaluation for a batch of rules
Add the rules to a conformance pack so that evaluation for these rules can be triggered at a time in the following scenarios:
Create or edit a conformance pack and save it.
Click **Evaluate** on the conformance pack details page.
![](https://qcloudimg.tencent-cloud.cn/raw/f5a906a04946df50ba071b8c5817a646.png)

2. Automatic trigger
Configure an execution frequency for a rule or conformance pack. Then the system will automatically execute the rule evaluation, with the first automatic start time each day being 00:30.
![](https://qcloudimg.tencent-cloud.cn/raw/df358dc94544c85bebfe6d5ebbbca3ee.png)
