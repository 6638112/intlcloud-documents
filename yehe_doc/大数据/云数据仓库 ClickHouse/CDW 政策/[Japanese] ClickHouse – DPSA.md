## 1.	背景

本モジュールは、お客様がクラウド データ ウェアハウス（「**機能**」）を使用する場合に適応されます。このモジュールは、https://intl.cloud.tencent.com/document/product/301/17347 （「**DPSA**」）にあるデータプライバシーおよびセキュリティ契約に組み込まれています。本モジュールで使用されているが定義されていない用語は、DPSAで定義された意味を持つものとします。DPSA と本モジュールの間に矛盾が生じた場合、その矛盾の範囲内で本モジュールが適用されるものとします。

## 2.	処理

当社は本機能に関連して、次のデータを処理します。

| **個人情報**                                     | **用途**                                                      |
| ------------------------------------------------------------ | ------------------------------------------------------------ |
| **構成データ**: クラスター構成情報(課金モード、リージョン、アベイラビリティーゾーン、 クラスター名、クラスターの説明、CVM の CPU の数、仮想プライベートクラウド (VPC)を含む)  ストレージデータ(お客様が 決定したストレージの割り当てデータを含む)、自動更新、購入時間 (注文が生成される時期)) | 弊社は、お客様の 特定の設定に基づいて本機能を提供する目的でのみ、このデータを 処理します。 このデータは、 当社の TencentDB for MySQL 機能に保存されますのでご注意ください。 |
| **データの監視**: CPU とメモリの使用率、 1 秒あたりのオープン ファイル数、データ ディスク使用率、キー インジケータ (1 秒あたりの処理クエリ数、処理時間、 1 秒あたりの接続数、クラスターによる 1 秒あたりのレコード出力数、 オペレータの合計計算時間など) | この情報を使用して、動作状態を監視し、 パフォーマンスや消費インジケーターなどの機能のさまざまなイベントに対してしきい値アラームを 設定できます。 このデータはクラウドモニター(CM)機能によって実行され、 保存され、MySQL 機能のテンセント DB に 格納されることに注意してください。 |
| **ビジネスデータ**: CBS および/または この機能にインポート/保存するビジネスデータおよび構成データのバイナリ データ ファイル | 当社は、このデータを、お客様に本機能を提供し、 お客様のクエリを実行する目的でのみ処理します。 このデータは、 この目的のために当社の CBS 機能と統合されていますのでご注意ください。階層化ストレージ機能を使用する場合、 このデータは COS フィーチャーにも保管されます。 |

## 5.	データ保持

当社は、本機能に関連して処理された個人データを以下のように保管します。

| **個人情報** | **保持ポリシー**                                         |
| ------------------------ | ------------------------------------------------------------ |
| 構成データ       | お客様が本機能を使用する限り、 当社は当該データを保持します。本機能のご使用が終了すると、 7日後にこのデータが削除されます。 |
| モニタリングデータ          | お客様が本機能を使用する限り、 当社は当該データを保持します。本機能のご使用が終了すると、 7日後にこのデータが削除されます。 |
| ビジネスデータ            | お客様が本機能を使用する限り、 当社は当該データを保持します。この機能では、  このようなデータを手動で削除できます。そのようなデータの削除を選択したり、 本機能の使用を終了したりすると、そのデータは7日後に削除されます。 |

お客様は DPSA に従って、当該個人データの削除を要請することができます。

## 6.	特別条件

お客様は、本機能を、個人データの処理に同意できる最低年齢に達しているエンドユーザーのみが使用することを保証する必要があります。これは、エンドユーザーが所在する法域によって異なる場合があります。
