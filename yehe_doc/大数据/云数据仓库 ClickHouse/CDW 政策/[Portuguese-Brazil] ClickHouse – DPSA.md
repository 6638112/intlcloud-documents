## 1.	CONSIDERAÇÕES

Este Módulo se aplica se você usar Cloud Data Warehouse (“**Recurso**”). Este Módulo está incorporado ao contrato de privacidade e segurança de dados localizado em https://intl.cloud.tencent.com/document/product/301/17347 (“**DPSA**”). Os termos utilizados, mas não definidos neste módulo, devem ter o significado dado a eles no DPSA. No caso de qualquer conflito entre o DPSA e este Módulo, este Módulo se aplicará conforme a inconsistência.

## 2.	PROCESSAMENTO

Processaremos os seguintes dados em conexão com o Recurso:

| **Informações Pessoais**                                     | **Uso**                                                      |
| ------------------------------------------------------------ | ------------------------------------------------------------ |
| **Dados de configuração:** informações de configuração do cluster  \[incluindo modo de faturamento, região zona de disponibilidade,  nome do cluster, descrição do cluster, número da CPU da CVM, nuvem virtual privada  (VPC), dados de armazenamento (incluindo dados para alocação de armazenamento determinados por  você), renovação automática, tempo de compra  (quando o pedido será gerado)\] | Apenas processamos esses dados  com a finalidade de fornecer o Recurso a você de acordo com a sua  configuração específica.  Observe que esses dados  estão armazenados em nosso recurso TencentDB for MySQL. |
| **Dados de monitoramento:** utilização de CPU  e memória, número de arquivos abertos por segundo, uso de disco de dados, indicadores-chave  (incluindo o número de consultas processadas por segundo, tempo de processamento,  número de conexões por segundo, número de registros de saída pelo cluster por  segundo, tempo total de cálculo do operador) | Nós usamos essas informações para dar suporte a você para monitorar o status da operação e definir alarmes  de limiar para vários eventos no Recurso, como indicadores de desempenho ou  consumo.  Observe que esses  dados são executados e armazenados em nosso recurso Cloud Monitor (CM), e armazenados  em nosso recurso TencentDB for MySQL. |
| **Dados de negócios:** arquivos de dados binários  de sua empresa e dados de configuração que você importa/armazena no CBS e/ou  neste Recurso | Apenas processamos estes dados  com a finalidade de fornecer o Recurso para você e para executar a consulta.  Observe que esses  dados estão integrados com nosso recurso CBS para este fim. Se você optar por  usar a função de armazenamento hierárquico, esses dados também continuaram armazenados no recurso COS. |

## 5.	RETENÇÃO DE DADOS

Armazenaremos dados pessoais processados em conexão com o Recurso da seguinte forma:

| **Informações Pessoais** | **Política de Retenção**                                         |
| ------------------------ | ------------------------------------------------------------ |
| Dados de Configuração       | Retemos esses dados  enquanto você usar o Recurso. Quando seu uso do Recurso é encerrado,  nós vamos excluir esses dados após 7 dias. |
| Dados de monitoramento          | Retemos esses dados  enquanto você usar o Recurso. Quando seu uso do Recurso é encerrado,  nós vamos excluir esses dados após 7 dias. |
| Dados de negócios            | Retemos esses dados  enquanto você usar o Recurso. Você pode excluir manualmente esses dados no  Recurso. Esses dados serão imediatamente excluídos do Recurso após 7 dias ao encerrar  o seu uso ou caso você opte por excluir os dados. |

Você pode solicitar a exclusão desses dados pessoais de acordo com o DPSA.

## 6.	CONDIÇÕES ESPECIAIS

Você deve garantir que este recurso seja usado somente por usuários finais com a idade mínima exigida para que possam consentir com o processamento de seus dados pessoais. Pode haver diferenças dependendo da jurisdição em que o usuário final está localizado.
