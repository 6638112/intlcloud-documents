Terms used but not defined in this Module shall have the meaning given to them in the DPSA. In the event of any conflict between the DPSA and this Module, this Module shall apply to the extent of the inconsistency.

## 2.	PROCESSING

We will process the following data in connection with the Feature:

| **Personal Information**                                     | **Use**                                                      |
| ------------------------------------------------------------ | ------------------------------------------------------------ |
| **Configuration Data:** cluster  configuration information (including billing mode, region, availability zone,  cluster name, cluster description, number of CVM’s CPU, virtual private cloud  (VPC), storage data (including data for allocation of storage determined by  you), automatic renewal, purchase time  (when order will be generated)) | We only process this data  for the purposes of providing the Feature to you in accordance to your  specific configuration.  Please note that this  data is stored in our TencentDB for MySQL feature. |
| **Monitoring Data:** CPU and memory  utilization, number of open files per second, data disk usage, key indicators  (including the number of queries processed per second, processing time,  number of connections per second, number of records output by the cluster per  second, total calculation time of the operator) | We use this information to support you to monitor operation status and set threshold alarms  for various events in the Feature, such as performance or consumption  indicators.  Please note that this  data is performed by and stored in our Cloud Monitor (CM) feature, and stored  in our TencentDB for MySQL feature. |
| **Business Data:** binary data files  of your business and configuration data that you import/store in CBS and/or  this Feature | We only process this data  for the purposes of providing the Feature to you, to execute your query.  Please note that this  data is integrated with our CBS feature for this purpose. If you choose to  use the tiered storage function, this data is also stored in the COS feature. |

## 5.	DATA RETENTION

We will store personal data processed in connection with the Feature as follows:

| **Personal Information** | **Retention Policy**                                         |
| ------------------------ | ------------------------------------------------------------ |
| Configuration Data       | We retain such data for  as long as you use the Feature. When your use of the Feature is terminated,  we will delete this data after 7 days. |
| Monitoring Data          | We retain such data for  as long as you use the Feature. When your use of the Feature is terminated,  we will delete this data after 7 days. |
| Business Data            | We retain such data for  as long as you use the Feature. You may manually delete such data on the  Feature. Once you choose to delete such data or terminate your use of the  Feature, such data will be after 7 days. |

You can request deletion of such personal data in accordance with the DPSA.
