## 2. PROCESSING

We will process the following data in connection with the Feature:

| **Personal Information**                                     | **Use**                                                      |
| ------------------------------------------------------------ | ------------------------------------------------------------ |
| **Storage Data:** binary data files of your business and configuration data that you import/store in your CBS feature and/or this Feature (if your files contain personal data) | We only process this data for the purposes of providing the Feature to you.Please note that this data is stored in our Cloud Block Storage (“**CBS**”) feature for this purpose. If you choose to use the tiered storage function, this data is also stored in the Cloud Object Storage (“**COS**”) feature.<br/>Please note that we have no access to the personal data, if any, stored in the database or control over the data. |
| **Running Log Data:** records of hardware, software and system problems, and monitoring data of events occurring in the system (including timestamp and messages such as connection messages, SQL execution messages, and error messages) | We only process this data for the purposes of providing the Feature to you, to enable you to locate issues and attacks.<br/>Please note that this data is stored in our Cloud Virtual Machine (“**CVM**”) feature. |
| **Configuration Data**: visitor access permissions created by you and management of resources determined by you (including names of the white list and the black list for permission and forbidden permission to specific databases created by you, names of the designated visitors, names of the databases and the IP address configured to access the databases) | We only process this data for the purposes of providing the Feature to you in accordance to your specific configuration.<br/>Please note that this data is stored in our CVM feature. |
