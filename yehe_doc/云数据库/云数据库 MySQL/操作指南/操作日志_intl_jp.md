## 操作シナリオ
指定時間を超えたSQLステートメントを「スロークエリー」といい、対応ステートメントは「スロークエリーステートメント」といいます。データベース管理者（DBA）はスロークエリーステートメントを解析して、スロークエリーが現れる原因を探します。このプロセスを「スロークエリー解析」といいます。

コンソールの操作ログページで、インスタンスのスローログ明細、エラーログ明細、ロールバックログ、スローログのダウンロードを確認できます。コマンドライン画面（CLI）か、またはクラウドデータベースAPIを使用して、データベースログを確認、ダウンロードすることもできます。[スロークエリーログの確認](https://intl.cloud.tencent.com/document/product/236/15845) と[バイナリーログの確認](https://intl.cloud.tencent.com/zh/document/product/236/15843)をご参照ください。

>?TencentDB for MySQL（単一ノード - 基本型は含まない）インスタンスは、操作ログ管理機能に対応します。
>

#### MySQLスロークエリー関連の説明
- long_query_time:スロークエリーしきい値パラメータ。精度はマイクロ秒レベルで、デフォルトでは10s。SQLステートメント実行時間がこの数値を超えると、スローログに記録されます。 
long_query_timeパラメータの調整は、オリジナルのスローログに影響を与えません。例えば、スローログのしきい値パラメータが10sの場合は、10sを超えるスローログ記録を報告し、その後1sに調整します。当初報告のログは依然表示されます。
- log_queries_not_using_indexes:インデックスを使用したことのない照会を記録するかどうか。デフォルトではOFFです。


## 操作手順
1. [MySQLコンソール](https://console.cloud.tencent.com/cdb)にログインし、インスタンスリストのインスタンスIDまたは「操作」列の【管理】をクリックして、インスタンス管理ページに進みます。
2. インスタンス管理ページで、【操作ログ】画面を選択すると、インスタンスのスローログ明細、エラーログ明細、ロールバックログ、スローログダウンロードを選択、確認できます。

<tr>
<td>スローログダウンロード</td><td>スローログダウンロードの提供</td></tr>
<tr>

<tr>

3. スローログダウンロードページで、「操作」列の【ダウンロード】をクリックしてスローログをダウンロードします。
4. ポップアップダイアログボックスで、ダウンロードアドレスをコピーして、[クラウドデータベースが存在するVPC内のCVM（Linuxシステム）にログイン](https://intl.cloud.tencent.com/document/product/213/10517)し、wgetコマンドを実行して、より効率的なプライベートネットワークの高速ダウンロードを行うことをお勧めします。
>?
>- ログサイズが0KBの場合は、ダウンロードできません
>- 【ローカルダウンロード】を選択して直接ダウンロードすることもできますが、時間がかかります。
>- wgetコマンド形式：wget -c 'ログファイルのダウンロードアドレス' -O カスタムファイル名.log
>
例：
```
wget -c 'http://szx.dl.cdb.tencentyun.com:303/cfdee?appid=1210&time=1591&sign=aIGM%3D' -O test.log
```
