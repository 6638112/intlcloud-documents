### 읽기 전용 인스턴스와 백업 인스턴스에는 어떤 차이가 있습니까?
- 읽기 전용 인스턴스: 읽기 기능만 제공하는 인스턴스로, 마스터 인스턴스와 동일한 리전에 있으며 마스터 인스턴스 1개당 최대 5개의 읽기 전용 인스턴스를 생성할 수 있습니다.
- 백업 인스턴스: 백업 데이터베이스 인스턴스로, 마스터 인스턴스와 동일한 리전에 있으며 마스터 인스턴스 1개당 1~2개의 백업 인스턴스가 있습니다.

### TencentDB for MySQL 읽기 전용 인스턴스는 외부 네트워크 연결이 가능합니까?
읽기 전용 인스턴스는 외부 네트워크 연결과 내부 네트워크 연결을 지원합니다.


### TencentDB for MySQL은 임시 인스턴스 추가가 허용되지 않습니까?
현재는 임시 인스턴스 추가를 지원하지 않습니다.

### 인스턴스 업그레이드에 실패합니다. 어떤 이유입니까?
인스턴스 1개의 테이블 수량이 100만 개를 초과하는 경우 업그레이드에 실패할 수 있습니다. 또한 데이터베이스 모니터링에도 영향을 미칠 수 있으므로, 테이블 수량을 합리적으로 기획하고 인스턴스 1개당 테이블 수량은 100만 개 이하로 제어하십시오.
