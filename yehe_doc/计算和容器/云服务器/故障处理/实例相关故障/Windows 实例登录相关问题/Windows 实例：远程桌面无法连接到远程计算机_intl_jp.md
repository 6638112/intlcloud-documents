## ユースケース
WindowsからリモートでWindowsインスタンスに接続しようとすると、以下のプロンプトが表示されます：
![](https://main.qcloudimg.com/raw/8c79cadc3e14c9c4e0cbb5303b79f74a.png)

リモートデスクトップは以下のいずれかの理由によりリモートコンピューターに接続できません：
1）サーバーへのリモートアクセスが有効になっていません
2）リモートコンピューターの電源がオフになっています
3）ネットワーク上でリモートコンピューターが利用できません

リモートコンピューターの電源がオンになっていて、ネットワークに接続してリモートアクセスが有効になっていることを確認してください。


## 考えられる原因

以上のプロンプトが表示される理由は以下の原因が含まれます（以下の状況に限らず、実際の状況に応じて分析してください）：
- インスタンスが異常な状態です
- CVMにパブリックIPがないか、パブリックネットワークの帯域幅が0です
- インスタンスにバインドされているセキュリティグループはリモートログインポート（デフォルトは3389）を開かれていません
- リモートデスクトップサービスが起動しない
- リモートデスクトップの設定に問題があります
- Windows ファイアウォールの設定に問題があります

## トラブルシューティングの手順

### インスタンスが実行されているかどうかを確認する
1. [CVMコンソール](https://console.cloud.tencent.com/cvm/index)にログインします。
2. 「CVM」ページのインスタンスリストで、インスタンスが【実行中】であるかを確認します。下図に示すように：
![CVMリストページ](https://main.qcloudimg.com/raw/03cf75228bc468d2e436f876f229ebc9.png)
 - はい、 [CVMにパブリックIPがあるかどうかを確認](#step01)してください。
 - いいえ、 Windows インスタンスを起動してください。

<span id="step01"></span>
### CVMにパブリックIPがあるかどうかを確認する
CVMコンソールでCVMにパブリックIPがあるかどうかを確認してください。下図に示すように：
![パブリックIPなし](https://main.qcloudimg.com/raw/58c75d68372069652ec09ab93cfdbdc0.png)
 - はい、[パブリックネットワークの帯域幅を購入したかを確認](#step02)してください。
 - いいえ、[Elastic IPを申請してCVMにバインド](https://intl.cloud.tencent.com/document/product/213/16586)してください。
 
<span id="step02"></span>
###  パブリックネットワークの帯域幅を購入したかを確認する
パブリックネットワークの帯域幅は0MBになっているかを確認します（少なくとも1MBのパブリックネットワークの帯域幅が必要です）。
 - はい、 [ネットワーク構成の調整](https://intl.cloud.tencent.com/document/product/213/15517)で帯域幅を1Mbps或は1Mbps以上に調整してください。
![](https://main.qcloudimg.com/raw/29b771d9de5d1ecdadb872c0378a31c7.png)
 - いいえ、[インスタンスのリモートログインポート（3389）がインターネットに開かれているかどうかを確認](#step03)してください。

<span id="step03"></span>
### インスタンスのリモートログインポート（3389）がインターネットに開かれているかどうかを確認する
1. コンソールでログインするインスタンスをクリックして、インスタンス情報ページに入ります。
2. 「セキュリティグループ」タブで、インスタンスにバインドされているセキュリティグループが、リモートログインポート（デフォルトでは3389）がインターネットに開かれているかどうかを確認します。下図に示すように：
![セキュリティグループ](https://main.qcloudimg.com/raw/28b5f0a038dd354346745bd97f724350.png)
 - はい、 [ Windows インスタンスのシステム設定を確認](#step04)してください。
 - いいえ、対応するセキュリティグループルールを編集して、ポートを開放します。操作方法については、[セキュリティグループ操作ガイド](https://intl.cloud.tencent.com/document/product/213/18197)をご参照ください。

<span id="step04"></span>
###  Windows インスタンスのシステム設定を確認する
1.  VNC を利用してインスタンスにログインして、 Windows インスタンスのシステム設定を確認します。
>? VNCによるログインの詳細については、 [ Windows インスタンスのログイン](https://intl.cloud.tencent.com/document/product/213/5435) の「VNCでのログイン」をご参照ください。
2. OSのインターフェイスで、【スタート】> 【ファイル名を指定して実行】をクリックし、【ファイル名を指定して実行】ボックスに **services.msc**と入力し、**Enter**キーを押してサービスウィンドウを開きます。
3.  「リモートデスクトップサービス」をダブルクリックして、プロパティを開き、リモートデスクトップサービスが起動されているかを確認します。下図に示すように：
![Remote Desktop Service](https://main.qcloudimg.com/raw/6c781636c69eacae76a08b88f9e32b99.png)
 -はい、 [ステップ4](#step04_4)を実行してください。
 - いいえ、「スタートアップの種類」を「自動」に設定し、「サービスの状態」を「実行中」に設定してください。（つまり【開始】をクリックして、サービスを起動する）。
4. <span id="step04_4">【スタート】をクリックして、【実行】で **sysdm.cpl**を入力し、 **Enter**キーを押して、「システムのプロパティ」ウィンドウを開きます。</span>
5. 「リモート」タブで、リモートデスクトップが「このコンピューターへのリモート接続を許可する(L)」に設定されているかを確認します。下図に示すように：
![リモートの設定](https://main.qcloudimg.com/raw/cbf2b2797bd48777008753f389574674.png)
 - はい、 [ステップ6](#step04_6)を実行してください。
 - いいえ、リモートデスクトップを「このコンピューターへのリモート接続を許可する(L)」に設定してください。
6. <span id="step04_6">【スタート】をクリックし、【コントロールパネル】を選択して、コントロールパネルを開きます。</span>
7. 「コントロールパネル」で、【システムとセキュリティ】>【Windows ファイアウォール】を選択して、 「Windows ファイアウォール」を開きます。
8. 「Windows ファイアウォール」で、 Windows ファイアウォールのステータスを確認します。下図に示すように：
![Windows ファイアウォールの状態](https://main.qcloudimg.com/raw/e74937594a03c141e6e5ac753f025d91.png)
 - ステータスが「オン」の場合は、 [ステップ9](#step04_9)を実行してください。
 - ステータスが「オフ」の場合は、 [チケットを送信](https://console.cloud.tencent.com/workorder/category?level1_id=6&level2_id=7&source=0&data_title=%E4%BA%91%E6%9C%8D%E5%8A%A1%E5%99%A8CVM&level3_id=142&radio_title=%E4%BA%91%E6%9C%8D%E5%8A%A1%E5%99%A8%E7%99%BB%E5%BD%95%E4%B8%8D%E4%B8%8A&queue=15&scene_code=12686&step=2) してフィードバックしてください。
9. <span id="step04_9">は 「Windows ファイアウォール」で、【Windows ファイアウォールを介したアプリまたは機能を許可する】をクリックして、「許可されたアプリ」ウィンドウを開きます。</span>
10. 「許可されたアプリ」ウィンドウで、「許可されたアプリと機能(A)」で「リモートデスクトップ」がチェックされているかを確認します。下図に示すように：
![リモートデスクトップをチェックする](https://main.qcloudimg.com/raw/dfed8792ce1dd8b32bee4aaa02ef6bbf.png)
 - はい、 [ステップ11](#step04_11)を実行してください。
 - いいえ、「リモートデスクトップ」をチェックして、「リモートデスクトップ」を開放してください。
11. <span id="step04_11"> 「Windows ファイアウォール」で、【 Windows ファイアウォールの有効化または無効化】をクリックして、「設定のカスタマイズ」ウィンドウを開きます。</span>
12. 「設定のカスタマイズ」ウィンドウで、「プライベートネットワークの設定」と「パブリックネットワークの設定」を「Windows ファイアウォールを無効にする(推奨されません)」に設定します。下図に示すように：
![閉じる](https://main.qcloudimg.com/raw/5d5f8c4d7b783bc1018e03368ae400bc.png)

上記の手順を実行してもリモートデスクトップ経由でWindows インスタンスに接続できない場合は、[チケットを送信](https://console.cloud.tencent.com/workorder/category?level1_id=6&level2_id=7&source=0&data_title=%E4%BA%91%E6%9C%8D%E5%8A%A1%E5%99%A8CVM&level3_id=142&radio_title=%E4%BA%91%E6%9C%8D%E5%8A%A1%E5%99%A8%E7%99%BB%E5%BD%95%E4%B8%8D%E4%B8%8A&queue=15&scene_code=12686&step=2) してフィードバックしてください。
