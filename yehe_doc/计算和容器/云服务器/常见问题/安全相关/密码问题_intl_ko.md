### CVM의 초기 비밀번호는 어떻게 획득하나요?

- [[**사용자 정의 설정**](https://buy.cloud.tencent.com/cvm?tab=custom)]을 통해 CVM을 구매한 경우, 로그인 방식에 따른 초기 비밀번호 설정 방법은 다음과 같습니다.
<table>
	<tr><th>로그인 방식</th><th>설명</th></tr>
	<tr><td>비밀번호 자동 생성</td><td>초기 비밀번호는 이메일과 콘솔 <a href="https://console.cloud.tencent.com/message">내부 메시지</a>로 전송됩니다.</td></tr>
	<tr><td>키 즉시 연결</td><td><b>기본 종료</b>사용자 이름/비밀번호로 로그인할 수 있으며, 초기 비밀번호는 이메일과 콘솔 <a href="https://console.cloud.tencent.com/message">내부 메시지</a>에도 전송됩니다.</td></tr>
	<tr><td>비밀번호 설정</td><td>사용자 정의 비밀번호가 초기 비밀번호입니다.</td></tr>
</table>

더 자세한 내용은 [로그인 비밀번호 관리](https://intl.cloud.tencent.com/document/product/213/17008)를 참조 바랍니다.

### 내부 메시지에서 초기 비밀번호 관련 메시지를 받지 못했다면 어떻게 하나요?
CVM을 구매할 때 시스템에서 이메일과 콘솔 [내부 메시지](https://console.cloud.tencent.com/message)로 초기 비밀번호를 전송합니다.
내부 메시지에 초기 비밀번호 관련 메시지를 받지 못했다면 현재 로그인한 계정이 루트 계정인지 확인하고, 루트 계정이 아니라면 루트 계정으로 다시 로그인한 다음 확인하시기 바랍니다.

### 비밀번호를 잊었을 때는 어떻게 해야 하나요?
비밀번호를 잊으셨다면 재설정할 수 있습니다. 자세한 작업 방식은 [인스턴스 비밀번호 재설정](https://intl.cloud.tencent.com/document/product/213/16566)을 참조 바랍니다.

### CVM 비밀번호는 어떻게 변경(재설정)하나요?
자세한 작업 방식은 [인스턴스 비밀번호 재설정](https://intl.cloud.tencent.com/document/product/213/16566)을 참조하여 재설정하시기 바랍니다.

### 비밀번호 재설정에 실패하면 어떻게 하나요?
Windows CVM이라면, [비밀번호 재설정 실패 및 미적용](https://intl.cloud.tencent.com/document/product/213/35720)을 참조하여 진단하시기 바랍니다.
Linux CVM이라면, [티켓 제출](https://console.cloud.tencent.com/workorder/category)을 통해 문의 바랍니다.

### CVM 비밀번호는 어떻게 일괄 변경(재설정)하나요?
자세한 작업 방식은 [인스턴스 비밀번호 재설정](https://intl.cloud.tencent.com/document/product/213/16566)을 참조 바랍니다.

### CVM의 기본 사용자 이름과 비밀번호는 무엇인가요?
기본 사용자 이름은 아래와 같이 인스턴스 유형과 관련되어 있습니다.
- Windows 인스턴스에 원격 로그인 시의 기본 사용자 이름은 Administrator입니다.
- Linux 인스턴스에 원격 로그인 시의 기본 사용자 이름은 root(Ubuntu 시스템의 기본 사용자 이름은 ubuntu)입니다.

비밀번호는 인스턴스 생성 시 사용자가 설정하며, [인스턴스 생성](https://intl.cloud.tencent.com/document/product/213/4855)을 참조 바랍니다. 비밀번호를 잊으셨다면 [인스턴스 비밀번호 재설정](https://intl.cloud.tencent.com/document/product/213/16566)을 참조하여 재설정할 수 있습니다.
