## 概要
停止済みインスタンスの非課金化は、従量課金インスタンスはシャットダウン操作によって**シャットダウン**状態になった後、インスタンス（CPU、メモリ）の料金が請求されなくなり、クラウドディスク（システムディスクとデータディスク）、インターネット帯域幅やイメージ等のコンポーネントは引き続き課金されます。　　

<dx-alert infotype="notice" title="">
停止済みインスタンスの非課金化機能を有効にすると、インスタンスのCPUとメモリが**保持されない**ようになり、パブリックIPアドレスが**自動的に解放されます**。その他機能の説明、使用制限および影響については、[停止済み従量課金インスタンスの非課金化についての説明](https://intl.cloud.tencent.com/document/product/213/19918)をご参照ください。
</dx-alert>



## 操作手順
<dx-tabs>
::: コンソールからの操作

#### 単一インスタンスのシャットダウン
1.  [CVMコンソール](https://console.cloud.tencent.com/cvm)にログインします。
2. インスタンスの管理画面で、実際に使用されているビューモードに従って操作します。
  - **リストビュー**：下図のように、シャットダウンしたいインスタンスを選択し、右側操作バーで、**その他** > **インスタンスの状態** > **シャットダウン**を選択します。
![](https://qcloudimg.tencent-cloud.cn/raw/6bc11c552db35a4882506449573cf239.png)
   - **タブビュー**：下図に示すように、シャットダウンしたいインスタンスページで、右上コーナーの**シャットダウン**を選択します。
 ![](https://qcloudimg.tencent-cloud.cn/raw/8602dbbdc2f30d072fe53d62e40963e4.png)
3. 「**停止済みインスタンスの非課金化**」にチェックを入れ、**確定**をクリックします。
    この機能をサポートしない場合は、インスタンスリストに「停止済みインスタンスの非課金化機能はサポートしていません」と表示されます。　



#### 複数インスタンスのシャットダウン
1.  [CVMコンソール](https://console.cloud.tencent.com/cvm)にログインします。
2. 下図のように、すべてのシャットダウンしたいインスタンスにチェックを入れ、リスト上部で、**シャットダウン**をクリックすれば、インスタンスを一括シャットダウンできます。
シャットダウンできないインスタンスについて原因が表示されます。
![](https://qcloudimg.tencent-cloud.cn/raw/7c3e7ab231472f3fe8fa8d0bdd7ff685.png)
2. **停止済みインスタンスの非課金化**にチェックを入れ、**確定**をクリックします。
この機能をサポートしない場合は、インスタンスリストに「停止済みインスタンスの非課金化機能はサポートしていません」と表示されます。　
:::
::: クラウド\sAPI\sからの操作
StopInstancesインターフェースを使用してインスタンスをシャットダウンできます。詳細については、[インスタンスのシャットダウン](https://cloud.tencent.com/document/product/213/15743)をご参照ください。次のパラメータを追加します。

| パラメータ名    | 選択必須 | タイプ   | 説明                                                         |
| ----------- | ---- | ------ | ------------------------------------------------------------ |
| StoppedMode | いいえ   | String | 従量課金のクラウドディスクインスタンスに対してのみ有効です。<br>**数値範囲：**<br>KEEP_CHARGING：停止済みインスタンスも引き続き課金されます。 <br>STOP_CHARGING：停止済みインスタンスの非課金化。<br>**デフォルト値：**<br>KEEP_CHARGING |

:::
