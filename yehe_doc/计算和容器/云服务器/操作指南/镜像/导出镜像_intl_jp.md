## 概要
Tencent Cloudは、作成されたカスタマイズイメージの[Cloud Object Storage COS](https://intl.cloud.tencent.com/document/product/436/6222)のバケットへのエクスポートをサポートし、この機能を使用して必要なイメージをエクスポートできます。

##  前提条件
- 現在、この機能を使用するには申し込む必要があります。[チケットを提出](https://console.intl.cloud.tencent.com/workorder/category)してこの機能に申し込んでください。
- [Cloud Object Storageコンソール](https://console.cloud.tencent.com/cos)へ進んで、Cloud Object Storageサービスを有効にします。
- カスタマイズイメージが配置されている地域にバケットを作成します。詳細については、[バケットの作成](https://intl.cloud.tencent.com/document/product/436/13309)をご参照ください。


## 注意事項
- 現在、Windowsカスタマイズイメージのエクスポートをサポートしていません。
- カスタマイズイメージのシステムディスクおよびデータディスクの1つのブロックの容量は、500GBを超えてはなりません。
- マシン全体のイメージをエクスポートする場合、データディスクの数は5を超えることはできません。


## 操作手順
1. Cloud Virtual Machine（CVM）コンソールにログインし、左側のナビゲーションバーで**[イメージ](https://console.cloud.tencent.com/cvm/image)**を選択します。
2. 「イメージ」ページの上部で、エクスポートを必要とするカスタマイズイメージが配置されている地域を選択し、**カスタマイズイメージ**タブをクリックします。
3. イメージが配置されている行の右側にある**その他** > **イメージのエクスポート**を選択します。以下の通りです。
![](https://qcloudimg.tencent-cloud.cn/raw/67a493d7ae96d92b514f5c124619821c.png)
4. ポップアップ表示された「イメージのエクスポート」ウィンドウで、以下の設定を行ってください。下図の通りです：
![](https://qcloudimg.tencent-cloud.cn/raw/4cf1d41610772605ac5867be12353b5a.png)
 - **COS Bucket**：エクスポートされたイメージが配置されているバケットを選択します。バケットはイメージと同一地域にある必要があります。
 - **ファイルプレフィックスのエクスポート**：ファイルプレフィックスをカスタマイズでエクスポートします。
 「CVMにCOS Bucketへのアクセスを許可することに同意する」にチェックを入れます。
5. **OK**をクリックして、イメージエクスポートタスクを開始します。
6. ポップアップ表示された確認ウィンドウで**OK**をクリックします。
エクスポート時間は、イメージのサイズとタスクキューの混雑度によって異なりますので、しばらくお待ちください。エクスポートタスクが完了すると、イメージファイルはターゲットバケットに保存されます。[バケットリスト](https://console.cloud.tencent.com/cos/bucket)ページへ進んで、バケットIDをクリックして、詳細ページへ進むことができます。`カスタマイズプレフィックス_xvda.raw`という名前のファイルは、エクスポートされたイメージファイルです。
