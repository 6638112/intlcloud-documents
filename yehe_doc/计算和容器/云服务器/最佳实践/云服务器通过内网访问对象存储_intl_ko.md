
본문은 클라우드 서버 CVM이 객체 스토리지 내 COS에 액세스할 때 사용하는 액세스 방식과 내부 네트워크 액세스의 판단 방식을 소개하고, 연결성 테스트 예시를 제공합니다. 본문을 참고하여 CVM 의 COS 액세스에 대해 자세히 알아볼 수 있습니다.

## 액세스 방식 설명
Tencent Cloud에 서비스를 배포하여 COS에 액세스하는 경우, 리전 내 액세스 방법에는 다음과 같은 차이점이 있습니다.
-**동일 리전 내 액세스**: 리전 내에서의 액세스는 자동으로 내부 네트워크 주소로 연결됩니다. 생성된 내부 네트워크 트래픽은 과금되지 않습니다. 그러므로 Tencent Cloud의 여러 제품을 구매할 때 최대한 같은 리전을 선택하여 비용을 줄이실 것을 권장합니다.
- **리전 간 액세스**：내부 네트워크 액세스를 지원하지 않으며, 기본적으로 외부 네트워크 주소로 리졸브됩니다.



## 내부 네트워크 액세스 판단 방법
이 단계를 통하여 CVM이 내부 네트워크를 통해 COS에 액세스하는지 테스트할 수 있습니다.
CVM을 통한 COS 액세스의 경우, 내부 인터넷을 이용하여 COS에 액세스했는지 판단하려면, CVM에서 `nslookup` 명령어를 사용하여 COS 도메인을 리졸브할 수 있습니다. 내부 IP를 반환한다면 CVM과 COS 간 내부 네트워크로 액세스한 것이고, 아니라면 외부 네트워크로 액세스한 것입니다.
1. 버킷의 도메인 이름을 얻고 주소를 기록합니다. 자세한 내용은 [버킷 개요](https://intl.cloud.tencent.com/document/product/436/38493)를 참고하십시오.
2. 인스턴스에 로그인하고 nslookup 명령을 실행합니다. `examplebucket-1250000000.cos.ap-guangzhou.myqcloud.com`이 타깃 버킷 주소라고 가정하면, 다음 명령을 실행하십시오.
```
nslookup examplebucket-1250000000.cos.ap-guangzhou.myqcloud.com
```
반환된 결과가 아래 이미지와 같다면, 여기서 `10.148.214.13` 및 `10.148.214.14` 이 두 IP는 내부 네트워크를 통해 COS에 액세스했음을 의미합니다.
>?내부 IP 주소는 일반적으로 `10.*.*.*`, `100.*.*.*` 형식이며, VPC 네트워크는 일반적으로 `169.254.*.*` 등의 형식입니다. 해당 두 가지 형식의 IP는 내부 네트워크에 속합니다.
