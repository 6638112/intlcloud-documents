Tencent CloudのCloud Virtual Machineはストレージハードウェアメディアの調整をサポートしています。ストレージハードウェアメディアを調整すると、異なる業務のストレージ要件に柔軟に対応できます。コンソールで操作して費用計算を実行すると、直ちに新しいディスクメディアに基づいてCloud Virtual Machineが実行されます。

ディスクメディアの調整の前提条件および注意事項については、《[ディスクメディアの調整](https://intl.cloud.tencent.com/document/product/213/32365)》の[前提条件](https://intl.cloud.tencent.com/document/product/213/32365#LocalDiskPrecondition)および[注意事項](https://intl.cloud.tencent.com/document/product/213/32365#LocalDiskNotice)を参照してください。

## 料金ルール

- Cloud Block Storageに対するローカルディスクの調整は「日単位の差額支払い」ルールに準拠します。
料金計算方法：前後に月単位の差額を設定（0以下の場合は0として計算） \* ライフサイクルの残りの日数 / (365 / 12) * 適用される割引 = 支払い料金。
 - 前後に月単位の差額を設定：新旧に原価の月単位単価の差額を設定します。
 - 差額料金は日単位で計算します。つまりライフサイクルの残りの日数 = リソース有効期限 - 現在の時間です。
 - 適用される割引：ライフサイクルの残りの日数を基準に、適用される割引を下方向に整合します。その割引は現在のネットワークで有効な割引です。
- Cloud Block Storageに合わせてローカルディスクを調節しても、リソースの有効期限は影響を受けません。
- Cloud Block Storageに合わせてローカルディスクを調節する操作では、クーポンおよびプラットフォーム贈呈残高（贈呈金）の割引料金を使用することができます。

>! 前後に月単位の差額を設定する場合、その新しく設定した価格が古い設定より低いときは、返金は実行されません。
