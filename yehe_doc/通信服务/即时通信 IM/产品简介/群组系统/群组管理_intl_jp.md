インスタントメッセージIMは一般的なユースケースに基づきデフォルトで友だちの作業グループ（Work）、知らない人の社交グループ（Public）、臨時会議グループ（Meeting）とライブ配信グループ（AVChatRoom）のグループ形態を構成します。詳細については[グループタイプの紹介](https://intl.cloud.tencent.com/document/product/1047/33529)をご参照ください。

グループに対しては、以下の操作を行うことができます。

| グループ操作 | 説明               | 備考             |
| -------- | -------------- | ---------------- |
| グループの作成 | 新しいグループを作成すると、グループタイプ、グループ名および参加するユーザリストを指定でき、作成完了後グループIDが返送されます。グループIDはグループの一意の識別子であり、グループIDを通じてメッセージを送受信するなどその他のグループ操作ができます。| 各Appの1日当たりの純消費グループ上限は10000です。      |
| グループの譲渡 | グループ所有者の身元を他の人に変更します。| Appの管理者はREST APIを介してグループを譲渡でき、この他はグループ所有者のみがグループを譲渡できます。|
| グループの解散 | Appで作成したとあるグループを解散し、グループが解散される時、グループの元のメンバーはグループを解散する旨のシステムからのメッセージを受信します。| App管理者はREST APIインターフェースを呼び出して任意のグループを解散できます。App側でグループを解散する権限はメンバーのロールにより制限され、<br>知らない人の社交グループ（Public）、臨時会議グループ（Meeting）とライブ配信グループ（AVChatRoom）はグループ所有者のみがグループを解散できます。<br>友だちの作業グループ（Work）はグループ内のいかなる人もグループを解散できません。|

>
> - グループ作成時、インスタントメッセージIMは@TGS#で始まるデフォルトのグループIDをアサインします。ただし、ユーザが指定するグループIDもサポートしています。詳細は[グループIDのカスタマイズ](https://intl.cloud.tencent.com/document/product/1047/33529)をご参照ください。
> - グループを作成した後、複数の端末のグループの同期を維持する目的で、グループ所有者は端末でグループを作成した旨のシステムからのメッセージを受信します（1つの端末でグループを作成すると、すべての端末はこのグループが作成されたことをすぐに認識します）。



### グループデータの管理

グループデータとは、グループ名、紹介、お知らせ、グループ所有者などを含む1つのグループ次元の属性およびグループの次元のカスタムフィールドを指します。

| グループデータの管理   | 説明            | 備考           |
| ------------ | ---------------------- | ------------- |
| グループデータの取得 | グループの基本データをプルし、カスタムデータをプルしたい場合、プルフィールドを設定するインターフェースより設定できます | **グループメンバーによるグループデータの取得**：メンバーが本グループデータを取得します。<br>**グループメンバー以外のメンバーによるグループデータの取得**：グループメンバー以外のメンバーによるグループデータの取得は、公開された情報しか取得できません。<br>**グループにおける本人データの取得**：すべてのグループにおける本人データを取得することも、1つのグループにおける本人データを取得することもできます。<br>**グループにおけるとある人のデータの取得**：ライブ配信グループ（AVChatRoom）はグループ所有者、管理者および一部のグループメンバーを含む一部のメンバーのデータしか取得できません |
| グループデータの修正 | グループ名、グループの紹介、グループのお知らせ、グループのプロファイルフォト、グループの名刺を修正することができ、グループ追加オプション、グループ次元のカスタムフィールド、ユーザのグループにおける身元、グループメンバー次元のカスタムフィールドとグループメッセージ受信オプションなどの情報を修正できます | 現在、Appはコンソールでグループ名、紹介、お知らせ、プロファイルフォトURLなどの変更コールバックを自身で構成できます。その他のグループデータ（グループ次元のカスタムフィールドを含む）の変更コールバックを有効にする必要がある場合、[作業依頼書サブミット](https://console.cloud.tencent.com/workorder/category?level1_id=29&level2_id=40&source=0&data_title=%E4%BA%91%E9%80%9A%E4%BF%A1%20%20IM&step=1) にて申請してください |



### グループメンバー/グループの管理

グループメンバーの管理は以下の2つの面を含みます。
- 自身のグループにおける情報の取得/修正で、これらの情報はユーザ自身によってのみ取得/設定できます（例：メッセージ受信オプションなど）。
- その他のグループメンバーの情報の取得/修正で、グループメンバーの身元、グループ参加日時、最後にメッセージを送信した日時、グループ名刺およびグループ次元のカスタムデータを含みます。

| グループメンバーの管理   | 説明            | 備考           |
| --------------------- | ------------------- | -------------------- |
| グループメンバーのデータの取得        | 自身またはその他グループメンバーの情報を取得します。      | グループメンバーの身元、グループ参加日時、最後にメッセージを送信した日時、グループ名刺およびグループ次元のカスタムデータを取得できます。|
| グループメンバーのデータの修正        | グループ所有者、管理者またはメンバーは対応するグループメンバーのデータを修正できます。    | <ul style="margin:0;"><li>グループ所有者または管理者は、グループ内の身元（管理者の設定/取消）修正、発言禁止、グループ名刺の修正、グループメンバー次元のカスタムフィールドなどを含め、その他のグループメンバーのデータを修正します。</li><li>グループメンバーは、メッセージ受信オプション、グループ名刺、グループメンバー次元のカスタムフィールドなどを含め、自身のグループにおけるデータを自発的に修正します。</li></ul>  |
| グループへの招待              | グループへの招待は他のユーザをとあるグループに引き込みたいということです。    | <ul style="margin:0;"><li>友だちの作業グループ（Work）では、どのグループメンバーも他の人をグループに招待でき、招待された側の同意もなく直接グループに入れられます。</li><li>知らない人の社交グループ（Public）と臨時会議グループ（Meeting）ではデフォルトでAppの管理者だけがその他のユーザをグループに招待できます。</li><li>ライブ配信グループ（AVChatRoom）では、いかなる人も他の人をグループに招待できません。</li></ul>|
| グループ参加の申請              | グループ参加の申請とは、ユーザがIM SDKによりとあるグループに自発的に参加するアクションです。    | <ul style="margin:0;"><li>友だちの作業グループ（Work）はグループ参加の申請ができず、直接エラーが戻されます。</li><li>その他の内蔵グループのタイプの場合、グループ参加の申請の処理結果は、グループデータ中のApplyJoinOptionフィールドにより決定されます。</li></ul>|
| グループメンバーの削除          | グループメンバーの削除とは、グループ所有者またはグループ管理者がグループメンバーをグループから削除する操作です。| 管理者またはグループ所有者がユーザをグループから削除した後、削除されたユーザはグループから削除された旨のシステムからのメッセージを受信し、グループ内の他のメンバーもこのユーザがグループから削除された旨のイベントメッセージを受信します。|
| 自発的なグループ退会              | 自発的なグループ退会とは、グループ内のメンバーが自発的に退会操作をすることです。   | グループメンバーが自発的にグループを退会した後、自発的に退会操作をしたユーザは自発的にグループを退会した旨のシステムからのメッセージを受信し、グループ内の他のメンバーはこのメンバーがグループを退会した旨のイベントメッセージを受信します。|
| ユーザが参加したグループの取得  | 現在のユーザが参加しているすべてのグループリストをプルし、戻される情報には一部の基本情報しか含まれません。| 詳細なグループ情報は、グループメンバーのグループデータ取得機能に基づき取得できます。   |
| グループ未決情報リスト        | グループ未決情報とは一般的にすべての審査承認が必要なグループ関連の操作を指します。      | グループ未決リストをプルし、グループ未決が既読であることを報告し、グループ未決情報を処理（同意または拒否）できます。|
