本ドキュメントでは、Linuxインスタンスのログイン失敗の考えられる原因とトラブルシューティング方法について説明し、問題のトラブルシューティング、特定及び解決に役立ちます。

## 考えられる原因
Linux インスタンスにログインできない主な原因は次のとおりです：
- [SSHの問題によるログインできない](#UseSSHLogin)
- [パスワードの問題によるログインできない](#CryptographicProblem)
- [帯域幅の利用率が高すぎる](#BandwidthUtilization)
- [サーバーの負荷が高すぎる](#HighServerLoad)
- [セキュリティグループルールの不具合](#SafetyGroupRule)

セルフ診断ツールを使用して問題を確認できない場合は、 [VNCによるCVMにログイン](#VNC) し、障害をトラブルシューティングすることを推奨します。

## トラブルシューティング
### VNCによるログイン
<span id="VNC"></span>
標準の方法（Webshell）またはリモートログインソフトウェアを使用してLinuxインスタンスにログインできない場合、Tencent Cloud VNCを使用してログインし、故障の原因を特定します。
1.  [Tencent Cloudコンソール](https://console.cloud.tencent.com/cvm/index)にログインします。
2. インスタンスの管理画面で、ログインするインスタンスを選択し、【ログイン】をクリックします。下記画像に示すように：。
![](https://main.qcloudimg.com/raw/a4cc736f2dc7f13bf39756b8e39532d4.png)
3. ポップアップした 「Linux インスタンスにログインする」ウインドウで、【その他の方式(VNC）】を選択し、【すぐにログインする】をクリックします。
> ログイン中にパスワードを忘れた場合は、コンソールで当該インスタンスのパスワードをリセットすることができます。具体的な操作は [インスタンスパスワードのリセット](https://intl.cloud.tencent.com/document/product/213/16566)をご参照ください。
>
4. ポップアップしたダイアログボックスで、ユーザー名とパスワードを入力してログインします。

<span id="UseSSHLogin"></span>
### SSHの問題によるログインできない
**障害現象**：[SSHを利用して Linux インスタンスにログイン](https://intl.cloud.tencent.com/document/product/213/32501) する時、接続できないまたは接続失敗が表示されます。
**処理手順**：[ SSHを利用してLinux インスタンスにログインできない](https://intl.cloud.tencent.com/document/product/213/32486) を参照して、トラブルシューティングを行います。

<span id="CryptographicProblem"></span>
### パスワードの問題によるログインできない
**障害現象**：誤ったパスワードを入力した、パスワードを忘れた又はパスワードのリセット失敗によるログイン失敗です。
**解決方法**： [Tencent Cloudコンソール](https://console.cloud.tencent.com/cvm/index) で当該インスタンスのパスワードをリセットしてから、インスタンスを再起動します。
**処理手順**：インスタンスパスワードをリセットする方法については、 [インスタンスパスワードのリセット](https://intl.cloud.tencent.com/document/product/213/16566)をご参照ください。

<span id="BandwidthUtilization"></span>
### 帯域幅の利用率が高すぎる
**障害現象**：セルフ診断ツールにより診断された問題は帯域幅の利用率が高すぎることです。
**処理手順**：
1.  [VNC ログイン](#VNC) を介してインスタンスにログインします。
2.  [帯域幅の利用率が高いためログインできない](https://intl.cloud.tencent.com/document/product/213/32542)を参考し、インスタンスの帯域幅利用状況を表示して障害を処理します。

<span id="HighServerLoad"></span>
### サーバーの負荷が高すぎる
**障害現象**：セルフ診断ツール又はクラウド監視により、サーバーのCPU負荷が高すぎるため、システムがリモート接続を実行できないか、或いはアクセスが遅いことが判明しました。
**考えられる原因**：ウィルスやトロイの木馬、サードパーティアンチウィルスソフト、アプリケーションの異常、ドライバーの異常又はソフトウェアバックグラウンドの自動更新により、CPU利用率が高くなり、CVMにログインできなかったり、アクセス速度が遅くなったりすることが発生しました。
**処理手順**：
1.  [VNC ログイン](#VNC) を介してインスタンスにログインします。
2.  [Linux インスタンス：CPU とメモリの使用率が高いためにログインできない](https://intl.cloud.tencent.com/document/product/213/32387)を参考し、「タスクマネージャー」で負荷が高いプロセスを特定します。


<span id="SafetyGroupRule"></span>
### セキュリティグループルールの不具合
**障害現象**：セルフ診断ツールで検査した結果、セキュリティグループルール設定の不具合によりログインできなくなったことが判明しました。
**処理手順**： [セキュリティグループ（ポート）検証ツール](https://console.cloud.tencent.com/vpc/helper) を使用して検査を行います。
![](https://main.qcloudimg.com/raw/278c7f0abd9b7224d32fa5402554544a.png)
問題の原因がセキュリティグループポートの設定である場合は、【ワンクリックポートオーペン】機能でポートを開くことができます。
![](https://main.qcloudimg.com/raw/e4a40dafcc9607ce18ee7001129d9655.png)
セキュリティグループルールをカスタマイズする必要がある場合は、[セキュリティグループルールの追加](https://intl.cloud.tencent.com/document/product/213/34272) を参考して、セキュリティグループルールを設定しなおしてください。



## その他ソリューション
上記のトラブルシューティング方法を使用してもLinux インスタンスにログインできない場合、セルフ診断結果を保存して、[チケットを送信](https://console.cloud.tencent.com/workorder/category) してください。
