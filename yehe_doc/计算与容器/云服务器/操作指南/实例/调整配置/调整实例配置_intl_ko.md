## 작업 시나리오

Tencent Cloud 인스턴스의 빠르고 편리한 하드웨어 장치 변경 기능은 CVM의 효율성을 나타내는 지표입니다. 본 문서에서는 인스턴스 구성 업그레이드 및 다운그레이드, 모델 간의 구성 변경 방법 및 관련 주의사항을 소개합니다.

## 전제 조건

종료 상태거나 시작 상태인 인스턴스 모두 구성을 변경할 수 있습니다. 시작 상태의 인스턴스는 강제 재시작한 후에 작업이 적용됩니다.
>! 
> - 작업할 인스턴스가 **종료** 상태인 경우 콘솔 변경 작업을 바로 진행할 수 있습니다.
> - 작업할 인스턴스가 **시작** 상태인 경우 온라인에서 구성을 변경할 수 있으며, 변경 완료 후에는 강제 재시작한 뒤에 변경한 구성이 적용되었는지 확인해야 합니다.
> - **일괄 변경** 작업을 온라인에서 진행할 수 있습니다. 일괄로 변경된 기기 중 **시작** 상태의 기기는 강제 재시작한 뒤에 해당 작업이 적용되었는지 확인해야 합니다.

## 제한 및 영향

### 구성 변경 제한

**시스템 디스크와 데이터 디스크가 모두 CBS**인 인스턴스만 구성을 변경할 수 있습니다.
- 구성 업그레이드
횟수 제한이 없으며, 업그레이드는 즉시 적용됩니다.
- 구성 다운그레이드
 - 종량제 인스턴스는 언제든 횟수 제한 없이 구성을 다운그레이드할 수 있습니다.
- 인스턴스 패밀리 간 변경: 인스턴스 패밀리끼리 구성을 변경할 수 있으므로 데이터 마이그레이션을 하지 않아도 됩니다.
구성 변경 시 인스턴스 사양 변경은 현재 사용 중인 가용존이 타깃 사양을 제공하는지 여부에 따라 달라지므로 다음 제한 사항에 유의하십시오.
 - **스팟 인스턴스**는 모델 간 구성 변경을 지원하지 않습니다.
 - **전용 인스턴스**는 모델 간 구성 변경을 지원하지 않으며, 구성 변경 범위는 해당 인스턴스가 있는 CDH의 남은 리소스 수량에 따라 달라집니다.
 - **GPU, FPGA 등의 이종 인스턴스**는 인스턴스 패밀리 간 구성 변경을 위한 소스 인스턴스 사양 및 타깃 인스턴스 사양으로 사용할 수 없습니다.
 - **기본 네트워크를 구성하는 인스턴스**는 VPC만 지원하는 인스턴스로 변경할 수 없습니다.
 - 타깃 인스턴스 사양이 현재 사양에 구성된 CBS 유형을 지원하지 않을 경우 구성을 변경할 수 없습니다.
 - 타깃 인스턴스 사양이 현재 사양에 구성된 이미지 유형을 지원하지 않을 경우 구성을 변경할 수 없습니다.
 - 타깃 인스턴스 사양이 현재 사양에 구성된 ENI 또는 ENI 수량을 지원하지 않을 경우 구성을 변경할 수 없습니다. 자세한 내용은 [엘라스틱 네트워크 사용 제한](https://intl.cloud.tencent.com/document/product/576/18527)을 참조하십시오.
 - 타깃 인스턴스 사양이 현재 사양의 공인 네트워크 대역폭 최댓값을 지원하지 않을 경우 구성을 변경할 수 없습니다. 자세한 내용은 [공용 네트워크 대역폭 최댓값](https://intl.cloud.tencent.com/document/product/213/12523)을 참조하십시오.

### 관련 영향

변경 후 극소수의 인스턴스에서 내부 IP가 변경될 수 있습니다. 내부 IP가 변경됐을 경우 변경 페이지에 안내 문구가 표시되며, 관련 문구가 나타나지 않으면 내부 IP가 변경되지 않았음을 나타냅니다.

## 작업 순서

>?
> - 비즈니스에 변동 사항이 생길 경우, 상황에 맞게 구성을 변경할 수 있습니다.
>  - 구성 업그레이드 시, 이에 따라 발생할 수 있는 비용을 지불해 주십시오.
>  - 구성 다운그레이드 시, 강제 재시작 후에 새로운 구성이 즉시 적용되며, CVM이 새로운 구성에 맞춰 실행됩니다.

### 콘솔을 통한 변경

#### 단일 인스턴스 구성 변경

1. [CVM 콘솔](https://console.cloud.tencent.com/cvm/index)에 로그인한 뒤 CVM 목록으로 이동합니다.
2. 아래 이미지와 같이, 변경하려는 인스턴스의 오른쪽 작업란에서 [더보기]>[리소스 변경]>[구성 변경]을 클릭합니다.
![](https://main.qcloudimg.com/raw/65541031befe2d0144227cf5a616e161.png)
3. 아래 이미지와 같이, '타깃 구성 선택'에서 인스턴스 상태 및 작업을 확인한 뒤, **필요한 모델 및 인스턴스 사양을 선택하고 사양 및 성능 매개변수를 자세히 확인합니다.** 확인이 끝나면 [다음]을 클릭합니다.
![](https://main.qcloudimg.com/raw/818fbf0dfa791ad5d5a76186eefba019.png)
4. 인스턴스의 과금 방식에 따라 요금을 확인한 뒤 [다음]을 클릭합니다.
	- 종량제 인스턴스: 새로운 사양 적용 시 동결해야 하는 금액을 확인합니다. 구성 변경 후, 종량제 과금이 첫 단계부터 적용되므로 관련 규칙을 자세히 확인한 뒤 작업을 진행하십시오.
	![](https://main.qcloudimg.com/raw/25f8630836acdfe274357142d8609c5d.png)
5. 인스턴스 실행 상태에 따라 '종료 알림' 안내 문구가 다르므로, 주의해서 읽으십시오.
 - 인스턴스가 실행 중일 경우, 아래 이미지와 같이 안내 문구를 확인한 뒤 '강제 종료에 동의'를 선택합니다.
![](https://main.qcloudimg.com/raw/e016f2cc674938acd0046115f007669b.png)
 - 인스턴스가 종료 상태일 경우, 아래 이미지와 같이 현재 상태를 다시 한 번 확인하는 안내 문구가 뜹니다.
![](https://main.qcloudimg.com/raw/8385495393237523d0d71460a7b7009b.png)
6. [변경 시작]을 클릭하여 주문 페이지로 이동한 뒤, 요금을 지불합니다. 

### API을 통한 변경 

ResetInstancesType 인터페이스를 사용하여 인스턴스 구성을 변경할 수 있습니다. 자세한 내용은 [인스턴스 구성 변경](https://intl.cloud.tencent.com/document/product/213/33239)의 API 문서를 참조하십시오.
