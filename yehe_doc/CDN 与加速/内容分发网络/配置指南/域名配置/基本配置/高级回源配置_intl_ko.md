

Tencent Cloud CDN은 경로별 원본 요청(파일/폴더/모든 경로 파일/메인 화면 원본 요청), Client IP 소재 리전에 따른 원본 요청과 같은 보다 세밀한 원본 요청 설정을 지원합니다.

>!Client IP의 소재 리전에 따라 원본 요청에 대해 A/B 테스트를 진행중인 관계로 아직 전량 배포되지 않았습니다. 지속적으로 배포할 예정이오니 잠시만 기다려주십시오.

## 설정 가이드

[CDN 콘솔](https://console.cloud.tencent.com/cdn)에 로그인한 후, 왼쪽 메뉴에서 [도메인 관리]를 선택한 뒤 도메인 작업 열의 [관리]를 클릭하여 도메인 설정 페이지로 들어갑니다. Tab의 [기본 설정] 페이지의 [원본 서버 정보] 모듈 최하단에 [고급 원본 요청 설정]이 있으며, 클릭하여 펼쳐보기 할 수 있습니다.

![](https://main.qcloudimg.com/raw/17e0385a7cea2cbdd0ae012cae8b2555.png)

### 규칙 추가

필요에 따라 [규칙 추가]를 클릭하여 고급 원본 요청 설정 규칙을 추가할 수 있습니다:
![](https://main.qcloudimg.com/raw/ef2ae0e51a0cb032d493f89b8029b316.png)

**설정 제한**

- 도메인당 최대 50개까지 규칙을 추가할 수 있습니다.
- 단일 규칙 중의 원본 요청 주소는 IP/도메인 이름 원본 서버 및 포트(0-65535) 입력을 지원하며, 포트는 기본값으로 설정 가능합니다. 원본 요청 프로토콜이 HTTPS 또는 프로토콜을 따르도록 선택되어 있는 경우, 포트는 443으로만 구성하거나 구성하지 않아야 합니다.
- 더 많은 조작: 여러 규칙에 대한 우선순위 조정 및 규칙의 일괄 편집과 삭제를 지원합니다.

>?
>- 하위 우선순위가 상위 우선순위보다 높음 - 이와 같은 상대적 위치의 우선순위 조정은 여러 경로별 원본 요청 규칙(지정 파일 유형/폴더/모든 경로 파일/메인 화면 원본 요청 규칙) 또는 Client IP 소재 리전에 따른 원본 요청 규칙과 같은 동일한 유형의 원본 요청 규칙에만 적용됩니다.
>- 동일한 요청이 서로 다른 유형의 원본 요청 규칙에 부합하는 경우, 유형별 우선순위에 따라 실행되며 그 순서는 경로별 > Client IP입니다.
예시: `Client IP의 소재지가 지앙쑤이고 1.1.1.1로 Origin-pull`하며 `/test는 2.2.2.2로 Origin-pull`하는 경우, 지앙쑤에 소재한 Client IP가 /test에 액세스할 때 2.2.2.2로 Origin-pull합니다.
