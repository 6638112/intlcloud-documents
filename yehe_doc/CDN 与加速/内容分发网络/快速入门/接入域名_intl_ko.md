서비스를 활성화하고 [CDN 콘솔](https://console.cloud.tencent.com/cdn)에 로그인한 후 메뉴의 [Domain Management]에서 가속 도메인을 추가하여 Tencent Cloud CDN의 가속 서비스를 이용할 수 있습니다. 도메인을 추가하면 관련 도메인 설정이 전체 네트워크의 CDN 가속 노드에 전달되며, 기존 네트워크 상의 비즈니스에는 영향을 주지 않습니다. 가속 서비스를 정식으로 활성화하려면 CNAME을 설정해야 하며, 자세한 작업 순서는 [CNAME 설정](https://intl.cloud.tencent.com/document/product/228/3121)을 참조 바랍니다.


## 도메인 추가
도메인 관리 페이지에서 [Create a Distribution]을 클릭합니다.
![](https://main.qcloudimg.com/raw/02784c06077ee97314ec146aa3369fb1.png)

도메인 추가 페이지는 다음 세 가지로 구성되어 있습니다.

- 도메인 속성 설정
- 원본 서버 설정
- 캐시 특성 설정

Tencent Cloud CDN은 선택된 도메인 속성 설정 및 원본 서버 설정에 따라 적합한 캐시 특성 설정을 추천해드리므로, 설정을 변경할 필요 없이 바로 제출할 수 있습니다.

### Part1: 도메인 속성 설정
도메인 란에 비즈니스의 서비스 도메인을 입력한 뒤, 프로젝트, 가속 지역 및 서비스 유형을 선택합니다.
![](https://main.qcloudimg.com/raw/4f2de8b5158a317b1913bd5f920dbf3b.png)

**설정 항목 상세 설명: **

| 설정 항목   | 설정 설명                                                     |
| -------- | ------------------------------------------------------------ |
| 도메인     | 1. 도메인 길이는 50자를 초과할 수 없습니다.<br/>2. 도메인은 공업정보부의 ICP비안 발급을 완료한 상태여야 합니다.<br/>3. 도메인은 a.test.com, a.b.test.com 등과 같은 형식의 서브 도메인이거나 \*.test.com, \*.a.test.com 형식의 와일드카드 서브도메인이어야 합니다.<br/>4. 도메인이 와일드카드 서브도메인이거나 다른 사용자가 이미 액세스했다면, [소유권 인증](#m1)을 한 후에 액세스 및 검색할 수 있습니다.<br/><br/><strong>주의 사항:</strong><br/>1. 현재 와일드카드 서브도메인에 액세스한 뒤 다른 계정의 서브 도메인 또는 서브 와일드카드 서브도메인 액세스를 지원하지 않습니다.<br/>2. 지금은 `*.test.com` 과 `*.a.test.com`의 동시 액세스를 지원하지 않습니다. |
| 프로젝트 | 프로젝트란 Tencent Cloud의 모든 클라우드 서비스에 대한 일종의 공유 리소스 셋을 의미하며, [프로젝트 관리](https://console.cloud.tencent.com/project)에서 프로젝트 관련 작업을 진행하실 수 있습니다. |
| 가속 지역 | 중국 내: 글로벌 사용자가 액세스할 경우 중국 내 가속 노드로 스케쥴링하여 서비스를 제공합니다.<br/>중국 외: 글로벌 사용자가 액세스할 경우 중국 외 가속 노드로 스케쥴링하여 서비스를 제공합니다.<br/>글로벌: 글로벌 사용자가 액세스할 경우 가장 가까운 노드로 스케쥴링하여 서비스를 제공합니다.<br/><br/><strong>주의 사항: </strong><br/>중국 내, 중국 외 가속 서비스는 별도로 과금되며, 자세한 과금 정책은 이곳에서 [조회](https://intl.cloud.tencent.com/document/product/228/2949)하실 수 있습니다. |
| 서비스 유형 | Tencent Cloud CDN은 서비스 유형에 따라 맞춤형 가속 성능 최적화를 진행했으므로,<br/>비즈니스에 따라 적합한 서비스 유형을 선택하여 더 나은 가속 효과를 얻을 수 있습니다.<br/><br/>정적 가속: 전자상거래, 웹 사이트 및 게임 이미지 등의 소규모 리소스 가속 시나리오에 적합합니다.<br/>다운로드 가속: 게임 설치 패키지, 멀티미디어 소스 파일 다운로드, 모바일 펌웨어 전송 등의 다운로드 시나리오에 적합합니다.<br/>스트림 미디어 VOD 가속: 온라인 교육, 온라인 비디오 VOD 등의 시나리오에 적합합니다. |
| 태그 | 최대 50개의 태그를 추가할 수 있습니다.<br/>인벤토리 태그(태그 콘솔에 이미 추가된 태그)만 선택할 수 있습니다.<br/>태그 추가 시 태그 키와 태그값은 필수 선택 항목입니다. [태그 관리](https://console.cloud.tencent.com/tag/taglist)에서 관련 작업을 진행할 수 있습니다. |
| 가속 프로토콜 | IPv4: 노드가 IPv4 액세스만 지원합니다.<br/>IPv4 + IPv6: 노드가 IPv4, IPv6 액세스를 동시에 지원하며, 해당 옵션을 선택했을 경우에만 IPv6 원본 서버를 설정할 수 있습니다.<br/><br/><strong>주의 사항:</strong><br/>IPv6는 중국 내에서만 지원됩니다. |

### Part2: 원본 서버 설정

서비스의 원본 서버 관련 정보를 설정하면, CDN 노드에 캐시되는 리소스가 없을 때 원본 서버에서 가져온 후 캐시합니다.
![img](https://main.qcloudimg.com/raw/4a051cfc61dbd0e64b5ccfb8b386829d.png)

**설정 항목 상세 설명: **

| 설정 항목   | 설정 설명                                                     |
| -------- | ------------------------------------------------------------ |
| 원본 서버 유형 | 외부 원본: 안정적으로 실행되는 비즈니스 서버(즉 원본 서버)를 보유한 상태의 시나리오.<br/><a href = "https://intl.cloud.tencent.com/product/cos">객체 스토리지(COS 원본)</a>: 리소스가 이미 Tencent Cloud COS에 저장되어 있다면 바로 bucket을 원본 서버로 선택하여 사용할 수 있습니다. |
| 원본 서버 주소 | 외부 원본: <br/>1. 여러 개의 IP를 원본 서버로 설정할 수 있으며, Origin-pull 시 폴링 방식의 Origin-pull을 진행합니다.<br/>2. 포트(0 - 65535) 및 가중치 (1 - 100) 설정을 추가할 수 있습니다. 형식은 `원본 서버:포트:가중치`입니다(포트가 생략된 `원본 서버::가중치` 형식도 지원합니다). HTTPS 프로토콜은 현재 443 포트만 지원합니다.<br/>3. 도메인을 원본 서버로 설정할 수 있습니다. 이때 도메인은 서비스 가속 도메인과 일치하지 않아야 합니다. |
| 원본 프로토콜 | 원본 서버의 지원 상황에 따라 원본 요청 프로토콜을 선택합니다:<br/>HTTP: HTTP / HTTPS 액세스가 모두 HTTP를 사용하여 Origin-pull 합니다.<br/>HTTPS: HTTP / HTTPS 액세스가 모두 HTTPS를 사용하여 Origin-pull 합니다.(원본 서버가 HTTPS 액세스를 지원해야 함)<br/>프로토콜 따라가기: HTTP 액세스는 HTTP를 사용하여, HTTPS 액세스는 HTTPS를 사용하여 Origin-pull 합니다.(원본 서버가 HTTPS 액세스를 지원해야 함) |
| 원본 도메인 | 원본 도메인이란 CDN 노드가 원본을 가져올 때 원본 서버에서 액세스하는 사이트 도메인을 말합니다.<br/>서브 도메인 액세스 시 가속 도메인과 동일하도록 기본 설정되어 있으며, 사용자 정의 수정이 가능합니다.<br/>와일드카드 서브도메인 액세스 시 실제 액세스한 서브도메인과 동일하도록 기본 설정되어 있으며, 사용자 정의 수정이 가능합니다. |

| 기본 설정 | 노드의 캐시 리소스는 Key-Value 매핑을 따르며, 그중 Key는 리소스 URL을 말합니다.<br/>매개변수 필터링을 활성화하면 Key가 URL 중 “?” 뒤에 있는 매개변수를 무시하여 매핑합니다.<br/>매개변수 필터링을 비활성화하면 Key는 전체 리소스 URL이 됩니다.<br/>정적 가속 유형에서는 비활성화 상태로 기본 설정되어 있으며, 다운로드 및 스트리밍 미디어 VOD 가속 유형에서는 활성화 상태로 기본 설정되어 있습니다. |

| 캐시 규칙 | 노드의 캐시 만료 시간을 설정하는 것으로, 모든 파일의 캐시 만료 기한은 30일로 기본 설정되어 있습니다.<br/>여기에서 설정하는 캐시 만료 시간은 최대 만료 시간이며, 노드 스토리지 리소스의 영향을 받으므로 실제 상황에 따라 설정하시기 바랍니다. |

## 도메인 소유권 인증
액세스한 도메인이 와일드카드 서브도메인이거나 다른 사용자가 이미 액세스한 경우, 도메인의 소유권을 인증한 후에 액세스 및 검색할 수 있습니다. 도메인의 소유권 인증은 DNS 인증 방식을 사용하며, 인증 순서는 다음과 같습니다.

1. [Verification Method]를 클릭하여 DNS 인증 시 추가해야 하는 리졸브 기록 정보를 불러옵니다. 인증이 끝나기 전에는 페이지를 닫지 마시기 바랍니다.
   ![img](https://main.qcloudimg.com/raw/a0bae11cfa188a1ec2a6ca3d5cd4edec.png)
2. 도메인 리졸브 업체 측에서 TXT 형식의 DNS 기록을 1건 추가하면, 호스트 기록에는 _cdnauth를, 기록 값에는 인증 방법에 랜덤하게 생성된 값을 입력합니다. [DNS 리졸브 DNSPod](https://console.cloud.tencent.com/cns)의 예시는 다음과 같습니다. ![1583209117135](https://main.qcloudimg.com/raw/dd02d7b7b18d753f7b9e9c72df617cdf.png)
3. TXT 리졸브가 적용되면 [인증] 버튼을 클릭하여 인증합니다. "인증 실패"라는 메시지가 뜰 경우, DNS 기록이 적용될 때까지 기다렸다가 다시 시도하시거나 TXT 기록을 정확하게 입력했는지 확인하시기 바랍니다.
