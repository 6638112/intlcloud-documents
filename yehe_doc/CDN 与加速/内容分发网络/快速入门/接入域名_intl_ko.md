서비스를 활성화하면 [CDN 콘솔(https://console.cloud.tencent.com/cdn)에 로그인한 후 메뉴의 [도메인 관리]에서 가속 도메인을 추가하여 Tencent Cloud CDN의 가속 서비스를 이용할 수 있습니다. 도메인을 추가하면 관련 도메인 설정이 전체 네트워크 CDN 가속 노드에 전달되며, 기존 네트워크상의 비즈니스에 영향을 주지 않습니다. 가속 서비스를 정식으로 활성화하려면 CNAME을 설정해야 하며, 자세한 작업 순서는 [CNAME 설정](https://intl.cloud.tencent.com/document/product/228/3121)을 참조 바랍니다.

## 도메인 추가
도메인 관리 페이지에서 [도메인 추가]를 클릭합니다.
![](https://main.qcloudimg.com/raw/02784c06077ee97314ec146aa3369fb1.png)

도메인 추가 페이지는 다음 세 가지로 구성되어 있습니다.

- 도메인 속성 설정
- 원본 서버 설정
- 캐시 특징 설정

Tencent Cloud CDN에서는 선택하신 도메인 속성 설정 및 원본 서버 설정에 따라 적합한 캐시 특징 설정을 추천해드리므로, 설정을 변경할 필요 없이 바로 제출할 수 있습니다.

### Part1: 도메인 속성 설정
도메인 란에 비즈니스의 서비스 도메인을 입력하고 프로젝트, 가속 리전 및 서비스 유형을 선택합니다.
![](https://main.qcloudimg.com/raw/4f2de8b5158a317b1913bd5f920dbf3b.png)

**설정 프로젝트 상세 설명: **

| 설정 프로젝트   | 설정 설명                                                     |
| -------- | ------------------------------------------------------------ |
| 도메인     | 1. 도메인 이름은 50자를 초과할 수 없습니다.<br/>2. 도메인이 공업정보부의 ICP비안 발급을 완료한 상태여야 합니다.<br/>3. 도메인이 a.test.com, a.b.test.com 등 형식의 서브도메인 또는 \*.test.com, \*.a.test.com 형식의 와일드카드 서브도메인이어야 합니다.<br/>4. 도메인이 와일드카드 서브도메인이거나 다른 사용자가 액세스한 경우에는 [모든 권한 인증](#m1)을 한 후에 액세스 혹은 검색할 수 있습니다.<br/><br/><strong>주의 사항:</strong><br/>1. 와일드카드 서브도메인에 액세스한 후 다른 계정으로 서브 도메인 또는 서브 와일드카드 서브도메인에 액세스할 수 없습니다.<br/>2. *.test.com과 *.a.test.com의 동시 액세스를 지원하지 않습니다. |
| 서브 프로젝트 | 프로젝트란 Tencent Cloud의 모든 클라우드 서비스에 대한 일종의 공유 리소스를 의미하며, [프로젝트 관리](https://console.cloud.tencent.com/project)에서 프로젝트 관련 작업을 진행할 수 있습니다. |
| 가속 리전 | 중국 내: 글로벌 사용자가 액세스할 경우 중국 내 가속 노드로 스케쥴링하여 서비스를 제공합니다.<br/>중국 외: 글로벌 사용자가 액세스할 경우 중국 외 가속 노드로 스케쥴링하여 서비스를 제공합니다.<br/>글로벌: 글로벌 사용자가 액세스할 경우 가장 가까운 노드로 스케쥴링하여 서비스를 제공합니다.<br/><br/><strong>주의 사항: </strong><br/>중국 내와 중국 외 가속 서비스는 별도로 과금하며, 자세한 과금 정책은 클릭하여 [조회](https://intl.cloud.tencent.com/document/product/228/2949)하실 수 있습니다. |
| 서비스 유형 | Tencent Cloud CDN은 각각의 서비스 유형에 맞는 맞춤형 가속 성능 최적화를 실행하여,<br/>비즈니스에 더 적합한 서비스 유형을 선택해 더 나은 가속 효과를 얻을 수 있습니다.<br/><br/>정적 가속: 전자상거래, 웹 사이트 및 게임 이미지 등 소규모 리소스 가속 시나리오에 적합합니다.<br/>다운로드 가속: 게임 설치 패키지, 멀티미디어 소스 파일 다운로드, 휴대폰 펌웨어 배포 등의 다운로드 시나리오에 적합합니다.<br/>스트림 미디어 VOD 가속: 온라인 교육, 온라인 비디오 VOD 등 시나리오에 적합합니다. |
| 가속 프로토콜 | IPv4: 노드가 IPv4 액세스만 지원합니다.<br/>IPv4 + IPv6: 노드가 IPv4, IPv6 액세스를 동시에 지원하며, 해당 옵션을 선택했을 경우에만 IPv6 원본 서버를 설정할 수 있습니다.<br/><br/><strong>주의 사항:</strong><br/> IPv6는 중국 내에서만 지원합니다. <br/>

### Part2: 원본 서버 설정

서비스의 원본 서버 관련 정보를 설정하면, CDN 노드에 캐시 리소스가 없을 때 원본 서버에서 가져온 후 캐싱합니다.
![img](https://main.qcloudimg.com/raw/4a051cfc61dbd0e64b5ccfb8b386829d.png)

**설정 프로젝트 상세 설명: **

| 설정 프로젝트   | 설정 설명                                                     |
| -------- | ------------------------------------------------------------ |
| 원본 서버 유형 | 외부 원본 서버: 안정적으로 실행되는 비즈니스 서버(즉 원본 서버)를 보유한 상태의 시나리오.<br/><a href = "https://intl.cloud.tencent.com/product/cos">객체 스토리지(COS)</a>: 리소스가 Tencent Cloud COS에 저장되어 있다면 바로 bucket을 원본 서버로 사용할 수 있습니다. |
| 원본 서버 주소 | 외부 원본 서버: <br/>1. 여러 개의 IP를 원본 서버로 설정할 수 있으며, 원본 요청 시 폴링 방식의 원본 가져오기를 진행합니다.<br/>2. 멀티 IP 시나리오에서는 가중 방식의 원본 요청을 설정할 수 있습니다. 형식은 IP: 포트: 가중치(1 - 100)이며, 포트는 생략할 수 있습니다. 즉, IP::가중치입니다.<br/>3. 한 도메인을 원본 서버로 설정할 수 있으며, 해당 도메인은 서비스 가속 도메인과 이름이 같을 수 없습니다.<br/><br/>객체 스토리지(COS): <br/>1. 체크 박스를 드롭다운하여 원본 서버로 설정할 버킷을 선택합니다.<br/>2. 버킷이 개인 읽기/쓰기로 설정되어 있으면 버킷에 대한 CDN 서비스의 액세스 권한을 먼저 허용해야 합니다. 그러지 않을 경우 원본 가져오기에 실패할 수 있습니다. |
| 원본 프로토콜 | 원본 서버의 지원 상황에 따라 Origin-pull 프로토콜을 선택합니다:<br/>HTTP: HTTP / HTTPS 액세스가 모두 HTTP를 통해 원본을 가져옵니다.<br/>HTTPS: HTTP / HTTPS 액세스가 모두 HTTPS를 통해 원본을 가져옵니다(원본 서버가 HTTPS 액세스를 지원해야 함).<br/>프로토콜 따라가기: HTTP 액세스는 HTTP를 통해 원본을 가져오고, HTTPS 액세스는 HTTPS를 통해 원본을 가져옵니다(원본 서버가 HTTPS 액세스를 지원해야 함). |
| 원본 도메인 | 원본 도메인이란 CDN 노드가 원본을 가져올 때 원본 서버에서 액세스하는 사이트 도메인을 말합니다.<br/>서브 도메인 액세스 시 가속 도메인과 동일하도록 기본 설정되어 있으며, 사용자 정의 수정이 가능합니다.<br/>와일드카드 서브도메인 액세스 시 실제 액세스한 서브도메인과 동일하도록 기본 설정되어 있으며, 사용자 정의 수정이 가능합니다. |

## 액세스 완료

## 도메인의 모든 권한 인증
액세스한 도메인이 와일드카드 서브도메인이거나 다른 사용자가 액세스한 경우에는 도메인의 모든 권한을 인증한 후에 액세스 또는 검색할 수 있습니다. 도메인의 모든 권한 인증은 DNS 인증 방식으로, 인증 순서는 다음과 같습니다.

1. [인증 방식]을 클릭하여 DNS 인증 시 추가해야 하는 리졸브 기록 정보를 불러옵니다. 인증이 끝나기 전에 페이지를 닫지 마시기 바랍니다.
2. 도메인 리졸브 업체 측에서 TXT 형식의 DNS 기록을 1건 추가합니다. 호스트 기록에는 _cdnauth를 입력하고, 기록 값에는 인증 방법에 랜덤 생성된 값을 입력합니다.
3. TXT 리졸브가 적용되면 [인증] 버튼을 클릭하여 인증합니다. "인증 실패"라는 메시지가 뜰 경우, DNS 기록이 적용될 때까지 기다렸다가 다시 시도하시거나 TXT 기록을 정확하게 입력했는지 확인하시기 바랍니다.
