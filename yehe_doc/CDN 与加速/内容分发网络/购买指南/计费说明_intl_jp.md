# 課金説明

## 課金一覧

### 課金地域

Tencent Cloud Content Delivery Network（CDN）には、**中国国内**と**中国国外**の2つのサービス地域に分けられています。

中国国内の地域は、中国全域の一律課金となります。

中国国外の地域は、Tencent Cloud CDNノードサーバーの所在地に応じて、アジア太平洋地域リージョン1、アジア太平洋地域リージョン2、アジア太平洋地域リージョン3、中東、欧州、北米、南米、アフリカに分けられています。

-アジア太平洋地域リージョン1:中国香港、中国マカオ、シンガポール、ベトナム、タイ
-アジア太平洋地域リージョン2:中国台湾、日本、マレーシア、インドネシア、韓国
-アジア太平洋地域リージョン3:フィリピン、インド、オーストラリア、およびその他のアジア太平洋の国と地域

注記：

-中国国内と中国国外の各主要地域のCDN使用費用は、地域（主要地域）の単価と使用量に応じて個別に決済されます。

> 説明：
>
> - 各CDNノードはトラフィックデータをリアルタイムで記録し、コンピューティングセンターに報告し、ドメイン名の合計トラフィックデータとして集計します。帯域幅課金の場合は、5分の粒度データで決済されます。つまり、対応する帯域幅の値= 5分の粒度の合計トラフィック/ 300秒です。たとえば、5分以内に生成されたトラフィックの合計が30MBで、変換ルール「1MB = 8Mb」に従って、対応する帯域幅は（30 * 8）/ 300 = 0.8Mbpsとなります。
> -単位換算ルール：1GB = 1000MB、1 MB = 1000 KB、1Gbps = 1000Mbps、1Mbps = 1000Kbps。

### ピーク帯域幅課金

CDNの帯域幅課金は**段階的課金モデル**方式を採用しています。価格の段階は下記のとおりです。

### 計算方法

前日に中国大陸でのCDNのピーク帯域幅がXであり、中国国外の各主要地域でトラフィック使用が発生しなかった場合、段階計算方法は次のようになります。

1. X < 500 Mbpsの場合は、請求書の金額はX * 0.094となります。
2. 500Mbps ≤ X < 5000Mbpsの場合は、請求書の金額はX * 0.092となります。
3. 5000Mbps ≤ X < 50000Mbpsの場合は、請求書の金額はX * 0.086となります。
4. X ≥ 50,000Mbpsの場合は、請求書の金額はX * 0.084となります。

他の主要地域のCDN費用は、対象地域の単価と使用量に応じて個別に決済されます。

### 計算方法

トラフィック課金は帯域幅課金とは異なり、月度のトラフィックにより段階的に累進します。計算例は下記のとおりです。

- ユーザーが1月1日に中国大陸で3TBのトラフィックデータを使用して、中国国外の各主要地域ではトラフィックを使用しなかった場合、下図に示すように、灰色の部分は実際の課金段階となり、緑色の部分は1月1日に生成されたトラフィックで、そのうちの2TBが0TB～2TBの課金段階に該当され、残りの1TBが2TB～10TBの課金段階に該当するため、1月1日の実際の費用は、2 * 1000 * 0.037 + 1 * 1000 * 0.035となります。
  ![img](https://mc.qcloudimg.com/static/img/bfdae242f6cca57421a65e46a96b0c67/image.png)
- ユーザーが1月2日にも中国大陸で3TBのトラフィックデータを使用して、中国国外の各主要地域ではデータを使用しなかった場合、下図に示すように、トラフィック課金は月間累計となるため、1月2日に発生したトラフィックは2TB～10TBの課金段階に該当するため、1月2日の実際の費用は3 * 1000 * 0.035となります。
  ![img](https://mc.qcloudimg.com/static/img/f62d1056c1c2cab249cec62ad6e74ddc/image.png)
- ユーザーが1月3日に中国大陸で7TBのトラフィックデータを使用して、中国国外の各主要地域ではデータを使用しなかった場合、下図に示すように、1月3日の7TBのうち、4TBが2TB～10TBの課金段階に、3TBが10TB～50TBの課金段階に該当するため、1月3日の実際の費用は4 * 1000 * 0.035 + 3 * 1000 * 0.032となります。
  ![img](https://mc.qcloudimg.com/static/img/954e2d483e31afd411f9a91ebd7f66c8/image.png)

このように類推し、当月の1日分の費用を計算します。2月1日から新しい請求サイクルが始まります。　　

### 1日のピーク帯域幅は月間平均で課金

1.たとえば、ユーザーが1月1日から正式に課金され、対象地域の契約価格はP USD/Mbps/月となります。
2.有効日数：使用量＞0なら、課金対象日となります。
3. ユーザーの1月の課金対象日数が14日間の場合、この14日間毎日の288箇所の統計ポイントの最大値がMax_1、Max_2、Max_3... Max_14であり、課金帯域幅がAverage(Max_1, Max_2, ..., Max_14)で、1月のコストはAverage(Max_1, Max_2, ..., Max_14) * P * 14 / 31となります。

### 月95パーセンタイル帯域幅課金

1.たとえば、ユーザーが1月1日から正式に課金され、対象地域の契約価格はP USD/Mbps/月となります。
2.有効日数：使用量＞0なら、課金対象日となります。
3. ユーザーの1月の課金対象日数が14日間の場合、課金対象帯域幅は、14日間の課金対象日のすべての統計ポイント（14 * 288箇所）からトップ5％のポイントを除去して残った統計ポイントの中で最大がMax95となり、Max95は課金対象帯域幅となり、1月の費用は、Max95 * P * 14 / 31となります。

## 課金方式の選択

> 説明：
>
> ご使用中に、選択された課金方式が実際のビジネス状態に適さないことが判明した場合、課金方式を変更することができます。詳細について[課金方式の変更]（https://cloud.tencent.com/document/product/228/3077)をご参照ください。

**オプション：**
CDNサービスはお客様に**トラフィック課金**と**帯域幅課金**の2種類の課金方式を提供しています。お客様はご自身のビジネス状況に応じて適切な課金方式を選択することが可能です。

**計算例：**

たとえば、ユーザーは昨日00:00～24:00に中国大陸でのみトラフィックが発生して、トラフィックが200GBで、ピーク帯域幅が40Mbpsの場合、グラフは次のようになります。

![img](https://mc.qcloudimg.com/static/img/3ecfe86a031782ebeaf0b1f7595cc69f/image.png)

トラフィック課金方式の場合、支払い金額は200 * 0.037 = 7.4 USDとなります。

帯域幅課金方式の場合、支払い金額は40 * 0.094 = 3.76 USDとなります 

帯域幅課金はより費用対効果が高い、この例では帯域幅課金方式でのご利用をお勧めします。

## 支払い延滞説明

ご利用のTencent Cloudアカウントは支払い延滞状態にある時に、Tencent CloudはSMSやメール等の方法で支払い延滞を通知するとともに、**24時間**の待ち時間を保持します。24時間後にお客様のCDN加速サービスが停止されます。お客様はアカウントをチャージした後、CDNコンソールで手動で加速サービスをアクティブにする必要があります。

> 注意：
>
> 支払い延滞により加速サービスが停止された場合、お客様のすべてのドメイン名が無効になり、アクセスはオリジンサーバーに転送されます。CDNコンソールはクエリー操作のみをサポートし、設定を変更することはできません。お客様のCDN関連ドメイン名、設定情報が12ヶ月間保持されます。
