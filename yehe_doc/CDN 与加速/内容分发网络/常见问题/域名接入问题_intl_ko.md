### 도메인 이름은 어떻게 액세스하나요?
CDN 콘솔에서 도메인 이름에 액세스 할 수 있습니다. 자세한 정보는 [도메인 이름 액세스](https://intl.cloud.tencent.com/document/product/228/5734)를 참조 바랍니다.

### CDN은 와일드카드 서브도메인 액세스를 지원하나요?
CDN은 현재 와일드카드 서브도메인 액세스를 지원하지만 인증이 필요합니다. Tencent Cloud에서 제공한 인증 파일을 웹 사이트의 루트 디렉터리에 업로드하고 인증이 완료되면 와일드카드 서브도메인에 액세스할 수 있습니다.
기타:
1. `*.test.com`과 같은 와일드카드 서브도메인이 Tencent Cloud에 액세스했다면 해당 와일드카드 서브도메인의 어떤 서브도메인도 다른 계정에서 액세스할 수 없습니다.
2. 와일드카드 서브도메인 `*.test.com`에 액세스했다면 이 계정에서`*.path.test.com`과 같은 형식의 와일드카드 서브도메인에 액세스할 수 없습니다.

### CDN 설정에 얼마의 시간이 걸리나요?
CDN 설정 시간은 30분을 넘지 않으므로 설정 후 대기 시간이 30분을 초과했다면 [Submit Ticket][https://console.cloud.tencent.com/workorder/category]을 통해 문의하시기 바랍니다.

###  CDN의 도메인 이름에 액세스하려면 어떤 요구 사항이 있나요?
CDN 가속 중인 도메인 이름에 액세스하려면 공업정보부에 ICP비안을 등록해야 하며 원본 서버의 서비스 콘텐츠가 합법적이어야 합니다. 그렇지 않으면 도메인 이름에 액세스할 수 없습니다.

### 원본 서버 IP를 여러 개 설정할 수 있나요?
여러 개의 원본 서버 IP를 설정할 수 있습니다. 여러 개의 IP를 설정했다면 CDN은 원본 요청 시 입력한 IP 중 하나에 무작위로 액세스합니다. 어느 한 IP의 원본 가져오기 실패 횟수가 임계 값을 초과하면 해당 IP를 300초 동안 격리하고, 다시 원본을 가져오지 않도록 기본 설정되어 있습니다.

### 도메인 이름이 CDN에 액세스한 후 어떻게 CNAME을 바인딩하나요?
[CNAME 설정](https://intl.cloud.tencent.com/document/product/228/3121) 문서의 작업 설명을 참조하여 DNS 서비스 제공 업체를 통해 CNAME을 바인딩 할 수 있습니다.

### 도메인 이름 비활성화만 가능하고, 삭제할 수 없는 이유는 무엇인가요?
사용자에게 협력 권한이 있는지 확인하세요. 협력자의 작업 권한은 CDN 서비스 생성자가 설정하므로 생성자가 해당 권한을 지정하지 않으면 협력자는 작업할 수 없습니다. 권한을 받았음에도 해당 작업을 실행할 수 없다면 [Submit Ticket](https://console.cloud.tencent.com/workorder/category)을 통해 유지보수 담당자에게 문의하여 처리할 수 있습니다.

### 가속 서비스 비활성화 후 도메인 이름 설정은 저장되나요?
비활성화된 도메인 이름 설정은 저장되지만, 가속 서비스가 계속해서 제공되지 않으므로 이때부터는 사용자의 요청에 직접 원본 가져오기를 진행합니다.

### 가속 도메인 이름을 삭제한 후 도메인 이름 설정은 저장되나요?
도메인 이름이 삭제되면 해당 설정은 저장되지 않습니다.

### 가속 서비스를 비활성화하려면 어떻게 해야 하나요?
CDN 콘솔에서 가속 서비스를 비활성화할 수 있습니다. 자세한 작업 순서는 [가속 도메인 비활성화](https://intl.cloud.tencent.com/document/product/228/5736)를 참조 바랍니다.

### 가속 도메인 이름은 어떻게 삭제하나요?
CDN 콘솔에서 가속 도메인을 삭제할 수 있습니다. 자세한 작업 순서는 [가속 도메인 삭제](https://intl.cloud.tencent.com/document/product/228/5736)를 참조 바랍니다.

### 차단한 도메인 이름은 어떻게 해제하나요?
[Submit Ticket](https://console.cloud.tencent.com/workorder/category)을 통해 전문 유지보수 담당자가 처리해야 합니다.
