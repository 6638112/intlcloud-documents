### Tencent Cloud CDNノードでは、デフォルトのタイムアウト時間はどれぐらいですか。
Tencent Cloud CDNノードのデフォルトのタイムアウト時間は10秒です。

### CDNコンソールで接続されたドメイン名を無効にすると、CDNノード上のファイルはどうなりますか。
すでにCDNに接続されているドメイン名のアクセラレーションサービスを無効にすると、CDNノードは対応するドメイン名のアクセス設定を保持しますが、CDNトラフィックが発生しなくなり、このドメイン名もアクセスできなくなります。

### CDNに接続した後、ウェブサイトを開くことができない場合はどうすればよいですか。
まず、接続されたドメイン名のCDNステータスが「無効」になっているかどうかを確認します。ステータスが「無効」の場合、対応するページが開けられません。ステータスが「無効」でない場合は、次の手順に進みます。
+ pingまたはnslookupを実行して、該当ドメイン名のCNAME解決が有効になっているかどうかを確認します。CNAMEがまだ追加されていない場合は、[CNAMEの設定](https://intl.cloud.tencent.com/document/product/228/3121)ドキュメントの操作説明を参照して、DNSプロバイダーでCNAMEを追加してください。
+ CNAMEが有効になった後、オリジンサーバーに正常にアクセスできるかどうかを確認します。

上記の手順でもこの問題を解決できない場合は、[チケットを送信](https://console.cloud.tencent.com/workorder/category)してください。

### ユーザーがどのCDNノードにアクセスしているかを判断する方法は。
pingおよびnslookupコマンドを実行することにより、ユーザーがアクセスしているCDNノードのIPアドレス、遅延、パケット損失などの基本的なトラブルシューティング情報を取得することができます。

### ヒット率が低い理由は何ですか。
ヒット率が低いのは、次の原因が考えられます。
+ キャッシュの有効期限が短いなど、キャッシュ設定の問題。
+ HTTP Headerにより、キャッシュできなくなります。オリジンサーバーのCache-ControlまたはExpires設定を確認してください。
+ キャッシュ可能なコンテンツが少ないなど、オリジンサーバータイプの問題。
+ Webサイトへのアクセスが少なく、有効期限が短く、ファイルのヒット率が低いと、頻繁にback-to-originリクエストが発生します。

### ユーザーがCDNにアクセスすると接続が遅くなるのはなぜですか。
大きなファイルのダウンロード速度と小さなファイルの遅延を確認する必要があります。まずアクセスが遅いURLを取得し、速度測定ウェブサイトでアクセスが遅いかどうかを判断します(推奨ツール： [17ce](http://www.17ce.com) ）。
テスト接続が遅く、オリジンサーバーが外部オリジンである場合、[チケットを送信](https://console.cloud.tencent.com/workorder/category)して、ユーザーがオリジンサーバー機器の負荷と帯域幅が制限されているかどうか調査に協力してあげます。

### ユーザーアクセスリクエストがCDNキャッシュにヒットしたかどうかを確認するにはどうすればよいですか。
アクセス応答パケットヘッダーのX-Cache-Lookup情報を確認し、複数のX-Cache-Lookupが同時に返されるのが正常ですが、Cache Hit/Hit From MemCache/Hit From Discktankが返される時にのみ、CDNキャッシュがヒットしていることを示します。
![](https://mc.qcloudimg.com/static/img/64ac912c895b36f0241a927df6da3543/image.png)
+ X-Cache-Lookup:Hit From MemCacheはCDNノードのメモリがヒットしていることを示します。
+ X-Cache-Lookup:Hit From DisktankはCDNノードのディスクがヒットしていることを示します。

### ノードから返される同じ名前のファイルのサイズが異なるのはなぜですか。
すべてのファイルタイプがデフォルトではキャッシュされるため、CDNノードに異なるファイルバージョンが存在する可能性があります。この問題を解決するために：
+ファイルを強制的に更新し、キャッシュをすぐに更新します。
+ バージョン番号を付けます。例えば、```http://www.xxx.com/xxx.js?version=1```。
+ 同じファイル名を使用せずに、新しいファイル名に変更します。

上記の手順でもこの問題を解決できない場合は、[チケットを送信]（https://console.cloud.tencent.com/workorder/category）してください。


### CDNでリンク不正アクセス防止のホワイトリストが設定されると、ウェブサイトにアクセスできない場合はどうすればよいですか。

リンク不正アクセス防止を設定してホワイトリストを有効にするとき、【空のrefererを含める】にチェックを入れてください、ブラウザーでWebサイトに直接アクセスできるようになります（ブラウザーで直接アクセスする場合、refererは空になります）。

### トラフィックの上限設定でDDOS攻撃から防御できますか。

CDNは、DDoS保護ではなく、主にコンテンツ配信の高速化に焦点を当てています。CDN帯域幅上限設定機能を使用してみてください。帯域幅の使用状況は5分以内に自動的に統計されます。上限のしきい値に達した場合、CDNは設定に従って応答します。**しきい値は最大10000Tbpsまで設定可能です** 。

### Tencent Cloud CDNのすべてのノードのIPアドレスを提供できますか。 
セキュリティ上の理由から、CDNは完全なノードIPリストを提供できません。ノードのIPリージョン照会ページでIPリージョンを照会できます。詳細については、［ノードIP帰属クエリー](https://intl.cloud.tencent.com/document/product/228/10747)をご参照ください。
