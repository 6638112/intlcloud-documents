### 객체 액세스 시 다운로드하지 않고 바로 보기 하려면 어떻게 설정하나요?

사용자 외부 도메인을 바인딩합니다. 도메인 구매가 필요한 경우 Tencent Cloud를 통해 도메인을 등록하고 정적 웹 사이트 기능을 활성화합니다. 자세한 내용은 [정적 웹 사이트 설정](https://intl.cloud.tencent.com/document/product/436/14984)을 참고하십시오.

### COS 콘솔로 사용자 정의 도메인 설정에 실패하면 어떻게 처리하나요?

1. 도메인 ICP비안을 완료했는지 확인합니다.
2. 도메인의 DNS 리졸브가 정확한지 확인합니다(CDN 가속 비활성화 상태에서 사전에 DNS 리졸브 콘솔에서 도메인을 버킷의 [기본 도메인](https://intl.cloud.tencent.com/document/product/436/18424)에 CNAME합니다.

### 정적 웹 사이트 기능을 활성화해도 이미지가 보이지 않습니다.

1. COS 액세스 시, 객체(이미지) 바로 보기는 정적 웹 사이트 기능을 활성화해야 하며 객체 헤더의 Content-Disposition:inline 설정을 통해 구현할 수는 없습니다.  
2. 브라우저, CDN 캐시가 있는지 확인합니다. curl, wget 명령어로 브라우저 캐시를 스킵할 수 있으며 CDN 주소의 경우 [CDN 콘솔](https://console.cloud.tencent.com/cdn)에서 캐시 정리 작업을 할 수 있습니다. 

### 사용자가 외부 도메인 바인딩 시, CDN 가속을 활성화하는 것과 비활성화하는 것은 어떤 차이가 있나요?

- **CDN 가속 활성화: **CDN으로 도메인을 관리하며 COS 콘솔로 CDN 가속을 활성화하는 것과 [CDN 콘솔](https://console.cloud.tencent.com/cdn)에서 도메인(원본 서버 선택 COS)을 추가하는 것은 같은 효과를 냅니다. 사용자는 도메인 리졸브 시, CDN에서 할당한 CNAME 레코드를 사용해야 하며 설정 시에는 도메인을 먼저 추가하고 나서 리졸브합니다. 
- **CDN 가속 비활성화: **COS로 도메인을 관리하며 도메인 설정은 해당 버킷 소속 리전의 모든 다운로드 액세스 기기로 전달됩니다. 사용자는 도메인 리졸브 시, 버킷의 기본 도메인을 CNAME 레코드로 사용해야 합니다. 

### 객체의 사용자 정의 헤더 Content-Disposition를 설정했는데 적용이 안되는 이유는 무엇인가요?

다른 사용자 정의 헤더는 설정 후 바로 적용되지만 Content-Disposition 헤더는 정적 웹 사이트 기능을 활성화하고 사용자 정의 도메인을 사용해 액세스할 때만 적용됩니다. 


### 정적 웹 사이트 기능을 활성화해도 이미지가 보이지 않습니다.

브라우저, CDN 캐시가 있는지 확인합니다. curl, wget 명령어로 브라우저 캐시를 스킵할 수 있습니다. CDN 도메인 액세스를 사용하는 경우 [CDN 콘솔](https://console.cloud.tencent.com/cdn)에서 캐시 퍼지 작업을 할 수 있습니다. 

### 설정한 정적 웹 사이트에 CDN 도메인으로 액세스할 수 없을 때는 어떻게 하나요?

다음 단계에 따라 CDN 가속 도메인 설정을 확인하고 진단합니다.

1. 원본 서버 유형에서 원본 서버는 정적 웹 사이트 원본 서버를 선택해야 합니다.
2. Origin-pull 인증, CDN 서비스 권한은 버킷 권한에 맞게 설정해야 합니다. 
 - 버킷 권한의 상태가 개인 읽기인 경우, CDN 서비스에 대한 권한을 부여하고 Origin-pull 인증을 활성화해야 합니다.
 - 버킷 권한의 상태가 공개 읽기인 경우, CDN 서비스에 대한 권한을 부여하고 Origin-pull 인증을 활성화할 필요가 없습니다.
3. CDN 인증은 버킷 권한에 맞게 설정해야 합니다. 
 1. 버킷 권한이 개인 읽기인 경우:  
 <table>
	<tr><th>CDN 인증 설정</th><th>CDN 가속 도메인 액세스</th><th>COS 도메인 액세스</th><th>일반적인 시나리오</th></tr>
	<tr><td>비활성화(기본)</td><td>액세스 불가</td><td> COS 인증 사용 필요</td><td> CDN 도메인에 직접 액세스 하여 원본 서버 데이터 보호 가능</td></tr>
	<tr><td>활성화</td><td>URL 인증 사용 필요</td><td>COS 인증 사용 필요</td><td>전체 링크에서 액세스를 보호하고 CDN 인증 링크 도용 방지 지원</td></tr>
</table>
 2. 버킷 권한이 공개 읽기인 경우:  
<table>
	<tr><th>CDN 인증 설정</th><th>CDN 가속 도메인 액세스</th><th>COS 도메인 액세스</th><th>일반적인 시나리오</th></tr>
	<tr><td>비활성화(기본)</td><td>액세스 가능</td><td>액세스 가능</td><td>전역에 공개 액세스를 허용하고 CDN 또는 원본 서버 모두 액세스 가능</td></tr>
	<tr><td>활성화</td><td>URL 인증 사용 필요</td><td>액세스 가능</td><td>CDN 액세스에 링크 도용 방지를 활성화하나 원본 서버 액세스는 보호하지 않음. 권장하지 않음.</td></tr>
</table>
4. 이상의 설정에 문제가 없으면 CDN 가속 도메인에 사용한 프로토콜 및 정적 웹 사이트의 **강제 HTTPS**설정을 확인하십시오. 
 - CDN 가속 도메인 액세스에  HTTP 프로토콜을 사용하는 경우**강제 HTTPS** 옵션을 활성화하지 마십시오.
 - CDN 가속 도메인 액세스에 HTTPS 프로토콜을 사용하는 경우 CDN 가속 도메인에 **301/302 원본 정책 따르기 활성화** 설정을 권장합니다. 참고 문서: [원본 가져오기 301/302 따르기 설정](https://intl.cloud.tencent.com/zh/document/product/228/7183).
5. 이상의 단계에 따라 진단했는데도 문제를 해결할 수 없다면 [고객센터](https://intl.cloud.tencent.com/contact-sales)를 통해 더욱 자세한 진단이 가능합니다. 

### 정적 웹 사이트 기능을 프런트 엔드 Vue 프레임워크와 같이 사용하고 라우팅을 History 모드로 설정한 경우, 페이지 새로고침 시 404 문제가 발생하면 어떻게 해야 하나요?

버킷의 정적 웹 사이트 설정 페이지에서 오류 문서를 Web 애플리케이션의 게이트(일반적으로 index.html임)로 설정하고 오류 문서 응답 코드를 200으로 설정합니다. 정적 웹 사이트 관련 설정 가이드는 [정적 웹 사이트 설정](https://intl.cloud.tencent.com/document/product/436/14984)을 참고하십시오.


>! 상기 방법에 따라 설정을 완료한 후에 정상적인 404 응답 기능이 필요한 경우 직접 Vue의 프런트 엔드 라우팅에 가장 하위 레이어를 설정할 수 있습니다(일반적으로 와일드카드는 사용자 정의 404 모듈에 매칭됨).
