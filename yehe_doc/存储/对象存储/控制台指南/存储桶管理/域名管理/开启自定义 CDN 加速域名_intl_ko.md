## 소개
본 문서에서는 COS 콘솔에서 사용자 정의 가속 도메인 추가 및 CDN 가속 활성화에 관한 내용을 소개합니다. CDN 콘솔에서 사용자 정의 도메인을 추가하려면 CDN [도메인 액세스](https://intl.cloud.tencent.com/document/product/228/5734) 문서를 참조하십시오. 
>COS 콘솔에서 사용자 정의 도메인은 최대 10개까지 추가할 수 있습니다.

## 작업 순서

1. [COS 콘솔](https://console.cloud.tencent.com/cos5)에 로그인한 뒤, 왼쪽 메뉴에서 [버킷 리스트]를 클릭하여 버킷 리스트 페이지로 이동합니다.
2. 도메인을 설정할 버킷을 클릭하여 버킷 설정 페이지로 이동합니다.

3. 왼쪽의 [도메인 및 전송 관리] > [사용자 정의 CDN 가속 도메인] 설정 항목을 클릭한 뒤, 다시 [도메인 추가]를 클릭하여 다음과 같은 옵션을 설정합니다.
   **도메인**: 바인딩할 사용자 정의 도메인(예: ` www.example.com`)을 입력합니다. 이때 입력한 도메인의 ICP비안이 등록되었는지 확인하고, DNS 서비스 제공 업체로부터 CNAME을 설정하십시오. 자세한 내용은 [CNAME 설정](https://intl.cloud.tencent.com/document/product/228/3121)을 참조하십시오.
   **가속 리전**: 중국대륙 리전 가속, 중국홍콩 가속, GCD 및 글로벌 가속을 지원합니다. 글로벌 가속은 모든 리전 간 버킷의 가속을 지원합니다.
   **원본 서버 유형**: 기본 원본 서버와 정적 웹 사이트 원본 서버의 두 종류가 있습니다. 버킷에 정적 웹 사이트 기능을 활성화한 경우, 정적 웹 사이트 원본 서버로 설정됩니다. 반대의 경우, 기본 원본 서버로 설정됩니다. 사용자 정의 가속 도메인을 정적 웹 사이트로 실행할 경우, 정적 웹 사이트 원본 서버를 사용하고, 버킷의 정적 웹 사이트 기능을 활성화하십시오.
   **인증**: Origin-pull 인증을 활성화합니다. 개인 읽기 버킷의 경우, 원본 서버 보호를 위해 **Origin-pull 인증**을 활성화하십시오.
   ![](https://main.qcloudimg.com/raw/1278367aed8eada850c00fa4d5e18a4b.png)

>개인 읽기 상태인 버킷에 대해 Origin-pull 인증을 활성화하고 CDN 서비스 라이선스를 추가한 경우, CDN을 통해 원본 서버로 액세스할 때 서명 없이도 CDN 캐시 리소스가 공용 네트워크에 전송되어 데이터 보안에 취약할 수 있으니, 5단계에 따라 CDN 인증을 활성화하길 권장합니다.

4. 설정을 완료한 뒤, 오른쪽 작업 열의 [저장]을 클릭하면 도메인이 추가됩니다. 저장을 하면 **인증**란에 CDN 인증 버튼이 나타나며, 이때 사용자 정의 도메인의 CDN 인증을 활성화할 수 있습니다.
   **CDN 인증:** 타임스탬프 인증 설정을 활성화하면 악성 사용자의 콘텐츠 도용을 방지할 수 있습니다. 도메인을 추가한 뒤 설정하십시오.
5. [CDN 콘솔](https://console.cloud.tencent.com/cdn/access)에 로그인한 뒤, 왼쪽 메뉴에서 [도메인 관리]를 클릭합니다.
6. 설정할 도메인을 찾은 뒤, 오른쪽 작업 열에서 [관리]를 클릭합니다. 도메인 관리 페이지로 이동하여 상단의 [보안 설정]을 클릭합니다. 자세한 설정 단계는 [설정 설명](https://intl.cloud.tencent.com/document/product/228/35237) 문서의 인증 설정 부분을 참조하십시오.

>사용자가 도메인의 CDN 가속을 활성화하면 누구나 해당 도메인으로 원본 서버에 바로 액세스할 수 있으므로, **인증 설정**을 통해 기밀 유지가 필요한 원본 서버의 데이터를 보호하십시오.
