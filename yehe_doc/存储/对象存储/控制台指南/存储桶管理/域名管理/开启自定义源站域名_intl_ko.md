## 소개
본 문서에서는 사용자 정의 도메인을 버킷에 바인딩하고 사용자 정의 도메인으로 버킷 내 파일에 액세스하는 방법을 소개합니다.
>COS 콘솔에서 추가할 수 있는 사용자 정의 도메인은 최대 20개입니다. 20개를 초과하여 사용자 정의 도메인을 추가하려면 [티켓 제출](https://console.cloud.tencent.com/workorder/category)을 통해 문의하십시오.

## 작업 순서

1. [COS 콘솔](https://console.cloud.tencent.com/cos5)에 로그인한 뒤, 왼쪽 메뉴에서 [버킷 리스트]를 클릭하여 버킷 리스트 페이지로 이동합니다.
2. 도메인을 설정할 버킷을 클릭하여 스토리지 설정 페이지로 이동합니다.
![](https://main.qcloudimg.com/raw/41251dd19a92b7492106b7225f0871b1.png)
3. 왼쪽의 [도메인 관리]를 클릭하여 **사용자 정의 원본 서버 도메인**을 찾아 [도메인 추가]를 클릭합니다. 사용자 정의 도메인의 ICP비안이 공업정보부에 등록되었으며, [도메인 서비스 콘솔](https://console.cloud.tencent.com/cns/domains)에서 리졸브를 추가했다면, [도메인] 입력란에 사용자 정의 도메인을 직접 작성한 뒤 [저장]을 클릭합니다.
![](https://main.qcloudimg.com/raw/d0c76b2e57cd6a1b6d395bd7d4d21216.png)
4. 사용자 정의 도메인에 리졸브를 추가하지 않았다면, 다음 순서에 따라 사용자 정의 도메인을 추가합니다.
 1. [도메인 서비스 콘솔](https://console.cloud.tencent.com/cns/domains)에 로그인한 뒤 왼쪽 메뉴바에서 [도메인 리졸브 리스트]를 클릭하여 전체 도메인 페이지로 이동합니다. 다시 [리졸브 추가]를 클릭하면 리졸브 추가를 위한 대화 상자가 뜹니다.
 2. 사용자 정의 도메인을 입력하고, 프로젝트 옵션을 기본 프로젝트로 선택한 뒤 [확인]을 클릭하여 저장합니다. **특별한 경우가 아니라면 프로젝트 옵션을 기본 프로젝트로 선택합니다.**
 3. 도메인을 추가한 뒤 도메인을 클릭하여 리졸브 기록 관리 페이지로 이동합니다. [기록 추가]를 클릭하면 기록 추가를 위한 대화 상자가 뜹니다.
 8. 호스트 기록 시 안내에 따라 기록 유형은 CNAME을 선택하고, 회선 유형은 기본을 선택합니다. 기록값에는 버킷 도메인을 입력하고, TTL은 기본으로 유지합니다. 정보를 작성한 뒤 [저장]을 클릭합니다.
 5. 리졸브를 추가하면 적용되기까지 약 10분이 소요됩니다. 적용되면, 3번 순서에 따라 설정합니다.

## 신구 버전 콘솔의 차이점	

구버전 콘솔과 최신 버전 콘솔의 버킷 도메인은 다소 차이가 있습니다. 광저우 리전의 경우, 구버전 콘솔의 버킷 리전 약칭은 cosgz였으나 최신 버전 콘솔에서의 해당 약칭은 ap-guangzhou입니다. 도메인에 관한 자세한 내용은 [리전 및 액세스 도메인](https://intl.cloud.tencent.com/document/product/436/6224)을 참조하십시오.	

 도메인의 차이가 있으므로 사용자 정의 원본 서버 도메인을 사용할 때 CNAME 설정에 유의하십시오. 최신 버전 버킷 도메인으로 CNAME과 매핑하길 **권장**합니다. 최신 버전 도메인으로 CNAME과 매핑하면 구버전 버킷 도메인을 사용할 때보다 다양하게 작업할 수 있습니다.	

 DNS 벤더로부터 CNAME 정보를 수정할 수 있습니다.	

- DNS 벤더가 Tencent Cloud라면 [Tencent Cloud 리졸브 콘솔](https://console.cloud.tencent.com/domain)에서 설정하십시오.	
- DNS 벤더가 Alibaba Cloud라면 Alibaba Cloud 리졸브 콘솔에서 설정하십시오.	
- DNS 벤더가 AWS Cloud라면 AWS Cloud 리졸브 콘솔에서 설정하십시오.	

>구버전 콘솔과 최신 버전 콘솔은 도메인 외에 호출하는 API에도 차이가 있습니다. 풍부한 경험과 안정적인 기능을 제공하는 최신 버전 콘솔 및 최신 버전 API, 툴, SDK의 사용을 **권장**합니다. COS의 최신 버전 콘솔이 호출하는 작업 객체 또는 버킷의 API는 XML API인 반면, 구버전 콘솔은 JSON API로 호출합니다. 신구 버전 API의 차이에 관한 내용은 [API FAQ](https://cloud.tencent.com/document/product/436/30746)를 참조하십시오.
