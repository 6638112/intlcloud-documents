## 소개
COS 콘솔에서 버킷을 정적 웹 사이트 호스팅으로 설정할 수 있으며, 버킷의 액세스 도메인에 액세스하여 정적 웹 사이트로 액세스할 수 있습니다. 정적 웹 사이트에 관한 설명은 [정적 웹 사이트 호스팅](https://intl.cloud.tencent.com/document/product/436/30958)을 참조하십시오.

>
>- 버킷의 정적 웹 사이트 호스팅을 사용하려면 먼저 버킷의 액세스 권한을 **공개 읽기/개인 쓰기**로 설정해야 합니다.
>- 정적 웹 사이트 설정을 활성화한 뒤, 정적 웹 사이트 도메인으로 COS 원본 서버에 액세스해야 적용됩니다. COS 기본 도메인으로 액세스하면 정적 웹 사이트의 효과가 발생하지 않습니다.

## 전제 조건
버킷을 생성한 상태여야 합니다. 자세한 내용은 [버킷 생성](https://intl.cloud.tencent.com/document/product/436/13309) 문서를 참조하십시오.

## 작업 순서
1. [COS 콘솔](https://console.cloud.tencent.com/cos5)에 로그인한 뒤, 왼쪽 메뉴바에서 [버킷 리스트]를 클릭하여 정적 웹 사이트로 호스팅할 버킷을 클릭하면 버킷 상세 인터페이스로 이동합니다.

2. 왼쪽의 [권한 관리] > [버킷 액세스 권한]을 클릭한 뒤, 공용 권한 칸에서 **공개 읽기/개인 쓰기**를 선택하여 저장합니다.
![](https://main.qcloudimg.com/raw/460f2cdd71d3a21a74911a52218e7670.png)
3. 왼쪽의 [기본 설정]을 클릭한 뒤, 하단의 **정적 웹 사이트** 설정 항목을 찾아 [편집]을 클릭합니다. [현재 상태]의 활성화 버튼을 열어 정적 웹 사이트 설정 항목을 차례로 입력합니다. 다음은 설정 항목에 관한 설명입니다.
   **강제 HTTPS(선택 사항)**: 강제 HTTPS를 활성화하면 사용자가 정적 웹 사이트에 액세스할 때 정적 웹 사이트의 액세스 노드에서 HTTPS 프로토콜을 강제로 실행하여 액세스합니다.
   **html 확장명 무시(선택 사항)**: index 경로로 액세스하면 index.html 객체를 자동 매칭하여 반환합니다.
   **인덱스 문서(필수)**: 정적 웹 사이트의 초기 화면인 인덱스 문서는 웹 사이트의 루트 디렉터리나 서브 디렉터리로 요청을 보낼 때 반환되는 웹 사이트로서, 이 페이지의 이름은 보통 index.html이라고 칭합니다.
   
 >! 버킷에 폴더를 생성한 경우, 폴더의 모든 계층에 인덱스 문서를 추가해야 합니다.

  **에러 문서(선택 사항)**: 에러 문서는 정적 웹 사이트로 액세스할 때 에러가 발생하여 반환된 페이지입니다. 해당 설정 항목으로 에러 문서를 직접 정의할 수 있습니다. 정적 웹 사이트가 사용자의 요청에 응답하지 못할 때, 지정한 사용자 정의 에러 페이지를 반환하게 됩니다. 예를 들어, 이름이 error.html인 에러 문서를 설정한 경우, 액세스 시 HTTP 에러가 발생하면 페이지에서 error.html 페이지를 반환하여 도움말 가이드를 제공합니다. 에러 문서를 설정하지 않은 경우, 액세스 시 HTTP 에러가 발생하면 페이지에서 기본값으로 설정된 에러 정보를 반환합니다.
  
>! 에러 문서 설정 시 버킷의 루트 디렉터리 또는 서브 디렉터리의 하위 파일을 지원합니다. 브라우저에서 인식할 수 있는 `.html` 또는 `.htm` 등과 같은 형식의 파일을 사용하십시오. `.zip` 파일처럼 브라우저가 인식하지 못하는 파일은 대부분 브라우저에서 액세스할 수 없거나 액세스 요청을 거부한다는 에러 메시지가 표시됩니다.

 **에러 문서 응답 코드**: 에러 문서를 설정하면 해당 항목이 표시됩니다. 문서 에러 시 반환되는 HTTP 응답 코드로, 기존 에러 코드나 200을 설정할 수 있습니다.
 **리디렉션 규칙(선택 사항)**: 리디렉션 규칙을 적용하면 특정한 파일 경로나 요청 시 포함된 접두사, 응답 코드를 통해 조건에 따라 리디렉션을 요청할 수 있습니다.
예를 들어, 버킷의 특정 파일을 삭제하거나 파일 이름을 변경할 때, 리디렉션 규칙을 추가하여 해당 파일에 대한 액세스 요청을 다른 파일로 리디렉션할 수 있습니다.
 - 에러 코드: 현재의 리디렉션 규칙에서는 `4xx` 에러 코드(예: 404)에 대해서만 리디렉션을 설정할 수 있으며, 에러 페이지는 선택적으로 사용자 정의할 수 있습니다. 사용자가 상황에 따른 HTTP 에러를 트리거하면 에러 페이지에 기타 가이드 정보가 표시됩니다.
 - 접두사 매칭: 접두사 매칭 규칙을 적용하여 버킷 내 파일이나 폴더에 리디렉션을 설정할 수 있습니다. 자세한 예시는 [리디렉션 규칙 예시](https://intl.cloud.tencent.com/document/product/436/30958#.E9.87.8D.E5.AE.9A.E5.90.91.E8.A7.84.E5.88.99)를 참조하십시오.
	![](https://main.qcloudimg.com/raw/55ffdab17e3e7781e9d580dafc4c9969.png)
4. 설정을 완료하면 [저장]을 클릭합니다.
