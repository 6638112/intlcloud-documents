## 소개
버킷 버전 제어 기능을 활성화하면 버킷에 여러 버전의 객체를 저장하고 지정한 버전의 객체를 인덱스, 삭제 또는 복구할 수 있습니다. 해당 기능은 사용자가 잘못 삭제하거나 응용 프로그램 장애로 인한 데이터 손실을 복구하는 데 도움이 될 수 있습니다. 본 문서는 콘솔에서의 버킷 버전 제어 활성화 방법을 소개하며, 버전 제어에 대한 자세한 내용은 [버전 제어 개요](https://intl.cloud.tencent.com/document/product/436/19883)를 참조하십시오.

>!
>- 버킷의 버전 제어를 활성화하면 버전 제어 비활성화 상태(초기 상태)로 되돌릴 수 없습니다. 다만, 해당 버킷의 버전 제어를 일시 정지할 수는 있습니다. 이 경우 동일한 이름의 객체를 새로 업로드해도 여러 개의 버전이 생성되지 않습니다.
>- 버전 제어 활성화 후에 동일한 이름의 객체를 새로 업로드하면 여러 개의 버전이 생성되며 스토리지 용량을 차지하게 됩니다. 따라서 이러한 버전의 객체에도 동일하게 스토리지 사용 요금이 부과됩니다.

## 작업 순서
1. [COS](https://console.cloud.tencent.com/cos5) 콘솔에 로그인 후, 왼쪽 메뉴에서 [버킷 리스트]를 클릭하여 버킷 리스트 페이지로 이동합니다.
2. 버전 제어 설정이 필요한 버킷을 클릭하여 버킷 상세 인터페이스로 이동합니다.
3. 왼쪽 [내고장성 및 재해 복구 관리]>[버전 제어]를 클릭해 현재 상태를 활성화로 수정하고 저장한 후, 시스템에서 확인 팝업이 뜨면 [확인]을 클릭하여 버전 제어를 활성화합니다. 버전 제어 기능이 필요 없을 경우 현재의 '활성화' 버튼을 '비활성화'로 수정하면 됩니다.
4. 버전 제어 활성화 후 같은 이름의 파일을 해당 버킷에 업로드할 경우, 파일 리스트 인터페이스에서 [이전 버전 나열]을 클릭하면 다른 시점에 업로드한 같은 이름의 파일을 조회할 수 있으며, 파일 삭제 후에도 삭제된 파일 기록을 볼 수 있습니다. 삭제 전 이름이 같은 파일은 그대로 보관되어 있습니다. 버전에 따라 파일을 삭제하거나 복구할 수 있습니다.
