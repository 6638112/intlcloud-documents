## 소개

[COS 콘솔](https://console.cloud.tencent.com/cos5)에서 버킷의 리스트 기능을 활성화할 수 있습니다. 리스트 기능을 활성화하면 객체 속성, 설정 상세 정보 등에 관한 리스트 보고서를 매일 또는 매주 귀하의 버킷에서 확인할 수 있습니다. 리스트에 대한 세부 사항은 [리스트 기능 개요](https://intl.cloud.tencent.com/document/product/436/30622)를 참조하십시오. 다음은 버킷 리스트 기능 활성화 방법입니다.

> !
>- 사용자는 하나의 버킷에 다수의 리스트 작업을 설정할 수 있습니다.
>- 리스트 작업 실행 중에는 객체의 내용은 읽지 않고 객체의 메타데이터와 같은 속성 정보만 스캔합니다.
>- 현재 리스트 기능은 금융 클라우드 리전은 지원하지 않습니다.

## 전제 조건

생성된 버킷에 대한 세부 사항은 [버킷 생성](https://intl.cloud.tencent.com/document/product/436/13309) 문서를 참조하십시오.

## 작업 순서

1. [COS 콘솔](https://console.cloud.tencent.com/cos5)에 로그인합니다.
2. 왼쪽 메뉴에서 [버킷 리스트]를 클릭한 후 리스트 기능을 활성화하고 싶은 버킷(원본 버킷)을 클릭합니다.
3. 왼쪽 [기본 설정]>[리스트 설정] 클릭 후 [리스트 추가]를 클릭합니다.
   ![](https://main.qcloudimg.com/raw/73aa7b3f19389dfdd42d4066e6437329.png)
4. **리스트 추가**페이지에서 다음과 같은 설정이 가능합니다.
  - **리스트 이름**: 출력 리스트 이름을 입력합니다.
  - **타깃 버킷**: 리스트를 저장하는 버킷입니다. 기본 설정된 타깃 버킷은 원본 버킷이며, 타깃 버킷은 반드시 원본 버킷과 같은 리전에 있어야 합니다.
 - **리스트 보고서 접두사(선택 사항)**: 타깃 버킷용 접두사를 입력하면 공용 위치의 매니페스트 파일을 그룹 나누기하여 기본값으로 초기화할 수 있습니다.
 - **상태**: 활성화/비활성화 리스트에서 선택할 수 있습니다.
 - **[고급 설정]**: 고급 설정에서 더 많은 리스트 정보를 설정할 수 있습니다. 고급 설정을 하지 않으면 기본값으로 설정됩니다.
   - **출력 포맷**: 기본적으로 CSV 포맷으로 출력됩니다.
   - **객체 버전**: 리스트에 객체 버전의 포함 범위를 선택합니다(모든 버전 포함/현재 버전만 포함). 선택하지 않을 경우 기본적으로 현재 버전만 포함합니다.
   - **생성 주기**: 리스트를 내보낼 주기를 선택합니다(매일/매주). 선택하지 않을 경우 기본적으로 매일 내보내게 됩니다.
   - **접두사 필터링(선택 사항)**: 필터링을 통해 객체 키 접두사가 동일한 객체를 리스트에 넣습니다. 입력하지 않을 경우 기본적으로 필터링되지 않습니다.
   - **시간 필터링(선택 사항)**: 필터링을 통해 수정 시간이 일정 시간 이후 또는 일정 기간 이내로 설정된 객체를 리스트에 넣습니다. 설정하지 않을 경우 기본적으로 필터링되지 않습니다.
   - **리스트 암호화**: 선택한 리스트의 서버 암호화 여부를 선택할 수 있습니다.
     - 비암호화: 리스트를 암호화 하지 않을 경우의 기본 항목입니다.
     - SSE-COS: COS 호스팅 키의 서버 암호화를 사용하여 보고서를 암호화합니다. 더 많은 정보는 객체 스토리지 개발자 가이드의 [SSE-COS 암호화](https://intl.cloud.tencent.com/document/product/436/18145)를 참조하십시오.
   - **리스트 정보**: 객체 크기, 스토리지 유형, ETag, 버킷 복사 상태, 멀티파트 업로드 상태, 최신 업데이트 날짜와 같은 리스트 보고서에 포함해야 하는 객체의 정보를 선택할 수 있습니다. 선택하지 않을 경우 기본값은 전체 선택입니다.
     ![](https://main.qcloudimg.com/raw/a219e0eb29661c254f0a70f094cf1ae0.png)

> !ETag(실체 태그)는 객체의 해시입니다. ETag는 객체의 내용 변경만 반영할 뿐, 객체의 메타데이터 변경은 반영하지 않습니다. ETag가 반드시 객체 데이터의 MD5 요약은 아니며, 이는 객체의 생성 방식과 암호화 방식에 따라 결정됩니다.

5. 설정 정보에 오류가 없음을 확인한 후 [저장]을 클릭합니다. COS는 앞으로 매일 또는 매주 자동으로 매니페스트 파일을 생성하여 귀하가 설정한 타깃 버킷으로 전송하게 됩니다.
>?리스트 기능이 생성한 리스트 보고서 포맷과 내용의 세부 사항은 [리스트 기능 개요](https://intl.cloud.tencent.com/document/product/436/30622)를 참조하십시오.
