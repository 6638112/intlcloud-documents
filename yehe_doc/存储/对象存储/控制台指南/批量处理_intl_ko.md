## 소개

COS 일괄 프로세스 기능으로 버킷에 저장된 객체에 대해 대규모 일괄 작업을 실현할 수 있습니다. 현재 다음 작업에 대해 일괄 프로세스 작업을 진행할 수 있습니다.

- 객체 복사
- 보관된 객체 복구

처리할 객체를 객체 매니페스트 파일에 정리합니다. 해당 매니페스트 파일은 리스트 기능으로 생성된 리스트 보고서(먼저 [리스트 기능 활성화](https://intl.cloud.tencent.com/document/product/436/30624) 필요)이거나 사용자가 지정한 포맷에 따라 직접 생성할 수도 있습니다. COS 일괄 프로세스 기능은 해당 객체 매니페스트 파일에 따라 일괄 프로세스를 진행합니다. COS의 일괄 프로세스 작업 사용에 대한 자세한 정보는 [일괄 프로세스 개요](https://intl.cloud.tencent.com/document/product/436/32958)를 참조하십시오.

## 작업 순서

1. [COS 콘솔](https://console.cloud.tencent.com/cos5)에 로그인한 후, 왼쪽 메뉴에서 [일괄 프로세스]를 클릭해 일괄 프로세스 작업 관리 인터페이스로 이동합니다.
2. [작업 생성]을 클릭하여 일괄 프로세스 작업 생성을 시작합니다. 설정 항목 설명은 다음과 같습니다.
 - **작업 리전**: 작업을 생성할 리전을 선택합니다. 작업 리전은 사용자 리스트의 처리 대기 중인 객체의 버킷 기전과 동일해야 하며, 그렇지 않을 경우 실패합니다.
 >?현재 일괄 프로세스 기능은 중국대륙 공유 클라우드 리전에서만 지원되며 다른 리전에서는 지원되지 않습니다.
 - **리스트 포맷**: 리스트 객체에 사용할 유형을 선택하며, 다음과 같은 두 가지 포맷이 있습니다.

 - **리스트 버킷**: 리스트가 있는 버킷을 선택합니다.
 - **매니페스트 파일 경로**: 매니페스트 파일 또는 CSV 파일이 있는 경로를 선택합니다. 포맷은 각각 json과 csv로, 예를 들어 리스트를 버킷 `examplebucket-1250000000`의 루트 디렉터리에 저장한 경우 리스트 경로는 `manifest.json`이 됩니다. 
![](https://main.qcloudimg.com/raw/5544edff70879dbee24dc2e3fef0c36e.png)
3. [다음 단계]를 클릭하여 작업 설정 페이지로 이동한 후, 작업 유형을 선택합니다. 설정 항목 설명은 다음과 같습니다.
  - **데이터 일괄 복사**:
    - 타깃 버킷: 리스트 리스트에 있는 객체를 복사하여 붙여넣을 버킷입니다.
    - 접두사 작업: 복사한 객체에 접두사를 추가, 변경 및 삭제할 수 있습니다.
    - 스토리지 유형: 복사한 객체에 스토리지 유형을 설정합니다. 표준 스토리지, 표준IA 스토리지, Cloud Archive Storage(CAS)를 선택할 수 있습니다.
    - 서버 암호화: 복사한 객체를 암호화할지 여부를 선택합니다. 암호화하지 않음, SSE-COS 암호화를 선택할 수 있습니다.
    - 액세스 권한: 복사한 객체에 액세스 권한을 설정합니다. 모든 권한 복사, 모든 권한 변경, 신규 권한 추가를 선택할 수 있습니다.
    - 객체 메타데이터: 복사한 객체에 메타데이터를 설정합니다. 모든 메타데이터 복사, 모든 메타데이터 변경, 신규 메타데이터 추가를 선택할 수 있습니다.
    - 객체 태그: 복사한 객체에 객체 태그를 설정합니다. 모든 태그 복사, 모든 태그 변경, 신규 태그 추가를 선택할 수 있습니다.
  - **CAS 일괄 복구**:
    - 복구 모드: 표준 모드와 일괄 모드를 선택할 수 있으며, 복구 모드에 대한 자세한 내용은 [보관된 객체 복구](https://intl.cloud.tencent.com/document/product/436/30961)를 참조하십시오.
    - 사본 유효 기간: 사본이 몇 일 후 자동 만료되어 삭제할지 설정하며, 설정 범위는 최소 1일 최대 365일입니다.
      ![](https://main.qcloudimg.com/raw/4a51de0288564cb70ac5249523637e3e.png)
4. [다음 단계]를 클릭하여 기타 설정 페이지로 이동한 후, 다음과 같은 관련 설정을 작성합니다.
 - **작업 설명(선택 사항)**: 해당 항목 작업에 대한 설명으로 작성하지 않아도 됩니다.
 - **작업 우선순위**: 우선순위가 높은 작업을 우선적으로 실행하며, 0을 제외한 자연수를 입력합니다. 숫자가 클수록 우선순위가 높아집니다.
 - **작업 보고서**: 작업 보고서 생성 여부를 선택합니다.
 - **CAM 역할**: CAM 역할을 생성하거나 기존의 CAM 역할을 선택하여 COS에 작업 권한을 부여합니다.
![](https://main.qcloudimg.com/raw/3a45c1180a241468084fcff59f1018e8.png)
> !일괄 프로세스 작업은 CAM 역할 생성 방식을 통해 COS에 일괄 프로세스 권한을 부여해야 하며, CAM 역할에 대한 자세한 사항은 [CAM 역할 개요](https://intl.cloud.tencent.com/document/product/598/19420)를 참조하십시오.
5. [다음 단계]를 클릭하여 정보 확인 페이지로 이동한 후, 작성한 일괄 프로세스 작업 설정을 확인합니다. 수정이 필요한 경우 해당 [수정] 또는 [이전 단계]를 클릭하고, 오류가 없는지 최종 확인한 후 [완료]를 클릭합니다.
![](https://main.qcloudimg.com/raw/2f84d4dafa2586e6ca9aacae5d5b7d47.png)
6. 일괄 작업을 생성한 후, 작업 리스트에서 방금 생성한 작업을 찾아 [확인]을 클릭합니다. 작업을 취소해야 하는 경우 오른쪽에 있는 [작업 취소]를 클릭합니다.
![](https://main.qcloudimg.com/raw/94aed43dc8ca6c02d3149fe12ee3883d.png)
