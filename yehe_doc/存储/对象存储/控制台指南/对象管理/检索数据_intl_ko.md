## 소개

COS Select 기능으로 스토리지 레벨에서 필요한 데이터를 선별하여 COS 전송 데이터양 최소화로 사용 비용을 줄이고 데이터 확보 효율을 높입니다. COS 콘솔에서 기본 SQL 템플릿을 사용하거나 구문 규칙에 부합하는 명령을 직접 입력하여 스토리지 버킷에 있는 단일 파일 콘텐츠를 인덱스합니다.

>?
>
현재 인덱스 기능은 json、csv, parquet 파일 포맷의 객체를 지원합니다. 그 중 json、csv 포맷은 중국대륙 공유 클라우드 리전에서만 지원되며 parquet 포맷은 베이징 리전에서만 지원됩니다.
>- 인덱스한 파일이 COS 규칙에 부합하는지 확인하십시오. COS Select 규칙 관련 정보를 알고 싶다면 [Select 개요](https://intl.cloud.tencent.com/document/product/436/32472)를 참조하십시오.
>- 콘솔의 인덱스 기능은 최대 128MB 파일을 지원하며 40M 이하 데이터를 추출합니다. 용량이 더 큰 파일을 처리하거나 더 많은 데이터를 추출하고 싶다면 [API](https://intl.cloud.tencent.com/document/product/436/32360)나 SDK를 사용하십시오.

## 작업 순서

1. [COS 콘솔](https://console.cloud.tencent.com/cos5)에 로그인합니다.
2. 왼쪽 메뉴에서 [버킷 리스트]를 클릭합니다.
3. 버킷 이름을 클릭해 객체에 있는 버킷으로 이동합니다.
4. 버킷의 “파일 리스트” 모듈에서 인덱스 작업이 필요한 객체를 찾고 오른쪽의 [더 많은 조작]>[인덱스]를 클릭합니다.
 >?현재는 표준 스토리지, 표준IA 스토리지, INTELLIGENT TIERING 객체만 인덱스 지원됩니다.
5. 콘솔 인덱스 기능 페이지로 이동하여 실제 상황에 따라 인덱스가 필요한 파일 유형, 제목 필드, 세퍼레이터, 압축 포맷, 내보내기 포맷을 선택합니다.
   ![](https://main.qcloudimg.com/raw/b6390a741fc0b3dd6880752ed78d7ca5.png)
6. [SQL 템플릿 선택]을 클릭하여 인덱스가 필요한 템플릿 문형을 선택할 수 있으며 [확인]을 클릭합니다.
   ![](https://main.qcloudimg.com/raw/0b3886fd501962d7eff5efd0b3bb4a89.png)
7. 실제 파일에 따라 텍스트 박스에서 명령 편집을 할 수 있으며 편집을 마친 후 [SQL 실행]을 클릭합니다.
   ![](https://main.qcloudimg.com/raw/b269aa12cd63d838559642b60f0f598c.png)
8. 실행을 마친 후 최하단에 있는 텍스트 박스에서 이전 결과 100건을 바로 조회할 수 있습니다. 완전한 데이터를 얻고 싶다면 [인덱스 결과 내보내기]를 클릭하십시오.
