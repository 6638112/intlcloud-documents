## 소개

COS 콘솔로 버킷의 파일 리스트 페이지에 객체를 업로드할 수 있습니다. 객체 관련 설명은 [객체 개요](https://intl.cloud.tencent.com/document/product/436/13324)를 참조하십시오.

> ?
> - 현재 INTELLIGENT TIERING 유형은 베이징, 상하이, 광저우, 충칭 리전만 지원합니다. 해당 유형을 업로드할 경우에는 우선 대응되는 지역의 버킷을 선택하세요 [INTELLIGENT TIERING 설정 활성화](https://intl.cloud.tencent.com/document/product/436/38306).
> - DEEP ARCHIVE 유형은 베이징, 상하이, 광저우, 청두 리전만 지원합니다.

## 전제 조건

객체 업로드 전 생성된 버킷을 확보하십시오. 생성된 버킷이 없을 경우 [버킷 생성](https://intl.cloud.tencent.com/document/product/436/13309) 문서를 참조해 작업을 진행하십시오.

## 작업 순서

1. [COS 콘솔](https://console.cloud.tencent.com/cos5)에 로그인하여 [버킷 리스트]를 클릭해 버킷 리스트로 이동합니다. 저장할 객체의 스토리지를 선택하고 스토리지의 파일 리스트 페이지로 이동합니다. 파일 리스트에서 [파일 업로드]를 클릭합니다.
![](https://main.qcloudimg.com/raw/1f499717168f5559337899f635e19a13.png)

2. 파일 업로드 화면에서 [파일 선택] 혹은 [폴더 선택]을 클릭하면 단일 혹은 다수의 로컬 파일(혹은 폴더)를 선택해 업로드할 수 있습니다. 선택 완료 후에는 [업로드] 혹은 [다음 단계]를 클릭해 객체를 설정한 후 업로드할 수 있습니다(3단계 참조).
![](https://main.qcloudimg.com/raw/8489d2bae2ba778bac87386093cd51e7.png)

3. (옵션) 업로드 대기 파일의 스토리지 유형, 액세스 권한, 서버 암호화, 객체 태그, 메타데이터(선택 입력)을 설정한 후 [업로드]를 클릭합니다. 설정 설명은 다음과 같습니다.
- **스토리지 유형**: 여러 응용 시나리오에 따라 각 객체마다 스토리지 유형을 설정할 수 있습니다. 기본 스토리지 유형은 표준 스토리지입니다. 스토리지 유형의 소개는 [스토리지 유형](https://intl.cloud.tencent.com/document/product/436/6222?lang=en&pg=#storage-classes)을 참조하십시오.

- **액세스 권한**: 각 객체에 여러 액세스 권한을 설정할 수 있습니다. 기본 액세스 권한은 상속 권한입니다(즉 상속 버킷 권한). 액세스 권한에 관한 더 많은 정보는 [액세스 제어의 기본 개념](https://intl.cloud.tencent.com/document/product/436/30581)을 참조하십시오.
- **서버 암호화**: 객체를 업로드하는 동시에 객체의 서버 암호화 속성을 설정할 수 있습니다. Tencent Cloud COS는 업로드한 객체에 데이터 암호화 보호 정책을 추가하여 데이터 입력 전 자동 암호화를 도우며, 해당 데이터 액세스 시 자동으로 암호화를 합니다. 현재는 SSE-COS, SSE-KMS(베이징, 상하이, 광저우 리전만 지원)를 지원합니다. 자세한 정보는 [서비스 암호화 개요](https://intl.cloud.tencent.com/document/product/436/18145)를 참조하십시오.
- **객체 태그**: 객체 태그는 태그 키(tagKey), 태그 값(tagValue)이 “=”로 연결되어 구성됩니다. 예: group = IT. 지정 객체의 태그 설정, 조회, 삭제 작업을 할 수 있습니다.
- **메타데이터**: 객체 메타데이터는 서버가 HTTP 프로토콜로 HTML 데이터를 브라우저에 전송하기 전에 보내는 문자열로, HTTP Header라고도 부릅니다. HTTP Header 수정으로 페이지 응답 방식을 변경하거나 캐시 시간 수정 등 정보를 전달할 수 있습니다. 객체의 HTTP Header 수정 시에도 객체는 수정되지 않습니다. 자세한 정보는 [객체 사용자 정의 Headers](https://intl.cloud.tencent.com/document/product/436/13361)를 참조하십시오.
  ![](https://main.qcloudimg.com/raw/0b7c7b6297a1098fe43074f85a138fbf.png)

> ?일부 브라우저는 파일 업로드를 지원하지 않으므로 IE10 이상, Firefox, Chrome 등 주요 브라우저를 권장합니다.

4. [업로드]클릭 후 페이지 오른쪽 위의 “작업 완료”에서 현재 업로드 진행도를 조회할 수 있으며, 업로드 완료 후 버킷의 “파일 리스트” 페이지에서 방금 업로드한 객체를 조회할 수 있습니다.
![](https://main.qcloudimg.com/raw/eab5784b108fc096dbe317fed25f7925.png)

> ?이미지의 작업 진행도는 현재 작업에 생성된 작업 개수 통계입니다. 예를 들어 한 작업에 10개의 파일을 업로드하고 모두 업로드에 성공한 경우에는 해당 임무 진행도가 “성공 1개, 실패 0개, 일시 정지 0개”로 표시됩니다.
