## 소개

COS 콘솔로 버킷의 파일 리스트 페이지에 객체를 업로드할 수 있습니다. 객체 관련 설명은 [객체 개요](https://intl.cloud.tencent.com/document/product/436/13324)를 참고하십시오.

>?
> - 현재 인텔리전트 티어링 유형은 베이징, 상하이, 광저우, 충칭, 도쿄, 싱가포르 리전만 지원합니다. 해당 유형을 업로드하려면 우선 해상 리전의 버킷 [INTELLIGENT TIERING 설정](https://intl.cloud.tencent.com/document/product/436/38306)을 활성화하십시오.
> - 현재 딥 아카이브 스토리지 클래스는 베이징, 난징, 상하이, 광저우, 청두, 충칭 및 도쿄 지역에서만 사용할 수 있습니다. 이 스토리지 클래스에 객체를 업로드하려면 먼저 리전에서 버킷을 선택하십시오.
> 

## 전제 조건

객체 업로드 전 생성된 버킷을 확보하십시오. 생성된 버킷이 없을 경우 [버킷 생성](https://intl.cloud.tencent.com/document/product/436/13309) 문서를 참고해 작업을 진행하십시오.

## 작업 단계

1. [COS 콘솔](https://console.cloud.tencent.com/cos5)에 로그인합니다.
2. 왼쪽 사이드바에서 **버킷 리스트**를 클릭합니다.
3. 대상 버킷을 클릭하여 파일 목록 페이지로 이동합니다.
4. 파일 목록에서 **파일 업로드**를 클릭합니다.
![](https://main.qcloudimg.com/raw/1f499717168f5559337899f635e19a13.png)
5. 팝업 창에서 **파일 선택** 또는 **폴더 선택**을 클릭하고 필요에 따라 하나 이상의 로컬 파일(또는 폴더)을 선택합니다.
>? 일부 브라우저는 파일 업로드를 지원하지 않으므로 IE10 이상, Firefox, Chrome 등 주요 브라우저를 권장합니다.
>
![](https://main.qcloudimg.com/raw/8489d2bae2ba778bac87386093cd51e7.png)
6. (옵션) **매개변수 설정**을 클릭하고 파일 업로드 창에서 객체 속성을 설정합니다.
 ![](https://main.qcloudimg.com/raw/0b7c7b6297a1098fe43074f85a138fbf.png)
설정은 다음 내용을 참고하십시오.
 - **스토리지 유형**: 필요에 따라 객체의 스토리지 클래스를 선택합니다. 이 필드는 기본적으로 표준 스토리지로 설정됩니다. 자세한 내용은 [스토리지 유형](https://intl.cloud.tencent.com/document/product/436/6222)을 참고하십시오.
 - **액세스 권한**: 각 객체에 여러 액세스 권한을 설정할 수 있습니다. 기본 액세스 권한은 상속 권한입니다(즉 상속 버킷 권한). 액세스 권한에 관한 더 많은 정보는 [액세스 제어 기본 개념](https://intl.cloud.tencent.com/document/product/436/30581)을 참고하십시오.
 - **서버 암호화**: 객체를 업로드하는 동시에 객체의 서버 암호화 속성을 설정할 수 있습니다. Tencent Cloud COS는 업로드한 객체에 데이터 암호화 보호 정책을 추가하여 데이터 입력 전 자동 암호화를 도우며, 해당 데이터 액세스 시 자동으로 암호화를 합니다. 현재는 SSE-COS, SSE-KMS(베이징, 상하이, 광저우 리전만 지원)를 지원합니다. 자세한 정보는 [서비스 암호화 개요](https://intl.cloud.tencent.com/document/product/436/18145)를 참고하십시오.
 - **객체 태그**: 객체 태그는 태그 키(tagKey), 태그 값(tagValue)이 “=”로 연결되어 구성됩니다. 예: group = IT. 지정 객체의 태그 설정, 조회, 삭제 작업을 할 수 있습니다.
 - **메타데이터**: 객체 메타데이터는 서버가 HTTP 프로토콜로 HTML 데이터를 브라우저에 전송하기 전에 보내는 문자열로, HTTP Header라고도 부릅니다. HTTP Header 수정으로 페이지 응답 방식을 변경하거나 캐시 시간 수정 등 정보를 전달할 수 있습니다. 객체의 HTTP Header 수정 시에도 객체는 수정되지 않습니다. 자세한 정보는 [객체 사용자 정의 Headers](https://intl.cloud.tencent.com/document/product/436/13361)를 참고하십시오.
7. **업로드**를 클릭합니다.
페이지 오른쪽 상단의 ‘완료한 작업’에서 업로드 진행 상황을 확인할 수 있습니다. 업로드가 완료되면 업로드된 객체가 ‘파일 목록’에 나타납니다.
![](https://main.qcloudimg.com/raw/eab5784b108fc096dbe317fed25f7925.png)
>? 이미지의 작업 진행률은 현재 업로드 작업에 의해 생성된 작업의 수를 나타냅니다. 예를 들어 10개의 파일이 모두 성공적으로 업로드되면 작업 진행률이 ‘성공 1개, 실패 0개, 일시 중지 0개’로 표시됩니다.
>
