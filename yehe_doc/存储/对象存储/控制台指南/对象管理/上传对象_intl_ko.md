## 소개

COS 콘솔을 통해 버킷 파일 리스트 페이지에서 객체를 업로드할 수 있습니다. 객체 관련 설명은 [객체 개요](https://intl.cloud.tencent.com/document/product/436/13324)를 참조하십시오.

> - 현재 INTELLIGENT TIERING 유형은 베이징, 상하이, 광저우, 충칭 리전에서만 지원합니다. 이 유형을 업로드해야 할 경우 버킷 [INTELLIGENT TIERING 활성화 설정](https://intl.cloud.tencent.com/document/product/436/38306)을 참조하십시오.
> - DEEP ARCHIVE 유형은 베이징, 상하이, 광저우, 청두 리전에서만 지원합니다.

## 전제 조건

객체 업로드 전 버킷이 생성되었는지 확인하십시오. 버킷이 생성되지 않았다면 [버킷 생성](https://intl.cloud.tencent.com/document/product/436/13309) 문서를 참고하여 작업하십시오.

## 작업 순서

#### 1. 파일 리스트로 이동

 [COS 콘솔](https://console.cloud.tencent.com/cos5)에 로그인하고 [버킷 리스트]를 클릭하여 버킷 리스트로 이동하십시오. 객체 저장이 필요한 버킷을 선택하고 버킷 파일 리스트 페이지로 이동하십시오. 파일 리스트에서 [파일 업로드]를 클릭하면 파일 업로드 인터페이스가 팝업되어 아래 그림처럼 보여집니다.
![](https://main.qcloudimg.com/raw/1f499717168f5559337899f635e19a13.png)

#### 2. 업로드 객체 선택

파일 업로드 인터페이스에서 [파일 선택] 혹은 [폴더 선택]을 클릭하면 단일 혹은 여러 로컬 파일(또는 폴더)을 선택 업로드할 수 있습니다. 선택 완료 후 [업로드]나 [다음 단계]를 클릭하여 객체 속성 설정 후 다시 업로드합니다.(3단계 참고)
![](https://main.qcloudimg.com/raw/8489d2bae2ba778bac87386093cd51e7.png)

#### 3. 객체 속성 설정(선택 사항)

업로드할 파일의 스토리지 유형, 액세스 권한, 서버 암호화, 객체 태그, 메타데이터(옵션)를 설정한 후 [업로드]를 클릭합니다. 설정 설명은 아래와 같습니다.

- **스토리지 유형**: 다양한 응용 시나리오에 따라 다양한 객체에 다양한 스토리지 유형을 설정할 수 있으며 스토리지 유형은 기본적으로 표준 스토리지입니다. 스토리지 유형 소개에 대한 자세한 내용은 [스토리지 유형](https://intl.cloud.tencent.com/document/product/436/6222)을 참조하십시오.

- **액세스 권한**: 다양한 객체에 다양한 액세스 권한을 설정할 수 있으며 액세스 권한은 기본적으로 권한 상속(즉, 버킷 권한 상속)으로 설정됩니다. 액세스 권한에 대한 자세한 내용은 [액세스 제어의 기본 개념](https://intl.cloud.tencent.com/document/product/436/30581)을 참조하십시오.
- **서버 암호화**: 객체를 업로드하며 객체의 서버 암호화 속성을 설정합니다. Tencent Cloud COS는 업로드하는 객체 애플리케이션 데이터 암호화의 보호 정책을 위해 데이터 입력 전 자동 암호화를 진행하며 해당 데이터에 액세스 할 때 자동 복호화합니다. 현재는 SSE-COS、SSE-KMS(베이징, 상하이, 광저우 리전에서만 지원)를 지원하며 자세한 내용은 [서버 암호화 개요](https://intl.cloud.tencent.com/document/product/436/18145)를 참조하십시오.
- **객체 태그**: 객체 태그는 태그 키(tagKey)와 태그 값(tagValue), “=”가 연결 구성됩니다. 예시로 group = IT가 있습니다. 지정 객체에 태그 설정, 조회, 삭제 작업을 진행할 수 있습니다.
- **메타데이터**: 객체 메타데이터는 서버가 HTTP 프로토콜로 HTML 자료를 브라우저에 보내기 전 전송하는 문자열로 HTTP Header라고도 불립니다. HTTP Header 수정을 통해 페이지 응답 형식을 수정하거나 설정 정보를 전달할 수 있습니다. 캐시 시간 수정이 대표적인 예시입니다. 객체의 HTTP Header 수정으로 객체 자체 수정은 불가하며 상세 정보는 [사용자 정의 객체 Headers](https://intl.cloud.tencent.com/document/product/436/13361)를 참조하십시오.
  ![](https://main.qcloudimg.com/raw/0b7c7b6297a1098fe43074f85a138fbf.png)

> ?일부 브라우저는 여러 파일 업로드를 지원하지 않으며 IE10 이상, Firefox, Chrome 등 주요 브라우저 사용을 권장합니다.

#### 4. 업로드 완료 확인

[업로드]를 클릭한 후 페이지 오른쪽 상단에 있는 “작업 완료”에서 현재 업로드 진도와 업로드 완료 여부를 조회할 수 있으며 업로드 완료 후에는 버킷 “파일 리스트” 페이지에서 방금 업로드한 객체를 조회할 수 있습니다.
![](https://main.qcloudimg.com/raw/eab5784b108fc096dbe317fed25f7925.png)

> ?그림에서 보이는 작업 진도는 현재 작업 시 만들어지는 작업 개수 통계 수치를 보여줍니다. 예시로 단일 작업으로 파일 10개를 업로드 완료하면 작업 진도에서 “성공 10개”, “실패 0개”, “일시중지 0개”라는 문구를 볼 수 있습니다.
