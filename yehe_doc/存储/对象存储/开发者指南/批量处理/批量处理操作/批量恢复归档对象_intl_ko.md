보관된 객체 일괄 복구 작업은 리스트 상의 Cloud Archive Storage(CAS) 유형의 객체를 복구하는데 사용합니다. 해당 작업은 POST Object restore 관련 매개변수를 사용자 지정 설정할 수 있으며, 해당 설정 정보는 복사본의 복구 시간 및 만료 시간에 영향을 미칩니다. POST Object restore에 대한 자세한 내용은 [POST Object restore](https://intl.cloud.tencent.com/document/product/436/12633)를 참고하십시오.

보관된 객체 일괄 복구 작업을 생성할 경우 다음 두 가지 매개변수를 지정해야 합니다.

- 복구 모드: 표준 모드와 일괄 모드로 나뉘며, 복구 모드에 대한 자세한 내용은 [보관된 객체 복구](https://intl.cloud.tencent.com/document/product/436/30961)를 참조하십시오.
- 복사본 유효기간: 아카이브 유형의 객체는 복구 후 임시 복사본이 생성되며, 해당 복사본은 지정된 시간 후 자동으로 만료되어 삭제됩니다. 복사본 유효기간에 대한 자세한 내용은 [보관된 객체 복구](https://intl.cloud.tencent.com/document/product/436/30961)를 참조하십시오.

## 주의사항

1. 보관된 객체 일괄 복구 작업에 이미 복구된 객체 복사본이 포함되어 있는 경우 해당 객체의 복사본 유효기간을 업데이트하여 작업 중 모든 복사본 객체의 유효기간을 동일하게 만듭니다.
2. 일괄 복구 작업은 객체 복구 요청에만 사용할 수 있습니다. 모든 요청 발송 완료 후 일괄 프로세스 페이지에 해당 작업 진행 상황이 표시됩니다. COS에서는 언제 복구 작업이 완료되었는지는 통지하지 않으며, 이벤트 알림 설정을 통해 알림을 수신할 수 있습니다. 자세한 내용은 [이벤트 알림](https://intl.cloud.tencent.com/document/product/436/31648)을 참조하십시오.
