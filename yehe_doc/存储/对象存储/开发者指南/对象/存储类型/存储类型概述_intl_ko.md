스토리지 유형은 COS에서 객체의 스토리지 레벨과 활성화 정도를 나타냅니다. 액세스 빈도의 높고 낮음과 재해 복구 정도에 따라, COS는 인텔리전트 티어링 스토리지, 표준 스토리지, 표준IA 스토리지, 아카이브 스토리지, 딥 아카이브 스토리지의 다양한 객체의 스토리지 유형을 제공합니다. 스토리지 유형별로 각각 객체 액세스 빈도, 데이터 지속성, 데이터 가용성 및 액세스 딜레이 시간 등의 특성을 가집니다. 사용자는 시나리오에 따라 스토리지 유형을 선택해 데이터를 COS에 업로드할 수 있습니다.

> !
>
> - 객체를 업로드할 때, 스토리지 유형을 설정하지 않으면 스토리지 유형은 **표준 스토리지**로 기본 적용됩니다.


## 아카이브 스토리지

아카이브 스토리지(ARCHIVE)는 콜드 데이터 유형에 속합니다. 데이터 검색 시 사전 복구(동결 해제)가 필요하며, 높은 신뢰성, 저렴한 비용, 장기간 저장이 장점인 객체 스토리지 서비스를 제공합니다. 아카이브 스토리지 최소 저장 기간은 90일입니다.

복구 작업은 세 가지 모드를 지원합니다.

- 고속 모드: 1~5분 내 복구 작업 완료.
- 표준 모드: 3~5분 내 복구 작업 완료.
- 대량 모드: 5~12분 내 복구 작업  완료.                      

> ? 
> - 복구 작업 가이드는 [보관된 객체 복구](https://intl.cloud.tencent.com/document/product/436/30961)를 참고하십시오.
> - 데이터 복구 요청에는 QPS 제한이 있으며, 100회/초로 제한됩니다.

**적용 시나리오**

아카이브 스토리지는 아카이브 데이터, 의료 영상, 과학 자료 등 컴플라이언스 파일 보관, 라이프사이클 파일 보관, 작업 로그 보관, 원격 재해 복구 등 데이터를 장기간 저장해야 하는 시나리오에 적합합니다.

> !아카이브 스토리지는 저장 시간 및 저장 단위에 최소한의 제한이 있으며, 저장 시간이 90일 미만일 경우, 90일을 기준으로 계산됩니다. 단일 저장 파일이 64KB 미만이면 64KB로 계산되며, 64KB보다 크거나 같을 경우에는 실제 크기로 계산됩니다. 가격 정보는 [COS 가격](https://intl.cloud.tencent.com/pricing/cos)을 참고하십시오.

## 딥 아카이브 스토리지

딥 아카이브 스토리지(DEEP_ARCHIVE)는 사용자에게 다른 스토리지 유형에 비해 신뢰성이 높고 비용이 저렴한 COS 장기 스토리지 서비스입니다. 딥 아카이브의 최소 저장 기간은 180일입니다. 데이터 읽기 전에 먼저 데이터를 복구해야 합니다. 딥 아카이브에 대한 자세한 내용은 [딥 아카이브 소개](https://intl.cloud.tencent.com/document/product/436/38304)를 참고하십시오.

복구 작업은 두 가지 모드를 지원합니다.

- 표준 모드: 복구 시간 12~24시간 소요.
- 대량 모드: 복구 시간 24~48시간 소요. 

> ? 데이터 복구 요청에는 QPS 제한이 있으며, 100회/초로 제한됩니다.

**적용 시나리오**

딥 아카이브는 의료 영상, 보안 모니터링 데이터, 로그 데이터와 같은 장기 저장이 필요한 비즈니스 시나리오에 사용됩니다.

> !딥 아카이브는 저장 시간 및 저장 단위에 최소한의 제한이 있으며, 저장 시간이 180일 미만일 경우, 180일을 기준으로 계산됩니다. 단일 저장 파일이 64KB 미만이면 64KB로 계산되며, 64KB보다 크거나 같을 경우 실제 크기로 계산됩니다. 가격 정보는 [COS 가격](https://intl.cloud.tencent.com/pricing/cos)을 참고하십시오.

## 스토리지 유형의 전환

COS는 객체의 스토리지 레벨과 활성화 정도를 나타내는 다양한 객체 스토리지 유형을 제공합니다. 실제 사용 도중 사용자는 실제 필요에 따라 객체의 스토리지 유형을 변경하거나 객체를 활성화 정도가 낮은 스토리지 유형(예: 표준IA, 아카이브 또는 딥 아카이브)으로 전환할 수 있습니다.

>?
>- 스토리지 유형 전환 시, 먼저 현재 객체의 버킷이 해당 스토리지 유형을 지원하는지 확인하십시오.
>- 객체가 **아카이브/딥 아카이브** 유형에 속하면 표준 스토리지 유형으로 복구되어야 기타 스토리지 유형으로 변경할 수 있습니다. 자세한 내용은 [보관된 객체 복구](https://intl.cloud.tencent.com/document/product/436/30961)를 참고하십시오.
