스토리지 유형은 COS에서 객체의 스토리지 레벨과 활성화 정도를 나타냅니다. 액세스 빈도의 높고 낮음에 따라, COS는 INTELLIGENT TIERING, 표준 스토리지, 표준IA 스토리지, CAS, DEEP ARCHIVE 등 다양한 객체의 스토리지 유형을 제공합니다. 저장 유형별로 각각 객체 액세스 빈도, 데이터 지속성, 데이터 가용성 및 액세스 딜레이 시간 등의 특성을 가집니다. 사용자는 시나리오에 따라 스토리지 유형을 선택해 데이터를 COS에 업로드할 수 있습니다.

>!
>객체를 업로드할 때, 스토리지 유형을 설정하지 않으면 기본 스토리지 유형은 **표준 스토리지**가 됩니다.

## 표준 스토리지

표준 스토리지(STANDARD)는 핫 데이터 유형으로, 낮은 액세스 딜레이 시간, 높은 처리량 등 성능으로 사용자에게 높은 신뢰성과 고가용성, 고성능의 COS 서비스를 제공합니다.

**적용 시나리오**

표준 스토리지는 모두 대량의 실시간 액세스가 많은 파일이나 데이터 인터랙션이 잦은 인기 비디오, 소셜 이미지, 모바일 애플리케이션, 게임 프로그램, 정적 웹 사이트 등의 비즈니스 시나리오에 적합합니다.

표준 스토리지(STANDARD)는 대부분의 사용 시나리오를 커버하므로 범용 스토리지 유형에 속합니다.


## 표준IA 스토리지

표준IA 스토리지(STANDARD_IA)는 사용자에게 높은 신뢰성, 저렴한 스토리지 비용 및 짧은 액세스 대기 시간을 자랑하는 COS 서비스를 제공합니다. 스토리지 단가를 낮춰 최초 바이트 액세스 시간을 밀리초 단위 내로 유지하고, 사용자에게 데이터 검색 시나리오에서 짧은 액세스 딜레이 시간과 고속 읽기를 보장합니다. 표준 스토리지와 명확히 구분되는 점은 사용자가 데이터에 액세스할 때 데이터 검색 비용이 과금된다는 것입니다.

**적용 시나리오**

표준IA 스토리지는 모두 클라우드 데이터, 빅 데이터 분석, 정부 및 기업 비즈니스 데이터, 저빈도 아카이브, 데이터 모니터링 같이 액세스 빈도가 낮은(예: 월평균 액세스 빈도 1~2회) 비즈니스 시나리오에 적합합니다.

>표준IA 스토리지 유형은 저장 시간 및 저장 단위에 최소한의 제한이 있으며, 저장 시간이 30일 미만일 경우, 30일을 기준으로 계산됩니다. 단일 저장 파일이 64KB 미만이면 64KB로 계산되며, 64KB보다 크거나 같을 경우에는 실제 크기로 계산됩니다. 가격 정보는 [COS 가격](https://intl.cloud.tencent.com/pricing/cos)을 참조하십시오.

## INTELLIGENT TIERING

INTELLIGENT TIERING 유형의 객체는 표준 스토리지 레이어와 표준IA 스토리지 레이어 두 개의 스토리지 레이어에 저장할 수 있습니다. COS는 INTELLIGENT TIERING 유형 객체의 액세스 빈도수에 따라 자동으로 두 개의 스토리지 레이어를 전환하며, 데이터 검색 과금이 없어 사용자는 스토리지 비용을 절감할 수 있습니다. INTELLIGENT TIERING에 대한 자세한 내용은 [INTELLIGENT TIERING 소개](https://intl.cloud.tencent.com/document/product/436/38305)를 참조하십시오.

**적용 시나리오**

INTELLIGENT TIERING은 데이터 액세스 모드가 고정되지 않은 시나리오에 적합합니다. 비즈니스가 비용에 대한 요구가 엄격하고 파일 읽기 성능에 민감하지 않을 경우, 이 스토리지 유형을 사용해 비용을 낮출 수 있습니다.

> !INTELLIGENT TIERING은 저장 시간 및 저장 단위에 최소한의 제한이 있으며, 저장 시간이 30일 미만일 경우, 30일을 기준으로 계산됩니다. 단일 저장 파일이 64KB 미만이면 64KB로 계산되며, 64KB보다 크거나 같을 경우에는 실제 크기로 계산됩니다. 가격 정보는 [COS 가격](https://buy.cloud.tencent.com/price/cos)을 참조하십시오.

## CAS

CAS(ARCHIVE)는 콜드 데이터 유형으로 데이터 검색 시 사전 복구(동결 해제)가 필요하며, 사용자에게 높은 신뢰성, 매우 낮은 스토리지 비용 및 장기 저장을 실현하는 COS 서비스를 제공합니다. CAS는 최소 90일 동안 저장되며, 데이터를 읽기 전에 먼저 데이터를 복구(동결 해제)해야 합니다.

복구 작업은 세 가지 모드를 지원합니다.

- 고속 모드: 복구 작업 1~5분 내 완료
- 표준 모드: 복구 작업 3~5분 내 완료
- 대량 모드: 복구 작업 5~12분 내 완료                      

> ? 
> - 복구 작업 가이드는 [보관된 객체 복구](https://intl.cloud.tencent.com/document/product/436/30961)를 참조하십시오.
> - 데이터 복구 요청에는 QPS 제한이 있으며, 100회/초로 제한됩니다.

**적용 시나리오**

CAS는 아카이브 데이터, 의료 영상, 과학 자료 등 컴플라이언스 파일 보관, 라이프사이클 파일 보관, 작업 로그 보관 및 원격 재해 복구 등 장기 데이터 저장이 필요한 비즈니스 시나리오에 적합합니다.

> !CAS는 저장 시간 및 저장 단위에 최소한의 제한이 있으며, 저장 시간이 90일 미만일 경우, 90일을 기준으로 계산됩니다. 단일 저장 파일이 64KB 미만이면 64KB로 계산되며, 64KB보다 크거나 같을 경우에는 실제 크기로 계산됩니다. 가격 정보는 [COS 가격](https://intl.cloud.tencent.com/pricing/cos)을 참조하십시오.

## DEEP ARCHIVE

DEEP ARCHIVE는 신뢰성이 높고 다른 스토리지 유형에 비해 비용이 저렴한 장기 스토리지를 사용자에게 제공하는 COS 서비스입니다. DEEP ARCHIVE는 최소 180의 저장 기간이 필수이며, 데이터를 읽기 전에 먼저 데이터를 복구해야 합니다. DEEP ARCHIVE에 대한 자세한 내용은 [DEEP ARCHIVE 소개](https://intl.cloud.tencent.com/document/product/436/38304)를 참조하십시오.

복구 작업은 두 가지 모드를 지원합니다.

- 표준 모드: 복구 시간 12~24시간
- 대량 모드: 복구 시간 24~48시간 

> - 데이터 복구 요청에는 QPS 제한이 있으며, 100회/초로 제한됩니다.

**적용 시나리오**

DEEP ARCHIVE는 의료 영상, 보안 모니터링 데이터, 로그 데이터와 같은 장기 저장이 필요한 비즈니스 시나리오에 사용됩니다.

> !DEEP ARCHIVE는 저장 시간 및 저장 단위에 최소한의 제한이 있으며, 저장 시간이 180일 미만일 경우, 180일을 기준으로 계산됩니다. 단일 저장 파일이 64KB 미만이면 64KB로 계산되며, 64KB보다 크거나 같을 경우 실제 크기로 계산됩니다. 가격 정보는 [COS 가격](https://intl.cloud.tencent.com/pricing/cos)을 참조하십시오.
