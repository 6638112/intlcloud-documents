TencentDB for MySQL는 데이터 스토리지의 높은 보안성을 보장합니다. 저장 과정 중 데이터의 기밀성과 완전성을 보장합니다.

## [자동 백업](id:ZDBF)
TencentDB for MySQL은 자동 백업과 수동 백업을 모두 지원하여 데이터의 완전성과 신뢰성을 보장하는 데이터 복원성을 보장합니다. MySQL은 기본적으로 데이터 백업과 로그 백업 기능을 제공하며, 자동 백업 주기는 한 주에 2회 이하로 설정할 수 없습니다. 기타 백업이 필요할 경우, 콘솔 혹은 API를 통해 언제든지 수동 백업을 실행할 수 있습니다.

이 기능을 사용하는 방법에 대한 자세한 내용은 [자동 백업 설정](https://intl.cloud.tencent.com/document/product/236/37796)을 참고하십시오.

## [아카이브 백업 보관](id:DQBFBL)
TencentDB for MySQL을 사용하면 아카이브 백업 보관을 활성화할 수 있습니다. 필요에 따라 백업 파일의 보관 기간을 유연하게 구성할 수 있습니다. 이 기간은 기본적으로 7일이며 최대 1830일까지 가능합니다. 보관 기간을 초과한 백업 파일은 자동으로 삭제됩니다. 데이터 보안을 위해 일주일에 두 번 이상 데이터를 백업하는 것이 좋습니다.

이 기능을 사용하는 방법에 대한 자세한 내용은 [정기 백업 설정 활성화](https://intl.cloud.tencent.com/document/product/236/37796)를 참고하십시오.

## [Transparent Data Encryption](id:TMSJJM)
TencentDB for MySQL는 Tencent Cloud 데이터베이스 팀에서 개발한 Transparent Data Encryption(TDE) 기능을 지원합니다. 투명한 암호화는 데이터 암호화 및 암호 해독이 사용자에게 감지되지 않음을 의미합니다.  암호화된 테이블을 생성할 때 암호화 키를 지정할 필요가 없으며 디스크에 쓰는 동안 데이터는 암호화되고 디스크에서 읽는 동안 해독됩니다.

TDE는 국제적으로 통용되는 AES 알고리즘과 256비트 암호화 키를 사용하며, Tencent Cloud [Key Management Service](https://intl.cloud.tencent.com/document/product/1030/31961)가 관리합니다. KMS 서비스에 액세스하려면 액세스 권한이 있어야 하며 KMS 콘솔에서 키를 교체하여 시스템 보안을 더욱 강화할 수 있습니다.

이 기능을 사용하는 방법에 대한 자세한 내용은 [TDE 활성화](https://intl.cloud.tencent.com/document/product/236/38491)를 참고하십시오.

## [데이터 보안](id:SJAQ)
**민감한 데이터 감지** 기능은 식별 규칙을 통해 민감한 인스턴스 데이터를 자동으로 감지하고 감지된 민감한 데이터에 대해 자동 분류 보호를 구현할 수 있습니다. 컴플라이언스 요구 사항을 충족하면서 모든 데이터 세트 필드에서 민감한 속성을 식별하는 20개의 기본 제공 규칙을 지원하여 긴 텍스트에 숨겨진 민감한 데이터가 적절하게 처리될 수 있도록 합니다. 또한 개인화된 데이터 보호 요구 사항을 충족하기 위해 사용자 지정 유형의 민감한 데이터 자동 검색을 지원합니다.

**데이터 마스킹** 기능에는 다양한 비즈니스 시나리오에서 마스킹 작업을 지능적으로 실행하고 관리하여 핵심 엔터프라이즈 데이터를 기밀로 유지하는 많은 고급 마스킹 알고리즘이 내장되어 있습니다. 처리 후 원시 데이터의 배포 또는 형식을 변경하지 않으면서 보안을 보장하여 통계 분석, 테스트, 개발 및 트레이닝에 효과적인 데이터 사용을 보장합니다.
- 컴플라이언스 측면에서 데이터 마스킹은 익명화 기술을 통해 마스킹된 데이터를 복원할 수 없도록 할 수 있습니다. 이는 기업이 개인 정보를 사용하는 동안 해당 법률 및 규정을 엄격하게 준수하고 고객의 컴플라이언스 요구 사항을 충족할 수 있도록 합니다.
- 시스템 성능 측면에서 데이터 마스킹은 데이터 마스킹 작업 시작할 때 최신 실시간 백업 데이터를 가져와 감지할 수 없는 방식으로 온라인에서 동적으로 작업을 완료합니다. 소스 데이터베이스를 중지하거나 데이터베이스 성능을 손상시키거나 추가 오버헤드를 발생시키지 않습니다.
