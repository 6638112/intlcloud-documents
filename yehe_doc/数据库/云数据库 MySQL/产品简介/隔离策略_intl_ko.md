
본 문서는 TencentDB for MySQL의 기본형, 범용형, 전용형 격리 정책을 소개합니다.

>?
>- 구 기본 버전 아키텍처는 '단일 노드-기본형'으로 업그레이드되고, 구 단일 노드 고속 IO는 '단일 노드-범용형'으로 업그레이드됩니다.
>- 이중 노드와 3중 노드 아키텍처는 범용형 및 전용형 격리 정책을 지원합니다. 전용형 격리 정책은 현재 베타 단계이므로 구매가 필요한 경우 [Submit Ticket](https://console.cloud.tencent.com/workorder/category)을 통해 문의하십시오.

| 격리 정책 | 설명                                                         |
| -------- | ------------------------------------------------------------ |
| 기본형   | 단일 노드만 기본형 격리 정책(구 기본 버전)을 지원합니다. 컴퓨팅과 스토리지가 분리되며 기본 레이어는 클라우드 스토리지를 사용합니다.
| 범용형   | <li>전용 할당된 메모리 및 디스크는 동일한 베어메탈의 기타 범용 규격 인스턴스와 CPU 리소스를 공유합니다. <li>리소스 재사용을 통해 '규모의 경제' 효과가 나타나면, 가성비가 높아지고 CPU 리소스 재사용률은 낮아집니다. <li>디스크 크기는 CPU, 메모리와 바인딩되지 않기 때문에 효율적인 설정이 가능합니다. |
| 전용형   | <li>전용 CPU 바인딩, 메모리, 디스크 리소스는 장기간 안정적인 성능을 유지하므로 베어메탈의 다른 인스턴스 동작에 영향을 받지 않습니다.  <li>전용형 최고 사양은 베어메탈을 독점하여 단독으로 베어메탈 한 대의 모든 리소스를 사용할 수 있습니다. |
