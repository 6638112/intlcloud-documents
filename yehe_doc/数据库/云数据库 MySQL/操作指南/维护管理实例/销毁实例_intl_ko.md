
## 개요
서비스 수요에 따라 사용자는 콘솔에서 종량제 인스턴스 및 정액 과금제 인스턴스를 직접 반환할 수 있습니다.
- 정액 과금제 인스턴스 반환 후 인스턴스는 CDB 휴지통에 7일 보관되며, 해당 기간 동안 액세스할 수 없습니다.
- 종량제 인스턴스 반환 후 인스턴스는 CDB 휴지통에 24시간 보관되며, 해당 기간 동안 액세스할 수 없습니다.

직접 반환하여 인스턴스 상태가 **격리 중**이 되면 더 이상 해당 인스턴스 관련 요금이 발생하지 않습니다.
>!
>- 인스턴스를 폐기하면 데이터를 복구할 수 없으며, 백업 파일이 동기화 폐기되어 클라우드에서 데이터를 복구할 수 없게 됩니다. 사전에 백업 파일을 전송하시기 바랍니다.
>- 해당 인스턴스에 관련 읽기 전용 또는 재해 복구 인스턴스가 있는 경우, 인스턴스가 폐기된 후 IP 리소스가 동시에 릴리스됩니다.
>  - 읽기 전용 인스턴스는 동시에 폐기됩니다.
>  - 재해 복구 인스턴스는 동기화 연결을 끊고 자동으로 마스터 인스턴스로 업그레이드합니다.
>- 인스턴스 폐기 후 환불 처리:
>  - 5일 이내 무조건 환불 정책을 충족하는 인스턴스의 경우 결제 금액이 귀하의 Tencent Cloud 계정으로 반환됩니다.
>  - 보통 직접 환불 신청한 금액은 구매 시 사용된 현금 및 크레딧의 결제 비율에 따라 Tencent Cloud 계정으로 환불됩니다.
>  - 프로모션 리워드 채널을 통한 주문의 경우 실제 현금 결제 금액의 25%가 환불됩니다. 현재 이러한 종류의 주문은 셀프 환불이 불가능하며, [Submit Ticket](https://console.cloud.tencent.com/workorder/category)을 통해 환불을 신청할 수 있습니다.
> 

## 작업 단계
1. [TencentDB for MySQL 콘솔](https://console.cloud.tencent.com/cdb/)에 로그인한 후 인스턴스 리스트에서 원하는 인스턴스를 선택한 다음 **작업** 열에서 **더 보기** > **폐기/반품** 또는 **폐기/반품 및 환불**을 선택합니다.
2. 팝업 창에서 ‘규정을 읽고 동의했습니다’를 읽고 선택한 후 **즉시 폐기**를 클릭합니다.
>?
>- 정액 과금제 인스턴스 폐기를 선택하고 폐기 버튼을 사용할 수 없는 경우는, 계정이 정액 과금제 직접 환불 할당량을 모두 사용했음을 의미하며 정액 과금제 인스턴스를 수동으로 폐기할 수 없으며 만료 후 자동으로 폐기됨을 나타냅니다.
>- 인스턴스 일괄 폐기를 지원하며, 인스턴스 목록에서 폐기할 인스턴스를 여러 개 선택한 후 상단의 **더 많은 작업** > **폐기/반환**을 선택합니다.
