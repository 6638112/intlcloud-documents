본문은 TencentDB for MySQL 콘솔의 인스턴스 리스트 및 관리 페이지에 대한 작업을 설명합니다.

## 인스턴스 리스트 페이지
[TencentDB for MySQL 콘솔](https://console.cloud.tencent.com/cdb)에 로그인하여 인스턴스 리스트 페이지에 들어가 인스턴스 정보를 확인하고 인스턴스를 관리할 수 있습니다.
![](https://staticintl.cloudcachetci.com/yehe/backend-news/DfYa317_3.png)

| 기능 | 설명 |
|---------|---------|
| 로그인 | 인스턴스 리스트에서 작업 열의 **로그인**을 클릭하여 new-gen 데이터베이스 툴을 사용하여 데이터베이스에 로그인합니다. 자세한 내용은 [DMC 개요](https://intl.cloud.tencent.com/document/product/236/39221)를 참고하십시오. |
| 관리 | 인스턴스 리스트에서 작업 열의 **관리**를 클릭하여 인스턴스 관리 페이지로 이동합니다. |
| 버전 업그레이드 | 인스턴스 리스트에서 작업 열의 더보기>버전 업그레이드를 클릭하여 데이터베이스 버전을 업그레이드합니다. 자세한 조건 및 지침은 [데이터베이스 엔진 버전 업그레이드](https://intl.cloud.tencent.com/document/product/236/8126)를 참고하십시오. |
| 구성 조정 | 인스턴스 리스트에서 작업 열의 **더보기** > **구성 조정**을 클릭하여 데이터베이스 인스턴스의 구성(예: 확장)을 조정합니다. 인스턴스 업그레이드와 다운그레이드가 모두 지원됩니다. 자세한 내용은 [데이터베이스 인스턴스 사양 변경](https://intl.cloud.tencent.com/document/product/236/19707)을 참고하십시오. |
| 종량제를 정액 과금제로 변경 | 인스턴스 리스트에서 작업 열의 **더보기** > **종량제를 정액 과금제로 변경**을 클릭하여 종량제 인스턴스의 과금 방식을 변경합니다. 자세한 내용은 [종량제를 정액 과금제로 변경](https://www.tencentcloud.com/document/product/236/52515)을 참고하십시오. |
| 태그 편집 | 인스턴스 리스트에서 작업 열의 **더보기** > **태그 편집**을 클릭하면 해당 인스턴스의 태그를 수정 및 관리할 수 있습니다. 자세한 내용은 [태그 편집](https://intl.cloud.tencent.com/document/product/236/31918)을 참고하십시오. |
| 폐기/반품 | 인스턴스 리스트에서 작업 열의 **더보기** > **폐기/반품**을 클릭하여 인스턴스를 반환합니다. 자세한 내용은 [인스턴스 폐기](https://intl.cloud.tencent.com/document/product/236/31895)를 참고하십시오. |
| 보안 그룹 | 인스턴스 리스트에서 작업 열의 **더보기** > **보안 그룹**을 클릭하여 인스턴스의 보안 그룹을 설정하거나 수정합니다. 자세한 내용은 [CDB 보안 그룹 관리](https://intl.cloud.tencent.com/document/product/236/14470)를 참고하십시오. |
| 연장 | 대상 인스턴스를 선택하고 인스턴스 리스트 상단의 **연장**을 클릭하여 인스턴스에 대한 수동 연장을 설정합니다. 자세한 내용은 [연장 안내](https://intl.cloud.tencent.com/document/product/236/47702)를 참고하십시오. |
| 재시작 | 하나 이상의 대상 인스턴스를 선택하고 인스턴스 리스트 상단의 **재시작**을 클릭하여 다시 시작합니다. <li>재시작하는 동안 인스턴스에 액세스할 수 없으며 인스턴스에 대한 기존 연결이 닫힙니다. <li>재시작하는 동안 비즈니스 쓰기 수와 더티 페이지 수가 많으면 다시 시작하지 못합니다. 이 경우 인스턴스는 재시작 전의 상태로 돌아가 액세스할 수 있게 됩니다. <li>재시작 성공을 보장하고 비즈니스에 미치는 영향을 최소화하려면 사용량이 적은 시간에 인스턴스를 다시 시작하십시오.</li>|
| 롤백 | 대상 인스턴스를 선택하고 인스턴스 리스트 상단의 **더보기** > **롤백**을 클릭하여 콜드 백업 및 binlog를 기반으로 지정된 시점으로 롤백합니다. 자세한 내용은 [데이터베이스 롤백](https://intl.cloud.tencent.com/document/product/236/7276)을 참고하십시오. |
| 매개변수 설정 | 하나 이상의 대상 인스턴스를 선택하고 인스턴스 리스트 상단의 **더보기** > **매개변수 설정**을 클릭하여 해당 매개변수 값을 수정합니다. 자세한 내용은 [인스턴스 매개변수 설정](https://intl.cloud.tencent.com/document/product/236/35793)을 참고하십시오.|
| 자동 연장 구성 | 하나 이상의 대상 인스턴스를 선택하고 인스턴스 리스트 상단의 **더보기** > **자동 연장 활성화**를 클릭하여 월별 자동 연장을 설정합니다. 자세한 내용은 [연장 안내](https://intl.cloud.tencent.com/document/product/236/47702)를 참고하십시오. |


## 인스턴스 관리 웹페이지
[TencentDB for MySQL 콘솔](https://console.cloud.tencent.com/cdb)에 로그인한 뒤, 인스턴스 리스트에서 대상 인스턴스 ID를 클릭하거나 **작업** 열의 **관리**를 클릭하면 인스턴스 관리 페이지로 이동하여 세부 정보를 확인하고 모니터링하며 데이터베이스를 관리할 수 있습니다.
![](https://staticintl.cloudcachetci.com/yehe/backend-news/Fvf9182_4.png)

| 기능 | 설명 |
|---------|---------|
| 인스턴스 세부 사항 |인스턴스 세부 사항 탭에서는 데이터베이스의 다양한 정보를 보고 조작할 수 있습니다. <img src="https://main.qcloudimg.com/raw/071659c8118f8c9b94d4ab90cebbd955.png"  style="margin:0;">을(를) 클릭하여 인스턴스의 기본 정보를 수정합니다. 여기서 공중망 주소는 기본적으로 비활성화되어 있습니다. 필요한 경우 공중망 주소 다음에 **활성화**를 클릭하여 활성화합니다. |
|인스턴스 모니터링 | 인스턴스 모니터링 탭에서는 액세스, 로드, 쿼리 캐시, 테이블, InnoDB, MyISAM 등 다양한 차원에서 현재 데이터베이스의 다양한 핵심 지표에 대한 모니터링 데이터를 볼 수 있습니다. 자세한 내용은 [모니터링 기능](https://intl.cloud.tencent.com/document/product/236/8455)과 [알람 정책(Cloud Monitoring)](https://intl.cloud.tencent.com/document/product/236/8457)을 참고하십시오.|
| 데이터베이스 관리 | <li>**데이터베이스 리스트**<br>이 탭에서 SQL 파일을 지정된 데이터베이스로 가져올 수 있습니다. 자세한 내용은 [SQL 파일 가져오기](https://intl.cloud.tencent.com/document/product/236/8466)를 참고하십시오. </li><li>**매개변수 설정**<br>이 탭에서 사용자 정의 가능한 데이터베이스 매개변수의 기록을 설정하고 볼 수 있습니다. 매개변수 값을 수정하려면 **매개변수 실행값** 열에서 <img src="https://main.qcloudimg.com/raw/071659c8118f8c9b94d4ab90cebbd955.png"  style="margin:0;">을(를) 클릭합니다. 자세한 내용은 [매개변수 템플릿 사용](https://intl.cloud.tencent.com/document/product/236/31906)을 참고하십시오. </li><li>**계정 관리**<br>이 탭에서 권한 수정 및 비밀번호 재설정과 같은 시스템의 기본 루트 계정을 관리할 수 있습니다. 계정을 생성 및 삭제하거나 암호 복잡성 기능을 활성화하여 데이터베이스 액세스 보안을 강화할 수도 있습니다. 자세한 내용은 [계정 관리](https://intl.cloud.tencent.com/document/product/236/31900) 및 [비밀번호 복잡성 설정](https://intl.cloud.tencent.com/document/product/236/49197)을 참고하십시오.</li>|
| 보안 그룹 | 보안 그룹 탭에서 데이터베이스에 대한 보안 그룹을 구성할 수 있습니다. 자세한 내용은 [CDB 보안 그룹 관리](https://intl.cloud.tencent.com/document/product/236/14470)를 참고하십시오.|
| 백업 및 복구 | 백업 및 복구 탭에서 binlog를 다운로드하고, 콜드 백업을 수행하고, 자동, 수동 및 리전 간 백업을 설정하고, 백업 암호화를 활성화할 수 있습니다. 자세한 내용은 [데이터베이스 백업](https://intl.cloud.tencent.com/document/product/236/37796), [리전 간 백업](https://www.tencentcloud.com/document/product/236/50652), [백업 암호화](https://www.tencentcloud.com/document/product/236/51881)를 참고하십시오.|
| 작업 로그 | 작업 로그 탭에서 슬로우 쿼리 로그, 오류 로그, 롤백 로그를 조회 및 다운로드할 수 있습니다. 자세한 내용은 [작업 로그](https://intl.cloud.tencent.com/document/product/236/34588)를 참고하십시오.|
| 읽기 전용 인스턴스 | 읽기 전용 인스턴스 탭에서 읽기/쓰기 분리 및 단일 소스 다중 복제 애플리케이션 시나리오에 적용하여 데이터베이스의 읽기 로드 용량을 높일 수 있는 하나 이상의 읽기 전용(RO) 인스턴스를 생성할 수 있습니다. 자세한 내용은 [읽기 전용 인스턴스 생성](https://intl.cloud.tencent.com/document/product/236/7270)을 참고하십시오.|
| 데이터베이스 프록시 |애플리케이션 서비스가 데이터베이스에 액세스할 때 프록시 요청에 사용됩니다. 자동 읽기/쓰기 분리, 연결 풀, 연결 유지 등의 고급 기능을 제공하며 고가용성, 고성능, Ops 지원 및 사용 편의성을 자랑합니다. 자세한 내용은 [데이터베이스 프록시 소개](https://www.tencentcloud.com/document/product/236/42048)를 참고하십시오. |
| 데이터 보안 | 데이터 보안 탭에서 인스턴스에 대해 TDE를 활성화할 수 있지만 물리적 백업에서 다른 서버의 자체구축 데이터베이스로 인스턴스를 복구할 수 없습니다. 또한 SSL 암호화를 활성화하여 데이터를 암호화된 방식으로 전송하고 가로채기, 변조 또는 도청을 방지할 수 있습니다. 자세한 내용은 [TDE 활성화](https://intl.cloud.tencent.com/document/product/236/38491) 및 [SSL 암호화 설정](https://intl.cloud.tencent.com/document/product/236/48452)을 참고하십시오. |
| 연결 진단 | 연결 진단 탭에서는 잠재적인 연결 및 액세스 문제를 확인하고 제공된 솔루션을 사용하여 데이터베이스에 정상적으로 액세스할 수 있도록 합니다. 자세한 내용은 [원클릭 연결 진단 툴](https://intl.cloud.tencent.com/document/product/236/31927)을 참고하십시오.|
