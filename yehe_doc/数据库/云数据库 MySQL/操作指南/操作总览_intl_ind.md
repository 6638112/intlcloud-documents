Dokumen ini menjelaskan operasi pada daftar instans dan halaman manajemen di konsol TencentDB for MySQL.

## Halaman Daftar Instans
Anda dapat login ke [konsol TencentDB for MySQL](https://console.cloud.tencent.com/cdb) dan masuk ke halaman daftar instans untuk melihat informasi instans dan mengelola instans Anda.
![](https://main.qcloudimg.com/raw/51001a389fec3fc1d62d45ff0f0895f9.png)

| Fitur | Deskripsi |
|---------|---------|
| Penyesuaian konfigurasi | Dalam daftar instans, klik **More** (Lainnya) di kolom **Operation** (Operasi) untuk menyesuaikan konfigurasi (yaitu, penskalaan) instans database Anda. Baik peningkatan maupun penurunan versi instans didukung. Untuk informasi selengkapnya, lihat [Menyesuaikan Spesifikasi Instans Database](https://intl.cloud.tencent.com/document/product/236/19707). |
| Kembalikan | Dalam daftar instans, pilih instans yang akan dikembalikan dan klik **More** (Lainnya) > **Roll back** (Kembalikan) di bagian atas untuk mengembalikannya ke titik waktu yang ditentukan berdasarkan cadangan dan binlog cold. Untuk informasi selengkapnya, lihat [Mengembalikan Database](https://intl.cloud.tencent.com/document/product/236/7276).|
| Mulai ulang | Dalam daftar instans, pilih sebuah instans dan klik **Restart** (Mulai ulang) di bagian atas untuk memulai ulang. Anda juga dapat memilih beberapa instans untuk memulai ulang batch. <li>Instans tidak akan dapat diakses selama proses mulai ulang, dan koneksi yang ada akan ditutup. <li>Jika jumlah penulisan bisnis dan jumlah halaman kotor tinggi selama proses mulai ulang, proses tersebut akan gagal. Dalam hal ini, instans akan kembali ke status sebelum memulai ulang dan dapat diakses. <li>Mulai ulang instans selama jam tidak sibuk untuk memastikan tingkat keberhasilan memulai ulang dan mengurangi dampaknya pada bisnis Anda.</li> |

## Halaman Manajemen Instans
Login ke [konsol TencentDB for MySQL](https://console.cloud.tencent.com/cdb). Setelah sebuah instans diinisialisasi, klik ID-nya dalam daftar instans atau klik **Manage** (Kelola) di kolom **Operation** (Operasi) untuk masuk ke halaman manajemen instans, tempat Anda dapat melihat detailnya, memantaunya, dan mengelola database.
![](https://main.qcloudimg.com/raw/167ee41bc241ef2e5a68c5fc160112d4.png)

| Fitur | Deskripsi |
|---------|---------|
| Detail instans | Pada tab **Instans Details** (Detail Instans), Anda dapat melihat dan memanipulasi berbagai informasi database. Klik <img src="https://main.qcloudimg.com/raw/071659c8118f8c9b94d4ab90cebbd955.png"  style="margin:0;"> untuk mengubah informasi dasar instans, tempat alamat jaringan publik dinonaktifkan secara default. Jika perlu, klik **Enable** (Aktifkan) setelah **Public Network Address** (Alamat Jaringan Publik) untuk mengaktifkannya. |
| Pemantauan instan | Pada tab **Instans Monitoring** (Pemantauan Instans), Anda dapat melihat data pemantauan dari berbagai metrik inti database saat ini dalam berbagai dimensi seperti akses, beban, cache kueri, tabel, InnoDB, dan MyISAM. Untuk informasi selengkapnya, lihat [Fitur Pemantauan](https://intl.cloud.tencent.com/document/product/236/8455) dan [Kebijakan Alarm (Cloud Monitor)](https://intl.cloud.tencent.com/document/product/236/8457). |
| Manajemen database | <li> **Database List** (Daftar Database) <br>Pada tab ini, Anda dapat mengimpor file SQL ke dalam database tertentu. Untuk informasi selengkapnya, lihat [Mengimpor File SQL](https://intl.cloud.tencent.com/document/product/236/8466). <li>**Parameter Settings** (Pengaturan Parameter)<br>Pada tab ini, Anda dapat mengatur dan melihat riwayat parameter database yang dapat disesuaikan. Klik <img src="https://main.qcloudimg.com/raw/071659c8118f8c9b94d4ab90cebbd955.png"  style="margin:0;"> di kolom **Current Value** (Nilai Saat Ini) untuk memodifikasi nilai parameter. Untuk informasi selengkapnya, lihat [Mengelola Templat Parameter](https://intl.cloud.tencent.com/document/product/236/31906). <li>**Account Management** (Manajemen Akun)<br>Pada tab ini, Anda dapat mengelola akun root default sistem, seperti mengubah izin dan mengatur ulang kata sandi. Anda juga dapat membuat dan menghapus akun. Untuk informasi selengkapnya, lihat [Manajemen Akun](https://intl.cloud.tencent.com/document/product/236/31900). |
| Grup keamanan | Pada tab **Security Group** (Grup Keamanan), Anda dapat mengonfigurasi grup keamanan untuk database Anda. Untuk informasi selengkapnya, lihat [Manajemen Grup Keamanan TencentDB](https://intl.cloud.tencent.com/document/product/236/14470).|
| Pencadangan dan pemulihan | Pada tab **Backup and Restoration** (Pencadangan dan Pemulihan), Anda dapat mengunduh binlog dan melakukan pencadangan cold. Untuk informasi selengkapnya, lihat [Mencadangkan Database](https://intl.cloud.tencent.com/document/product/236/37796).|
| Log operasi | Pada tab **Operation Log** (Log Operasi), Anda dapat melihat dan mengunduh log kueri lambat, log kesalahan, dan log pengembalian. Untuk informasi selengkapnya, lihat [Log Operasi](https://intl.cloud.tencent.com/document/product/236/34588). |
| Instans Baca Saja | Pada tab **Read-Only Instance** (Instans Baca Saja), Anda dapat membuat satu atau beberapa instans baca saja (RO), yang dapat diterapkan pada pemisahan baca/tulis dan skenario aplikasi satu-sumber-beberapa-replika untuk meningkatkan beban baca kapasitas database Anda. Untuk informasi selengkapnya, lihat [Membuat Instans Baca Saja](https://intl.cloud.tencent.com/document/product/236/7270).|
| Proksi database | Proksi ini digunakan untuk permintaan proksi ketika layanan aplikasi mengakses database. Proksi ini menyediakan fitur canggih seperti pemisahan baca/tulis otomatis, kumpulan koneksi, dan persistensi koneksi dan menawarkan ketersediaan tinggi, performa tinggi, dukungan OPS, dan kemudahan penggunaan. Untuk informasi selengkapnya, lihat [Proksi Database](https://intl.cloud.tencent.com/document/product/236/42048). |
| Enkripsi data | Instans dengan enkripsi data yang diaktifkan tidak dapat dipulihkan dari cadangan fisik ke database yang dibuat sendiri di server lain. Untuk informasi selengkapnya, lihat [Enkripsi Data Transparan](https://intl.cloud.tencent.com/document/product/236/43495). |
| Pemeriksaan Koneksi | Pada tab **Connection Check** (Pemeriksaan Koneksi), Anda dapat memeriksa kemungkinan konektivitas dan masalah akses dan mengatasinya menggunakan solusi yang disediakan untuk memastikan bahwa database Anda dapat diakses secara normal. Untuk informasi selengkapnya, lihat [Pemeriksa Konektivitas Sekali Klik](https://intl.cloud.tencent.com/document/product/236/31927).|
