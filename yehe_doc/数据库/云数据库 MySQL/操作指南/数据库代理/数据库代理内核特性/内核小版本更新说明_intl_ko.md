
본 문에서는 TencentDB for MySQL 데이터베이스 프록시의 버전 업데이트를 소개합니다.

## 데이터베이스 프록시 버전 조회
인스턴스의 [데이터베이스 프록시](https://intl.cloud.tencent.com/document/product/236/42052) 페이지의 **개요** > **프록시 버전**에서 조회할 수 있습니다.
![](https://qcloudimg.tencent-cloud.cn/raw/6868726aa0bdb77251d41979120986ae.png)

## 버전 업그레이드 설명
| 버전 | 설명 |
|---------|---------|
| 1.1.2 | **기능 업데이트**<br>1. MySQL 8.0 버전을 지원합니다. <br>2. 비지속 연결 비즈니스에서 데이터베이스와 빈번한 연결 시나리오를 처리할 수 있는 연결 수준 연결 풀 기능을 지원합니다. 데이터베이스 프록시는 연결을 저장하고 다음에 연결이 설정될 때 연결을 재사용합니다. <br>3. 읽기 전용 인스턴스에 대한 재연결을 지원합니다. 장기간 연결 시나리오에서 읽기 전용 인스턴스가 재시작되거나 새 읽기 전용 인스턴스가 추가되면 데이터베이스 프록시는 자동으로 읽기 전용 인스턴스에 대한 연결을 복구하여 라우팅합니다. <br>4. 내부 메모리 관리 메커니즘을 업데이트하여 새 버전의 메모리 소모가 더 낮습니다. <br>**문제 수정**<br>1. 백엔드 연결이 시간 초과로 끊어진 후에도 클라이언트 연결이 계속 끊어지지 않는 문제를 수정하였습니다. <br>2. 내부 캐시로 인해 메모리가 너무 빨리 증가할 수 있는 문제가 수정되었습니다. <br>3. 낮은 확률로 부정확한 형식의 메시지가 반환되는 문제가 수정되었습니다.|
| 1.0.1 | **기능 업데이트**<br>1. MySQL 5.7 버전을 지원합니다. <br>2. 읽기/쓰기 분리를 지원합니다. <br>3. 읽기/쓰기 분리에서 읽기 가중치 설정을 지원합니다. <br>4. 프라이머리/세컨더리 복사 딜레이 임계값 설정 지원, 읽기 전용 인스턴스는 딜레이가 임계값을 초과하면 라우팅에서 제거되고 딜레이가 임계값 아래로 떨어지면 라우팅이 복구됩니다. 프라이머리/세컨더리 복사가 중단되면 중단된 읽기 전용 인스턴스는 직접 제거됩니다. <br>5. 최소 보관 수 설정을 지원하며, 읽기 전용 인스턴스를 제거할 때 최소 보관 수를 n으로 설정하면 n개 이상의 읽기 전용 인스턴스가 라우팅에 보관됩니다. <br>6. 기본적으로 활성화된 장애 조치 설정을 지원합니다. 장애 조치가 비활성화되고 프라이머리 인스턴스의 읽기 가중치가 0이면 모든 읽기 전용 인스턴스가 제거된 후 읽기 요청에서 오류를 보고합니다. 장애 조치가 활성화되어 있거나 프라이머리 데이터베이스 가중치가 0이 아닌 경우 요청이 프라이머리 인스턴스로 라우팅됩니다. <br>7. 라우팅 노드를 지정하는 hint 구문을 지원합니다.|
