
본 문서는 TencentDB for MySQL 데이터베이스 프록시 서비스의 특성 및 적용 시나리오에 대하여 소개합니다. 

데이터베이스 프록시는 CDB 서비스와 애플리케이션 서비스 사이에 위치한 네트워크 프록시 서비스입니다. 애플리케이션 서비스가 CDB에 액세스할 때 발생하는 모든 요청을 중계하는 데 사용됩니다.
데이터베이스 프록시 액세스 주소는 기존의 데이터베이스 액세스 주소와 별개이며, 데이터베이스 프록시 주소의 요청을 통해 프록시 클러스터 내 데이터베이스의 마스터/슬레이브 노드에 액세스하여 읽기/쓰기를 분리하고, 읽기 요청을 읽기 전용 인스턴스에 전달해 마스터 데이터베이스의 부하를 낮춥니다. 
>?데이터베이스 프록시는 현재 베타 테스트 중이며 무료로 사용할 수 있습니다. 
>
## 데이터베이스 프록시 특성
- **높은 안정성**
클러스터 아키텍처 배포 방식을 적용하여, 멀티 노드가 장애를 판단하여 조치합니다. 
- **강한 격리성**
데이터베이스 프록시는 독립된 리소스를 사용하여 현재의 인스턴스에 프록시 서비스를 제공합니다(각 프록시 리소스는 독립적이며, 리소스를 공유하지 않습니다).
- **초고성능**
각 프록시는 초당 최고 10여 개의 요청을 처리할 수 있습니다. 
- **간편하고 빠른 확장**
1개-60개의 프록시 노드 상태 확장을 지원합니다(베타 테스트 기간 중에는 6개의 노드만 지원합니다).
- **완벽한 성능 모니터링**
읽기/쓰기 요청 수량, CPU, 메모리 등 성능 지표의 초 단위 모니터링을 제공하며, [모니터링 데이터](https://intl.cloud.tencent.com/document/product/236/41090) 및 서비스 계획에 따라 프록시 개수를 조정할 수 있습니다. 
- **핫 스와프**
마스터 인스턴스에 전환, 변경, 읽기 전용 인스턴스의 추가 혹은 감소 등 상황이 발생할 경우 데이터베이스 프록시는 동적으로 설정을 로딩할 수 있습니다. 네트워크 끊김이나 재시작 현상이 발생하지 않습니다.
- **[자동 읽기/쓰기 분리]를 지원합니다(https://intl.cloud.tencent.com/document/product/236/41089)**
데이터베이스 프록시의 읽기/쓰기 분리 기능을 통하여 효과적으로 마스터 인스턴스의 읽기 부하를 줄일 수 있으며, 읽기 전용 인스턴스 추가를 통해 데이터베이스 클러스터의 수평적 확장 기능을 제공합니다. 동시에 사용자가 스스로 읽기/쓰기 분리를 하도록 도와, 사용자의 서비스 읽기/쓰기 분리 요청의 복잡도를 줄여줍니다. 특히 대량 읽기 부하가 있는 상황에 더 적합합니다. 
예를 들어, 데이터베이스 프록시의 읽기/쓰기 기능을 활성화하고 응용 프로그램에 하나의 프록시 연결 주소만 설정하면 해당 주소가 자동으로 읽기/쓰기 분리를 하고, 읽기 요청을 읽기 전용 인스턴스로 전송하며 쓰기 요청을 마스터 인스턴스로 전송합니다. 읽기 전용 인스턴스를 추가하거나 삭제하여도 응용 프로그램의 설정을 조정하지 않아도 됩니다. 

## 적용 시나리오
- 대량의 단기 연결이 메인인 비즈니스, 성능 부족.
- 다수의 읽기 전용 인스턴스를 사용하여 응용 프로그램에서 수동으로 읽기/쓰기 분리를 해야 하는 비즈니스, 점검 비용과 리스크가 높은 경우.
- 지나치게 많은 연결 수로 인해 인스턴스의 부하가 과도히 높은 경우.
