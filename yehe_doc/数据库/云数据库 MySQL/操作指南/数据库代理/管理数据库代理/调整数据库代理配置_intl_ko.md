데이터베이스 프록시 기능은 설정 변경을 지원하며, 콘솔을 통해 데이터베이스 프록시의 프록시 사양 및 노드 수를 조정할 수 있습니다.

## 전제 조건
[데이터베이스 프록시 활성화](https://intl.cloud.tencent.com/document/product/236/42052)가 완료 되어 있어야 합니다.

## 주의 사항
설정 변경 작업 시 데이터베이스 프록시 버전이 업데이트되면 설정 변경 과정에서 데이터베이스 프록시 버전이 자동으로 업그레이드됩니다.

## 작업 단계
1. [MySQL 콘솔](https://console.cloud.tencent.com/cdb)에 로그인한 후, 위의 리전을 선택한 후 객체 인스턴스 ID를 클릭하여 인스턴스 관리 페이지로 이동합니다.
2. 인스턴스 관리 페이지에서 **데이터베이스 프록시** 페이지를 선택합니다.
3. 데이터베이스 프록시 페이지의 **개요** 페이지에서 오른쪽의 **설정 변경**을 클릭합니다.
![](https://qcloudimg.tencent-cloud.cn/raw/e1d6cb6714059f661e9b67373279b5ea.png)
4. 팝업 창에서 전환 시간을 선택하고 **확인**을 클릭합니다.
   - **프록시 사양**: 현재 옵션 사양은 2코어 4000MB, 4코어 8000MB, 8코어 16000MB입니다.
   - **노드 수**: 데이터베이스 프록시의 고가용성 확보를 위해 2개 이상의 노드를 선택할 것을 권장합니다.
   - **전환 시간**:
     - 점검 시간 내: 점검 시간 내 전환하며, 점검 시간은 인스턴스 상세 페이지에서 수정 가능합니다.
     - 업그레이드 완료 시: 업그레이드 완료 시 전환됩니다.
![](https://qcloudimg.tencent-cloud.cn/raw/efe6bb6da5af0df4a90cf52a6ed4486e.png)
5. **데이터베이스 프록시** 페이지의 **기본 정보**에서 작업 상태가 **업그레이드 중**에서 **실행 중**으로 변경되고 설정 조정이 완료될 수 있습니다.
>?
>- 업그레이드 완료 시 전환을 선택하여 설정 조정이 완료된 후 시스템이 자동으로 새 설정으로 전환합니다.
>- 점검 시간 내 전환을 선택하여 설정 조정이 완료된 후 설정한 점검 시간 내에 시스템이 전환됩니다.
>- 점검 시간 내 전환을 선택했으나 업무상 사전에 전환해야 하는 경우 설정 조정이 완료된 후 데이터베이스 프록시 페이지에서 **개요** > **기본 정보** > **상태 / 작업** 업그레이드 전환 대기 후 **지금 완료**를 클릭하여 전환할 수 있습니다.
