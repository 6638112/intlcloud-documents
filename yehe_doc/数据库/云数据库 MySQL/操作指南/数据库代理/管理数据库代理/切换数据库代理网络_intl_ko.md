본문에서는 TencentDB for MySQL 콘솔을 통한 데이터베이스 프록시의 네트워크 수정 방법에 대해 소개합니다. 

## 전제 조건
[데이터베이스 프록시 활성화](https://intl.cloud.tencent.com/document/product/236/42052)가 완료되어 있어야 합니다.

## 주의 사항
- 네트워크를 변경하면 인스턴스 데이터베이스 프록시 IP가 변경됩니다. IP 기본 보관 시간은 24시간이며, 최대 168시간까지 설정할 수 있습니다. 이전 액세스 IP는 효력을 상실하게 됩니다. 즉시 클라이언트 프로그램을 수정하십시오.
- 기존 IP 주소의 회수 시간을 0시간으로 설정하면 네트워크 변경 직후 기존 IP 주소를 회수합니다.
- ​MySQL 인스턴스가 위치한 리전의 VPC 네트워크만 선택할 수 있지만 서브넷 가용존의 선택을 제한하지 않으며, 서브넷 주소 범위를 조회할 수 있습니다.

## 작업 단계
1. [MySQL 콘솔](https://console.cloud.tencent.com/cdb)에 로그인한 후 위의 리전을 선택한 후 객체 인스턴스 ID를 클릭하여 인스턴스 관리 페이지로 이동합니다.
2. 인스턴스 관리 페이지에서 **데이터베이스 프록시** 페이지를 선택합니다.
3. 데이터베이스 프록시 페이지의 **개요** > **연결 주소** > **네트워크 유형**에서 ![](https://qcloudimg.tencent-cloud.cn/raw/0286958e4f0522141375f4080708ceec.png)을(를) 클릭합니다.
![](https://qcloudimg.tencent-cloud.cn/raw/ee84c770775763a4c4d20beee04a36ae.png)
4. 팝업 창에서 관련 설정을 선택하고 **확인**을 클릭합니다.
   - 기존 IP 주소 회수 시간을 설정합니다. 설정 범위는 0 - 168시간입니다.
   - 시스템을 선택하여 자동으로 IP 주소를 할당하거나 수동으로 IP 주소를 지정합니다.
![](https://qcloudimg.tencent-cloud.cn/raw/f48ad92683bd8eae4baa76580af03d7b.png)
5. 네트워크 변경이 성공적으로 완료되면 **연결 주소**에서 변경된 네트워크를 쿼리할 수 있습니다.
