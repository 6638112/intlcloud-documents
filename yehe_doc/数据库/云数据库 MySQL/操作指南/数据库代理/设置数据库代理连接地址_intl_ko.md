
본문에서는 TencentDB for MySQL 콘솔을 통한 데이터베이스 프록시 주소 연결 설정 방법에 대해 소개합니다. 

데이터베이스 프록시 액세스 주소는 기존의 데이터베이스 액세스 주소와 별개이며, 데이터베이스 프록시 주소를 통하는 요청은 모두 프록시 클러스터를 통해 데이터베이스의 마스터/슬레이브 노드를 액세스하여 읽기/쓰기를 분리하고, 읽기 요청을 읽기 전용 인스턴스에 포워딩하여 마스터 데이터베이스의 부하를 낮춥니다.

## 전제 조건
- [데이터베이스 프록시 활성화]가 완료되어야 합니다. (https://intl.cloud.tencent.com/document/product/236/42052)

## 작업 순서
1. [MySQL 콘솔](https://console.cloud.tencent.com/cdb)에 로그인한 뒤, 인스턴스 리스트에서 프록시를 활성화한 마스터 인스턴스를 선택하고, 인스턴스 ID 혹은 '작업'열의 [관리]를 클릭하여 인스턴스 관리 페이지로 이동합니다.
2. 인스턴스 관리 페이지에서 [데이터베이스 프록시] 페이지를 선택하고 ‘데이터베이스 프록시 주소’뒤의 <img src="https://main.qcloudimg.com/raw/be716b5360d5256a9d5e816e29872ec1.png"  style="margin:0;">아이콘을 클릭합니다.
![](https://main.qcloudimg.com/raw/63015c402bdd31e04e2597af84013a74.png)
3. 팝업된 대화 상자에서 프록시 주소 수정 후 [확인]을 클릭합니다.
>!내부 네트워크 주소를 수정하면 현재 액세스중인 데이터베이스의 작업에 영향을 줄 수 있으니 사용량이 적을 때 주소 수정을 권장하며, 비즈니스의 재연결 메커니즘을 확보하시기 바랍니다.
>
