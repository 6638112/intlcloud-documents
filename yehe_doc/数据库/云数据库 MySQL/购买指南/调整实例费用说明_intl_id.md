
## Metode Penagihan untuk Instans yang Ditingkatkan
Setelah peningkatan, instans bayar sesuai pemakaian akan ditagih berdasarkan spesifikasi instans baru mulai dari siklus penagihan berikutnya.

<span id = "degrade_billing"></span>
## Metode Penagihan untuk Instans yang Diturunkan Versinya

#### Bayar sesuai pemakaian
Jika Anda memilih untuk menurunkan versi instans database bayar sesuai pemakaian, instans tersebut akan ditagih dengan harga bayar sesuai pemakaian tingkat 1 untuk konfigurasi baru.

## Dokumentasi
TencentDB for MySQL mendukung penyesuaian cepat dari spesifikasi instans dan memungkinkan operasi penskalaan yang fleksibel. Untuk informasi selengkapnya, harap lihat [Menyesuaikan Spesifikasi Instans Database](https://intl.cloud.tencent.com/zh/document/product/236/19707).
