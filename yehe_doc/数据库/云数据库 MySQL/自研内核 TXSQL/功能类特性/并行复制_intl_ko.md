## 기능 소개
공식 MySQL 5.6 이하 버전은 slave 노드에서 재생되며, master 노드가 binlog를 동기화하면 모두 싱글 스레드 모드가 됩니다. 5.6 및 이후 버전은 병렬 모드로 변경됩니다. 그러나 공식 병렬 처리는 database와 logical clock을 기반으로 하며, 병렬 처리 세분성이 너무 커서 병렬 효과가 만족스럽지 못한 경우가 많습니다.

Tencent Cloud의 TXSQL 커널 팀은 병렬 복사 기능을 최적화하고 table 병렬화를 지원하여, 병렬 세분성을 테이블 단위로 높이고 마스터/슬레이브 딜레이를 줄입니다.

## 지원 버전
- 커널 버전 MySQL 5.7 20180530 이상
- 커널 버전 MySQL 5.6 20170830 이상

## 적용 시나리오
이 기능은 주로 slave 머신의 binlog 재생 속도를 향상시키고 마스터와 슬레이브의 delay를 줄이는 것을 목적으로 합니다.

## 사용 설명
slave_parallel_type 매개변수를 새로 추가된 값 TABLE로 설정하여 이 기능을 켭니다.
또한 information_schema 아래에 새로운 상태 테이블 cdb_slave_thread_status를 추가하여 상태 정보를 표시합니다.

| 매개변수 이름              | 동적 | 유형  | 기본값     | 매개변수 값 범위                   | 설명                                                         |
| ------------------- | ---- | ----- | -------- | ---------------------------- | ------------------------------------------------------------ |
| slave_parallel_type | YES  | char\* | DATABASE | DATABASE/TABLE/LOGICAL_CLOCK | 슬레이브 머신 병렬 복사 수준으로, <li>DATABASE는 데이터베이스 수준 복사이며 각각의 데이터베이스의 복사 이벤트를 병렬로 완료할 수 있습니다.<li>TABLE은 테이블 수준 복사로 각각의 테이블의 복사 이벤트를 병렬로 수행할 수 있습니다.<li>LOGICAL_CLOCK은 논리적 클럭 수준 복사이며 호스트에서 동일한 논리적 클럭에 속하는 이벤트를 동시 실행할 수 있습니다. |
