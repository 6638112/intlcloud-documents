## 기능 소개
이 기능은 주로 IO 지터를 피하기 위해 대용량 데이터 파일이 있는 테이블을 삭제하는 데 사용됩니다.

DROP TABLE은 원본 데이터베이스 파일 (.ibd)의 이름을 새 임시 파일로 변경하고 성공적으로 반환합니다. 임시 파일은 innodb-async-drop-tmp-dir에서 지정한 디렉터리에 저장되며 백그라운드에서 일괄적으로 truncate하고 각 truncate의 파일 크기는 innodb-async-truncate-size에 의해 제어됩니다. 비동기화 테이블 삭제 기능의 활성화는 innodb-async-drop-tmp-dir 매개변수에 의해 제어되며 비어 있지 않으면 활성화됩니다.

이 기능은 사용자 작업이 필요하지 않으며 커널에 의해 자동 완성되며, 테이블 삭제 시 테이블의 데이터 파일에 대해 다른 디렉터리에 하드 링크를 생성하는 것이 원칙입니다. drop table이 실행되면 파일에 대한 하드 링크만 삭제됩니다. 그 후, 백그라운드 스레드는 하드링크 디렉터리에서 삭제가 필요한 파일을 스캔하고 백그라운드 truncate 전에 테이블 데이터 파일을 자동으로 drop합니다.

## 지원 버전
커널 버전 MySQL 5.7 20190203 이상

## 적용 시나리오
이 기능은 삭제할 테이블 데이터 파일이 매우 큰 시나리오에 적합합니다.

## 사용 설명
innodb_async_truncate_work_enabled를 ON으로 설정하면 DROP TABLE이 비동기화 모드가 되며 기본값은 OFF입니다.
임시 파일은 innodb_async_drop_tmp_dirr에서 지정한 디렉터리에 저장되며 각 truncate의 파일 크기는 innodb_async_truncate_size에서 제어합니다.

| 매개변수 이름                             | 동적 | 유형   | 기본 | 매개변수 값 범위                        | 설명                            |
| ---------------------------------- | ---- | ------ | ---- | ------------------------- | ------------------------------ |
| innodb_async_truncate_work_enabled | Yes  | string |  OFF   | ON/OFF      | 대용량 테이블 비동기화 truncate 활성화 여부          |
| innodb_async_truncate_size         | Yes  | int    | 5.7, 20210630 이하 버전, 기본값: 128<br>5.7, 20211030 이상 버전, 기본값: 64  | 5.7, 20210630 및 이전 버전, 범위: 128MB-2048MB<br>버전 5.7, 20211030 이상 버전, 범위: 16MB-256MB | 비동기화 DROP TABLE 백그라운드 truncate 파일 크기/회(단위 MB) |
