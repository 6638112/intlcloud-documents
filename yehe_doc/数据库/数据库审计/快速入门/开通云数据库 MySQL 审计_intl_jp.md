Tencent CloudはTencentDB for MySQLにデータベースの監査機能を提供し、データベースへのアクセスおよびSQLステートメントの実行状況を記録し、企業のリスク管理に貢献し、データのセキュリティレベルを高めます。  

>!データベース監査は、TencentDB for MySQL 5.6/5.7/8.0の2ノードと3ノードをサポートしており、TencentDB for MySQL 5.5とTencentDB for MySQLシングルノードをサポートしません。

## [監査ルールの作成](id:cjsjgz)
1.[MySQLコンソール](https://console.cloud.tencent.com/dls/mysql)にログインし、左側ナビゲーションバーで**データベース監査**をクリックし、上側でリージョンを選択して、**監査ルール**画面をクリックします。
2. 監査ルール画面で、**ルールの作成**をクリックします。
![](https://qcloudimg.tencent-cloud.cn/raw/687766a51c68c1b5ee483716c6097bac.png)
3. 監査ルールの作成画面で、ルール名と詳細を入力して**次へ**をクリックします。
4. パラメータ設定画面で、監査方法とパラメータを選択して、**保存**をクリックします。
>!
>?ルールを作成した後、監査ポリシーと関連を付けないと、有効になりません。
>- SQL監査ルールの詳細な使用方法については、[SQL監査ルール](https://intl.cloud.tencent.com/document/product/1102/47770)をご参照ください。

## SQL監査サービスの有効化
1.[MySQLコンソール](https://console.cloud.tencent.com/dls/mysql)にログインし、左側ナビゲーションバーで**データベース監査**をクリックし、上側でリージョンを選択し、**監査インスタンス**画面で**無効**をクリックして、監査が無効になっているインスタンスを絞り込みます。
![](https://qcloudimg.tencent-cloud.cn/raw/c64c25e2c28775636390079ba2971c91.png)
>?または、**監査ログ**画面の監査インスタンスで、監査が無効になっているインスタンスをそのまま検索して、有効にします。
>![](https://qcloudimg.tencent-cloud.cn/raw/8c8ed0771f0c0e81babe362e8d98c663.png)
2.**監査インスタンス**画面で、監査機能を有効にするインスタンスのIDをクリックして有効化画面に入ります。利用規約を同意するようにチェックを入れて、**次へ**をクリックします。
3. **SQL監査サービス設定**画面で、監査ログの保存時間を選択して、**次へ**をクリックします。
>?
>- 選択可能な監査ログの保存時間は、7日、30日、3カ月、6カ月、1年、3年、5年です。有効にした後、コンソールで保存時間を変更できます。詳しくは「ログ保存時間の変更」(https://intl.cloud.tencent.com/document/product/1102/47771)をご参照ください。
>- SQLログの保存時間に対するセキュリティとコンプライアンス上の要求を満たすために、保存時間は180日以上を推奨します。
4. **ポリシー作成**画面で、ポリシー名を設定し、作成された[監査ルール](https://intl.cloud.tencent.com/document/product/1102/47770)を選択して、**ポリシー作成**をクリックします。

## 監査ポリシーの作成
1.[MySQLコンソール](https://console.cloud.tencent.com/dls/mysql)にログインし、左側ナビゲーションバーで**データベース監査**をクリックし、上側でリージョンを選択して、**監査ポリシー**画面をクリックします。
2. **監査ポリシー**画面で、**ルールの作成**をクリックします。
![](https://qcloudimg.tencent-cloud.cn/raw/d5260ffd47d4318825a3ef152385dddd.png)
3. 表示されたダイアログボックスでポリシー名を設定し、作成された[監査ルール](https://intl.cloud.tencent.com/document/product/1102/47770)を選択して、**OK**をクリックします。

## 監査ログの確認
監査機能が有効になっている場合、**監査ログ**画面でSQL監査ログを確認できます。詳しくは、「監査ログ」(https://intl.cloud.tencent.com/document/product/1102/47772)をご参照ください。
