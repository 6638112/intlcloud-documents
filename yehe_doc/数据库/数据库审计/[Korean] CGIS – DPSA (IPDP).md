
## 1\. 배경

이 모듈은 Database Audit("**기능**”)을 사용하는 경우 적용됩니다. 이 모듈은 https://intl.cloud.tencent.com/document/product/301/17347 의 데이터 개인 정보 보호 및 보안 계약(“**DPSA**”)에 포함됩니다. 이 모듈에서 사용되었지만 정의되지 않은 용어는 DPSA에 정의된 의미를 갖습니다. DPSA와 이 모듈 간에 충돌이 발생할 경우, 이 모듈은 상충하는 범위까지만 적용됩니다.

## 2\. 처리

당사는 기능과 관련하여 다음과 같은 데이터를 처리합니다.

| **개인 정보**                                     | **이용**                                                      |
| ------------------------------------------------------------ | ------------------------------------------------------------ |
| **구성 데이터**  (귀하가 정한 감사 정책, 감사 정책 이름, 리소스 ID, 리소스  이름, 지역, 리소스 구성 세부 정보, Elasticsearch Service  데이터베이스 암호) | 당사는  사용자의 개별 구성에 따라  사용자에게 기능을 제공하기 위해서만 이 데이터를 처리합니다.  이 데이터는 MySQL용 TencentDB,  MongoDB용 TencentDB, TDSQL-C 기능과 통합되며 Elasticsearch  Service 기능에 저장됩니다. |
| **감사 로그 데이터**  (SQL 문 실행 시간, 스레드 ID, 행 세부 정보, 처리 시간의  세부 정보, 실패한 SQL 문 오류 코드, 감사 규칙 번호,  클라이언트 IP, 인스턴스 이름, 인스턴스 ID, 사용자 이름(사용자가 만들었는지 또는  임의인지 여부), 데이터베이스 이름, 감사 정책 이름, SQL 문 및 SQL 문  유형 포함) | 당사는  데이터베이스 작업을 기록하기 위해,  사용자에게 기능을 제공하는 목적으로만 이 데이터를 처리합니다.  이  데이터는 대상 데이터베이스 기능 및 Elasticsearch Service 기능에 저장된  (인스턴스 ID 제외)에서 수신됩니다. |

DPSA에 명시된 바와 같습니다.

## 5\. 데이터 보유

당사는 다음과 같이 기능과 관련하여 처리된 개인 데이터를 저장합니다.

| **개인  정보**                                    | **보유 정책**                                         |
| ------------------------------------------------------------ | ------------------------------------------------------------ |
| 구성 데이터  감사 로그 데이터(저장되지 않은  인스턴스 ID 제외) | 당사는 선택한  데이터 보존 시간 제한 동안 이러한 데이터를 유지합니다.  데이터 보존 시간 제한을  선택하지 않으면 저장 후 30 일 후에 자동으로 해당 데이터를 삭제합니다.   데이터 보존 시간 제한이 종료되기 전에  이러한 데이터를 삭제하도록 요청하는 경우 삭제 요청이 이루어진 후 3 일 이내에 이러한 데이터가 삭제됩니다.  이 기능이 종료되면 이러한 데이터는 3일 동안 보관된 후  삭제됩니다.   서비스 만료 시 서비스를 갱신하지 않는 경우, 만료되고 나서  3일 후에 해당 데이터가 자동으로 삭제됩니다. |

DPSA에 따라 이러한 개인 데이터의 삭제를 요청할 수 있습니다.

## 6\. 특수 조건

개인 데이터 처리에 동의할 수 있는 최소 연령을 만족하는 개인인 최종 사용자만 이 기능을 사용할 수 있습니다. 이는 최종 사용자의 해당 지역에 따라 다를 수 있습니다.
