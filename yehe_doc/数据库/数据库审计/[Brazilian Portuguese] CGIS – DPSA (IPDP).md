
## 1\. CONSIDERAÇÕES

Este Módulo se aplica se você usar o Database Audit (“**Recurso**”). Este Módulo está incorporado ao Contrato de privacidade e segurança de dados localizado em https://intl.cloud.tencent.com/document/product/301/17347 (“**DPSA**”). Os termos utilizados, mas não definidos neste Módulo, devem ter o significado dado a eles no DPSA. No caso de qualquer conflito entre o DPSA e este Módulo, este Módulo deve ser aplicado conforme a inconsistência.

## 2\. PROCESSAMENTO

Processaremos os seguintes dados em conexão com o Recurso:

| **Informações pessoais**                                     | **Uso**                                                      |
| ------------------------------------------------------------ | ------------------------------------------------------------ |
| **Dados de configuração**  (política de auditoria definida por você, nome da política de auditoria, ID de recursos, nome do  recurso, região, detalhes de configuração de recurso, sua senha do banco de dados do  Elasticsearch Service) | Apenas  processamos esses dados com a finalidade de fornecer o Recurso a você  de acordo com a sua configuração específica.  Observe que esses dados estão integrados com o nosso TencentDB for MySQL,  TencentDB for MongoDB, recursos TDSQL-C e armazenados no recurso  Elasticsearch Service. |
| **Dados de registro de auditoria**  (incluindo o tempo de execução de comunicado SQL, ID de thread, detalhes das linhas, detalhes  dos tempos de processamento, comunicado de código de erro de falha SQL, número da regra da auditoria,  IP do cliente, nome da instância, ID da instância, nome de usuário (criado por você ou  aleatório), nome do banco de dados, nome da política de auditoria, comunicado SQL e tipo de comunicado  SQL) | Apenas  processamos esses dados com a finalidade de fornecer o Recurso para você  com o objetivo de gravar as operações do banco de dados.  Observe  que esses dados são recebidos a partir de recursos de banco de dados de destino e  (além do ID de instância) armazenados em nosso recurso Elasticsearch Service. |

## 5\. RETENÇÃO DE DADOS

Armazenaremos dados pessoais processados em conexão com o Recurso da seguinte forma:

| **Informações  pessoais**                                    | **Política de retenção**                                         |
| ------------------------------------------------------------ | ------------------------------------------------------------ |
| Dados de configuração  Dados de registro de auditoria (além do ID da instância,  quem não é armazenado) | Retemos tais dados de acordo com o seu limite de  tempo de retenção de dados escolhido.  Se você não  escolher o limite de tempo de retenção de dados, excluiremos automaticamente tais dados depois de 30  dias do armazenamento.   Se você solicitar a exclusão de tais dados antes  do limite de tempo de retenção de dados chegar ao fim, tais dados serão excluídos 3  dias após o pedido de exclusão ser feito.  Se este Recurso for encerrado, tais dados serão retidos por 3 dias antes de  serem excluídos.   Quando você não renovar o serviço após a expiração do serviço, tais dados  serão excluídos automaticamente 3 dias após a expiração. |

Você pode solicitar a exclusão desses dados pessoais de acordo com o DPSA.

## 6\. CONDIÇÕES ESPECIAIS

Você deve garantir que este Recurso seja usado somente por usuários finais com a idade exigida para que possam consentir com o processamento de seus dados pessoais. Pode haver diferenças dependendo da jurisdição em que o usuário final está localizado.
