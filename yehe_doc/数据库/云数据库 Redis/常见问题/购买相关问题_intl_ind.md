### Bagaimana cara memilih spesifikasi TencentDB for Redis? 
Spesifikasi TencentDB for Redis dapat dipilih berdasarkan dua faktor: kapasitas dan performa. Kapasitas ditentukan sesuai dengan kebutuhan penyimpanan data. Untuk informasi selengkapnya tentang performa, silakan lihat [Metrik Performa](https://intl.cloud.tencent.com/document/product/239/17952).

### Wilayah mana yang didukung oleh TencentDB for Redis? 
Sebagai infrastruktur, TencentDB for Redis mendukung wilayah tempat instans CVM di-deploy. Untuk informasi selengkapnya, silakan lihat [Wilayah dan AZ](https://intl.cloud.tencent.com/document/product/239/4106).

### Bagaimana TencentDB for Redis ditagih?
Tagihan sesuai pemakaian. Untuk informasi selengkapnya, silakan baca [Ikhtisar Penagihan](https://intl.cloud.tencent.com/document/product/239/9894).

### Apa metrik performa TencentDB for Redis? 
TencentDB for Redis tersedia dalam arsitektur standar dan klaster. Untuk persyaratan QPS antara 80.000 dan 120.000, Anda dapat memilih arsitektur standar. Untuk performa QPS yang lebih tinggi, gunakan arsitektur klaster.

### Versi apa yang didukung TencentDB for Redis? 
Saat ini, TencentDB for Redis kompatibel dengan Redis 2.8, 4.0, dan 5.0.

### Bagaimana cara meminta pengembalian dana untuk instans yang dibeli jika saya tidak membutuhkannya lagi?
- Instans bayar sesuai pemakaian: TencentDB for Redis akan mengembalikan sumber daya secara langsung di [konsol](https://console.cloud.tencent.com/redis). Pengembalian dana tidak dapat dilakukan.
