## 과금 방식

TDSQL for MySQL은 2가지 과금 방식을 제공합니다.

| 과금 방식 | 결제 모드                                   | 적용 시나리오                                      |
| -------- | ------------------------------------------ | ----------------------------------------------- |
| 정액 과금제 | 선불. 요금은 인스턴스가 생성될 때 지불됩니다. | 종량제 옵션보다 요금이 낮기 때문에 장기적으로 안정적인 수요가 있는 기업에 적합합니다. 긴 시간을 구매할수록 할인율이 높아집니다. |
| 종량제 과금 | 후불. 리소스는 주문형으로 요청할 수 있으며 정산 시 실제 사용량에 따라 요금이 부과됩니다. | 수요가 변동하는 비즈니스에 적합합니다. 비용 절감을 위해 사용 후 즉시 인스턴스를 해제할 수 있습니다. |

종량제 과금에 대한 메모리 가격은 사용 시간을 기반으로 하며 총 세 단계로 구분합니다.

| 사용 시간               | 단계별 가격             |
| ---------------------- | -------------------- |
| 0시간＜사용 시간 ≤ 96시간   | 종량제 1단계 가격 |
| 96시간＜사용 시간 ≤ 360시간 | 종량제 2단계 가격 |
| 사용 시간＞360시간          | 종량제 3단계 가격 |
