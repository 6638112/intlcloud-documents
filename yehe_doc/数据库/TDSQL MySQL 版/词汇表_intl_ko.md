

### 수직 샤딩

데이터베이스는 여러 테이블로 구성되며 각 테이블은 서로 다른 비즈니스에 해당합니다. 수직 샤딩은 비즈니스 기능에 따라 테이블을 분류하고 다른 데이터베이스에 배포합니다.



### 샤딩되지 않은 테이블

샤딩되지 않은 테이블은 샤딩이 필요하지 않은 테이블을 저장하는 데 사용됩니다. 이 테이블의 모든 데이터는 첫 번째 물리적 샤드에 저장되고 이 유형의 모든 테이블은 MySQL과 정확히 동일한 구문 및 사용 방법으로 첫 번째 물리적 샤드에 배치됩니다. 분산되지 않은 테이블로 간주할 수 있습니다.

### 샤딩된 테이블

샤딩된 테이블은 방대한 양의 데이터가 있는 원본 테이블을 여러 데이터베이스 노드로 분할하여 각 물리적 샤드가 일부 데이터를 전달하고 모든 물리적 샤드가 완전한 데이터를 제공할 수 있도록 합니다.

### TDSQL for MySQL

TDSQL for MySQL은 자동 샤딩(수평 샤딩) 및 Shared Nothing 아키텍처를 지원하는 Tencent Cloud에 배포된 분산 데이터베이스 서비스입니다. 분산 데이터베이스를 사용하면 백엔드의 여러 물리적 샤드 노드에 균등하게 분할 및 분산되는 완전한 논리적 데이터베이스 테이블을 얻을 수 있습니다. TDSQL은 기본적으로 프라이머리-세컨더리 아키텍처를 배포하고 재해 복구, 백업, 복원, 모니터링 및 마이그레이션을 위한 전체 솔루션 세트를 제공하므로 테라바이트(TB)에서 페타바이트(PB)의 데이터를 저장하는 데 이상적입니다.



### 브로드캐스트 테이블

브로드캐스트 테이블은 스몰 테이블 브로드캐스트 기술을 기반으로 합니다. 테이블을 브로드캐스트 테이블로 설정하면 이 테이블에 대한 모든 작업이 모든 물리적 샤드에 브로드캐스트되며 각 샤드는 테이블의 모든 데이터를 소유합니다.
<span id="OLAP1"></span>
### 온라인 분석 처리(OLAP)

데이터 웨어하우스 시스템의 주요 응용 프로그램인 OLAP(On-Line Analytical Processing)는 복잡한 분석 작업을 지원하고 의사 결정 지원에 중점을 두고 직관적인 쿼리 결과를 제공합니다.
<span id="OLTP1"></span>
### 온라인 트랜잭션 처리(OLTP)

OLTP(On-Line Transaction Processing)는 전통적인 관계형 데이터베이스의 주요 애플리케이션으로 주로 은행 거래와 같은 기본 및 일상 거래를 처리하는 데 사용됩니다.

### Percona

Percona는 MySQL 프로토콜과 완벽하게 호환되며 MySQL에 비해 기능과 성능이 크게 향상되었습니다. 현재 TDSQL은 Percona 5.7을 지원합니다.

### Proxy

Tproxy를 통해 TDSQL은 자동 데이터베이스 및 테이블 샤딩을 달성하고 기본 물리적 데이터베이스 인스턴스를 관리하며 MySQL 데이터베이스와 호환되는 고유한 서비스 포트를 제공합니다.



### 강제 동기화

강제 동기화는 MySQL 기반의 다중 스레드 비동기 복제 프로그램입니다.

### 고유한 디지털 시퀀스

전 세계적으로 고유한 디지털 시퀀스로 약칭은 sequence입니다. 시퀀스는 unsigned long 유형의 8바이트를 포함합니다. 현재 TDSQL은 이 필드의 전역 내 고유성과 증분을 보장합니다.

### 수평 샤딩
