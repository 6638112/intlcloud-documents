
본문은 인스턴스 네트워크 유형을 변경하고 인스턴스 액세스 주소를 수정하는 방법을 설명합니다.

>!인스턴스의 네트워크 구성을 수정하는 것은 매우 위험합니다. 사용량이 적은 시간에만 하십시오. 수정 후 다른 서비스에 할당하지 않는 한 원래 IP는 24시간 동안 유효합니다. 이에 따라 가능한 빨리 비즈니스 구성을 수정하는 것이 좋습니다.

## 내부 IP 수정
VPC에서 TencentDB 인스턴스의 내부 IP를 수정할 수 있습니다.
1. [TDSQL for MySQL 콘솔](https://console.cloud.tencent.com/tdsqld)에 로그인하고 인스턴스 목록에서 인스턴스 ID를 클릭하고 인스턴스 세부 정보 페이지로 들어갑니다.
2. 기본 정보 섹션에서 **내부 IP** 옆에 있는 <img src="https://main.qcloudimg.com/raw/071659c8118f8c9b94d4ab90cebbd955.png"  style="margin:0;">을(를) 클릭하여 수정합니다. 현재 서브넷에 사용 가능한 IP가 있는 경우에만 그렇게 할 수 있습니다.
![](https://main.qcloudimg.com/raw/25931632d09fe7f90719db45bdaad8b5.png)
3. 팝업 창에서 내부 IP를 수정하고 **확인**을 클릭합니다.

## VPC 서브넷 간 전환
VPC 서브넷 간에 인스턴스를 전환할 수 있습니다.
1. [TDSQL for MySQL 콘솔](https://console.cloud.tencent.com/tdsqld)에 로그인하고 인스턴스 목록에서 인스턴스 ID를 클릭하고 인스턴스 세부 정보 페이지로 들어갑니다.
2. 기본 정보 페이지에서 **네트워크** 옆에 있는 **서브넷 변경**을 클릭합니다.
3. 팝업 창에서 서브넷을 선택하고 **IP 자동 할당** 또는 IP 지정을 선택한 후 **확인**을 클릭합니다.
>?
>- 원래 VIP 주소는 VPC가 수정된 후 24시간 동안 유효합니다. 24시간 이내에 비즈니스 IP 주소를 적절하게 수정하십시오.
>- TDSQL for MySQL은 로컬 듀얼 액티브 아키텍처를 지원하므로 비즈니스 서버 또는 기본 데이터베이스 노드와 동일한 리전에서 VPC 서브넷을 선택하는 것이 좋습니다.
>
![](https://main.qcloudimg.com/raw/c228774130d9932505ae8661d5a7e4fe.png)

## VPC 간 전환
1. [TDSQL for MySQL 콘솔](https://console.cloud.tencent.com/tdsqld)에 로그인하고 인스턴스 목록에서 인스턴스 ID를 클릭하고 인스턴스 세부 정보 페이지로 들어갑니다.
2. 기본 정보 페이지에서 **네트워크** 옆에 있는 **서브넷 변경**을 클릭합니다.
3. 팝업 창에서 VPC 및 서브넷을 선택하고 **IP 자동 할당** 또는 IP 지정을 선택한 후 **확인**을 클릭합니다.
>?
>- 원래 VIP 주소는 VPC가 수정된 후 24시간 동안 유효합니다. 24시간 이내에 비즈니스 IP 주소를 적절하게 수정하십시오.
>- TDSQL for MySQL은 로컬 듀얼 액티브 아키텍처를 지원하므로 비즈니스 서버 또는 기본 데이터베이스 노드와 동일한 리전에서 VPC 서브넷을 선택하는 것이 좋습니다.


## 클래식 네트워크에서 VPC로 전환
1. [TDSQL for MySQL 콘솔](https://console.cloud.tencent.com/tdsqld)에 로그인하고 인스턴스 목록에서 인스턴스 ID를 클릭하고 인스턴스 세부 정보 페이지로 들어갑니다.
2. 기본 정보 페이지에서 **네트워크** 옆에 있는 **VPC 네트워크로 전환**을 클릭합니다.
![](https://main.qcloudimg.com/raw/5d7751c75d3d8109b33f4676e414e42c.png)
3. 팝업 창에서 VPC 및 서브넷을 선택하고 **IP 자동 할당** 또는 IP 지정을 선택한 후 **확인**을 클릭합니다.
>!
>- 기본 네트워크를 VPC 네트워크로 변경한 후에는 다시 되돌릴 수 없습니다.
>- 전환 후 VPC 액세스는 즉시 적용됩니다. 원래의 클래식 네트워크 액세스는 24시간 동안 유지됩니다. 따라서 TDSQL for MySQL 인스턴스와 연결된 다른 인스턴스는 중단 없는 액세스를 보장하기 위해 24시간 이내에 VPC로 마이그레이션해야 합니다.
>- TDSQL for MySQL은 로컬 듀얼 액티브 아키텍처를 지원하므로 비즈니스 서버 또는 기본 데이터베이스 노드와 동일한 리전에서 VPC 서브넷을 선택하는 것이 좋습니다.
>
