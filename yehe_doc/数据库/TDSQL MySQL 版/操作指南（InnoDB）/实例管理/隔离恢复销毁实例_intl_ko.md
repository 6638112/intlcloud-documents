## 인스턴스 격리
더 이상 사용하지 않을 때 인스턴스를 격리할 수 있습니다. 일단 격리되면 인스턴스를 사용하거나 액세스할 수 없으며(아직 제거되지는 않음) 휴지통으로 이동되어 복구하거나 제거할 수 있습니다. 그렇지 않으면 만료되면 자동으로 제거됩니다. 인스턴스가 격리되더라도 해당 리소스가 차지하는 공간은 해제되지 않으며 여전히 가장 기본적인 데이터 복제본이 있습니다.

- 종량제 인스턴스: [콘솔](https://console.cloud.tencent.com/mariadb/instance/index)에 로그인하고 인스턴스 목록에서 종량제 인스턴스를 선택한 다음 **폐기/반환**을 클릭하여 수동으로 반환할 수 있습니다. 인스턴스가 반환된 후에는 **격리됨** 상태가 되며 액세스할 수 없는 24시간 동안 유지됩니다. 복구하려면 휴지통 목록에서 복구할 수 있습니다.

인스턴스가 반환된 후 상태가 **격리됨**으로 변경되면 관련 요금이 발생하지 않습니다.

>!
>- 인스턴스가 격리되면 해당 IP가 해제되며 인스턴스가 복구된 후에는 원래 IP를 되찾지 못할 수 있습니다.
>- 인스턴스가 격리된 후에는 업그레이드, 매개변수 수정, 계정 생성 또는 수정, 롤백 또는 이름 변경이 불가능합니다.

### 작업 단계
1. [TDSQL for MySQL 콘솔](https://console.cloud.tencent.com/tdsqld)에 로그인합니다. 인스턴스 목록에서 인스턴스를 선택하고 상단의 **더보기**>**폐기/반환**을 클릭합니다.
2. 팝업 창에서 동의를 선택한 뒤, **확인**을 클릭합니다.
![](https://main.qcloudimg.com/raw/16f8187856e09385505c70bde7fb3912.png)
휴지통으로 이동하면 인스턴스가 **격리됨** 상태인 것을 확인할 수 있습니다.

## 인스턴스 복구
격리된 인스턴스는 정상 실행 상태로 복구할 수 있습니다. 복구까지 몇 분 정도 소요될 수 있습니다. 복구된 인스턴스는 격리 전의 원래 IP가 아닌 새 IP를 가질 수 있습니다.

### 작업 단계
1. [TDSQL for MySQL 콘솔](https://console.cloud.tencent.com/tdsqld)에 로그인하고 휴지통 목록에서 인스턴스를 찾은 다음 **복구/시작**을 클릭합니다.
2. 팝업 창에서 **확인**을 클릭합니다.

## 인스턴스 폐기
더 이상 인스턴스가 필요하지 않으면 반환할 수 있습니다. 반환되면 **격리됨** 상태가 되며 휴지통으로 이동되어 만료되면 자동으로 제거됩니다. 또는 **지금 완전 삭제**를 클릭하여 완전히 종료할 수 있습니다.

### 주의 사항
- 인스턴스가 제거된 후에는 해당 데이터를 복구할 수 없습니다. 데이터를 미리 백업하십시오.
- 인스턴스가 제거되면 해당 IP 리소스가 동시에 해제되고 재해 복구 인스턴스는 동기화 연결을 중지하고 기본 인스턴스로 자동 승격됩니다.
