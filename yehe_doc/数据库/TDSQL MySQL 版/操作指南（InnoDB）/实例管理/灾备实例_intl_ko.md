본 문서에서는 콘솔에서 재해 복구/읽기 전용 인스턴스를 만들고 관리하는 방법을 설명합니다.

## 작업 시나리오
TDSQL for MySQL은 가용존/리전 간 재해 복구 읽기 전용 인스턴스를 제공하여 서비스 연속성, 데이터 안정성 및 규정 준수 요구 사항이 더 높은 애플리케이션에 대한 데이터 안정성을 개선하는 동시에 저렴한 비용으로 지속적인 서비스를 제공할 수 있도록 지원합니다.
>?재해 복구/읽기 전용 인스턴스 요금은 원본 인스턴스와 동일합니다. [제품 가격](https://intl.cloud.tencent.com/document/product/1042/35777)을 참고하십시오.

## 적용 시나리오
- 원격 재해 복구: 데이터 보안을 보장하기 위해 재해 복구 인스턴스를 사용하여 여러 리전의 비즈니스 및 데이터를 백업할 수 있습니다. 가용존/리전 장애로 인해 인스턴스를 사용할 수 없게 된 경우 가용존/리전 간 재해 복구 인스턴스로 신속하게 전환하여 비즈니스에 미치는 영향을 최소화할 수 있습니다.
- 근거리 액세스: 특정 가용존의 인스턴스를 원본 인스턴스로 사용하고 다른 가용존/리전의 인스턴스를 읽기 전용 인스턴스로 사용할 수 있어 사용자에게 근거리 액세스, 원격 읽기 기능 및 향상된 액세스 속도를 제공합니다.
- 다중 리전 배포: TDSQL for MySQL 인스턴스는 여러 지역에 배포할 수 있습니다. 인스턴스가 가용존/리전에서 네트워크 변동 또는 가용성이 없는 경우 비즈니스 요구 사항에 따라 다른 가용존/리전으로 전환할 수 있습니다.

## 기능
- 재해 복구/읽기 전용 인스턴스는 읽기 전용 액세스를 위해 별도의 데이터베이스 연결 주소를 제공합니다. 더 낮은 비용의 장치 이중화로 근거리 액세스 및 데이터 분석에 사용할 수 있습니다.
- 원본 인스턴스는 다른 리전 및 가용존에 배포할 수 있는 재해 복구 읽기 전용 인스턴스 하나를 생성할 수 있습니다.
- 재해 복구/읽기 전용 인스턴스는 고가용성(1-원본-1-복제본 및 1-원본-2-복제본) 아키텍처를 지원하여 데이터베이스에 대한 단일 장애점을 방지합니다.
- 원본 인스턴스가 실패하면 재해 복구/읽기 전용 인스턴스가 몇 초 만에 활성화되어 전체 읽기/쓰기 기능을 제공할 수 있습니다.
- 재해 복구/읽기 전용 인스턴스의 데이터는 공용 네트워크보다 대기 시간이 짧고 안정성이 뛰어난 사설망을 통해 동기화됩니다.
- 현재는 프로모션 기간으로 전용 회선 트래픽을 무상으로 이용할 수 있습니다. 상용화 후 요금 부과 시 별도로 공지합니다. 

## 기능 제한
- 재해 복구/읽기 전용 인스턴스는 매개변수 설정 및 계정 관리 기능을 지원하지 않습니다.
- 재해 복구/읽기 전용 인스턴스의 데이터베이스 버전은 기본적으로 원본 인스턴스와 동일합니다. 인스턴스 사양 및 디스크 크기는 원본 인스턴스보다 크거나 같아야 합니다.

## 작업 단계
### 재해 복구/읽기 전용 인스턴스 생성
1. [TDSQL for MySQL 콘솔](https://console.cloud.tencent.com/tdsqld)에 로그인한 후, 인스턴스 목록에서 인스턴스 ID를 클릭하여 인스턴스 관리 페이지로 이동합니다.
2. 인스턴스 세부 정보 페이지의 인스턴스 아키텍처 다이어그램에서 **재해 복구/읽기 전용 인스턴스 추가**를 클릭하고 인스턴스 구매 페이지로 들어갑니다.
![](https://qcloudimg.tencent-cloud.cn/raw/cc2132fa43b38fab37e5ed2761d3a0be.png)
3. 구매 페이지에서 재해 복구/읽기 전용 인스턴스의 청구 모드, 리전 및 기타 기본 정보를 선택하고 **구매하기**를 클릭합니다.
>?
>- 생성을 완료하는 데 필요한 시간은 데이터 양에 따라 다르며 생성 중에는 콘솔의 원본 인스턴스에 대해 어떤 작업도 수행할 수 없습니다. 적절한 시기에 하는 것이 좋습니다.
>- 현재 전체 인스턴스 데이터 동기화만 지원합니다. 디스크 용량이 충분한지 확인하십시오.
>- 원본 인스턴스가 실행 중 상태이고 실행 중인 작업이 없는지 확인합니다. 그렇지 않으면 동기화 작업이 실패할 수 있습니다.  
3. 결제 후 인스턴스 목록으로 돌아가 인스턴스를 초기화하면 다음 작업을 진행할 수 있습니다.

### 재해 복구/읽기 전용 인스턴스 관리
- **재해 복구/읽기 전용 인스턴스 보기**
 재해 복구/읽기 전용 인스턴스가 있는 리전에서 보고, 인스턴스 목록에서 필터링할 수 있습니다.
![](https://qcloudimg.tencent-cloud.cn/raw/12a0753a3f9e67b4ebbffe99efe4818f.png)
- **원본 인스턴스와 재해 복구 읽기 전용 인스턴스 간의 관계 보기**
인스턴스 세부 정보 페이지의 인스턴스 아키텍처 다이어그램에서 원본 인스턴스와 재해 복구 읽기 전용 인스턴스 간의 관계를 볼 수 있습니다.
![](https://qcloudimg.tencent-cloud.cn/raw/de145e7a78a76fb7902a7388e6e56de0.png)
- **재해 복구 읽기 전용 인스턴스 기능**
 재해 복구/읽기 전용 인스턴스는 인스턴스 세부 정보, 샤드 관리, 모니터링 및 알람, 매개변수 설정, 데이터 보안, 백업 및 복원, 성능 최적화 기능을 제공합니다.


### 재해 복구/읽기 전용 인스턴스를 원본 인스턴스로 업그레이드
필요에 따라 재해 복구/읽기 전용 인스턴스를 콘솔에서 원본 인스턴스로 전환할 수 있습니다.
1. [TDSQL for MySQL 콘솔](https://console.cloud.tencent.com/dcdb)에 로그인하여 인스턴스 목록에서 타깃 재해 복구/읽기 전용 인스턴스를 선택한 후 인스턴스 ID를 클릭하여 인스턴스 관리 페이지로 들어갑니다.
2. 오른쪽 상단 모서리에 있는 **원본 인스턴스로 전환**을 클릭하여 재해 복구/읽기 전용 인스턴스를 원본 인스턴스로 업그레이드합니다. 전환 후에는 원본 인스턴스와의 동기화 링크가 연결 해제되어 업그레이드된 인스턴스가 데이터 쓰기 기능과 전체 TDSQL for MySQL 기능을 사용할 수 있습니다.
>!동기화 연결이 끊어지면 재연결할 수 없습니다. 작업 시 주의하십시오.
