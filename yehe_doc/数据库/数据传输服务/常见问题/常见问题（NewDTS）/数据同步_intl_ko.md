
### 원본/대상 인스턴스 HA(High Availability) 전환 시 동기화 서비스가 영향을 받습니까?
- 원본 인스턴스가 GTID(Global Transaction Identifier)를 지원 및 활성화할 때 증분 동기화 단계에서 원본 인스턴스에서 HA가 발생하면 서비스가 자동으로 재연결되고 HA가 완료된 후 동기화 데이터 스트림이 빠르게 복구됩니다.
- 증분 동기화 단계에서 대상 인스턴스 HA가 발생하면 서비스도 자동으로 다시 연결되며 HA가 완료된 후 동기화 데이터 스트림이 빠르게 복구됩니다.

### 상위 버전 인스턴스의 데이터를 하위 버전 인스턴스로 동기화하는 기능을 지원합니까?
지원하지 않습니다. 동일한 유형의 데이터베이스에 대해 대상 인스턴스의 주요 버전은 원본 인스턴스보다 낮아서는 안 됩니다. MySQL 인스턴스를 예를 들면, MySQL 5.7 인스턴스은 MySQL 5.6으로 동기화를 지원하지 않습니다.

### 원본/대상이 오프 클라우드 인스턴스가 될 수 있습니까?
가능합니다. 온 클라우드의 데이터베이스는 오프 클라우드로 동기화할 수 있으며 액세스 모드는 공중망, CCN 등을 지원합니다.

### 양방향 동기화 토폴로지에서 DDL을 양방향으로 동기화할 수 있습니까?
아니오. 동기화 인스턴스를 생성할 때 인스턴스 중 하나만 DDL 동기화를 수행하도록 허용할 수 있습니다. 그렇지 않으면 점검 알고리즘이 DDL 루프를 점검하고 인스턴스 중 하나의 생성을 금지합니다.

### 비(Non) 트랜잭션 엔진이 지원됩니까?
현재 기술 솔루션은 트랜잭션의 원자성에 따라 달라지는 트랜잭션 출처를 표시하기 위해 트랜잭션의 라우팅 정보를 사용합니다. 비(Non) 트랜잭션 엔진 기반의 DB 테이블은 트랜잭션의 원자성을 훼손하여 데이터의 일관성을 보장할 수 없으므로 사용하지 않는 것이 좋습니다.
