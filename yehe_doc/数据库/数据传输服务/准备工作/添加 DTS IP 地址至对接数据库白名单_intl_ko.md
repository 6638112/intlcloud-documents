## 작업 시나리오
데이터 마이그레이션, 동기화 또는 구독 작업을 생성할 때 서로 통신할 수 있도록 DTS IP 주소를 원본 및 대상 데이터베이스의 얼로우리스트에 추가해야 합니다.

데이터베이스 얼로우리스트에 DTS IP 주소를 추가하지 않으면 연결 테스트가 실패할 수 있으며 테스트가 실패하면 추가하라는 메시지가 표시됩니다.

![](https://qcloudimg.tencent-cloud.cn/raw/631f9a6a9910f954516bdf69bda71736.png)

## 원칙

원본 또는 대상 데이터베이스 얼로우리스트에 각 리전의 IP 주소를 추가합니다. IP 주소 정보는 [DTS IP 주소](#1)를 참고하십시오.

> ?원본 또는 대상 데이터베이스가 MySQL/PostgreSQL/TDSQL-C PostgreSQL/TDSQL TDStore, TDSQL PostgreSQL/TDSQL-A PostgreSQL인 경우 DTS IP 주소가 데이터베이스의 보안 그룹 규칙에 자동으로 추가됩니다. 이 외의 시나리오에서는 얼로우리스트에 수동으로 추가해야 합니다.

예시1: 원본 데이터베이스가 베이징 리전의 자체 구축 MySQL이고 대상 데이터베이스가 광저우 리전의 TencentDB for MySQL인 경우 [DTS IP 주소](#1)에서 베이징 리전의 IP 주소를 가져와 추가해야 합니다. 원본 데이터베이스 얼로우리스트에 추가합니다. 대상 데이터베이스가 TencentDB for MySQL이므로 DTS IP 주소가 대상 데이터베이스 얼로우리스트에 자동으로 추가됩니다.

예시2: 원본 데이터베이스가 베이징 리전에 자체 구축 MongoDB이고 대상 데이터베이스가 광저우 리전의 TencentDB for MongoDB인 경우 [DTS IP 주소](#1)에서 두 리전의 IP 주소를 각각 가져와 데이터베이스 얼로우리스트에 추가해야 합니다.

## 작업 단계

[DTS IP 주소](#1)에서 데이터베이스 리전의 IP 주소를 가져와 원본 또는 대상 데이터베이스 얼로우리스트에 추가합니다.

- 자체 구축된 데이터베이스의 경우 방화벽 설정 시 DTS IP 주소가 데이터베이스에 액세스할 수 있도록 허용합니다.
  - Windows: 제어판을 열고 Windows Defender 방화벽을 찾아 방화벽 정책을 확인합니다.
  - Linux: ‘iptables -L’ 명령을 실행하여 서버의 방화벽 정책을 확인합니다.

- TencentDB 데이터베이스 또는 CVM에 자체 구축된 데이터베이스의 경우 아래 지침에 따라 DTS IP 주소를 보안 그룹에 추가합니다.
  1. 원본 데이터베이스에 로그인하고 인스턴스 목록에서 인스턴스 ID를 클릭하여 인스턴스 관리 페이지로 이동합니다.
  2. 인스턴스 관리 페이지에서 **보안 그룹** 또는 **데이터 보안** 탭을 선택하고 DTS IP 주소를 보안 그룹에 추가합니다.
     ![](https://qcloudimg.tencent-cloud.cn/raw/276d6b7c7f112d3630d040496d9dbc07.png)
  
- 타사 클라우드 데이터베이스의 경우 해당 데이터베이스의 보안 그룹에 DTS IP 주소를 추가합니다.
