
## 확인 사항
- 요구 사항 확인: 대상 데이터베이스가 샤딩된 인스턴스인 경우 샤드 키를 미리 설정할 수 있습니다. 원본 및 대상 데이터베이스의 테이블 샤드 키가 다른 경우 경고가 트리거됩니다. 작업이 차단되지는 않지만 비즈니스에 영향을 미칩니다. 경고 무시 여부를 평가 및 결정해야 합니다.
- 비즈니스 영향: 일부 샤드 키가 다른 경우 마이그레이션 또는 동기화 작업이 실패됩니다.

## 수정 방법
대상 데이터베이스에 사전 설정된 샤드 키가 있는 경우 다음 명령을 실행하여 원본 데이터베이스를 샤딩합니다.
```
sh.shardCollection("<database>.<collection>", { <shard key> : "hashed" } , false, {numInitialChunks: 사전 설정 chunk 수}) 
```
확인 작업을 다시 실행합니다.
