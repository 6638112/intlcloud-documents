
## 확인 사항
#### MySQL/TDSQL-C MySQL/TDSQL MySQL/PostgreSQL 확인 요건

대상 데이터베이스의 저장 공간은 원본 데이터베이스에서 마이그레이션할 테이블 크기의 1.2배 이상이어야 합니다.
전체 데이터 마이그레이션은 INSERT 작업을 동시에 실행하여 대상 데이터베이스의 일부 테이블에 데이터 조각을 생성합니다. 따라서 전체 마이그레이션이 완료된 후 대상 데이터베이스의 테이블 크기가 원본 데이터베이스의 테이블 크기보다 클 수 있습니다.

#### MongoDB 확인 요건
대상 데이터베이스의 저장 공간은 원본 데이터베이스에서 마이그레이션할 테이블 크기의 1.3배 이상이어야 합니다.

#### Redis 확인 요건

대상 데이터베이스의 저장 공간은 원본 데이터베이스 저장 공간 크기의 1.5배 이상이어야 합니다.

## 수정 방법
- 대상 데이터베이스에서 일부 데이터를 삭제하여 충분한 공간을 확보합니다. 
- 마이그레이션을 위해 더 큰 용량의 인스턴스를 사용하려면 [대상 데이터베이스의 스토리지 사양을 업그레이드](https://intl.cloud.tencent.com/document/product/236/19707) 하십시오.
