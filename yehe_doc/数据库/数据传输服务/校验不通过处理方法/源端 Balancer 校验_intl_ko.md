
## 확인 사항
- 확인 요건: 원본 데이터베이스가 샤딩된 인스턴스인 경우 마이그레이션을 시작하기 전에 해당 Balancer를 비활성화해야 합니다.
- 확인 설명: 증분 마이그레이션 작업은 Oplog를 가져옵니다. Balancer가 활성화된 경우 원본 데이터베이스의 moveChunk로 인해 대상 데이터베이스의 데이터 불일치가 발생할 수 있습니다.

## 수정 방법
1. 원본 데이터베이스에 로그인합니다.
2. 다음 명령을 실행하여 원본 데이터베이스 Balancer를 비활성화합니다.
```
sh.stopBalancer()
sh.getBalancerState()
```
3. 확인 작업을 다시 실행합니다.
