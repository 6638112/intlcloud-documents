## 확인 사항
원본 데이터베이스는 Slave 노드여야 합니다. 그렇지 않으면 점검 항목이 알람을 보고합니다. 알람 항목은 작업을 계속하는 것을 방해하지 않으며 사용자는 이를 무시하고 작업을 계속할 수 있지만 영향에 주의해야 합니다.

마이그레이션 프로세스 동안 DTS는 원본 데이터베이스에서 bgsave 작업을 수행하며, 원본 데이터베이스의 메모리와 리소스를 사용합니다. 원본 데이터베이스가 Master 노드인 경우 입력 작업에 더 큰 영향을 미칩니다.

- 원본 데이터베이스가 Tencent Cloud Redis 데이터베이스인 경우 기본 데이터 마이그레이션 프로세스는 Slave 노드를 사용하여 마이그레이션하므로 사용자는 주의할 필요가 없습니다.
- 원본 데이터베이스가 로컬에서 자체 구축된 Redis 데이터베이스인 경우 Master 마이그레이션을 사용하는 시나리오가 있을 수 있으므로 점검 결과 알람이 보고되면 원본 데이터베이스를 복제본 데이터베이스로 수정하는 것이 좋습니다. 


## 수정 방법
마이그레이션 작업을 재구성하고 원본 데이터베이스의 매개변수를 Slave 노드 정보로 구성합니다.
