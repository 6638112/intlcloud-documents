## 확인 사항
대상 데이터베이스 사용자가 원본 데이터베이스 사용자와 충돌하는지 확인합니다.

## 수정 방법
전체 데이터베이스 마이그레이션에서 대상 데이터베이스와 원본 데이터베이스의 계정이 동일한 경우 해당 계정을 삭제해야 합니다.
- 대상 데이터베이스의 계정이 초기화 계정인 경우 이를 사용하여 데이터베이스에 로그인하고 다음 명령을 실행합니다.
```
create user 신규 사용자 with password ‘비밀번호’;
grant pg_tencentdb_superuser to 신규 사용자 이름;
alter user 신규 사용자 with CREATEDB;
alter user 신규 사용자 with CREATEROLE;
```
- 대상 데이터베이스의 계정이 신규 사용자인 경우 이를 사용하여 데이터베이스에 로그인하고 충돌하는 사용자를 삭제합니다.
```
drop user 사용자 충돌;
# 충돌하는 사용자에게 리소스 종속성이 있는 경우 다음 명령문을 실행하여 먼저 리소스 owner를 수정합니다(아래 예시와 같은 테이블 사용).
alter table 테이블 이름 owner to 신규 사용자
```
충돌하는 사용자를 삭제한 후 확인 작업을 다시 실행합니다.
