
## MySQL/TDSQL MySQL 확인 사항 
- 확인 요건: 대상 데이터베이스 버전은 원본 데이터베이스 버전 이상이어야 하고 마이그레이션 및 동기화 작업의 모든 버전은 버전 요구 사항을 충족해야 합니다.
- 확인 설명: 여기에서 버전은 주 버전 번호로 구분됩니다. 예를 들어 5.6.x는 5.6.x, 5.7.x 이상으로의 마이그레이션 또는 동기화를 지원합니다. 마지막 숫자는 제한되지 않는 보조 버전 번호입니다. 예를 들어 5.6.5를 5.6.4로 마이그레이션하거나 동기화할 수 있지만 호환성 문제가 있을 수 있습니다.

## PostgreSQL 확인 사항
- PostgreSQL 10.x 이하 버전(예: 9.x)은 전체 + 증분 마이그레이션을 지원하지 않습니다. 원본 데이터베이스에 대해 증분 마이그레이션 작업이 설정된 경우 확인에 실패합니다.

- 버전이 다른 경우 특별한 호환성 문제가 있을 수 있으며 마이그레이션 중에 알람이 표시됩니다. 각 버전의 호환성 보고서를 읽고 비즈니스에서 일부 호환되지 않는 기능을 사용하는지 확인할 수 있습니다.

## MongoDB 확인 사항
원본 및 대상 데이터베이스 버전은 MongoDB 지원 버전이어야 합니다.

## SQL Server 확인 사항

기본 버전에서 고가용성 버전(듀얼 서버 고가용성 에디션 및 클러스터 에디션 포함)으로의 마이그레이션만 지원되며 대상 데이터베이스의 버전 번호는 원본 데이터베이스의 버전 번호보다 높아야 합니다. 

## 수정 방법
[데이터 마이그레이션이 지원되는 데이터베이스](https://intl.cloud.tencent.com/document/product/571/42580) 및 [데이터 동기화가 지원되는 데이터베이스](https://intl.cloud.tencent.com/document/product/571/42579)에 따라 원본 및 대상 데이터베이스를 확인합니다. 원본 또는 대상 데이터베이스 버전이 지원되지 않는 경우 대상 데이터베이스 버전을 업그레이드하거나 더 높은 버전의 데이터베이스 인스턴스를 사용하십시오.
