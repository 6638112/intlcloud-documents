## 현상 설명
마이그레이션/동기화 작업이 너무 오래 걸리거나 중단됩니다. 

## 예상 원인
- 마이그레이션된 데이터의 양이 많습니다.
- 원본 데이터베이스에 느린 SQL 문이 있습니다.
- 원본 데이터베이스 콘텐츠가 호환되지 않습니다.
- 대역폭이 제한적이거나 네트워크 지터가 있습니다.
- 증분 마이그레이션 또는 동기화 중에는 원본 데이터베이스에 데이터가 기록되지 않습니다.
> ?마이그레이션 유형으로 ‘전체 + 증분 마이그레이션’을 선택하면 전체 마이그레이션 작업이 완료된 후 증분 마이그레이션 작업이 계속 진행되며, **작업** 열에서 **완료**를 클릭하여 수동으로 중지해야 합니다. 그렇지 않으면 작업이 계속 실행되며 작업이 중단된 경우가 아닙니다.

## 솔루션
#### 마이그레이션된 데이터 볼륨이 높음
데이터 볼륨이 높아 마이그레이션/동기화 작업이 느려집니다.

#### 원본 데이터베이스의 느린 SQL 문
원본 데이터베이스에 느린 SQL 문이 있는지 확인합니다. 그렇다면 명령문을 처리하십시오. 그렇지 않은 경우 다른 원인을 확인하십시오.

#### 원본 데이터베이스 비준수 콘텐츠
원본 데이터베이스의 콘텐츠가 비준수 콘텐츠입니다. 예를 들어 원본 데이터베이스에 기본 키가 없는 테이블이 있는 경우 이러한 테이블과 관련된 대규모 쿼리는 작업 속도를 저하시킵니다. 이러한 테이블에 기본 키를 추가하거나 마이그레이션하지 않는 것이 좋습니다.

#### 네트워크 문제
- CCN을 사용하여 접속하는 경우 CCN에서 설정한 대역폭을 확인하십시오. CCN은 모든 리전 간에 10Kbps 미만의 대역폭만 무료로 제공하므로 DTS가 데이터를 전송하기에는 부족합니다. 이 경우 더 높은 대역폭을 구성해야 합니다.
- 자체 구축한 데이터베이스를 사용하는 경우 네트워크 대역폭이 제한되어 있는지 확인하십시오.

#### 증분 마이그레이션 또는 동기화 중에 원본 데이터베이스에 데이터가 기록되지 않음
증분 마이그레이션 또는 동기화에서 원본 데이터베이스에 장기간 기록된 데이터가 없거나 빈 Binlog가 있는 경우 원본 데이터베이스에 데이터를 기록하여 작업을 재개할 수 있습니다.
