본문은 DTS의 데이터 마이그레이션 기능을 사용하여 SQL Server에서 TencentDB for SQL Server로 데이터를 마이그레이션하는 방법을 설명합니다.

## 주의 사항 
- 전체 데이터 마이그레이션 중에 DTS는 원본 데이터베이스 리소스를 사용하므로 원본 데이터베이스의 로드와 부담이 증가할 수 있습니다. 데이터베이스 설정이 낮으면 사용량이 적은 시간에 데이터를 마이그레이션하는 것이 좋습니다.
- 전체 마이그레이션은 잠긴 테이블로 구현되어 몇 초 동안 쓰기 작업이 차단됩니다.

## 전제 조건
- [TencentDB for SQL Server](https://intl.cloud.tencent.com/document/product/238/31571)를 생성해야 합니다.
- 원본 및 타깃 데이터베이스는 [데이터 마이그레이션 지원 데이터베이스](https://intl.cloud.tencent.com/document/product/571/42647)에 설명된 대로 마이그레이션 기능 및 버전에 대한 요구 사항을 충족해야 합니다.
- 모든 [준비 작업](https://intl.cloud.tencent.com/document/product/571/42652)을 마쳐야 합니다.
- 원본 데이터베이스가 있는 서비스는 파일 공유 포트 445를 열어야 합니다.
- 원본 데이터베이스는 ‘전체 복구 모드’로 설정해야 하며 마이그레이션 전에 전체 백업을 수행하는 것이 좋습니다.
- 원본 데이터베이스의 로컬 디스크 공간은 나머지 여유 공간이 마이그레이션할 데이터베이스 크기에 맞도록 충분히 커야 합니다. 
- 원본 데이터베이스가 TencentDB for SQL Server 또는 TencentDB for SQL Server Basic Edition 인스턴스(예: 공용 네트워크/CVM 기반 자체 구축 인스턴스 또는 다른 클라우드의 인스턴스)가 아닌 경우 타깃은 마이그레이션에 sysadmin 권한이 있는 계정을 사용해야 하며 xp_cmdshell 저장 프로시저를 실행할 수 있어야 합니다. 원본 데이터가 SQL Server High-Availability Edition 또는 Cluster Edition 인스턴스용 TencentDB인 경우 권한 제한이 없습니다.
- 마이그레이션 원본의 SQL 서비스 시작을 local로 변경해야 하며 원본에서 마이그레이션할 데이터베이스 계정은 무제한이지만 sysadmin 권한이 있어야 합니다.
이미지와 같이 마이그레이션 원본의 SQL 서비스가 시작되고, 시작 구성의 로그인 ID 내장 계정은 Local System으로 수정해야 합니다.

## 제한
- 동일한 원본 인스턴스에 대해 한 번에 하나의 마이그레이션 작업만 시작할 수 있습니다.
- 현재 중국 본토와 중국홍콩 간의 리전 간 마이그레이션은 지원되지만 다른 리전 간의 마이그레이션은 지원되지 않습니다.
- 데이터베이스 수준 마이그레이션만 지원되며(즉, 데이터베이스의 모든 객체는 함께 마이그레이션되어야 함) 단일 테이블 마이그레이션은 지원되지 않습니다.
- 인스턴스 수준의 login, job agent, 트리거 및 db link(link server)는 마이그레이션할 수 없습니다.

## 작업 제한
- 마이그레이션하는 동안 원본 및 대상 데이터베이스와 포트 번호의 사용자 정보(사용자 이름, 암호 및 권한 포함)를 수정하거나 삭제하지 마십시오. 그렇지 않으면 마이그레이션 작업이 실패합니다.
- 증분 동기화 중에는 트랜잭션 로그 백업을 수행하지 마십시오. 그렇지 않으면 트랜잭션 로그가 잘리고 비연속 상태가 됩니다.
- 전체 데이터 마이그레이션만 수행하는 경우 마이그레이션 중에 원본 데이터베이스에 새 데이터를 쓰지 마십시오. 그렇지 않으면 원본 및 대상 데이터베이스의 데이터가 일치하지 않습니다. 데이터 쓰기가 있는 시나리오에서 실시간으로 데이터 일관성을 보장하려면 전체+증분 데이터 마이그레이션을 선택하는 것이 좋습니다.
- 전체 + 증분 데이터 마이그레이션의 경우 **완료**를 클릭하고 작업 상태가 **완료됨**이 된 후 원본 데이터베이스에 새 데이터를 쓰지 말고 **완료**를 클릭하십시오. 2분 동안 쓰기를 중단하는 것이 좋습니다. 그렇지 않으면 원본 및 대상 데이터베이스의 데이터가 일치하지 않을 수 있습니다.

5. 확인 작업 페이지에서 확인을 진행하고 확인 통과 후 **작업 실행**을 클릭합니다.
    확인이 실패한 경우 [확인 실패 수정 방법](https://intl.cloud.tencent.com/document/product/571/42552)의 설명 대로 문제를 수정하고 확인 작업을 다시 시작합니다.
 - 실패: 확인 항목 실패로 작업이 차단되었음을 나타냅니다. 문제를 수정하고 확인 작업을 다시 실행해야 합니다.
 - 알람: 확인 항목이 요구 사항을 완전히 충족하지 못함을 나타내며, 작업을 계속할 수 있지만 비즈니스에 영향을 미칩니다. 알람을 무시할지 아니면 문제를 해결하고 계속할지 선택합니다.
6. 데이터 마이그레이션 작업 목록으로 돌아가면 작업이 준비 중 상태로 들어간 것을 볼 수 있습니다. 1 - 2분 후 데이터 마이그레이션 작업이 시작됩니다.
   - **전체 마이그레이션** 선택: 완료되면 작업이 자동으로 중지됩니다.
   - **전체 + 증분 마이그레이션** 선택: 전체 마이그레이션이 완료된 후 마이그레이션 작업은 자동으로 중지되지 않는 증분 데이터 동기화 단계에 자동으로 들어갑니다. 증분 데이터 동기화를 수동으로 중지하려면 **완료**를 클릭해야 합니다.
      - 적절한 시기에 증분 데이터 동기화 및 비즈니스 전환을 수동으로 완료합니다.
      - 마이그레이션 작업이 **준비 완료** 단계인지 확인하고 몇 분 동안 원본 데이터베이스에 쓰기를 중지한 후 증분 동기화를 수동으로 완료합니다.
7. (옵션) 작업 조회, 삭제 등 작업을 수행하려면 해당 작업을 클릭하고 **작업** 열에서 해당 작업을 선택합니다. 자세한 내용은 [작업 관리](https://intl.cloud.tencent.com/document/product/571/42637)를 참고하십시오.
8. 마이그레이션 작업 상태가 **작업 성공**이 되면 정식으로 컷오버할 수 있습니다. 자세한 내용은 [컷오버 설명](https://intl.cloud.tencent.com/document/product/571/42612)을 참고하십시오.
