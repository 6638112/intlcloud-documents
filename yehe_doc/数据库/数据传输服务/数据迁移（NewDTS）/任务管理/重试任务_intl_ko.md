
## 작업 시나리오
마이그레이션 작업이 시작된 후 예외로 인해 중단된 경우 재시도할 수 있습니다.

>?현재 MySQL/TDSQL-C MySQL/MariaDB/Percona/TDSQL MySQL 간의 마이그레이션만 데이터 마이그레이션 작업을 재시도할 수 있습니다.

작업을 다시 시도하기 위해 DTS는 Binlog 오프셋을 사용하여 작업이 중지된 데이터 포인트를 찾은 다음 원본 데이터베이스에서 실행되지 않은 첫 번째 GTID 위치에서 Binlog를 가져오고 계속해서 데이터를 사용합니다.

다양한 데이터 마이그레이션 시나리오에서 작업 재시도에 대한 지원은 다음과 같습니다.
- 원본 데이터베이스 내보내기: 작업이 잠기기 전에 원본 인스턴스에서 시간이 많이 걸리는 SQL 문으로 인해 데이터베이스 잠금 실패로 인해 작업이 중단된 경우에만 작업 재시도가 지원되고, 다른 경우 재시도가 지원되지 않습니다.
- 데이터 가져오기: 재시도가 지원되지 않습니다.
- 증분 동기화: 재시도가 지원됩니다.

## 작업 단계
[DTS 콘솔](https://console.cloud.tencent.com/dts/migration)에 로그인하고 왼쪽 사이드바에서 **데이터 마이그레이션**을 선택하고 지정된 마이그레이션 작업을 선택한 후 **작업** 열에서 **재시도**를 클릭합니다.
