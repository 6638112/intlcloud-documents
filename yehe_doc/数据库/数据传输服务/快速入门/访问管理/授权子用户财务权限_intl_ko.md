
## 작업 시나리오  
서브 계정은 일반적으로 재무 권한이 없으며, DTS 정액 과금제 인스턴스 구매 시 주문서를 제출하면 아래와 같은 안내가 나타납니다. 주문서 결제를 위해서는 루트 계정이 필요합니다. 서브 계정에게 재무 권한을 부여한 후 서브 계정으로 직접 정액 과금제 인스턴스를 구입할 수 있으며, 루트 계정의 계정 잔액으로 결제할 수 있습니다.

## 전제 조건
[서브 계정 생성 및 권한 부여](https://intl.cloud.tencent.com/document/product/571/47358)를 완료해야 합니다.

## 작업 단계
1. 루트 계정으로 [CAM 콘솔](https://console.cloud.tencent.com/cam)에 로그인합니다.
2. 왼쪽 사이드바에서 **정책**을 클릭한 다음 오른쪽에서 **사용자 정의 정책 생성**을 클릭하고 **정책 구문으로 생성**을 선택합니다.      
3. **빈 템플릿**을 선택하고 **다음**을 클릭합니다.  
4. 정책을 생성하고 필요에 따라 정책 이름 및 설명을 입력합니다. 정책 내용에 샘플 코드를 복사합니다.<br>
   
   <br>샘플 정책 구문: 
```
{
    "version": "2.0",
    "statement": [
        {
            "effect": "allow",
            "action": "finance:*",
            "resource": "qcs::dts:::*"
        }
    ]
}
```
5. **완료**를 클릭하고 정책 목록 페이지로 돌아간 다음 **사용자/그룹 연결**을 클릭합니다.   
6. 권한을 부여할 서브 계정(위 단계에서 새로 생성한 서브 계정)를 선택하고 **확인**을 클릭합니다.
