## 전체 프로세스
| **작업 프로세스**        | **설명**                                                     |
| ------------------- | ------------------------------------------------------------ |
| 1. 준비 작업         | 데이터 구독 작업 생성 이전에 네트워크 환경 [준비 작업](https://intl.cloud.tencent.com/document/product/571/42652)을 해야 합니다.                       |
| 2. 데이터 구독 작업 생성 | 이 섹션에서는 기본 샘플 구독 작업만 제공합니다. 더 많은 시나리오는 [데이터 구독](https://intl.cloud.tencent.com/document/product/571/42664)을 참고하십시오. |
| 3. 구독 작업 관리     | 구독 객체 수정과 같은 작업에 대한 자세한 내용은 [작업 관리](https://intl.cloud.tencent.com/document/product/571/42660)를 참고하십시오.                    |
| 4. 구독한 데이터를 소비     | 소비자 그룹을 관리하고 Kafka 클라이언트를 사용하여 데이터를 구독합니다. 자세한 내용은 [구독 데이터 소비](https://intl.cloud.tencent.com/document/product/571/39538)를 참고하십시오. |
| 5. 작업 중지         | 구독 작업이 완료된 후 수동 중지해야 합니다.                           |

## 데이터 구독 작업 예시

1. [DTS 콘솔](https://console.cloud.tencent.com/dts/dss)에 로그인한 뒤 왼쪽 사이드바에서 **데이터 구독**을 선택하고 **데이터 구독 생성**을 클릭합니다.
2. 데이터 구독 생성 페이지에서 적합한 설정을 선택하고 **즉시 구매**를 클릭합니다.
3. 구매 성공 후, 데이터 구독 리스트로 돌아가 **작업**열의 **구독 설정**을 클릭해 새로 구입한 구독의 설정을 진행합니다. 설정 완료 후 사용할 수 있습니다.
![](https://qcloudimg.tencent-cloud.cn/raw/e3d28df3f5199d18abcb0d789994bb4c.png)
4. 데이터베이스 구독 설정 페이지에서 적합한 설정을 선택하고 **다음 단계**를 클릭합니다.
![](https://qcloudimg.tencent-cloud.cn/raw/8fdd91f5a9071bbcb0e8471738113c07.png)
5. 구독 유형 및 대상 선택 페이지에서 구독 유형을 선택하고 **설정 저장**을 클릭합니다.
>?DTS는 시스템 테이블 구독을 지원하지 않고 ‘test’ 데이터베이스는 DTS에 의해 테스트 데이터로 인식되므로 구독 객체는 ‘test’라는 시스템 테이블과 데이터베이스를 제외합니다.
>
![](https://qcloudimg.tencent-cloud.cn/raw/e3bd14c693fa91096ef47f254de7200a.png)
6. 사전 인증 페이지에서 2 - 3분에 걸쳐 사전 인증 작업이 실행됩니다. 사전 인증이 완료된 후, **실행**을 클릭하여 데이터 구독 작업 설정을 완료합니다.
>?인증에 실패하면 오류 알림에 따라 구독할 인스턴스에서 수정하고 다시 인증을 진행합니다.
>
![](https://qcloudimg.tencent-cloud.cn/raw/458f6897c7b200a59c5c7fa58ecd6f81.png)
7. 시동을 클릭하면 구독 작업이 초기화되며 3 - 4분동안 실행됩니다. 초기화에 성공하면 **실행 중** 상태로 변합니다.
