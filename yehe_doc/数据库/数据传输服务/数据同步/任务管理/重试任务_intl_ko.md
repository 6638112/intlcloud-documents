
## 작업 시나리오
동기화 작업이 시작된 후 예외로 인해 중단된 경우 다시 시도할 수 있습니다.

>?현재 원본 데이터베이스가 MySQL 또는 TDSQL-C인 경우에만 동기화 작업을 재시도할 수 있습니다.

작업을 재시도하기 위해 DTS는 Binlog 오프셋을 기반으로 작업이 중단된 데이터 포인트를 식별하고 원본 데이터베이스에서 실행되지 않은 첫 번째 GTID 위치에서 Binlog를 가져오기 시작하고 데이터를 계속 소비합니다.

다양한 데이터 동기화 시나리오에서 작업 재시도에 대한 지원은 다음과 같습니다.
- 원본 데이터베이스 내보내기: 재시도가 지원되지 않습니다.
- 데이터 가져오기: 재시도가 지원되지 않습니다.
- 증분 동기화: 재시도가 지원됩니다.

## 작업 단계
[DTS 콘솔](https://console.cloud.tencent.com/dts/migration)에 로그인하고 왼쪽 사이드바에서 **데이터 동기화**를 선택하고 대상 동기화 작업을 선택한 후 **작업** 열에서 **더보기** > **재시도**를 클릭합니다.
