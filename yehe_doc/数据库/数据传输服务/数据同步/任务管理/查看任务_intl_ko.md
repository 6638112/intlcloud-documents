## 작업 시나리오

작업 세부 정보에서는 동기화 작업, 동기화 구성, 구조 초기화, 데이터 초기화, 모니터링 데이터 등의 다양한 정보를 볼 수 있습니다.

## 전제 조건
데이터 동기화 작업을 성공적으로 생성하고 [DTS 콘솔](https://console.cloud.tencent.com/dts/replication)에 로그인해야 합니다.

## 작업 단계
- 방법1: [데이터 동기화](https://console.cloud.tencent.com/dts/replication) 작업 리스트에서 대상 동기화 작업을 선택하고 해당 이름을 클릭합니다.
- 방법2: [데이터 동기화](https://console.cloud.tencent.com/dts/replication) 작업 목록에서 대상 동기화 작업을 선택하고 **작업** 열에서 **보기**를 클릭합니다.

### 동기화 작업 페이지
태스크, 원본 데이터베이스 및 대상 데이터베이스의 정보를 표시합니다.
![](https://qcloudimg.tencent-cloud.cn/raw/1b9c5f80502aafacfb6abf214e248b29.png)

### 동기화 설정 페이지
동기화 작업 설정을 표시합니다.
![](https://qcloudimg.tencent-cloud.cn/raw/37044c19f87ec1aaedc183de3398f36f.png)

### 구조 초기화 페이지
동기화 구성에서 **구조 초기화**를 선택하면 이 페이지에 관련 정보가 표시됩니다.

| 매개변수   | 설명                                             |
| ------ | ------------------------------------------------ |
| 객체 이름 | 대상 데이터베이스의 테이블 이름                               |
| 원본 데이터베이스   | 데이터베이스 및 테이블과 같은 원본 데이터베이스 객체의 이름         |
| 대상 데이터베이스 | 데이터베이스 및 테이블과 같은 대상 데이터베이스 객체의 이름           |
| 상태   | 현재 상태. 작업 실패 시 실패 원인 표시             |
| 작업   | 대상 데이터베이스에서 실행된 생성 구문을 보려면 [생성 명령문 보기] 클릭 |

![](https://qcloudimg.tencent-cloud.cn/raw/c6771f7eb82a16a24da4ef3d2b6055ad.png)

### 작업 초기화 페이지
동기화 구성에서 **데이터 초기화**를 선택하면 이 페이지에 관련 정보가 표시됩니다.
>?작업 실패 시 **실패 세부 정보 보기**를 클릭하여 문제를 해결할 수 있습니다.
>
![](https://qcloudimg.tencent-cloud.cn/raw/253e5bb2692158df8b5ce7f84c823e89.png)

### 모니터링 데이터 표시
모니터링 데이터가 수집되지 않으면 기본적으로 -가 표시됩니다.
