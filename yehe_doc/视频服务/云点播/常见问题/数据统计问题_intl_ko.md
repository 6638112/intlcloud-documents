### 어떤 통계 데이터를 볼 수 있습니까?
VOD 콘솔에서 대역폭/트래픽, 저장 용량 및 트랜스코딩에 대한 통계 데이터를 볼 수 있습니다.

- 대역폭/트래픽 통계: VOD 대역폭 및 도메인/지역/ISP별 트래픽 사용 내역, 트래픽 사용량 TOP10 지역, ISP 트래픽 사용량 비교.
- 스토리지 용량 통계: 현재 사용 중인 총 파일 수 및 저장 용량을 포함하여 시간대별 VOD 스토리지 용량 사용.
- 트랜스코딩 통계: 시간대별 VOD 트랜스코딩 작업 유형, 세부 정보 및 비율.

또한 VOD는 도메인/지역/ISP별 고유 IP 접속 요청 건수 및 접속 요청 건수에 대한 데이터 분석을 제공합니다. 재생 통계의 경우 VOD FileId 차원의 파일 재생 현황 쿼리 및 재생 횟수 TOP100 동영상 쿼리도 지원합니다.

### 요청 건수는 무엇입니까?

요청 건수는 일정 기간 동안 CDN으로 전송된 모든 재생 요청 건수를 나타냅니다. 이 데이터에는 Tencent Cloud 플레이어, 웹 페이지 및 사용자 정의 플레이어를 포함한 모든 소스의 통계가 포함되어 있습니다.
요청 건수는 동영상 형식과 관련이 있습니다. 형식이 MP4인 경우 재생 횟수와 같습니다. 형식이 HLS인 경우 M3U8 및 TS 부분에 대한 요청 건수와 같습니다. 이 값은 5분마다 한 번씩 업데이트됩니다.

### 재생 횟수 및 트래픽 통계는 무엇입니까?
- 파일 재생 횟수는 Tencent Cloud 플레이어에 의해 리포트되며 Tencent Cloud 플레이어를 사용하는 소스 통계만 포함됩니다. 3시간마다 한 번씩 업데이트됩니다.
- 트래픽 통계는 모든 소스(Tencent Cloud 플레이어, 웹 페이지 및 사용자 정의 플레이어 포함)의 통계를 포함하는 CDN 노드에 의해 보고되며 1시간마다 업데이트됩니다.

### 사용 통계의 대역폭 통계는 어떻게 집계됩니까?
VOD의 각 CDN 엣지 노드는 실시간으로 트래픽 데이터를 수집 및 컴퓨팅 센터에 보고하여, 해당 데이터를 도메인 이름의 총 트래픽 데이터로 집계하고, 집계된 대역폭 통계를 시간, 사용 트래픽 또는 시간별로 표시합니다.
- 1분 동안 생성된 총 트래픽이 6MB인 경우 해당 대역폭은 (6 x 8) ÷ 60 = 0.8Mbps입니다.
- 5분 단위의 통계를 기반으로 대역폭별 사용량을 계산하므로 해당 대역폭 값은 5분 단위 총 트래픽 ÷ 300초입니다.


### 콘솔의 사용 통계에 표시되는 트래픽 통계가 로그의 통계와 다른 이유는 무엇입니까?

VOD의 가속 도메인 로그의 다운스트림 바이트를 기반으로 계산된 트래픽은 애플리케이션 레이어의 데이터에 관한 것입니다.
네트워크를 통한 실제 데이터 전송 중에 생성된 트래픽은 애플리케이션 레이어 트래픽보다 약 `5%`-`15%` 더 많습니다.

- TCP/IP 헤더에 의한 소비: TCP/IP 기반 HTTP 요청에서 각 패킷의 최대 크기는 40바이트의 TCP/IP 헤더를 포함하여 최대 1,500바이트입니다. **이 헤더는 전송 중에 트래픽을 생성하지만 애플리케이션 레이어에서 계산될 수 없습니다**. 이 부분의 간접비는 약 `3%`입니다.
- TCP 재전송: 네트워크를 통한 정상적인 데이터 전송 중에 인터넷에서 약 `3%`-`10%`의 패킷이 손실되고 서버는 손실된 패킷을 재전송합니다. 이러한 유형의 트래픽은 전체 트래픽의 `3%`-`7%`를 차지하며, 애플리케이션 레이어에서 계산될 수 없습니다.

업계 표준에 따라 청구 가능한 트래픽은 위에서 설명한 애플리케이션 레이어 트래픽과 오버헤드의 합계입니다. VOD에서 애플리케이션 레이어 모니터링 통계는 `10%` 증가하므로, 과금되는 가속 트래픽(사용 통계에 표시됨)은 로그에 계산된 가속 트래픽의 약 `110%`입니다.
