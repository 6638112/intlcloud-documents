### ‘미디어 자산 관리’의 ‘Web 플레이어 코드 생성’에 표시되는 ‘HTML’ 및 ‘IFRAME’의 용도는 무엇입니까?

‘HTML’ 및 ‘IFRAME’은 사용자 정의 코드를 배포하는 방식으로, 용도는 다음과 같습니다.

**HTML**은 HTML 코드로, 웹 페이지 편집 시 플레이어를 삽입하는데 사용됩니다. 사용자가 웹 페이지를 방문할 때 이 플레이어가 표시되며 동영상을 재생합니다. 이 코드는 약간의 맞춤형 수정이 필요할 수 있으므로 HTML 프로그래밍 언어에 대한 지식이 있는 사용자에게 적합합니다. 코드는 모바일 장치 및 PC에 자동으로 적용될 수 있으며, Web SDK를 제공하여 매우 유연한 2차 개발을 진행할 수 있습니다. 자세한 내용은 [Web SDK 개발 가이드](http://video.qcloud.com/download/docs/QVOD_Player_Web_SDK_Developer_Guide.pdf)를 참고하십시오.

**IFRAME**을 사용하면 수정 없이 직접 플레이어 코드를 쉽게 배포할 수 있으며, 코드의 내용을 알 필요가 없이 HTML 페이지의 해당 위치에 코드를 복사하여 붙여넣기만 하면 됩니다. IFRAME 태그는 Chrome, IE, Safari 및 Firefox와 같은 주요 브라우저와의 호환성이 매우 높습니다.

>? 사용자가 iOS 또는 Android 모바일 장치에서 웹 페이지를 방문하는 경우, HTML5만 재생에 사용됩니다. PC에서 방문하는 경우, Flash가 우선적으로 사용됩니다. 브라우저가 Flash를 지원하지 않으면 HTML5가 사용됩니다. HTML5가 지원되지 않는 경우, 플레이어는 최신 브라우저를 다운로드하라는 메시지가 표시됩니다.

### Web 플레이어는 동적 워터마킹을 지원합니까?

아직 지원하지 않습니다. 추후 이 기능이 지원되면 API를 통해 시청자의 ID를 업로드하고, 이를 비디오에 동적으로 무작위 표시되게 하여 비디오 콘텐츠를 보호할 수 있습니다.


### Web 플레이어 코드는 모바일 장치를 지원합니까?

Web 플레이어 코드는 다양한 유형의 모바일 장치에서의 재생을 지원합니다.

### Web 플레이어는 롤 이미지를 지원합니까?

비디오 시작 부분에 롤 이미지를 추가할 수 있습니다.

### Web 플레이어와 비디오 파일의 관계는 무엇입니까? 얼마나 많은 Web 플레이어를 만들 수 있습니까?

웹 플레이어에는 플레이어 외관 및 롤과 같은 웹 플레이어 코드를 사용하여 재생할 때 비디오 파일에 대한 재생 매개변수 설정이 포함됩니다. 웹 플레이어를 사용하여 재생되지 않는 비디오 파일의 경우(예: 브라우저에서 URL에 직접 액세스하거나 서드 파티 또는 자체 개발 플레이어 사용), 관리 인터페이스의 플레이어 매개변수 설정은 적용되지 않습니다.

현재 Web 플레이어는 최대 10개까지 생성할 수 있습니다.


### 플레이어의 외관을 커스터마이징할 수 있습니까?

관리 페이지에서 Logo 및 플레이어 크기를 커스터마이징할 수 있으며, SDK를 통해 iOS 또는 Android 플레이어 크기도 설정할 수 있습니다.



### 워터마크와 Logo의 차이점은 무엇입니까?

워터마크는 트랜스코딩 과정에서 비디오 파일에 완전히 삽입된 고정 표식입니다. 트랜스코딩이 완료되면 취소할 수 없습니다.
Logo는 비디오 재생 중에 플레이어에 표시되는 아이콘입니다. 언제든지 이동하거나 취소할 수 있습니다.
