
VOD 콘솔을 통해 기본 정보 편집, 오디오/비디오 배포, Player+ 미리보기 및 Web 플레이어 코드 생성 등 오디오/비디오 관리가 가능합니다. 본 문서에서는 오디오/비디오 관리 방법에 대해 소개합니다.

## 더 많은 페이지
1. [VOD 콘솔](https://console.cloud.tencent.com/vod)에 로그인하여 왼쪽 사이드바에서 **애플리케이션 관리**를 선택합니다.
2. 오디오/비디오 파일을 처리할 애플리케이션 이름을 클릭하여 애플리케이션 관리 페이지로 들어갑니다.
3. 기본적으로 **미디어 자산 관리**>**오디오/비디오 관리**, ‘업로드됨’ 페이지로 이동합니다.
4. **더 보기**를 클릭합니다. 비디오 상태 변경, 스토리지 등급 변경, 데이터 검색 등 VOD의 서비스 기능을 더 많이 얻을 수 있습니다.

### 오디오/비디오 상태
사용자는 미디어 상태 변경을 통해 오디오/비디오 액세스 가용성을 제어할 수 있습니다.
![](https://qcloudimg.tencent-cloud.cn/raw/a76a5157b15f15ae4b9616b5ce20f778.png)

- **재생 차단**: 주문형 콘솔 미리보기를 제외하고 다른 시나리오의 오디오/비디오 리소스에 액세스하는 URL(원본 파일, 트랜스 코딩된 출력 파일, 스크린샷 등)은 모두 403이 반환됩니다.
- **차단 해제**: 재생 차단된 오디오/비디오 차단 해제를 진행합니다. 차단 해제 후 미디어 파일에 정상적으로 액세스할 수 있습니다.
>? 차단/해제 작업은 전체 적용까지 약 5 - 10분 정도 소요됩니다.

### 스토리지 유형 변경
사용자는 스토리지 유형을 직접 수정하여 적절한 스토리지 유형을 선택할 수 있습니다. 현재 지원되는 스토리지 유형 및 그 특징은 다음 표와 같습니다.

| 스토리지 유형   | 스탠다드 스토리지 | 스탠다드IA 스토리지 | 아카이브  | 딥 아카이브   |
| ------ | ---- | ---- | ----- | -------- |
| VOD 기본값  | 예    | 아니오    | 아니오     | 아니오        |
| 스토리지 비용   | 높음    | 중간    | 낮음     | 매우 낮음       |
| 액세스 성능   | 높음    | 낮음    | 액세스 미지원 | 액세스 미지원    |
| 데이터 검색 요금 | 없음    | 없음    | 있음     | 있음        |
| 지원 단지   | 전체   | 전체   | 전체    | 베이징, 상하이, 충칭 |
| 최단 과금 시간 | 없음    | 30일    | 90일   | 180일     |

#### 특징 설명:
1. VOD는 다양한 업로드 방식(API/SDK/콘솔 업로드)을 사용하며 기본 스토리지 유형은 모두 스탠다드 스토리지 유형입니다.
2. 스토리지 비용은 [구매 가이드](https://intl.cloud.tencent.com/document/product/266/14666)를 참고하십시오.
2. 액세스 가이드는 영상 시청 시 **첫 화면 시간**, **랙** 등의 지표에 영향을 미치므로 **자주 접속하는 온라인 비즈니스의 경우 스토리지 유형 변경을 권장하지 않습니다**. 아카이브 유형과 딥 아카이브 유형은 재생, 비디오 처리 시작 등 직접 액세스가 불가하며 검색 후에만 액세스할 수 있습니다.
4. 데이터 검색은 아카이브/딥 아카이브를 스탠다드 스토리지로 단기 또는 영구 검색할 수 있습니다. 검색 모드, 소스 스토리지 유형별로 요금이 상이합니다.
5. 딥 아카이브는 현재 베이징, 상하이 및 충칭 단지만 지원됩니다.
6. **스탠다드IA 스토리지/아카이브/딥 아카이브 파일은 최소 스토리지 시간이 있으며, 사전 삭제 시에도 최소 스토리지 시간에 따라 요금이 부과됩니다.**

>?
>- 스탠다드IA 스토리지 미디어 파일의 최소 스토리지 기간은 30일이며, 사전에 삭제해도 사용자에게 30일에 해당하는 요금이 부과됩니다.
>- 아카이브 미디어 파일의 최소 스토리지 기간은 90일이며, 사전에 삭제해도 사용자에게 90일에 해당하는 요금이 부과됩니다.
>- 딥 아카이브 미디어 파일의 최소 스토리지 기간은 180일이며, 사전에 삭제해도 사용자에게 180일에 해당하는 요금이 부과됩니다.
>- 일 결산 사용자는 사전 삭제량이 익일에 청구되고, 월 결산 사용자는 사전 삭제량이 익월에 청구됩니다.

#### 사용 제한:
아카이브 또는 딥 아카이브를 다른 유형으로 변경하려면 데이터 검색 기능이 필요합니다.
>?사용 제한 이내의 비디오는 유형 변경 시 무시됩니다.

### 데이터 검색 기능
사용자는 데이터 검색 기능을 사용하여 미디어의 아카이브/딥 아카이브 유형을 스탠다드 스토리지로 변경할 수 있습니다. 아카이브 및 딥 아카이브에 대한 스토리지 유형 및 검색 작업을 변경하는 여러 모드가 있습니다. 모드별 최종 효과는 동일하며 속도와 비용(즉, 검색 비용 관련 자세한 내용은 [구매 가이드-데이터 검색](https://intl.cloud.tencent.com/document/product/266/14666)을 참고하십시오.)의 차이가 있습니다.

| 동결 해제 모드 | 아카이브 검색 소요 시간 | 딥 아카이브 검색 소요 시간 |
| ---- | -------- | ---------- |
| 초고속 모드 | 5분      | 미지원        |
| 표준 모드 | 5시간      | 24시간       |
| 배치 모드 | 12시간     | 48시간       |

>?
>- 매체(FileId로 표시)는 원본 파일, 트랜스코딩된 파일 및 스크린샷 파일 등과 같이 스토리지에 있는 많은 파일을 포함할 수 있습니다. 각 파일의 실제 동결 해제 시간은 순차적일 수 있습니다. VOD는 각 파일의 특정 동결 해제 완료 상태를 유지하지 않고 미디어 전체를 가져와서 가능한 최대 동결 해제 시간으로 미디어의 동결 해제 완료 시간을 계산합니다. 이 시간에 도달하기 전에는 미디어 아래의 모든 파일이 실제로 동결 해제되었더라도 미디어는 여전히 동결 해제가 되지 않은 상태로 표시되어 액세스가 허용되지 않습니다.
>- 미디어가 동결 해제 완료로 표시되는 시간은 실제 동결 해제 완료 시점보다 늦기 때문에, 미디어의 동결 해제 사본의 사용 가능 시간은 예상 시간보다 짧습니다. 동결 해제 후 복사본의 사용 가능 시간을 충분히 확보하려면 딥 아카이브 동결 해제 시 1일의 유효 기간을 추가하는 것이 좋습니다.

2. 오디오/비디오 파일을 처리할 애플리케이션 이름을 클릭하여 애플리케이션 관리 페이지로 들어갑니다.
3. 기본적으로 **미디어 자산 관리**>**오디오/비디오 관리**, ‘업로드됨’ 페이지로 이동합니다.
4. 타깃 비디오 파일이 있는 행에서 **관리**를 클릭하면 ‘기본 정보’ 페이지로 이동합니다.

## Superplayer 미리보기
>!Superplayer는 ABS 트랜스 코딩된 비디오에 대한 미리보기만 지원합니다. Superplayer 요건에 부합하는 비디오를 빠르게 생성하려면 [Superplayer로 재생하기 - 액세스 가이드](https://cloud.tencent.com/document/product/266/43629#.E5.88.9D.E7.BA.A7.E6.AD.A5.E9.AA.A4)를 참고하십시오.

ABS 트랜스코딩된 비디오는 Superplayer로 미리보기가 가능하며, VOD는 **Web 플레이어**와 **단말 플레이어**의 미리보기를 지원합니다.

### 매개변수 정보
**재생 설정**: Superplayer 설정 내의 모든 플레이어 설정이 포함되어 있으며, default와 basicDrmPreset로 기본 설정되어 있습니다. **default 재생 설정 외의 모든 재생 설정은 기본 배포 도메인의 [Key 링크 도용 방지 활성화](https://console.cloud.tencent.com/vod/distribute-play/domain)를 실행해야 사용할 수 있습니다**.
![](https://qcloudimg.tencent-cloud.cn/raw/e46e31631d7a92cbe12cb0d015364951.png)


### 설정 항목
**설정 항목**: 선택한 재생 구성 항목을 표시합니다.
- 재생을 위한 어댑티브 비트 레이트 스트리밍 템플릿: 현재 플레이어가 템플릿으로 트랜스 코딩된 비디오를 미리 볼 수 있도록 어댑티브 비트 레이트 스트리밍 템플릿을 지정합니다.
- 썸네일 미리보기용 이미지 스프라이트 템플릿: 플레이어가 템플릿에서 생성된 이미지 스프라이트를 미리 볼 수 있도록 이미지 스프라이트 템플릿을 지정합니다.
![](https://qcloudimg.tencent-cloud.cn/raw/762dee1c8d7a1515a0f7e7eebf889465.png)

### 재생 제어
Key 링크 도용 방지의 도메인 재생 설정 활성화: 매개변수를 설정하여 **재생 링크 만료 시간**, **미리보기 시간** 및 **최대 재생 가능 IP 수**를 제어할 수 있습니다.
- 재생 링크 만료 시간: 비디오 링크가 해당 시점에서 만료되도록 설정합니다.
- 미리보기 시간: 비디오 미리보기 시간을 지정합니다. 시간은 최소 30초를 초과하여야 하며, 입력하지 않을 경우 무제한 시청이 가능합니다.
- 최대 재생 가능 IP 수: 재생이 허용되는 서로 다른 IP의 최대 수입니다.

## Web 플레이어 코드 생성
1. [VOD 콘솔](https://console.cloud.tencent.com/vod)에 로그인하여 왼쪽 사이드바에서 **애플리케이션 관리**를 선택합니다.
2. 오디오/비디오 파일을 처리할 애플리케이션 이름을 클릭하여 애플리케이션 관리 페이지로 들어갑니다.
3. 기본적으로 **미디어 자산 관리**>**오디오/비디오 관리**, ‘업로드됨’ 페이지로 이동합니다.
4. 타깃 비디오 파일이 있는 행에서 **관리**를 클릭하면 ‘기본 정보’ 페이지로 이동합니다.
5. **Web 플레이어 코드 생성**을 클릭하여 ‘Web 플레이어 코드 생성’ 페이지로 들어갑니다. 해당 페이지에서 플레이어 정보 및 Web 플레이어 코드를 관리할 수 있습니다.
	- ‘매개변수 설정’ 섹터에서 **변경**을 클릭하면 ‘플레이어 변경’ 대화 창이 팝업됩니다. 플레이어 드롭다운 리스트에서 설정 완료한 플레이어(플레이어 관련 설정은 [Web 플레이어 관리](https://intl.cloud.tencent.com/document/product/266/14056) 참고)를 선택하고 **확인**을 클릭합니다.
	- ‘Web 플레이어 코드’ 섹터의 ‘비디오 크기’ 드롭다운 목록에서 적합한 비디오 크기를 선택하고 필요에 따라 자동 재생을 활성화 또는 비활성화합니다. 코드 유형은 ‘HTML’ 또는 ‘IFRAME’으로 설정합니다.
6. **코드 복사**를 클릭하면 생성된 Web 플레이어 코드가 복사됩니다.
