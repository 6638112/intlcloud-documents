
VOD에 대한 모든 요청의 세부 정보가 포함된 지난 30일 동안의 CDN 액세스 로그를 다운로드할 수 있습니다. 로그는 매시간 기록됩니다.
## CDN 로그 다운로드
1. [VOD 콘솔](https://console.cloud.tencent.com/vod)에 로그인하고 왼쪽 사이드바에서 [로그 다운로드]>[CDN 로그]를 클릭합니다.
2. 원하는 시간 범위와 도메인 이름을 선택하고 [확인]을 클릭하여 로그 다운로드 링크를 가져옵니다.
![](https://main.qcloudimg.com/raw/5da8ab89237962c2900798525de41ba2.png)
>?
>- 요청이 수신되지 않은 날에는 액세스 로그가 생성되지 않으며 페이지에 데이터 없음이 표시됩니다.
>- 기본적으로 CDN은 시간 단위로 요청을 기록합니다. 즉, 하루에 최대 24개의 로그 파일이 생성될 수 있습니다. 요청이 수신되지 않은 시간에 대해서는 로그가 생성되지 않습니다.
>- CDN 로그는 약 30분 지연될 수 있습니다.

## 로그 필드 설명
다운로드 한 로그 데이터 패키지를 압축 해제하고 텍스트 모드로 로그 파일을 열면 각 필드 정보가 빈 칸으로 구분되어 있음을 볼 수 있습니다. 일부 정보는 아래 이미지와 같습니다.
![](https://main.qcloudimg.com/raw/db73fc1837d5e6522d137993cfaff50b.png)

| 순서 | 로그 내용                                    |
| --- |------------------------------------------ |
| 1    | 요청 시간                                    |
| 2    | 도메인 이름에 액세스한 클라이언트 IP                         |
| 3    | 액세스된 도메인 이름                                |
| 4    | 파일 요청 경로                                |
| 5    | 이번 액세스 요청의 바이트 수                          |
| 6    | 지역. 자세한 내용은 하기 [지역 매핑](#province) 참고      |
| 7    | ISP. 자세한 내용은 아래의 [ISP 매핑](#carrier) 참고 |
| 8    | HTTP 상태 코드                                 |
| 9    | Referer 정보                                |
| 10   | 응답 시간(밀리초)                            |
| 11   | User-Agent 정보                             |
| 12   | Range 매개변수                                  |
| 13   | HTTP Method                                 |
| 14   | HTTP 프로토콜 식별자                                |
| 15   | 캐시 HIT/MISS                               |
