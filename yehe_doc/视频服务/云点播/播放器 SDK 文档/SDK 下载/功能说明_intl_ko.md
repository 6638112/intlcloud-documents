## Superplayer
Superplayer는 아래와 같이 다양한 플랫폼에 연결할 수 있는 풍부한 기능을 제공합니다.

| 기능         | 설명                                               | iOS & Android | Web |
| ----------- | -------------------------------------------------- | ------------- | --- |
| 다양한 형식        | RTMP, FLV, HLS, MP4, WebRTC 등 다양한 오디오 및 비디오 형식 지원               | &#10003             | &#10003   |
| URL 재생      | 웹 비디오의 URL 재생 지원                                   | &#10003             | &#10003   |
| DASH 프로토콜     | 표준 DASH 형식의 비디오 지원                                  | ×             | &#10003   |
| FileID 재생   | VOD FileID 방식 재생 지원                               | &#10003             | &#10003   |
| 첫 번째 화면 사전 로딩     | 비디오 콘텐츠 사전 로딩 지원, 첫 번째 프레임 바로 재생               | &#10003             | &#10003   |
| 빠른 seek     | 재생을 위해 지정된 위치로 정확하고 빠른 seek 지원                               | &#10003             | &#10003   |
| H.265 하드웨어 디코딩    | H.265 하드 디코딩 및 재생 지원                                    | &#10003             | &#10003   |
| 소프트웨어와 하드웨어 디코딩 간의 자동 전환     | 단말에서 하드웨어 디코딩을 지원하지 않으면 자동으로 소프트 디코딩으로 전환                                 | &#10003             | &#10003   |
| 어댑티브 비트레이트 스트림       | HLS 어댑티브 비트스트림 재생 시 네트워크 대역폭에 따라 재생을 위한 해상도 스트림 수동 지정 또는 자동 선택 지원            | &#10003             | &#10003   |
| 해상도 전환      | 사용자가 여러 해상도 스트림을 지연 없이 원활하게 전환할 수 있도록 지원                                 | &#10003             | &#10003   |
| 해상도 이름 생성       | 다양한 해상도 스트림 이름 사용자 정의 지원                             | &#10003             | &#10003   |
| 재생 제어       | 시작, 종료, 일시 중지, 자동 재생, 연속 재생, 중단점 재생 재개, 중복 재생 등과 같은 재생 제어 기능 지원                | &#10003             | &#10003   |
| 배속 재생        | 오디오 가변 속도가 변경되지 않도록 0.5x - 2x 비디오 가변 속도 재생 지원          | &#10003             | &#10003   |
| 시작 시간 사용자 정의     | 비디오 재생 시작 시간 사용자 정의 지원                                 | &#10003             | &#10003   |
| 미리보기        | 미리보기가 활성화된 비디오 재생 지원                                      | &#10003             | &#10003   |
| 진행 표시줄 작업      | 진행 표시줄을 드래그하여 진행률 전환                                          | &#10003             | &#10003   |
| 진행 표시줄 표시 및 썸네일 미리보기 | 진행 표시줄에 마크 정보 추가 지원 및 썸네일(스프라이트 이미지) 미리보기 지원                        | &#10003             | &#10003   |
| 플레이어 크기       | 플레이어 크기 사용자 정의 지원                                       | &#10003             | &#10003   |
| 화면 채우기/맞추기     | 화면 크기에 맞게 비디오 이미지에 대해 다른 표시 모드 지원                             | &#10003             | &#10003   |
| 작은 창 재생        | 작은 창에서 동영상 재생 지원                                          | &#10003             | &#10003   |
| 비디오 미러링        | 비디오 이미지를 수평 또는 수직으로 미러링 지원                                      | &#10003             | &#10003   |
| 비디오 회전        | 비디오 이미지 지정된 각도로 회전 또는 비디오 파일의 회전 매개변수에 따라 자동 회전 지원          | &#10003             | ×   |
| 밝기 조정        | 비디오 재생 중 시스템 밝기 조정 지원                                      | &#10003             | \-  |
| 볼륨 조정        | 비디오 재생 중에 시스템 볼륨 조정 및 음소거 지원                                 | &#10003             | &#10003   |
| 듀얼 사운드 채널             | 듀얼 사운드 채널 재생 지원                                           | &#10003             | &#10003   |
|퓨어 오디오 재생|MP3 및 기타 파일에 대해 퓨어 오디오 재생 지원| &#10003| &#10003|
| 잠금 화면       | 회전 잠금 및 숨기기 인터페이스 요소를 포함한 화면 잠금 기능 지원                               | &#10003             | \-  |
| 댓글 자막          | 비디오 위에 댓글 표시 지원                                        | &#10003             | &#10003   |
| 롤 이미지       | 일시 중지 시 광고 표시를 위한 이미지 스티커 추가 지원                         | &#10003             | &#10003   |
| 비디오 화면 캡처       | 비디오 이미지 스크린샷 캡처 지원                                   | &#10003             | ×   |
| 자막 가져오기        | 사용자 정의 자막 파일 가져오기 지원                                        | &#10003             | &#10003   |
| 썸네일 구성        | 재생할 비디오의 썸네일 설정 지원                                        | &#10003             | &#10003   |
| 멀티 인스턴스         | 동시 재생을 위해 여러 플레이어를 동일한 페이지에 추가 지원                                 | &#10003             | &#10003   |
| 버퍼링      | 비디오 재생 시 나머지 콘텐츠 동시 다운로드 및 버퍼링 지원                         | &#10003             | &#10003   |
| Referer 링크 도용 방지 | 재생 요청 소스는 요청의 Referer 필드를 통해 식별할 수 있으므로 블록리스트 또는 얼로우리스트를 통해 요청 소스를 제어 지원 | &#10003             | &#10003   |
| Key 링크 도용 방지     | 재생 URL에 제어 매개변수를 추가하여 URL 유효 기간, 미리 보기 시간, 재생이 허용된 IP 수 등 제어 지원          | &#10003             | &#10003   |
| HLS 암호화      | 동영상 데이터는 키 및 HLS 기반 AES encryption 방식으로 암호화 지원       | &#10003             | &#10003   |
| 사유 프로토콜 암호화      | 동영상은 클라우드의 사유 프로토콜을 통해 암호화할 수 있으며 암호화된 동영상은 재생용 플레이어 SDK를 통해서만 복호화 가능     | &#10003             | &#10003   |
| 오프라인 다운로드        | 오프라인으로 다운로드한 암호화된 비디오는 재생용 플레이어 SDK를 통해서만 해독 가능                 | &#10003             | \-  |
| 재생 콜백        | 재생 상태, 첫 번째 프레임 및 재생 완료 또는 실패 재호출 지원                           | &#10003             | &#10003   |
| HTTPS 지원    | HTTPS 비디오 리소스 재생 지원                                   | &#10003             | &#10003   |
| 사용자 정의 HTTP 헤더 | 비디오 리소스 요청의 HTTP Headers 사용자 정의 가능                        | &#10003             | \-  |

## Superplayer Adapter
Superplayer Adapter는 아래와 같이 다양한 플랫폼에 연결할 수 있는 풍부한 기능을 제공합니다.

| 특징      | 설명                                         | iOS/Android | Web |
| -------- | -------------------------------------------- | ----------- | --- |
| QUIC 프로토콜   | QUIC 프로토콜 지원                                     | ×           | &#10003   |
| FileID 재생 | VOD FileId로 비디오 재생 지원                         | &#10003           | &#10003   |
| 해상도 전환    | 사용자가 여러 해상도 스트림을 지연 없이 원활하게 전환할 수 있도록 지원                         | &#10003           | &#10003   |
| 해상도 이름 생성    | 다양한 해상도 스트림 이름 사용자 정의 지원                             | &#10003           | &#10003   |
| 진행 표시줄 표시    | 진행률 표시줄에 마커 추가 지원                                | &#10003           | &#10003   |
| 썸네일 미리보기    | 미리보기를 위해 진행률 표시줄에 축소판(스프라이트) 표시 지원                         | &#10003           | &#10003   |
| 썸네일 구성     | 재생할 비디오의 썸네일 설정 지원                                  | &#10003           | &#10003   |
| HLS 암호화    | 동영상 데이터는 키 및 HLS 기반 AES encryption 방식으로 암호화 가능  | &#10003           | &#10003   |
| 사유 프로토콜 암호화   | 동영상은 클라우드의 사유 프로토콜을 통해 암호화할 수 있으며 암호화된 동영상은 재생용 플레이어 SDK를 통해서만 복호화 가능 | &#10003           | &#10003   |
