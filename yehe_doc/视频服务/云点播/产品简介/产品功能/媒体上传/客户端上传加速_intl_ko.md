## 기능 소개
클라이언트 업로드 가속은 다음의 몇가지 조치를 통해 고품질 업로드 서비스를 제공합니다.
<table ><thead ><tr>
<th style="width:200px">조치</th><th >설명</th></tr>
</thead><tbody ><tr>
<td>엣지 노드 근거리에서 데이터 수신</td>
<td>전 세계에 배포된 엣지 노드는 가장 가까운 단말로부터 업로드 요청을 수신합니다.</td>
</tr>
<tr>
<td>지능형 가속 네트워크</td>
<td>Tencent Cloud를 기반으로 네트워크를 가속화하고 최적의 링크를 지능적으로 선택하여 데이터를 스토리지 센터로 전송합니다.</td>
</tr>
<tr>
<td>QUIC 프로토콜 전송 지원</td>
<td>QUIC 프로토콜은 다중화, 연결 마이그레이션 등의 특성을 가지며 데이터 전송 효율이 높으며 약한 네트워크 환경에서 더 안정적입니다.</td>
</tr>
</tbody>
</table>



## 적용 시나리오
<table ><thead ><tr>
<th style="width:200px">시나리오</th><th >설명</th></tr>
</thead><tbody ><tr>
<td>원격 데이터 업로드</td>
<td>일부 단말 사용자는 VOD 스토리지 센터와의 거리가 멀기 때문에 지역 간 및 해상 간의 업로드 시나리오에서 업로드 품질이 저하됩니다. 클라이언트 업로드 가속을 사용하면, 사용자의 요청은 가까운 Tencent Cloud 엣지 노드를 통해 수신되고, 가속 네트워크를 이용하여 데이터를 전송하여 업로드 품질이 크게 향상됩니다.</td>
</tr>
<tr>
<td>약한 네트워크 환경 데이터 업로드</td>
<td>모바일 사용자는 종종 네트워크 전환을 경험하거나 기지국 신호가 약한 지역에 위치하기 때문에 네트워크 연결이 불안정하고 패킷 손실률이 높습니다. 업로드 가속은 QUIC 프로토콜 사용을 지원하므로 약한 네트워크 환경에서 데이터 전송이 더 안정적입니다.</td>
</tr>
<tr>
<td>일반 데이터 업로드</td>
<td>업로드 가속을 활성화하지 않으면 데이터 전송에 HTTP 1.1이 사용되는데, 이는 대용량 데이터를 다룰 때 취약합니다. 이에 비해 QUIC 프로토콜은 다중화를 허용하고 0-RTT 등 기능을 지원하여 데이터를 더 효율적으로 전송할 수 있습니다.</td>
</tr>

</tbody>
</table>
