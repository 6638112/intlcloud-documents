## 소개

멀티 엔드 업로드 기능을 사용하면 클라이언트, 서버, 공중망 URL을 통한 업로드, 라이브 녹화 등 다양한 방법을 통해 오디오, 비디오, 이미지 등의 미디어 파일을 VOD에 업로드할 수 있습니다.

구체적으로, VOD는 다음 업로드 방법을 지원합니다.

| 기능  | 설명  |
| -- | -- |
| 클라이언트에서 업로드 | 클라이언트의 미디어 파일을 VOD에 업로드합니다. iOS, Android, Web, 미니 프로그램 등 다양한 플랫폼용 SDK를 지원하며, 해당 SDK를 통합하는 것만으로도 이 기능을 구현할 수 있습니다. |
| 서버에서 업로드 | 서버의 미디어 파일을 VOD로 업로드합니다. Java, C#, PHP, Python, Node.js 및 Go를 포함한 다양한 프로그래밍 언어용 SDK를 지원합니다. |
| 콘솔을 통해 업로드 | 로그인 후 VOD 콘솔에서 직접 미디어 파일을 업로드합니다. |
| URL에서 풀링 | URL을 통해 네트워크의 미디어 리소스를 VOD로 풀링합니다. |
| 라이브 녹화 업로드 | CSS에서 녹화가 활성화된 경우 라이브 미디어를 VOD에 직접 녹화합니다. |
| 원본 서버 마이그레이션 도구를 통한 업로드 | VOD에서 제공하는 툴을 통해 다른 클라우드 벤더의 미디어 파일을 VOD로 업로드합니다. |

VOD의 업로드 방식은 거의 모든 미디어 소스를 커버하므로 모든 소스에서 파일을 업로드할 수 있습니다.

## 적용 시나리오

| 기능  | 설명  |
| -- | -- |
| 클라이언트에서 업로드 | UGC 및 PGC 시나리오에서 대부분의 미디어 파일은 일반 사용자가 만듭니다. 사용자가 휴대폰이나 PC와 같은 장치를 사용하여 콘텐츠를 만든 후 클라이언트의 미디어를 VOD에 업로드할 수 있습니다. |
| 서버에서 업로드 | 미디어의 저작권을 소유한 대형 비디오 포털 또는 플랫폼은 일반적으로 자체 서버에 미디어를 저장합니다. 이 경우 서버에 있는 미디어 파일을 일괄적으로 VOD에 업로드할 수 있습니다. |
| 콘솔을 통해 업로드 | 로컬 미디어 파일을 PC에 업로드하고 싶다면 콘솔에서 빠르게 업로드할 수 있습니다. |
| URL에서 풀링 | 업로드하려는 동영상이 이미 네트워크에 있는 경우 해당 동영상의 미디어 URL을 사용하여 VOD에 직접 업로드할 수 있습니다. |
| 원본 서버 마이그레이션 툴을 통한 업로드 | 다른 클라우드 벤더의 기존 동영상을 모두 마이그레이션하거나 로컬 저장소에서 많은 수의 동영상을 마이그레이션하려는 경우 원본 서버 마이그레이션 툴을 사용하여 VOD에 업로드할 수 있습니다. |
