## 액세스 모니터링
[VOD 콘솔](https://console.cloud.tencent.com/vod)에 로그인 후 왼쪽 디렉터리의 [데이터 센터]>[실시간 모니터링]를 클릭하여 관리 페이지에 들어가면 실시간 모니터링 정보가 표시됩니다. 기본적으로 지난 6시간 동안의 모든 도메인 이름에 대한 모니터링 곡선이 1분 통계 단위로 반환되며 지표는 다음과 같습니다.
- 대역폭: 1분 간의 총 트래픽을 시간(60초)으로 나누어 환산합니다.
- 요청 수 상태 코드 비율: 선택한 시간 구간에서 2XX/3XX/4XX/5XX 비율 이미지.
- 요청 수 상태 코드 2XX: 2XX 상태 코드 모니터링, 생성된 상태 코드는 모두 집계됩니다.
- 요청 수 상태 코드 3XX: 3XX 상태 코드 모니터링, 생성된 상태 코드는 모두 집계됩니다.
- 요청 수 상태 코드 4XX: 4XX 상태 코드 모니터링, 생성된 상태 코드는 모두 집계됩니다.
- 요청 수 상태 코드 5XX: 5XX 상태 코드 모니터링, 생성된 상태 코드는 모두 집계됩니다.

### 상세 정보 페이지 데이터 설명

각 지표 하단의 [상세 보기]를 클릭하면 지표 상세 정보 페이지로 이동합니다.

상세 정보 페이지의 왼쪽 상단에서도 지표 빠른 전환을 수행할 수 있습니다.

상세 정보 페이지에서 다음 데이터를 조회할 수 있습니다.

- 대역폭: 총 피크 대역폭, 실시간 대역폭 곡선, 도메인 대역폭 순위(높은 순).
- 트래픽: 총 트래픽, 실시간 트래픽 곡선, 도메인 트래픽 순위(높은 순), URL 트래픽 순위(높은 순).
- 요청 수: 총 요청 수, 실시간 요청 수 곡선, 도메인 요청 수 순위(높은 순), URL 요청 수 순위(높은 순).
- 상태 코드 비율: 2XX, 3XX, 4XX, 5XX 상태 코드 비율 원형 차트 및 각 상세 상태 코드 수량 및 비율.
- 상태 코드 2XX: 2XX 상태 코드 실시간 모니터링 곡선 및 구성, 2XX의 각 서브 상태 코드 모니터링 곡선, 2XX 상태 코드 도메인 순위(높은 순).
- 상태 코드 3XX: 3XX 상태 코드 실시간 모니터링 곡선 및 구성, 3XX의 각 서브 상태 코드 모니터링 곡선, 3XX 상태 코드 도메인 순위(높은 순).
- 상태 코드 4XX: 4XX 상태 코드 실시간 모니터링 곡선 및 구성, 4XX의 각 서브 상태 코드 모니터링 곡선, 4XX 상태 코드 도메인 순위(높은 순).
- 상태 코드 5XX: 5XX 상태 코드 실시간 모니터링 곡선 및 구성, 5XX의 각 서브 상태 코드 모니터링 곡선, 5XX 상태 코드 도메인 순위(높은 순).

### 세분성 설명

#### 전체보기 페이지 시간 세분성 설명

모니터링 페이지에 1분, 5분, 1시간 및 1일의 단위 곡선에 선택 항목 표시를 제공하며, 선택한 시간에 따라 최소 표시 가능 시간 세분성이 달라집니다.

- 시간 구간 ≤ 6시간의 경우 최소 시간 단위는 1분이며, 1분 시간 단위 모니터링 곡선의 현재 딜레이는 약 3분입니다.
- 시간 구간 ＞ 6시간, ≤ 24시간의 경우 최소 시간 단위는 5분이며, 5분 데이터의 딜레이는 5 - 10분입니다.
- 시간 구간 ＞ 24시간 및 ≤ 31일의 경우 최소 시간 단위는 1시간입니다.
- 시간 구간 ＞ 31일의 경우 최소 시간 단위는 1일입니다.

#### 상세 정보 페이지 세분성 설명

지표 상세 정보 페이지 상의 시간 단위는 다음과 같습니다.

- 시간 구간 ≤ 1일의 경우 최소 시간 단위는 1분이며, 1분 시간 단위 모니터링 곡선의 현재 딜레이는 약 3분입니다.
- 시간 구간 ＞ 1일 및 ≤ 31일의 경우 최소 시간 단위는 5분, 1시간, 1일입니다(선택 가능).
- 시간 구간 ＞ 31일의 경우 최소 시간 단위는 1일입니다.

>!1분 단위 데이터의 새 버전은 온라인 상태여야만 조회할 수 있습니다. 기록 데이터의 최소 쿼리 가능 단위는 5분이고 최대 쿼리 가능 시간 간격은 90일입니다.

#### 취합 설명

데이터 지표에 따라 1분 단위 취합은 5분, 1시간 및 1일 방식으로 달라집니다.

+ 대역폭: 제공하는 대역폭 모니터링의 최소 단위 데이터는 1분 데이터이며, 업계 표준에 따라 과금에 일반적으로 사용하는 5분 단위 데이터는 1분 데이터 AVG에서 오므로, 1시간 및 1일 주기의 대역폭 데이터는 5분 단위를 사용하여 MAX를 구합니다.
+ 트래픽: 5분, 1시간 및 1일 주기의 트래픽 데이터는 모두 1분 단위 트래픽 데이터를 사용하여 누적됩니다.
+ 요청 수 및 상태 코드: 5분, 1시간 및 1일 데이터는 모두 1분 단위 데이터를 사용하여 누적됩니다.

### 필터 설명
현재 [통계 지역], [ISP] 두 개 항목의 지정 조회를 지원하지 않으며, 특정 지역의 모든 ISP 또는 특정 ISP의 전체 지역 조회만 지원합니다.

## 패널 설정

새로운 버전의 실시간 모니터링 페이지는 관심 지표의 모니터링 곡선을 조회할 수 있도록 필요에 따라 지표 화면 조정을 지원합니다.

1. [VOD 콘솔](https://console.cloud.tencent.com/vod)에 로그인하고 왼쪽 디렉터리에서 [통계 분석]>[실시간 모니터링]을 선택하여 관리 페이지로 이동합니다.
2. 오른쪽 설정 아이콘을 클릭하여 설정 페이지로 이동합니다.
3. 필요에 따라 전체보기 페이지에 표시할 데이터 지표 선택: 체크된 지표가 개요 페이지에 직접 표시되며, 체크를 해체하면 기본으로 표시되지 않습니다.
   [액세스 모니터링]과 [Origin-pull 모니터링] 전체보기 페이지의 실시간 모니터링은 사용자 지정 화면에서 각각 설정할 수 있습니다.
   

## 데이터 대조

새로운 버전의 실시간 모니터링 페이지의 서브 페이지는 모두 데이터 곡선 대조 기능을 지원합니다.

1. [VOD 콘솔](https://console.cloud.tencent.com/vod)에 로그인하고 왼쪽 디렉터리에서 [통계 분석]>[실시간 모니터링]을 선택하여 관리 페이지로 이동합니다.
2. 특정 시간 구간의 모니터링 곡선을 조회한 다음 [데이터 대조]를 클릭하여 시간 주기를 지정하면 데이터 대조가 표시됩니다.
   
   편리한 사용을 위하여 시작 시간을 지정하면 시스템에서 자동으로 종료 시간을 지정하고, 종료 시간을 지정하면 시스템에서 자동으로 시작 시간을 지정하도록 하여 대조 시간 주기를 일치하게 합니다.
