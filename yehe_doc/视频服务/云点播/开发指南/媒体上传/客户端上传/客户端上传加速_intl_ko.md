클라이언트 업로드 가속은 고객에게 고품질 업로드 서비스를 제공할 수 있습니다. Tencent Cloud가 전 세계적으로 배포한 가속 네트워크를 기반으로 사용자의 요청에 따라 최적의 액세스 포인트와 최적의 링크를 지능적으로 선택하여 업로드 속도와 성공률을 향상시키는 기능입니다. 동시에 데이터 전송을 위한 QUIC 프로토콜의 사용을 지원하여 데이터 전송의 효율성과 약한 네트워크 환경에서의 안정성을 향상시킵니다.

## 업로드 품질에 영향을 미치는 주요 요인
### 장거리 전송
VOD는 전 세계 여러 지역에 스토리지 센터를 배포하였으며, 고객은 이러한 스토리지 센터를 선택적으로 활성화하여, 가장 가까운 스토리지에 업로드할 수 있습니다. 자세한 내용은 [근거리 업로드](https://intl.cloud.tencent.com/document/product/266/37548)를 참고하십시오. 그러나 그럼에도 일부 단말 사용자는 스토리지 센터에서 너무 멀리 떨어져 있고 일부 고객은 비즈니스 시나리오에서 여러 지역 또는 바다를 가로질러 업로드해야 하는 것이 불가피합니다. 장거리 데이터 업로드는 일반적으로 네트워크 링크가 길어지고 전송 지연이 커짐을 의미하며 중간에 특정 링크에서 네트워크 지터, 패킷 손실 등의 문제가 발생하면 전체 링크의 업로드 속도와 성공률이 낮아집니다.
### 약한 네트워크 문제
간단히 말해서 약한 네트워크는 지연이 크고 패킷 손실률이 높은 네트워크와 같이 품질이 좋지 않은 네트워크 환경입니다. 오늘날 모바일 네트워크는 매우 대중적이며 모바일 클라이언트의 업로드 볼륨이 매우 큰 비율을 차지합니다. 모바일 네트워크의 약한 네트워크 문제는 기지국 신호 범위가 약한 지역의 사용자 또는 네트워크 장치가 작동 중에 네트워크를 자주 전환하는 것과 같이 매우 일반적입니다. 네트워크가 약한 환경에서 어떻게 안정적인 데이터 전송을 유지하느냐는 업로드 품질 향상에 있어 중요한 문제입니다.
### 비효율적인 네트워크 프로토콜
VOD 고객이 업로드하는 파일은 대부분 데이터가 많은 동영상 파일이며, 업로드에 가장 많이 사용되는 네트워크 프로토콜은 여전히 ​​HTTP1.1입니다. 프로토콜은 기본적으로 직렬 모델을 기반으로 하며 HOL(head-of-line) 블로킹과 같은 문제도 있으며 대규모 데이터 전송 시나리오에서는 성능 병목 현상에 도달하기 쉽습니다.


## 클라이언트 업로드 가속 솔루션
### 글로벌 링크 가속, 높은 보장 채널
장거리 전송 시나리오의 경우, 네트워크 링크가 너무 길어서 업로드 품질이 좋지 않은 문제에 대해, VOD는 Tencent Cloud가 전 세계에 분포한 가속 네트워크와 엣지 노드를 기반으로 고객에게 데이터 업로드의 글로벌 링크 가속 채널을 제공합니다. Tencent Cloud의 지능형 글로벌 트래픽 관리 플랫폼의 도움으로 사용자의 업로드 요청이 사용자와 가장 가까운 엣지 노드에 전달되고 사용자의 데이터가 가까운 곳에서 수신됩니다. 그런 다음 Tencent Cloud가 수년간 발전시켜온 가속 네트워크를 통해 최적의 링크를 선택해 데이터를 저장소로 전송합니다.
### 더 빠르고 안정적인 QUIC 프로토콜
약한 네트워크 환경과 네트워크 프로토콜의 비효율성 문제에 대한 보완책으로 VOD는 클라이언트의 업로드에 QUIC 프로토콜을 제공합니다. QUIC 프로토콜은 UDP 기반의 신뢰도가 높은 저지연 통신 프로토콜로, 현재 표준 HTTP3 프로토콜은 QUIC를 기반으로 구현됩니다. QUIC는 0-RTT 연결 설정 및 HOL(head-of-line) 블로킹 없는 다중화 기능을 지원하므로 실제 데이터 전송을 위한 네트워크 대역폭 활용을 최대화할 수 있으며, 패킷 손실률과 네트워크 지연이 높은 약한 네트워크 환경에서도 고품질의 데이터 통신을 제공합니다. 동시에 QUIC는 연결 마이그레이션을 지원하며 모바일이 자주 네트워크를 전환하는 시나리오에서도 원활하게 전환하여 네트워크가 중단되지 않도록 할 수 있습니다.
### 사용 진입 장벽 없는 지능형 선택
VOD로 제공되는 업로드 가속 솔루션은 고객이 콘솔에서 해당 기능을 활성화하는 것만으로 간편하게 사용할 수 있습니다. SDK를 사용하여 업로드할 때 일반 채널과 가속 채널을 지능적으로 속도를 비교하고 우수한 것을 선택하며 데이터 업로드에 QUIC 프로토콜을 사용할지 여부를 자동으로 감지하고 결정합니다.

## 사용 방법
다음 두 가지 간단한 단계를 통해 클라이언트 업로드 가속 기능을 활성화할 수 있습니다.
1. 클라이언트 업로드 가속에 대해서는 [콘솔 작업](https://intl.cloud.tencent.com/document/product/266/18874) 가이드를 참고하여 ‘글로벌 링크 가속’을 활성화하고 필요에 따라 ‘QUIC 전송’을 활성화합니다.
2. Android 및 iOS 플랫폼의 경우 App이 시작될 때 [사전 업로드](https://intl.cloud.tencent.com/document/product/266/37548)가 호출되는지 확인하십시오. ‘QUIC 전송’이 활성화된 경우 Android 플랫폼은 9.6 이상의 SDK 버전을 사용해야 하고 iOS 플랫폼은 10.4 이상의 SDK 버전을 사용해야 합니다.
>?
>- Android 및 iOS용 SDK는 업로드 가속과 QUIC 기반 전송을 모두 지원합니다.
>- 현재 Web 및 미니프로그램용 SDK는 업로드 가속만 지원하고, QUIC 기반 전송은 지원하지 않습니다.
