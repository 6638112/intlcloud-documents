VOD는 업로드 및 트랜스코딩된 비디오를 재생하는 여러 방법을 지원하며 비디오 재생에는 주로 다음 세 가지 시나리오가 포함됩니다.

| 시나리오 | 설명 | 권장 재생 방법 |
| -- | -- | -- |
| 쇼트 비디오 재생 | 길이가 5분 미만인 동영상 재생 | 기본 플레이어 |
| 긴 비디오 재생 | 전문 기관에서 제작하여 동영상 웹사이트에 게시한 동영상 재생 | Superplayer |
| 암호화된 비디오 재생 | 저작권 보호를 위해 암호화가 필요한 긴 동영상 재생 | Superplayer |

본문에서는 세 가지 시나리오의 특징과 권장 재생 방법에 대해 설명합니다.

## 쇼트 비디오 재생

쇼트 비디오는 일반적으로 다음을 포함하여 길이가 5분 미만인 동영상을 말합니다.

* 쇼트 비디오 소셜 플랫폼(예: WeSee, Kuaishou 및 TikTok)에서 공유된 비디오.
* 전자상거래 플랫폼(예: JD.com 및 Pinduoduo)에서 공유된 제품 프로모션 비디오.
* WeChat 공식 계정 및 당사 미디어에서 공유된 동영상.

<img src="https://qcloudimg.tencent-cloud.cn/raw/3a8215d3897533ec756bdd4b04e6b811.png" width="800" />

### 재생 아키텍처
쇼트 비디오 재생 시나리오의 경우 VOD는 URL을 매개변수로 사용하여 VOD 비디오를 재생하는 **기본 플레이어 SDK**를 제공합니다.

<img src="https://main.qcloudimg.com/raw/a9d4a24cefcdb3ccba1b316ea6be9ee0.png" width="700" />

기본 플레이어 SDK를 사용한 전체 재생 아키텍처는 다음과 같습니다.
1. **서버에서 업로드**: 비즈니스 백엔드는 콘솔, 서버 API 또는 기타 수단을 통해 비디오를 VOD에 업로드합니다.
2. **비디오 처리 트리거**: 비디오 업로드 중 트랜스코딩을 지정할 수 있습니다. 비디오가 업로드되면 트랜스코딩 프로세스가 시작됩니다.
3. **트랜스코딩 및 스토리지에 쓰기**: 비디오가 트랜스코딩된 후 출력 비디오 콘텐츠가 VOD 스토리지에 기록됩니다.
4. **콘텐츠 다운로드**: 기본 플레이어는 다운로드 주소의 VOD CDN에서 콘텐츠를 다운로드합니다.
5. **비디오 재생**: 기본 플레이어로 비디오를 재생합니다.

### 기본 플레이어 통합

## 긴 비디오 재생

긴 비디오는 일반적으로 전문 기구에서 제작하고 주로 다음을 포함하는 비디오 웹사이트에 게시된 비디오를 나타냅니다.

* 비디오 소셜 플랫폼(예: Tencent Video, Youku 및 iQIYI)에 게시된 독점 TV 시리즈 및 버라이어티 쇼.
* 온라인 교육 웹사이트(예: Tencent Class 및 Penguin Tutoring)에 게시된 강의 동영상.
* 온라인 TV 플랫폼(CNTV 및 Mongo TV 등)에서 재생되는 TV 프로그램.

### 재생 아키텍처

긴 비디오 재생 시나리오의 경우 VOD는 FileId를 매개변수로 사용하여 VOD 비디오를 재생하는 **Superplayer SDK**를 제공합니다.

<img src="https://main.qcloudimg.com/raw/f6b52d9429111812b0ee0e78654f2e34.png" width="700" />

Superplayer SDK를 사용한 전체 재생 아키텍처는 다음과 같습니다.
1. **서버에서 업로드**: 비즈니스 백엔드는 콘솔, 서버 API 또는 기타 수단을 통해 비디오를 VOD에 업로드합니다.
2. **비디오 처리 트리거**: 비디오 업로드 중 어댑티브 비트 레이트 스트리밍을 지정합니다. 동영상이 업로드되면 비디오 처리가 시작됩니다.
3. **어댑티브 비트스트림으로 코드 변환 및 스토리지에 쓰기**: 비디오가 어댑티브 비트스트림으로 변환된 후 출력 비디오 콘텐츠가 VOD 스토리지에 기록됩니다.
4. **미디어 자산 업데이트**: 출력 비디오 정보가 미디어 자산 관리 모듈에 기록됩니다.
5. **다운로드 주소 요청**: Superplayer는 비디오의 FileId가 지정된 후 VOD 재생 서비스에서 비디오의 다운로드 주소를 가져옵니다.
6. **콘텐츠 다운로드**: Superplayer는 다운로드 주소의 VOD CDN에서 콘텐츠를 다운로드합니다.
7. **재생**: Superplayer는 출력 어댑티브 비트스트림을 재생합니다.

재생할 비디오에 대해 [Key 링크 도용 방지](https://intl.cloud.tencent.com/document/product/266/33986)가 활성화된 경우 재생 중에 [Superplayer 서명](https://intl.cloud.tencent.com/document/product/266/38099)을 지정해야 합니다. 이 경우 아키텍처는 다음과 같습니다.

1. ‘서명 배포’ 서비스는 비즈니스 백엔드에 구축되어야 하며, 서명은 [플레이어 서명 계산 규칙](https://intl.cloud.tencent.com/document/product/266/38099#.E7.AD.BE.E5.90.8D.E8.AE.A1.E7.AE.97)에 따라 생성됩니다.
2. Superplayer는 비디오를 재생하기 전에 플레이어 서명을 가져와야 합니다(아래 5단계 참고).
3. Superplayer가 다운로드 주소를 요청하면(아래 6단계 참고) 재생 서비스는 서명이 유효한지 확인한 후에만 다운로드 주소를 반환합니다.

<img src="https://main.qcloudimg.com/raw/e5ae52f1b5f15f289b6f54aa28917da4.png" width="700" />
