## 기능 소개
- HTTP 프로토콜에서 지원하는 Referer 메커니즘을 기반으로 HTTP 헤더의 Referer 필드를 통해 요청 소스를 식별할 수 있습니다. Referer 블록리스트 또는 얼로우리스트를 구성하여 비디오 요청 소스를 식별하고 인증할 수 있습니다.
- 블록리스트 및 얼로우리스트 모드가 지원됩니다. 비디오 재생 요청이 CDN 노드에 도달하면 노드는 구성된 Referer 블록리스트 또는 얼로우리스트에 따라 요청 소스를 인증합니다. 요청이 규칙을 충족하면 CDN은 비디오 데이터를 반환하고, 그렇지 않으면 403 응답 코드를 반환하고 재생 요청을 거부합니다.

>?Referer 링크 도용 방지 활성화에 대한 자세한 내용은 [링크 도용 방지 설정](https://intl.cloud.tencent.com/document/product/266/14060#referer-.E9.98.B2.E7.9B.97.E9.93.BE)을 참고하십시오.

## 주의 사항
* 이 기능은 선택 사항이며 기본적으로 활성화되어 있지 않습니다.
* 이 기능을 활성화한 후, 블록리스트 또는 얼로우리스트를 선택 및 구성합니다. 두 가지 모드는 상호 배타적이며 동시에 하나만 지원됩니다.
* 블록리스트 또는 얼로우리스트에 1 - 10개의 도메인 이름을 입력할 수 있습니다(한 줄에 하나의 항목).
* 도메인 이름 앞에 프로토콜 이름(`http://` 및 `https://`)을 넣지 마십시오. 도메인 이름 일치는 접두사를 기반으로 합니다(예를 들어, `abc.com`을 입력하면 `abc.com/123`과 `abc.com.cn`이 모두 일치됨). 와일드카드`(*.abc.com)`가 지원됩니다.
