ここでは、モニタリングダッシュボードで、进入指定ルームに入った後に通話詳細画面にてどのようなデータ情報を取得できるかについて述べます。



##　操作手順

1. TRTCコンソールにログインし、【[モニタリングダッシュボード](https://console.cloud.tencent.com/trtc/monitor)】の中で確認したいルームを選択します。
2. ルーム IDまたは右側の【通話詳細の確認】をクリックして、通話詳細画面に入ります。
   ![](https://main.qcloudimg.com/raw/fb52699e808f400969a6816d5457cc4d.png)

## 情報の解読

### 基本情報

- **通話ルーム情報：**現在のルームの基本情報。 SDKAppID、アプリケーション名、ルーム ID、ルーム累計ユーザー数、通話開始・終了時間、持続時間などのデータが含まれます。
  ![](https://main.qcloudimg.com/raw/417aecad2089acc5ac1f38480dba5d66.png)
- **現在の表示時間範囲：**その回のクエリーに対応する時間区間を表示します。1回のクエリーの最大表示時間は5時間です。
  ![](https://main.qcloudimg.com/raw/9d8e9db11232fd9f4c19d60676551373.png)

### 通話ユーザーリスト

表示内容にはユーザー ID、ユーザーロール、ユーザータイプ、リージョン、ユーザーの現在の時間範囲内の滞在状態、ユーザー入退室時間、持続時間、SDK バージョン、SDK タイプ、デバイス情報およびネットワーク情報が含まれます。
![](https://main.qcloudimg.com/raw/d69b22d292699374390ca3909bd4669a.png)

| データ項目                       | 説明                                                         |
| ---------------------------- | ------------------------------------------------------------ |
| ユーザー ID                      | デフォルトでは直近の6名ユーザーを表示し、最大で同時に20名のユーザーを表示します。当該ユーザー数が6人を超えた場合、開発者は手動で【ユーザー追加】、または【全ユーザー確認】を選択し追加できます。 |
| ユーザーロール                     | 分为キャスター、視聴者の2種類に分かれます。キャスターは上り、下りに権限があり、視聴者は下りのみ権限があります。 |
| ユーザータイプ                     | 展示ユーザーの上り、下りタイプを表示します。緑色の上矢印は、ユーザーに上りデータがあったことを表し、黄色の下矢印はユーザーに下りデータがあったことを表します。 |
| リージョン                         | ユーザー対応する地理的リージョン。                                         |
|ユーザーの現在時間範囲内の滞在状況 | その回のクエリーの表示時間範囲の中で、ユーザーがルーム内に滞在した時間帯を青色のブロックで表示します。ユーザーがルーム内にいなかった時間帯は灰色のブロックで表示します。 |
| ユーザー入退室時間               | ユーザーが最初に入室し、最後に退室するまでの時間。                 |
| 持続時間                     | ユーザーがルーム内にいた累計滞在時間。                                 |
| SDK バージョン                     | ユーザーの現在のクライアントのSDK バージョン番号。                                |
| SDK タイプ                     | ユーザーが使用するプラットフォームまたはOSのタイプ。                               |
| デバイスディスプレイ                     | ユーザーが使用するシステム型番またはコード。                                   |
| ネットワーク表示                     | ユーザーが使用するネットワークタイプ（ミニプログラム端末はNULLと表示される）。                 |



### 受発信端末データ

通話ユーザーについては、在接收リモートの他の人のデータを受信すると同時に、也会将自分のローカルのデータも送信します。このため受信と送信の両方向のデータ情報が必要です。デフォルトの状態では、モニタリングダッシュボードの通話詳細画面に表示されるのは、【[受信端末ビュー](#receive)】でのデータ概要となりますが、【[送信端末ビュー](#send)】をクリックして切り替え、確認することができます。

#### 受信端末ビュー<span id="receive"></span>

![](https://main.qcloudimg.com/raw/36f243b1f17e2c5d2ef393530f2c714b.png)

- 【全て】、【ビデオ】、【音声】、【画面共有】の4つのサブビューに分かれ、対応するタイプのデータを受信した時は、モニタリングダッシュボード画面に対応するタグ項目が表示されます。ない場合は表示されません。。
- 概要表には、デフォルトでは、ユーザーが受信した他のユーザーのデータ状況を全て表示します。異なる色のリージョンマーケティングによって、開発者は指定する送信端末ユーザーを選択し、単独のユーザーデータ概要を確認できます。
- 概要表に出現する赤線の部分は、表示この時間点でネットワークジッターが起きたことを表します。受信端末の右上角の【送信端末を選択して詳細を確認】をクリックして指定するユーザー IDを選択すれば、 [全リンク詳細](https://intl.cloud.tencent.com/zh/document/product/647/39071)に入って詳細な通話データを確認できます。
- 表示する時間区間が長い時は、データチャートの中でズームのスクロールを使用してより詳細な時間点データを確認することができます。



#### 送信端末ビュー<span id="send"></span>
![](https://main.qcloudimg.com/raw/67419d268bea1b7d8e0ba0b7c4a67164.png)
- 【全て】、【ビデオ】、【音声】、【画面共有】の4つのサブビューに分かれ、対応するタイプのデータを惣村する時は、モニタリングダッシュボード画面に対応するタグ項目が表示されます。ない場合は表示されません。
- 概要表に出現する赤線の部分は、表示この時間点でネットワークジッターが起きたことを表します。送信端末の右上角の【詳細の確認】をクリックすると [全リンク詳細](https://intl.cloud.tencent.com/zh/document/product/647/39071) 画面にジャンプし、送信端末の詳細データとイベントが確認できます。
- 表示する時間区間が長い時は、データチャートの中でズームのスクロールを使用してより詳細な時間点データを確認することができます。
