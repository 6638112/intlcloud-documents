이벤트 콜백 서비스는 TRTC 서비스 이벤트를 HTTP/HTTPS 요청 형식으로 사용자 서버에 공지합니다. 해당 서비스에는 방 이벤트 그룹(Room Event)과 미디어 이벤트 그룹(Media Event)의 일부 이벤트가 통합되어 있습니다. 아래 가이드에 따라 TRTC 콘솔에서 콜백 설정 정보를 입력하면 콜백 이벤트 공지를 수신할 수 있습니다.

<span id="step0"></span>
## 전제 조건
1. TRTC 콘솔에 접속하여 [애플리케이션 관리](https://console.cloud.tencent.com/trtc/app)를 선택합니다.
2. 콜백 설정할 애플리케이션을 선택하고 오른쪽 작업 열의 [애플리케이션 정보]를 클릭합니다.
3. 상세 페이지로 이동하여 [콜백 설정] 탭을 클릭하면 콜백 설정 정보를 설정할 수 있습니다.


<span id="setkey"></span>
## 콜백 키 설정
1. [【콜백 설정】 탭](#step0)의 '콜백 키' 모듈에서 오른쪽 상단에 있는 [편집]을 클릭합니다.
2. 실제 서비스 수요에 따라 콜백 키(옵션)를 작성합니다.
>? 콜백 키는 대소문자 알파벳 및 숫자로 구성되며 최대 32자까지 입력할 수 있습니다.
5. [확인]을 클릭하면 콜백 키 설정이 완료됩니다.

<span id="setadd"></span>
## 콜백 주소 설정
1. [【콜백 설정】 탭](#step0)의 '콜백 주소' 모듈에서 오른쪽 상단에 있는 [편집]을 클릭합니다.
2. 실제 서비스 수요에 따라 콜백 주소(필수)를 입력합니다.
	- 방 콜백: 방 생성/삭제 및 입장/퇴장 등의 이벤트 공지를 지원합니다.
	- 미디어 콜백: 비디오 데이터 푸시 시작/일시 정지, 오디오 데이터 푸시 시작/일시 정지, 서브 채널 데이터 푸시 시작/ 일시 정지 등의 이벤트 공지를 지원합니다.
>? 콜백 URL 프로토콜 헤더: HTTP, HTTPS 등 최대 2083자까지 지원합니다.
3. [확인]을 클릭하면 콜백 주소 설정이 완료됩니다.
