자체구축한 원본 서버와 라이브 스트리밍 소스가 있고 Tencent Cloud를 통해 콘텐츠를 스트리밍하려는 경우 CSS 재생 도메인 이름에 대한 원본 서버 정보를 설정할 수 있습니다. 설정이 성공하면 원본 서버에서 라이브 스트림을 가져와 CSS를 통해 배포할 수 있습니다. 이 문서에서는 원본 서버를 설정하는 방법을 설명합니다.

## 주의 사항
- 원본 서버 설정은 설정이 완료된 후 약 1시간 후에 적용됩니다.
- 재생 도메인에 대한 원본 서버 설정 후에는 더 이상 'StreamName'을 지정하여 푸시 도메인을 재생 도메인에 바인딩할 수 없습니다. 재생 도메인에 대해 워터마크, 트랜스 코딩, 녹화, 화면 캡처, 음란물 감지 작업을 설정할 수도 없습니다. 


## 전제 조건
- [CSS 콘솔](https://console.cloud.tencent.com/live)에 로그인되어 있어야 합니다.
- 라이브 방송 원본 서버가 구축되어 있어야 합니다.
- **재생 도메인**이 추가되어 있어야 합니다.

## 작업 단계
1. CSS 콘솔에서 왼쪽 사이드바의 [**도메인 관리**](https://console.cloud.tencent.com/live/domainmanage)를 선택합니다. **재생 도메인 이름**을 클릭하거나 오른쪽에서 **관리**를 클릭합니다.
2. **고급 설정** 탭을 선택합니다.
3. 원본 서버 설정 영역에서 **편집**을 클릭하고 아래 단계를 따릅니다.
 1. **원본 서버 설정**을 켭니다.
 2. 다음과 같이 설정을 완료합니다.

<li>포워딩 프로토콜이 FLV인 경우 기본적으로 FLV 재생이 지원됩니다. HLS를 선택할 수도 있습니다.</li>

 4. **저장**을 클릭하면 설정이 저장됩니다. 
>? **포워딩 프로토콜**은 원본 서버에서 스트림을 가져올 때 CSS에서 지원하는 프로토콜을 나타냅니다. **재생 프로토콜**은 CSS CDN을 통해 라이브 콘텐츠를 배포하는 데 사용되는 프로토콜을 나타냅니다.
