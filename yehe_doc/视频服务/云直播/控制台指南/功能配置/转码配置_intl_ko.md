본 문서에서는 [LVB 콘솔](https://console.cloud.tencent.com/live)을 통해 트랜스 코딩 설정을 생성하는 방법을 소개합니다. 생성 완료 후 해당 푸시 스트리밍 도메인에서 [트랜스 코딩 설정](https://intl.cloud.tencent.com/document/product/267/31062)을 연결해야 하며, 연결하고 약 5~10분 후 적용됩니다. API를 통해서도 라이브 방송 채널에 트랜스 코딩 템플릿을 생성할 수 있습니다.

시스템에서는 일반 트랜스 코딩과 고속 HD 트랜스 코딩 두 가지 트랜스 코딩 서비스를 지원하며, 각각 일반, SD, HD, 오디오 4가지 템플릿을 제공합니다.

## 트랜스 코딩 템플릿 생성
LVB 콘솔에 로그인하여 왼쪽 메뉴바에서 [기능 템플릿]>[트랜스 코딩 설정](https://console.cloud.tencent.com/live/config/transcode)을 선택하여 트랜스 코딩 템플릿을 추가/삭제할 수 있습니다.

일반, SD, HD 템플릿 중에서 한 개를 선택하면 시스템이 자동으로 비디오 비트레이트와 높이를 입력하며, 수동으로 비디오 비트레이트와 높이를 설정하여 사용자 정의 비트레이트 템플릿을 생성할 수도 있습니다.       
![](https://main.qcloudimg.com/raw/c3208fe3e06c785041954f176960dc8a.png)

> 트랜스 코딩 이름은 문자와 숫자 조합만 지원되며, 10자를 초과할 수 없습니다.


## 도메인 연결

트랜스 코딩 템플릿 생성 후 [Domain Management](https://console.cloud.tencent.com/live/domainmanage)에서 해당 푸시 스트리밍 도메인을 선택해 [템플릿 설정]으로 이동한 후, [편집]을 클릭하여 해당 도메인에 트랜스 코딩 템플릿을 지정해 줍니다. 자세한 방법은 [트랜스 코딩 설정](https://intl.cloud.tencent.com/document/product/267/31062)을 참조하십시오.
![](https://main.qcloudimg.com/raw/7c78f911a92f05293777f19957695874.png)

> 트랜스 코딩 템플릿을 지정하면 백그라운드에서 해당 비트레이트의 서로 다른 재생 주소를 생성하여 편리하게 선택하여 호출할 수 있습니다. 푸시 스트리밍 원본 해상도가 최대한 원본 비율과 비슷해야 화면이 늘어나는 변형이 발생하지 않습니다.
> 사용자가 처음 새로운 비트레이트 주소에 액세스하는 경우, 처음 트리거된 링크에 접속한 사용자의 로딩 시간이 조금 길어지는 것은 정상적인 현상입니다.


**실시간 트랜스 코딩 기능 사용 방법은 다음과 같습니다.**

1. Tencent Cloud Web 플레이어 SDK를 사용하는 경우 인코딩 후 플레이어 오른쪽 하단에 화질 선택란에 자동으로 채널 설정에 따라 해당하는 가용 비트레이트가 표시되며, 사용자가 수동으로 조정할 수 있습니다. 기본 판단 로직은 다음과 같습니다.
 - PC: 원본 해상도 우선 적용
 - 모바일: [HD 전환], 즉 900kbps 해상도를 우선 적용하며, 가용 불가 시 원본 해상도 우선 적용
2. 각 비트레이트에 해당하는 주소를 직접 획득하여 3rd party 플레이어에서 직접 재생합니다.
