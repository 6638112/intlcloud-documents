### ライブブロードキャストに同時接続数の上限がありますか。	
LVBでは、デフォルトでライブブロードキャストの視聴者数が制限されません。ネットワークなどの条件が許す限り、ライブブロードキャストを視聴できます。ユーザー側で帯域幅制限が設定されている場合、視聴者が多すぎて帯域幅を超えた場合、新規ユーザーは視聴できなくなります。この場合、オンラインユーザーの数は制限されます。

<span id="que2"></span>
### 再生トランスコーディングをどのように使用すればいいですか。
異なるネットワーク要因を考慮して、異なるビットレートと解像度を使用する要件に対応するために、[Transcoding Configuration](https://console.cloud.tencent.com/live/config/transcode)に移動して、異なるビットレートと異なる解像度のトランスコーディングテンプレートを設定します。トランスコーディングの詳細については、[Best Practice - LVB Encapsulating and Transcoding](https://intl.cloud.tencent.com/document/product/267/31561）をご参照ください。

 #### <span id="multirate">オリジナル、HD、SDシーン</span>
サービス再生のシナリオでは、一般的に、オリジナル、HD、SDの3つのビットレートが使用されます。
 - 元のストリームの解像度は、プッシュスビットレートの解像度と同じです。
 - HDストリーミングの推奨ビットレート：2000kbps、推奨解像度：1080p。
 - SDストリーミングの推奨ビットレート：1000kbps、推奨解像度：720p。

### タイムシフトプレイバックをどのように使用すればいいですか。
過去の特定時間帯のハイライトをプレイバックしたい場合は、現在HLSプロトコルのみをサポートしているタイムシフト機能を使用できます。

### HTTPSを使用して再生するにはどうすればよいですか。
再生ドメイン名がHTTPS をサポートする必要がある場合は、有効な証明書コンテンツと有効な秘密鍵コンテンツを準備する必要があります。[Manage Domain]（https://console.cloud.tencent.com/live/domainmanage）に移動し、【Play Domain Management】 > 【Advanced Configuration】 > 【HTTPS Configuration】を選択して設定を追加します。追加が完了してから有効になるまで2時間の有効化時間かかります、有効になるとライブブロードキャストストリームはHTTPSプロトコルを使用して再生できます。

### 中国本土以外のアクセラレーションノード再生を使用するにはどうすればよいですか。
LVB CDNノードは、中国本土だけでなく、世界のすべての大陸でノードカバレッジがあります、広く安定したカバレッジを実現しています。ユーザーが中国香港、中国マカオ、中国台湾、または中国本土以外の他の地域にいる場合は、[Manage Domain](https://console.cloud.tencent.com/live/domainmanage) でドメイン名を設定するときに、アクセラレーションリージョンを【Global Acceleration】または【Outside Mainland China and Hong Kong/Macao/Taiwan】を選択して、中国本土以外ノードカバレッジのサポートを利用します。
>! LVB GCDは現在、HTTP-FLV + HLSプロトコルのみをサポートしています。

### 再生リンク不正アクセス防止をどのように有効にすればいいですか。
不正なユーザーが再生URLを盗用して他のサイトで再生することによるトラフィック損失を防ぐために、再生URLにリンク不正アクセス防止を追加して、リンク不正アクセスによる損失を防ぐことを強くお勧めします。LVBの再生リンク不正アクセス防止は、主に4つのパラメータ値（txTime、キー（ハッシュキー）、txSecret、および有効時間）によって制御されます。

| リンク不正アクセス防止パラメータ | 説明 | 補足説明|
|---------|---------|---------|
| txTime |再生URLの有効時間　|形式は16進数のUNIX時間です。<br>現在のtxTime値が現在リクエストされている時間よりも大きい場合、正常に再生できます。それ以外の場合、再生はバックグラウンドによって拒否されます。|
| キー | MD5 計算方法のキー　|　カスタマイズ可能で、プライマリキーとバックアップキーの両方を設定できます。<br>プライマリキーが漏えいされた場合、バックアップキーを使用して、再生URLを結合し、またプライマリキーの値を変更できます。|
| txSecret | 再生URLの暗号化パラメータ　|　値は、key、StreamName、txTime を順次に連結した文字列をMD5暗号化アルゴリズムを使用して取得されるものです。<br>txSecret=MD5（key+StreamName+txTime）。|
|　有効時間　|　アドレスの有効時間　|　有効時間設定は0より大きく設定する必要があります。<br> txTimeが現在の時刻に設定され、有効化時間が300秒に設定されているとすると、再生URLの期限切れは現在の時刻+300秒になります。|

#### リンク不正アクセス防止計算
リンク不正アクセス防止の計算には、key（ランダムな文字列）、StreamName（ストリーム名）、およびtxTime（16進形式）の3つのパラメータが必要です。
キーを**somestring**に、ストリーム名（StreamName）を**test**に、txTimeを**5c2acacc**（2019-01-01 10:05:00）に設定するとします。HDビットレートは**900kbps**で、トランスコーディングテンプレート名は**900**です。
元のストリーミングアドレス：
```
txSecret = MD5(somestringtest5c2acacc) = b77e812107e1d8b8f247885a46e1bd34
http://domain/live/test.flv?txTime=5c2acacc&txSecret=b77e812107e1d8b8f247885a46e1bd34
http://domain/live/test.m3u8?txTime=5c2acacc&txSecret=b77e812107e1d8b8f247885a46e1bd34
```
HDストリーミング再生アドレス：
```
txSecret = MD5(somestringtest_9005c2acacc) = 4beae959b16c77da6a65c7edda1dfefe
http://domain/live/test_900.flv?txTime=5c2acacc&txSecret=4beae959b16c77da6a65c7edda1dfefe
http://domain/live/test_900.m3u8?txTime=5c2acacc&txSecret=4beae959b16c77da6a65c7edda1dfefe
```

#### 再生リンク不正アクセス防止を有効にする
1.[Manage Domain] （https://console.cloud.tencent.com/live/domainmanage）にログインします。
2.再生ドメイン名を選択するか、当該行の【Manage】をクリックして、ドメイン名の詳細ページに入ります。
3. 【Access Control】を選択し、【Edit】をクリックします。
4. 【Play Authentication】を有効に設定し、【Save】をクリックします。

>!
>- 再生認証の設定が完了してから有効になるまで**30分**かかります。
>- HTTP-FLV：再生中のURLは、txTimeの期限切れ後も正常に再生でき、txTimeの期限切れ後に再生の再再リクエストは拒否されます。
>- HLS：HLSはショートリンクであるため、常にm3u8をリクエストして最新のtsパートを取得しようとします。txTimeの値を現在の時刻+10分に設定すると、HLS再生URLリクエストは10分後に拒否されます。この問題を解決するには、サービス側でHLSリクエストアドレスを動的に更新するか、HLS再生アドレスの期限切れを長く設定します。

### 再生認証設定のマスターキーの形式の要件は何ですか。有効時間の長さに制限はありますか。
認証設定のメインキー値は、大文字、小文字、および数字のみをサポートし、最大長は256桁です。英字と数字のランダムな組み合わせが可能です。
有効期間は、ライブブロードキャストの継続時間に設定することをお勧めします。


<span id="que8"></span>
### 固定のライブブロードキャストプッシュアドレスを作成できますか。アドレスの有効期間は最大でどのくらい設定できますか。
プッシュスアドレスは主に不正アクセス防止と認証のために有効時間が設定されます。固定プッシュスアドレスを作成すると、プッシュの盗用が発生し、現在のサービスに損失を与える可能性があります。
LVBプッシュアドレスの有効時間に制限はなく、個人業務要件に応じてアドレスの有効期限を設定できます。またルール結合により有効期間の長いプッシュURLも生成できます。結合方法の詳細については、[How to Splice a Push URL](https://intl.cloud.tencent.com/document/product/267/32480)をご参照ください。

>? プッシュアドレスの有効期間を長すぎないように設定することはお勧めしません。長すぎると、プッシュアドレスの使用中にエラーが発生して、認証の失敗が通知されます。

### ライブブロードキャストの遅延はどれくらいですか。 
通常の状況では、RTMPプロトコルを使用してプッシュし、FLVプロトコルを介して再生します。遅延は約2秒〜3秒です。長すぎる場合は、問題が発生する可能性があります。
<span id="que11"></span>
### ライブブロードキャスト中に最大ビットレートを設定できますか。
  いいえ、最大ビットレートはプッシュ側で設定されますが、プッシュ側で最高ビットレートを設定するのも、ご利用のネットワークのアップロード速度にも依存します。ビットレート（または最大ビットレート）の上限はご利用のネットワークアップロード速度です。あまりにも高く設定すると、LVB画面でフレーム損失やラグが発生することがあります。

### Tencent Cloud LVBで使用されていないライブブロードキャストルームを削除するにはどうすればよいですか。 
LVBは現在、ライブストリームIDを使用して、ライブブロードキャストプッシュと再生を関連付けており、削除する必要があります。インスタントメッセージングIMサービスをご利用され、上限に達しないようにIMルームを削除する場合は[Disbanding Groups](https://intl.cloud.tencent.com/document/product/1047/34896を参照して、操作してください。
チャネルモードをご利用されている場合は、DeleteLVBChannel-LVBチャネルを削除　APIを呼び出して、削除するLVBチャネルのID番号を入力して（一括削除可能）削除できます。

>! チャンネルモードは古いバージョンで、更新もメンテナンスもされなくなりました。

 ### ライブブロードキャストでプッシュインターフェースを有効化・無効化する枠割は何ですか。 
LVBのプッシュインターフェースの有効化・無効化は、主にポルノ検出する時のライブブロードキャスト禁止に使用されます。例えば、キャスターがポルノ絡みコンテンツまたは反社会的なコンテンツを拡散していることがバックグラウンドで判明された場合、このストリーミングはいつでも中断またはブロックできます。

### バックグラウンド再生を実装する方法とは何ですか。
バックグラウンド再生は端末側の機能であり、お客様の実際のサービスロジックに基づいて開発する必要があります。ライブブロードキャストストリームが中断されない限り、バックグラウンドでのオーディオ再生をサポートできます。

### HTTPS設定を変更する際に、証明書情報を追加した後、「証明書が正しくありません」というメッセージが表示されますが、どうすればよいですか。 
 LVBの暗号化サービスにNginxを使用しているため、証明書タイプはNginxである必要があります。現在の証明書タイプが正しいかどうかを確認してください。

### 再生ドメイン名の認証を無効にした後、元々生成された再生アドレスを再生できなくなりましたが。 
 再生認証に有効期間を設定できます。有効期間内にアクセス認証を無効にしても、元のアドレスは再生できません。 

<span id="que17"></span>
### インターフェースのアクセス回数の制限をどのように理解すればよいですか。
LVBには、アカウント内のすべてのSecretIdから送信されるリクエストの総数にアクセス回数の制限があります。制限を超えたリクエストは、正常に応答されません。
例えば、1秒間に200回以内の制限とは、ご利用のアカウントのすべてのSecretIdから送信されたこのようなリクエストは、Tencent Cloud CVMでは1秒以内に200件しか受信しないことです。200件のリクエストが1人の利用者から200回送信されたものか、200名の利用者から1回ずつ送信されたものか、1つか複数のストリームへのクエリーかについて、サーバー側にも制限がありません、リクエストの合計数のみがカウントされます。 

<span id="que18"></span>
### プッシュ中に「RTMP close」が表示され、プッシュが失敗しますが、ログはプッシュが成功したことを示すのはなぜですか。 
現在のプッシュアドレスが不正である可能性があります。ミニプログラムDemoを使用して、プッシュアドレスが正常にプッシュできるかどうかをテストすることをお勧めします。

<span id="que19"></span>
### フレームレートを変更した後、正常にプッシュできず、サービスをローカルで数回再起動し、頻繁に切断する必要があります。どうすればよいですか。 
現在のフレームレート設定値が高すぎる可能性があります。15fps以上であればスムーズなビデオ再生が可能なので、フレームレートを下げることをお勧めします。  

<span id="que20"></span>
### どのような状況で、システムにより長時間にデータのないプッシュを切断しますか。 
プッシュ中にデバイスに障害が発生した場合に、このような問題が発生します。
例えば、アプリのクラッシュ、携帯電話のシャットダウンなどの外部要因による異常により、バックグラウンドで70秒以内にデータプッシュストリームを収集できなくなり、システムにより切断されます。
