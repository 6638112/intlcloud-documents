<span id="que1"></span>
### 라이브 방송은 접속 인원 수 제한이 있습니까?	
Tencent Cloud CSS는 기본적으로 라이브 방송 시청 접속 인원 수 제한이 없습니다. 네트워크 등 조건이 허용되면 모두 다 라이브 방송을 시청할 수 있습니다. 사용자가 대역폭 제한을 설정한 경우 시청자 수가 너무 많거나 대역폭 제한을 초과하면 새로운 이용자는 시청할 수 없으며, 이런 경우 인원 제한이 발생합니다.

<span id="que2"></span>
### 재생 트랜스 코딩 사용 방법은 어떻게 됩니까?
다양한 네트워크 요소를 고려하여 각종 비트레이트 및 해상도를 제공합니다. [트랜스 코딩 설정](https://console.cloud.tencent.com/live/config/transcode)에서 비트레이트 및 해상도별 트랜스 코딩 템플릿을 설정할 수 있습니다. 트랜스 코딩과 관련된 더 많은 정보는 [모범 사례-라이브 방송 캡슐화 및 트랜스 코딩](https://intl.cloud.tencent.com/document/product/267/31561)을 참고하십시오.

 #### <span id="multirate">저화질, 고화질, 표준 화질 시나리오</span>
비즈니스 재생 시나리오에는 일반적으로 3가지 비트레이트(저화질, 고화질, 표준 화질)를 사용합니다.
 - 저화질 스트림과 푸시 스트림의 비트레이트 해상도는 일치합니다.
 - 고화질 스트림의 경우 2000kbps 비트레이트, 1080p 해상도 사용을 권장합니다.
 - 표준 화질 스트림의 경우 1000kbps 비트레이트, 720p 해상도 사용을 권장합니다.

<span id="que3"></span>
### 타임 시프트 다시보기의 사용 방법은 어떻게 됩니까?
지나간 시점의 콘텐츠를 다시 보고 싶을 경우 타임 시프트 기능을 사용할 수 있습니다. 현재 타임 시프트 기능은 HLS 프로토콜만을 지원합니다.
<span id="que4"></span>
### HTTPS 재생 사용 방법은 어떻게 됩니까?
재생 도메인에 HTTPS 지원이 필요할 경우 유효한 인증서, 프라이빗 키를 준비하여 [도메인 이름 관리](https://console.cloud.tencent.com/live/domainmanage)에서 [재생 도메인 관리]>[고급 설정]>[HTTPS 설정]을 선택해 설정을 추가합니다. 추가 후 적용 시간(2시간)이 필요하며, 적용 후 라이브 방송 스트림에서 HTTPS 프로토콜 재생이 지원됩니다.

<span id="que5"></span>
### 해외 가속(GCD) 노드 재생 사용 방법은 어떻게 됩니까?
CSS CDN 노드는 중국대륙뿐만 아니라 전 세계 각 대륙에 분포되어 있으며, 범위가 넓고 안정적입니다. 사용자가 중국홍콩, 중국마카오, 중국대만 또는 중국 외 다른 지역에 분포하고 있을 경우 [도메인 이름 관리](https://console.cloud.tencent.com/live/domainmanage)에서 도메인 설정 시 가속 구역으로 [글로벌 가속] 또는 [중국홍콩, 마카오, 대만 지역 및 해외 지역]을 선택하면 해외 노드가 지원됩니다.
>! 현재 CSS GCD는 HTTP-FLV + HLS 프로토콜만을 지원합니다.

<span id="que6"></span>
### 재생 링크 도용 방지 활성화 방법은 어떻게 됩니까?
불법 사용자가 재생 URL을 도용하여 다른 곳에서 재생해 트래픽 손실을 야기한 경우 재생 주소에 재생 링크 도용 방지를 추가할 것을 강력히 권장합니다. CSS의 재생 링크 도용 방지는 txTime, key(해시 키), txSecret, 유효 시간 등 4가지 매개변수로 제어됩니다.

| 링크 도용 방지 매개변수 | 설명 | 보충 설명 |
|---------|---------|---------|
| txTime | 재생 URL의 유효시간 |형식은 16진법 UNIX 시간.<br> 현재 txTime값이 요청한 시간보다 클 경우 정상적으로 재생됩니다. 그렇지 않을 경우 백그라운드에서 재생을 거부합니다.|
| key | MD5 컴퓨팅 방식의 키 | 사용자 정의가 가능하며, 마스터/슬레이브 2개 Key를 설정할 수 있습니다. <br> 마스터 key가 유출된 경우 슬레이브 key를 사용하여 재생 URL 스플라이싱이 가능하며, 마스터 Key의 값을 변경할 수 있습니다.|
| txSecret | 재생 URL의 암호화 매개변수 | 값은 key, StreamName, txTime을 순서대로 스플라이싱한 스트링으로 MD5 암호화 알고리즘으로 도출합니다.<br>txSecret = MD5（key+StreamName+txTime). |
| 유효시간 | 주소 유효시간 | 유효시간은 0보다 커야 합니다. <br> txTime을 현재 시간으로, 유효시간을 300초로 설정할 경우 재생 URL 만료시간은 현재 시간+300초입니다.|

#### 링크 도용 방지 계산
링크 도용 방지 계산에는 key(랜덤 스트링), StreamName(스트림명), txTime(16진법 형식)의 3가지 매개변수가 필요합니다.
예: key: **somestring**, 스트림명(StreamName): **test**, txTime: **5c2acacc**(2019-01-01 10:05:00)으로 설정할 경우 고화질 비트레이트: **900kbps**, 트랜스 코딩 템플릿명: **900**입니다.
저화질 스트림 재생 주소:
```
txSecret = MD5(somestringtest5c2acacc) = b77e812107e1d8b8f247885a46e1bd34
http://domain/live/test.flv?txTime=5c2acacc&txSecret=b77e812107e1d8b8f247885a46e1bd34
http://domain/live/test.m3u8?txTime=5c2acacc&txSecret=b77e812107e1d8b8f247885a46e1bd34
```
고화질 스트림 재생 주소:
```
txSecret = MD5(somestringtest_9005c2acacc) = 4beae959b16c77da6a65c7edda1dfefe
http://domain/live/test_900.flv?txTime=5c2acacc&txSecret=4beae959b16c77da6a65c7edda1dfefe
http://domain/live/test_900.m3u8?txTime=5c2acacc&txSecret=4beae959b16c77da6a65c7edda1dfefe
```

#### 재생 링크 도용 방지 활성화
1. [도메인 이름 관리](https://console.cloud.tencent.com/live/domainmanage)에 로그인합니다.
2. 재생 도메인을 선택하거나 속한 행의 [관리]를 클릭하여 도메인 상세 페이지에 진입합니다.
3. [접근 관리]를 선택한 후 [편집]을 클릭합니다.
4. [재생 인증]을 활성화한 후 [저장]을 클릭합니다.

>!
>- 재생 인증 설정 성공 후 **30분**이 지나야 적용됩니다.
>-  HTTP-FLV: 현재 재생 중인 URL은 txTime 만료 후에도 정상적으로 재생되며, txTime 만료 후 다시 재생을 요청하면 거부합니다.
>- HLS: HLS는 단축 링크로 계속 m3u8을 요청하여 최신 ts 멀티파트를 획득합니다. txTime값을 현재 시간+10분으로 설정할 경우 10분 후 HLS 재생 URL 요청을 거부합니다. 따라서 비즈니스 측에서 HLS의 요청 주소를 동적으로 갱신하거나 HLS의 재생 주소 만료 시간을 길게 설정합니다.

<span id="que7"></span>
### 재생 인증 설정 중 마스터 Key 형식 조건 및 유효 시간 제한 여부는 어떻게 됩니까?
인증 설정 중 마스터 Key 값은 알파벳 대문자, 소문자, 숫자만을 지원하며 길이는 최대 256자입니다. 알파벳, 숫자를 자유롭게 조합하면 됩니다. 자세한 내용은 [재생 인증 설정](https://intl.cloud.tencent.com/document/product/267/3246)을 참고하십시오.
유효 시간은 1회 재생 시간으로 설정하는 것을 권장합니다.


<span id="que8"></span>
### 고정된 라이브 방송 푸시 스트림 주소를 생성할 수 있습니까? 주소 유효 시간은 최대 얼마까지 설정할 수 있습니까?
푸시 스트림 주소 유효 시간 설정은 인증 도용 방지를 위한 것으로 고정된 푸시 스트림 주소를 생성할 경우 인증 도용 현상이 발생해 현재 비즈니스에 피해를 입힐 수 있습니다.
라이브 방송 푸시 스트림 주소 유효 시간에는 제한이 없으며, 개인의 비즈니스 필요에 따라 주소 만료 시간을 설정할 수 있습니다. 또한 스플라이싱 규칙에 따라 유효 기간이 더욱 긴 푸시 스트림 주소를 생성할 수 있습니다. 구체적인 방법은 [라이브 방송 URL 직접 조합](https://intl.cloud.tencent.com/document/product/267/32480)를 참고하십시오.

>? 푸시 스트림 주소 유효 기간은 너무 길게 설정하지 않는 것이 좋습니다. 길게 설정할 경우 해당 푸시 스트림 주소 사용 중에 오류가 발생하여 인증 무효 알림이 발생합니다.

<span id="que9"></span>
###  라이브 방송 서비스 중 비디오에 Tencent Cloud 로고가 표시됩니까? 
라이브 방송 서비스 중 비디오에 Tencent Cloud 로고가 표시되지 않습니다. 

<span id="que10"></span>
###  라이브 방송의 딜레이 시간은 얼마나 됩니까? 
일반적으로 RTMP 프로토콜 푸시 스트림을 사용하여 FLV 프로토콜을 통해 재생할 경우 2초~3초 정도의 딜레이가 발생합니다. 딜레이가 너무 길다면 문제가 있는 것입니다.
<span id="que11"></span>
###  라이브 방송 시 최대 비트레이트를 설정할 수 있습니까?
 불가능합니다. 푸시 스트림 엔드에서 자체적으로 설정되며, 푸시 스트림 엔드에서 설정된 최대 비트레이트도 네트워크 업로드 속도에 따라 결정됩니다. 비트레이트(또는 최대 비트레이트) 최댓값은 네트워크 업로드 속도이며, 이를 너무 높게 설정할 경우 라이브 방송 화면 프레임이 떨어지며 랙이 발생합니다. 

<span id="que12"></span>
###   Tencent Cloud 라이브 방송에서 사용하지 않는 라이브 방송 방은 어떻게 삭제합니까? 
현재 CSS는 라이브 방송 스트림 id로 라이브 방송 푸시 스트림 및 재생을 연결하므로 별도의 삭제 작업이 필요하지 않습니다. 인스턴트 메시징 IM 서비스를 사용하며 IM 방이 최댓값에 도달하지 않도록 삭제하고자 할 경우 [그룹 해제](https://intl.cloud.tencent.com/document/product/1047/34896)를 참고하여 작업을 진행할 수 있습니다.
채널 모드를 사용한 경우 DeleteLVBChannel 라이브 방송 채널 삭제 인터페이스를 호출하여 삭제할 수 있으며, 삭제하고자 하는 라이브 방송 채널의 ID 번호(배치 작업 지원)를 입력합니다.

> ! 채널 모드는 구 버전 방법으로 현재 업데이트 및 유지 보수가 이루어지지 않습니다.

<span id="que13"></span>
 ### 라이브 방송에서 푸시 스트림 인터페이스 비활성화의 역할은 무엇입니까? 
라이브 방송에서 푸시 스트림 비활성화는 음란물 감지 시 시나리오 재생을 금지할 때 사용합니다. 예를 들면 백그라운드에서 호스트의 외설적이거나 불온한 콘텐츠를 발견한 경우 언제든지 스트림을 차단하거나 해당 스트림을 비활성화할 수 있습니다.

<span id="que14"></span>
### 백그라운드 재생 방법은 어떻게 됩니까?
백그라운드 재생은 단말 기능으로 고객의 실제 비즈니스 로직에 따라 개발해야 합니다. 라이브 방송 스트림이 중단되지 않는다면 백그라운드 오디오 재생을 지원할 수 있습니다.


<span id="que15"></span>
### HTTPS 설정 수정 시 인증서 정보 추가 후 ‘부정확한 인증서’라는 알림이 나타날 경우 어떻게 해야 합니까? 
 CSS의 암호화 서비스는 Nginx를 사용하므로 인증서 유형은 Nginx여야 합니다. 현재 인증서 유형이 정확한지 확인해 보십시오.

<span id="que16"></span>
###  재생 도메인 인증 비활성화 후 기존에 생성된 재생 주소를 재생할 수 없습니다. 
 재생 인증은 유효 기간을 설정할 수 있으며, 유효 기간 동안 액세스 인증을 비활성화하더라도 기존 주소는 재생할 수 없습니다. 

<span id="que17"></span>
###  인터페이스 액세스 횟수 제한은 무엇입니까?
CSS는 계정에 속한 SecretId로 발송하는 총 요청 수에 대한 액세스 횟수 제한이 있으며, 제한을 초과한 요청에는 정상적으로 응답하지 않습니다.
예를 들어 1초 동안 200회를 초과하지 않는다고 할 경우, 계정의 SecretId로 발송한 해당 요청은 1초 동안 Tencent Cloud 서버에서 200회만 수신합니다. 200회는 1명의 고객이 200회 발송하거나 200명의 고객이 인당 1회 발송하는 경우, 또는 여러 쿼리이거나 하나의 스트림이더라도 상관이 없으며 총 요청 수만을 계산합니다. 

<span id="que19"></span>

###  프레임 레이트 수정 후 정상적으로 푸시 스트림이 되지 않아 로컬에서 서비스를 여러 번 재시작해야 하며 자주 끊깁니다. 어떻게 처리해야 합니까? 
현재 프레임 레이트가 너무 높게 설정된 것 같습니다. 15 프레임 이상이면 비디오가 원활하게 재생되니 프레임 레이트를 낮춰 보십시오.  

<span id="que20"></span>
###  어떤 경우에 시스템에서 장시간 데이터가 없는 푸시 스트림을 차단합니까? 
푸시 스트림 시 디바이스에 문제가 생긴 경우 이와 같은 상황이 발생합니다.
예: App 튕김, 휴대폰 꺼짐 등 기타 비주관적 원인으로 이상이 발생하여 백그라운드가 70초 동안 데이터 푸시 스트림을 수집하지 못할 경우 시스템에서 차단합니다.
