## 시나리오 배경
Open Broadcaster Software(OBS)는 사용하기 좋은 3rd party 오픈 소스 프로그램 라이브 방송 스트림 미디어 콘텐츠 제작 소프트웨어입니다. 사용자에게 무료로 제공됩니다. OS X, Windows, Linux 운영체제를 지원하여 각종 라이브 방송 시나리오에 적용할 수 있어 대부분의 라이브 방송 작업 수요를 충족합니다. [OBS 홈페이지 다운로드](https://obsproject.com/download?spm=a2c4g.11186623.2.15.6aac1445JPlKR8)에서 최신 버전 소프트웨어를 다운로드하십시오.
이 작업은 OBS 툴을 설치한 후 PC에서 푸시 스트리밍을 설정하는 방법을 알려줍니다.

## 준비 과정
- [LVB 콘솔](https://console.cloud.tencent.com/live)에 로그인하여 푸시 스트리밍 주소를 생성하십시오. 자세한 내용은 [푸시 스트리밍 설정](https://intl.cloud.tencent.com/document/product/267/31059)을 참조하십시오.
- [OBS](https://obsproject.com/download?spm=a2c4g.11186623.2.15.6aac1445JPlKR8) 툴을 설치하십시오.

## 작업 순서

### OBS 푸시 스트리밍 사용 방법
 1. **<span id="step1">푸시 스트리밍 주소 설정</span>**
OBS를 실행하고 아래쪽 툴바에서 [제어]>[설정] 버튼을 눌러 설정 인터페이스로 진입합니다.
![](https://main.qcloudimg.com/raw/493a19e0f2bbea80983c341dd742a044.png)
[푸시 스트리밍]을 클릭하여 스트리밍 설정 탭에 들어가 스트리밍 종류를 사용자 정의 스트림 미디어 서버로 선택합니다. 아래 그림의 푸시 스트리밍 주소를 예로 들어보겠습니다.
![](https://main.qcloudimg.com/raw/55512ddf58bf32014424fa33b6f3d31f.png)
푸시 스트리밍 주소를 URL과 스트림 이름으로 나눠서 입력합니다. URL 부분에서 Path 앞의 주소 정보를 작성하고 스트림 이름 부분에서 StreamName 뒤의 주소 정보를 작성하면 매개변수 설정은 아래와 같습니다.
```
URL：rtmp://push.livetest.myqcloud.com/live/
스트림 이름： StreamName?txSecret=xxxxx&txTime=5C1E5F7F
```
푸시 스트리밍 주소를 가져오는 방법은 [빠른 스트리밍 주소 가져오기](https://intl.cloud.tencent.com/document/product/267/31056#step-3.-view-and-configure-information)를 참조하십시오.

 2. **<span id="step2">푸시 스트리밍 설정</span>**
아래 툴바의 [소스] 창에서 [+]를 클릭해 입력 소스를 선택합니다. 이때 부호율, 녹화 등 기타 설정을 할 필요가 없으며 바로 툴바의 [제어]>[푸시 스트리밍 시작]을 클릭하면 비디오 스트림이 설정의 푸시 스트림 주소에 보내집니다.
![](https://main.qcloudimg.com/raw/f8136fdb794595c8c491c995a74cdfcc.png)
혹은 위쪽의 툴바를 클릭해 [도구]>[자동 안내 설정]을 선택해 OBS의 안내에 따라 푸시 스트리밍 설정을 한 후 다시 툴바의 [제어]>[푸시 스트리밍 시작]을 누릅니다.

 3. **기타 푸시 스트리밍 관련 설정**
- 라이브 방송 딜레이에 영향을 주는 관련 설정
[제어]>[설정]>[내보내기]를 통해 [내보내기 모드]의 [고급]을 선택하면 [키 프레임 간격] 등으로 설정을 진행할 수 있습니다. 설정 방법은 아래 그림과 같습니다.
![](https://main.qcloudimg.com/raw/5f48205da162f1230723729c36369f65.png)
동시에 왼쪽 메뉴에서 [고급]을 선택하여 [스트리밍 딜레이] 설정을 진행할 수 있습니다.
![](https://main.qcloudimg.com/raw/c3ec9a71a014548eb680009d9798b1ac.png)
- 로컬 라이브 방송 녹화 관련 설정
 라이브 방송 시 로컬 녹화가 필요하면 라이브 방송 시 로컬에서 녹화 백업을 진행할 수 있습니다. 설정 방법은 아래 그림과 같습니다.
![](https://main.qcloudimg.com/raw/a3c90994ef7447766914dc41605623c3.png)
[제어]>[설정]>[내보내기]를 클릭하여 내보내기 설정 탭에 들어갑니다. [녹화] 설정바에서 관련 설정을 하면 녹화 파일을 로컬에 저장할 수 있습니다.
해상도는 시청자가 보는 화질을 결정합니다. 해상도가 높을수록 화면이 선명합니다. FPS는 비디오 프레임 레이트를 뜻하며 비디오의 재생 품질과 관련됩니다. 보통 비디오 프레임 레이트는 24 ~ 30프레임이며 16프레임 이하 화면은 딜레이되는 듯한 느낌을 줍니다. 그래서 비교적 높은 프레임 레이트를 요구하는 게임의 경우 30프레임 이하로 내려가면 일반적으로 자연스럽지 못한 느낌을 줍니다. 해상도와 프레임 레이트 설정 방법은 아래 그림과 같습니다.
![](https://main.qcloudimg.com/raw/f736ef195b26a81f46d1e26d7935a763.png)


## 관련 작업
### 자주 볼 수 있는 작업 소개
1. **자주 사용하는 라이브 방송 소스 입력 소개**
 - **영상**: 단일 영상 라이브 방송에 활용합니다.
 - **영상 슬라이드 쇼**: 복합 이미지를 순환 혹은 순차적으로 재생합니다.
 - **시나리오**: 라이브 방송에서 다양한 고효율을 실현합니다. 이 경우 다른 시나리오가 소스의 형태로 현재 시나리오에 추가되어 전체 시나리오에 포함될 수 있습니다.
 - **미디어 소스**: 로컬 비디오를 업로드하여 로컬 VOD 비디오 파일로 라이브 방송 처리할 수 있습니다.
 - **텍스트**: 라이브 방송 창에 실시간으로 텍스트를 추가합니다.
 - **캡처 보여주기**: 컴퓨터 데스크톱 작업을 실시간으로 동적 캡처하여 데스크톱의 모든 작업을 생방송 할 수 있습니다.
 - **게임 캡처**: 지정한 소스의 게임에 한해서 라이브 방송할 수 있습니다. 여러가지 게임 실황을 라이브 방송하는 데 적합합니다.
 - **창 캡처**: 선택한 창을 실시간 캡처합니다. 라이브 방송 중에 다른 창은 캡처하지 않으며 현재 창만 보여줍니다.
 - **컬러 소스**: 이 소스는 배경색으로 시나리오에 색상을 추가합니다. 창의 투명도를 조절하여 전체 창에 투명한 색상을 만들어 줍니다.
 - **비디오 캡처 디바이스**: 촬영 디바이스를 실시간 동적 캡처하여 촬영 후 화면을 라이브 방송할 수 있습니다.
 - **음성 입력 캡처**: 음성 라이브 방송 이벤트(음성 입력 디바이스)에 사용됩니다.
 - **음성 출력 캡처**: 음성 라이브 방송 이벤트(음성 출력 디바이스)에 사용됩니다.

2. **스튜디오 모드 소개**
사용자는 스튜디오 모드에서 현재 라이브 방송한 콘텐츠를 실시간 편집하고 편집 과정에서 오버 애니메이션화 된 장면 전환으로 사용자 환경에 미치는 영향을 최소화하여 라이브 콘텐츠의 편집을 진행할 수 있습니다.
[제어]>[스튜디오 모드]를 클릭하여 아래와 같이 인터페이스에서 [시나리오], [소스], [오버 애니메이션] 설정을 완료하면 라이브 방송 중 바로 왼쪽의 소재 편집 창에서 라이브 방송 인터페이스 편집을 할 수 있으며 [오버 애니메이션] 버튼을 눌러 라이브 방송 창으로 전환할 수 있습니다.
![](https://main.qcloudimg.com/raw/80e9e902e0e02902d78779ca5505b2f8.png)

### 퓨어 오디오 푸시 스트리밍
OBS 운영사 포럼의 답변에 따르면 현재 OBS Studio 23.2.1과 이전 버전은 퓨어 오디오 푸시 스트리밍을 지원하지 않습니다.
유사한 오디오 푸시 스트리밍을 구현하기 위해 아래 순서로 설정하십시오. 정적 캔버스(검은 화면 혹은 이미지)로 비디오를 대체하는 방법입니다. 대역폭을 낮춰야 하는 경우 비디오의 프레임 레이트와 부호율을 낮춰 퓨어 오디오 푸시 스트리밍에 가까워질 수 있지만 라이브 스트리밍 중 비디오 데이터가 여전히 남아있을 수 있습니다. 
1. [OBS 정상 푸시 스트리밍 설정](#step2)을 참조하여 [오디오 입력 캡처] 입력 소스를 추가합니다. 비디오 입력 소스를 추가하거나 이미지를 비디오 입력 소스로 추가할 수 없습니다.
![](https://main.qcloudimg.com/raw/c74e3e5058c69356095c761e1e572da6.png)
2. [제어]>[설정]>[비디오] 페이지에 들어가 [해상도(확대/축소) 출력]과 [자주 사용하는 FPS 값(프레임 레이트)]을 가장 낮게 설정하고 [저장]을 클릭하여 설정을 저장합니다.
![](https://main.qcloudimg.com/raw/cb77c3deb8d3e9831aaa1421b5b13412.png)
3. [출력]을 클릭하여 입력 설정 페이지로 들어가 아래와 같이 [비트레이트]를 가장 낮은 수준으로 설정합니다. 설정을 완료한 후 [저장] 혹은 [애플리케이션]을 클릭하여 설정을 저장합니다.
![](https://main.qcloudimg.com/raw/008e1ab02e2d08ee42f3a64e2e09d473.png)
4. [OBS 푸시 스트리밍 설정](#step1) [푸시 스트리밍] 순서에 따라 [푸시 스트리밍]을 시작하면 오디오를 들을 수 있고 비디오 콘텐츠는 검은 화면이거나 이미지로 표시됩니다. 비디오의 부호율이 가장 낮은 경우 대역폭 사용량도 크게 줄어듭니다.

### 비디오 루핑
**단일 파일 순환 재생 설정 방법**
1. OBS [소스]바에서 [+]를 눌러 [미디어 소스]를 추가합니다. 팝업창의 [로컬 파일]에서 루핑한 비디오 파일을 선택하고 [순환]을 선택한 다음 [확인]을 클릭합니다.
![](https://main.qcloudimg.com/raw/b41371c6674d4d8a2cd84fe37e3fbab4.png)
2. [OBS 푸시 스트리밍 설정](#step1) 순서에 따라 [푸시 스트리밍] 페이지의 [서버]와 [스트림 키] 항목을 설정하여 푸시 스트리밍으로 단일 파일 루핑을 실행할 수 있습니다.

**다중 파일 순환 루핑 방법**
OBS 자체에서는 다중 파일 순환 재생을 지원하지 않지만 3rd party 비디오 플레이어의 순환 재생 기능과 창 캡처를 이용하여 다중 파일 순환 재생을 구현할 수 있습니다.
