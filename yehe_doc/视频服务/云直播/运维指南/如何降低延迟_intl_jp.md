通常、RTMPプッシュ+FLV再生の通常のディレイは約2秒～3秒ですので、長すぎる場合は問題です。ライブストリーミングのディレイ時間が特に長い場合は、次の方法に従い、原因調査を行うことができます。
## 原因調査の考え方
### Step 1. 再生プロトコルのチェック
再生プロトコルにHLS（m3u8）を使用している多くのクライアントは、ディレイが比較的大きいと感じていますが、これは正常です。Appleが推奨するHLSは、大粒子のTSフラグメントに基づくストリーミングメディアプロトコルで、各フラグメントの持続時間は5s以上で、フラグメント数は通常3～4個であるため、総ディレイ時間が20～30sであることは驚くようなことではありません。
FLVを再生プロトコルとして切り替えることで、この問題を解決することができますが、スマートフォンのブラウザでCSSを視聴する場合は、再生プロトコルとしてHLS（m3u8）しか選択できないことに注意してください。その他のCSSプロトコルはAppleのSafariブラウザではいずれもサポートされていません。

### Step 2. プレーヤー設定のチェック
Tencent Cloud RTMP SDKのプレーヤーは、超高速、スムーズ、自動の3つのモードをサポートしています。具体的な設定については、[ディレイの調節](https://cloud.tencent.com/document/product/454/7886#Delay)をご参照ください。
- **超高速モード**：ほとんどのシーンでディレイは2秒～3秒以内であることを保証できます。ビューティーショーにはこのモードが最適です。
- **スムーズモード**：ほとんどのシーンでディレイは5秒以内です。ゲームCSSなど、ディレイには敏感ではないものの、スムーズさが要求されるシーンに最適です。

### Step 3. バックグランドへのウォーターマークの追加は不要
Tencent Cloudはバックグラウンドへのウォーターマークの追加をサポートしています。Tencent Cloud RTMP SDK（ライブストリーミングでウォーターマークをサポート）は使用できないがウォーターマークを追加したい一部の顧客のニーズを満たすことが目的ですが、この方法ではさらに3秒のディレイが発生してしまいます。したがって Tencent Cloud RTMP SDKを使用してプッシュする場合は、バックグラウンドのウォーターマークをオフにした後、キャスターのアプリでウォーターマークを追加します。

### Step 4. サードパーティのプッシュストリーミング装置？
Tencent Cloud統合ソリューションでのみ理想的な効果の維持を保証できます。サードパーティのプッシュソフトを使用する場合は、比較のために、まずTencent Cloud RTMP SDKのプッシュDemoに変更し、サードパーティのプッシュストリーミング装置のエンコードのバッファによって大幅なディレイが引き起こされる可能性を排除することをお勧めします（多くのサードパーティのプッシュストリーミング装置が無制限のバッファリング方式を暴力的なまでに使用し、上りの帯域幅が不足する問題を解決するため）。

### Step 5.  OBS設定のチェック
 OBSプッシュを使用する多くのクライアントは、再生側のディレイが大きいとフィードバックしています。 [PCプッシュ](https://cloud.tencent.com/document/product/267/32732#pc-.E7.AB.AF.E6.8E.A8.E6.B5.81) ドキュメントに従い、キーフレーム間隔を1または2に設定することに注意しつつ、対応するパラメータを設定することをお勧めします。
