### 푸시 스트리밍
 호스트가 로컬 비디오 소스와 오디오 소스를 Tencent 비디오 CVM으로 푸시하는 것이며, 일부 시나리오에서는 “RTMP 배포”라고도 부릅니다. 



[](id:play)

### 풀 스트리밍
라이브 방송이란 라이브 푸시 스트리밍을 구현한 뒤 Tencent 비디오 CVM 내의 비디오 소스와 오디오 소스를 지정된 주소로 불러와 재생하는 과정을 말합니다. 비디오 소스는 실시간 생성되고 사용자가 라이브를 푸시 스트리밍해야 의미가 있으며, 호스트가 방송을 중단하면 라이브 URL이 유효하지 않습니다. 또한 실시간 라이브의 특성상, 비디오 라이브 재생 시 플레이어에 진행 바가 존재하지 않습니다. 


[](id:push_domain)
### 푸시 스트리밍 도메인
라이브 스트리밍 푸시에 사용하는 도메인을 말하며, 필수 사양이므로 라이브 서비스 이용 전에 반드시 도메인을 등록하고 ICP비안을 완료해야 합니다. 푸시 스트리밍 도메인 설정이 완료되면 라이브 서비스에 해당하는 푸시 스트리밍 주소가 생성되며, 연동 규칙은 [푸시 스트리밍 URL 수동 연동](https://intl.cloud.tencent.com/document/product/267/38393)을 참조하십시오.

### 재생 도메인
라이브 스트리밍 재생에 사용하는 도메인을 말하며, 필수 사양이므로 라이브 서비스 이용 전에 반드시 도메인을 등록하고 ICP비안을 완료해야 합니다. 재생 도메인 설정이 완료되면 라이브 서비스에 해당하는 재생 주소가 생성되며, 연동 규칙은 [재생 URL 수동 연동](https://intl.cloud.tencent.com/document/product/267/38393)을 참조하십시오.


[](id:cname)
### CNAME 도메인
CNAME 도메인은 Tencent LVB 콘솔에서 가속 도메인에 액세스한 뒤 시스템에서 해당하는 도메인에 `.liveplay.myqcloud.com`을 접미사로 할당하는 도메인입니다. 사용자는 도메인 서비스 제공 업체에서 CNAME 기록을 설정해야 하고, 기록이 적용되면 리졸브 작업이 정식으로 Tencent Cloud LVB에 전달되며, 이 도메인의 모든 요청은 Tencent LVB의 엣지 노드로 전달됩니다.

[](id:streamname)
### StreamName
StreamName은 스트림의 식별자이며, 일반적으로 도메인과 함께 스트림을 식별하는 역할을 합니다.

### AppName[](id:appname)
라이브 방송 애플리케이션 이름으로, 라이브 방송 스트림 미디어 파일의 저장 경로를 구분하는 데 사용합니다. 기본값은 live이고, 사용자 정의 가능합니다.

[](id:trans)
### 트랜스 코딩
트랜스 코딩은 비디오 비트 스트림을 다른 비디오 비트 스트림으로 변환하는 과정이며, 오프라인 작업입니다. 트랜스 코딩을 통해 원본 비트 스트림의 코딩 포맷, 해상도, 비트 레이트 등의 매개변수를 변경하여 터미널과 네트워크 환경에 맞게 재생할 수 있습니다. 다음과 같이 트랜스 코딩 기능을 사용할 수 있습니다.
- 더 많은 터미널에 적용: 원본 비디오를 보다 강력한 터미널 적용 능력이 있는 포맷으로 트랜스 코딩하여, 비디오 리소스를 더 많은 디바이스에서 재생할 수 있도록 합니다.
- 다양한 대역폭에 적용: 비디오를 일반, 표준 화질, 고화질, 초고화질 출력으로 변환하여 사용자가 네트워크 환경에 적합한 비트 레이트를 선택해 비디오를 재생할 수 있습니다.
- 대역폭 절감: 보다 진보된 코딩 방식으로 트랜스 코딩하여, 원본 화질을 손상하지 않으면서 비트 레이트를 현저하게 낮추어 재생 대역폭을 절감합니다.

[](id:h264)
### H.264
H.264는 ITU-T 영상 부호화 전문가 그룹(VCEG)과 ISO/IEC 동영상 전문가 그룹(MPEG)이 공동으로 구성한 비디오 그룹(JVT, Joint Video Team)에서 제출한 높은 압축 효율의 디지털 비디오 코딩/디코딩 표준이며, MPEG-4의 Part 10과 같은 표준안입니다. H.264 트랜스 코딩의 장점은 다음과 같습니다.
- 1Mbps 이하의 속도로 표준 화질(해상도 1280*720 이하)의 디지털 영상 전송이 가능합니다.
- 기존의 비디오 코딩 표준 대비, 동일한 대역폭에서 보다 우수한 영상 품질을 제공합니다.
- 기존 압축 기술의 장점과 핵심을 유지하며, 다른 압축 기술에 비해 뛰어난 장점이 있습니다.


[](id:h265)
### H.265
H.265 표준은 기존의 비디오 코딩 표준 H.264에서 일부 기술을 유지하면서 최적화했습니다. H.265 트랜스 코딩의 장점은 다음과 같습니다.
- 1Mbps~2Mbps의 전송 속도로 720P(해상도 1280*720)의 일반 고화질 멀티미디어를 전송할 수 있습니다.
- 비트 스트림, 코딩 품질, 딜레이, 알고리즘 복잡도의 관계를 개선하여 최적화 설정을 구현합니다.


[](id:message)
### 이벤트 정보 알림
푸시 스트리밍 중 라이브 방송에서 이벤트 알림이 발생하면 Tencent Cloud에서 설정된 템플릿 정보에 따라 고객의 서버에 요청을 발송하고, 고객의 서버가 인증 요청하는 것을 말합니다. 관련 응답 프로토콜은 [이벤트 정보 알림 프로토콜](https://intl.cloud.tencent.com/document/product/267/38080)을 참조하십시오. 인증에 성공하면 콜백 정보가 포함된 JSON 데이터 패키지를 가져올 수 있으며, 가져오기 완료 후 관련 정보를 리졸브하여 기록하십시오. 


[](id:link)
### 링크 도용 방지
 푸시 스트리밍 URL과 재생 URL에 있는 txSecret 필드를 말하며, 공격자가 백그라운드를 위조하여 푸시 스트리밍 URL을 생성하거나 재생 주소를 불법으로 탈취하여 이익을 추구하는 것을 방지할 수 있습니다.

[](id:record)
 ### 라이브 방송 녹화
푸시 스트리밍 중에 라이브 방송 원본 스트림이 트랜스 멀티미디어 캡슐화(오디오, 비디오 데이터 및 대응하는 타임스탬프 등 정보를 수정하지 않음)를 거쳐 얻은 비디오 파일을 VOD 플랫폼에 저장합니다. 이 기능을 이용하려면 먼저 [VOD 서비스](https://intl.cloud.tencent.com/zh/product/vod)를 활성화해야 합니다.

[](id:watermark)
### 워터마크
라이브 방송 푸시 스트리밍 중에 비디오 저작권이 침해되지 않도록 트랜스 코딩 과정에서 설정된 워터마크를 비디오 스트림에 삽입하여 워터마크가 삽입된 비디오 스트림을 출력합니다. 워터마크는 텍스트나 이미지로 제작할 수 있습니다.

[](id:screenshot)
### 화면 캡처
라이브 방송 푸시 스트리밍 비디오 화면을 일정 시간 간격으로 캡처하여 생성된 이미지 파일을 COS에 저장합니다. 화면 캡처 기능을 활성화하려면 먼저 COS bucket에서 LVB 서비스의 데이터 쓰기 권한을 설정해야 합니다. 자세한 내용은 [COS bucket 화면 캡처 저장 권한 부여](https://intl.cloud.tencent.com/document/product/267/33384)를 참조하십시오.

### 음란물 감지
화면 캡처 기능을 기반으로 시스템에서 푸시 스트리밍 도메인에 연결된 화면 캡처/음란물 감지 템플릿을 통해 화면 캡처를 식별하고 결과를 콜백합니다. 자세한 내용은 [화면 캡처/음란물 감지 설정](https://intl.cloud.tencent.com/document/product/267/31072)을 참조하십시오.
