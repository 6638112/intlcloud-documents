### プッシュ
 -**プッシュ**：キャスターは、ローカルのビデオソースとオーディオソースをTencent Video CVMにプッシュします。一部のシナリオでは「RTMP公開」とも呼ばれます。 
<span id="play"></span>
### プル
 プル（LVB再生）のビデオソースはリアルタイムで生成されます。誰かがライブブロードキャストをプッシュする場合にのみ表示されます。キャスターによりライブブロードキャストを終了すると、ライブブロードキャストのURLは無効になります。リアルタイムのブロードキャストであるため、ライブブロードキャストビデオを再生するときに、プレーヤーにプログレスバーが表示されません。 
<span id="push_domain"></span>
### プッシュドメイン名
ライブブロードキャストストリームのプッシュに使用されるドメイン名(必須設定)を示します。ライブブロードキャストサービスを使用する前に、対象ドメイン名を登録してICP申告する必要があります。プッシュドメイン名を設定した後、LVBサービスは対応するプッシュアドレスを生成します。
<span id="play_domain"></span>
### 再生ドメイン名
LVBストリームの再生に使用されるドメイン名(必須設定)を示します。ライブブロードキャストサービスを使用する前に、ドメイン名を登録してICP申告する必要があります。再生ドメイン名を設定した後、LVBサービスは対応する再生アドレスを生成します。
<span id="cname"></span>
### ドメイン名CNAME
CNAMEドメイン名は、Tencent Cloud LVBコンソールでアクセラレーションドメイン名にアクセスした後、対応するドメイン名に割り当てられた`.liveplay.myqcloud.com`のサフィックスが付けられたドメイン名です。ユーザーは、ドメイン名サービスプロバイダーでCNAMEレコードを設定する必要があります。レコードが有効になると、ドメイン名解決がTencent Cloud LVBに転送され、当該ドメイン名に対するすべてのリクエストがTencent LVBのエッジサーバーに転送されます。
<span id="streamname"></span>
### StreamName
StreamNameはストリームの識別子であり、通常はドメイン名とともにストリームを一意に識別します。
<span id="appname"></span>
### AppName
ライブブロードキャストのアプリケーション名であり、ライブブロードキャストメディアファイルの格納パスを区別するために使用されます。デフォルトはliveで、カスタマイズできます。
<span id="trans"></span>
### トランスコーディング
トランスコーディングは、特定のビデオビットストリームを別のビデオビットストリームに変換するオフラインタスクです。コーデック、解像度、ビットレートなどの元のビットストリームのパラメータを変更して、さまざまなデバイスやネットワーク環境での再生に適応させます。トランスコーディングを使用して以下の機能を実現できます。
- 多様な端末に適応：元のビデオを端末適応性の高い形式にトランスコードして、ビデオリソースをより多くのデバイスで再生できるようにします。
- さまざまな帯域幅に適応：ビデオを滑らか、SD、HD、またはFHD出力に変換します。ユーザーは、現在のネットワーク環境に応じて、ビデオ再生に適したビットレートを選択できます。
- 帯域幅の削減：より高度な符号化方式を使用してトランスコーディングすると、元の画質が低下しない前提で、ビットレートを大幅に削減し、再生帯域幅を削減できます。
<span id="h264"></span>
### H.264
H.264は、ITU-T Video Coding Experts Group（VCEG）とISO/IEC Moving Picture Experts Group（MPEG）によって設立されたJoint Video Team（JVT）が推進している高度な圧縮デジタルビデオのコーデック標準です。また、MPEG-4のPart 10でもあります。H.264トランスコーディングを使用する利点は以下のとおりです。
- 1Mbps以下の速度でSD（1280x720未以下の解像度）デジタル画像伝送を実現できます。
- 他の既存のビデオコーディング標準と比較して、同じ帯域幅でより良い画質を提供します。
- 従来の圧縮技術の強みと精髄を活かした上、他の圧縮技術にはない利点もあります。
<span id="h265"></span>
### H.265
H.265標準は、既存のビデオコーディング標準H.264を中心に展開し、一部の技術を継承しながら、最適化と改善を行いました。H.265トランスコーディングを使用する場合の利点は次のとおりです。
- 1Mbps〜2Mbpsの伝送速度で720P（解像度1280*720）の通常のHDオーディオおよびビデオを伝送できます。
- ビットストリーム、エンコーディング品質、遅延、アルゴリズムの複雑さの関係を改善して、最適な設定を実現します。

<span id="message"></span>
### イベントメッセージ通知
プッシュ中にライブストリームによってイベント通知がトリガーされ、Tencent Cloudが設定されたメッセージテンプレートに基づいて顧客のサーバーにリクエストをプロアクティブに送信し、顧客のサーバーが認証リクエストに応答します。応答プロトコルの詳細については、[Event Message Notification Protocol](https://intl.cloud.tencent.com/document/product/267/31566)を参照してください。認証が完了すると、コールバック情報を含むJSONパケットを取得して、解析と情報ロギングを行うことができます。 
<span id="link"></span>
### リンク不正アクセス防止
 プッシュURLと再生URLのtxSecretフィールドを指します。攻撃者が顧客のバックエンドをなりすましてプッシュURL生成したり、顧客の再生アドレスを不正で盗用したりすることを防ぐことができます。

 <span id="record"></span>
 ### ライブブロードキャストレコーディング
プッシュ中に、LVBの元のストリームをオーディオ・ビデオ変換パッケージング（オーディオ・ビデオデータや対応するタイムスタンプなどの情報を変更しない）することによって生成されたビデオファイルをVODプラットフォームに保存できます。この機能を使用するには、事前に[Cloud on Demand Service]（https：//intl.cloud.tencent.com/product/vod）をアクティブ化する必要があります。

 <span id="watermark"></span>
### ウォーターマーク
LVBプッシュ中にビデオの著作権が侵害されないように、トランスコーディング中に設定済みのウォーターマークをビデオストリームに追加して、ウォーターマークが入ったビデオストリームを出力できます。ウォーターマークは、テキストまたは画像のいずれかです。

 <span id="screenshot"></span>
### スクリーンキャプチャ
プッシュされたライブストリーム内のビデオのスクリーンショットを一定の間隔でキャプチャし、生成された画像ファイルをCOSに保存します。スクリーンキャプチャ機能をアクティブにするには、最初にLVBにCOSバケットへの書き込み権限を付与する必要があります。詳細については、[Granting LVB Access to COS Bucket for Screenshot Storage]（https：//intl.cloud.tencent.com/document/product/267/33384）をご参照ください。 

 <span id="yellow_confidence"></span>
### ポルノ検出
スクリーンキャプチャ機能に基づいて、システムは、プッシュドメイン名に関連付けられたスクリーンキャプチャポルノ検出テンプレートを利用して、キャスターがLVBプッシュで非準拠のライブコンテンツを配信しているかどうかを識別できます。関連ドキュメントについては、[Screencapturing and Porn Detection Configuration]（https：//intl.cloud.tencent.com/document/product/267/31072）をご参照ください。
