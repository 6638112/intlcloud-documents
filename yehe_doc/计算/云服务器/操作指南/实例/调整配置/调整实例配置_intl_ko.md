## 작업 시나리오

Tencent Cloud 인스턴스의 빠르고 편리한 하드웨어 장치 변경 기능은 CVM의 효율성을 나타내는 지표입니다. 본 문서는 인스턴스 구성 업그레이드 및 다운그레이드, 모델 간의 구성 변경 방법 및 관련 주의사항을 소개합니다.

## 전제 조건

종료 상태거나 시작 상태인 인스턴스 모두 구성을 변경할 수 있습니다. 시작 상태의 인스턴스는 강제 재시작한 후에 작업이 적용됩니다.
> 
> - 작업할 인스턴스가 **종료** 상태인 경우 콘솔 변경 작업을 바로 진행할 수 있습니다.
> - 작업할 인스턴스가 **시작** 상태인 경우 온라인에서 구성을 변경할 수 있으며, 변경 완료 후에는 강제 재시작한 뒤에 변경한 구성이 적용되었는지 확인해야 합니다.
> - **일괄 작업** 변경을 온라인에서 진행할 수 있습니다. 일괄로 변경된 기기 중 **시작** 상태의 기기는 강제 재시작한 뒤에 해당 작업이 적용되었는지 확인해야 합니다.

## 제한 및 영향

### 구성 변경 제한

**시스템 디스크와 데이터 디스크가 모두 CBS**인 인스턴스만 구성을 변경할 수 있습니다.
- 구성 업그레이드
횟수 제한이 없으며, 업그레이드는 즉시 적용됩니다.
- 구성 다운그레이드
 - 종량제 인스턴스는 언제든 횟수 제한 없이 구성을 다운그레이드할 수 있습니다.
- 인스턴스 패밀리 간 변경: 인스턴스 패밀리끼리 구성을 변경할 수 있으므로, 데이터를 마이그레이션하지 않아도 됩니다.
구성 변경 작업 시 현 사용 가용존이 변경할 인스턴스 사양을 제공하는지에 대해 다음의 제한 사항을 확인하시기 바랍니다.
 -  **스팟 인스턴스**는 모델 간 구성 변경을 지원하지 않습니다.
 -  **전용 인스턴스**는 모델 간 변경을 지원하지 않으며, 구성 변경 범위는 해당 인스턴스가 있는 CDH의 남은 리소스 수량에 따라 제한됩니다.
 -   **GPU, FPGA 등의 이종 인스턴스**는 인스턴스 패밀리 간 구성 변경 시의 원본 인스턴스 사양과 타깃 인스턴스의 사양으로 선택할 수 없습니다.
 - **기본 네트워크로 구성된 인스턴스**는 VPC만 지원하는 인스턴스로 변경할 수 없습니다.
 -  타깃 인스턴스 사양이 현재 사양의 CBS 유형을 지원하지 않을 경우 구성을 변경할 수 없습니다.
 -  타깃 인스턴스 사양이 현재 사양의 이미지 유형을 지원하지 않을 경우 구성을 변경할 수 없습니다.
 -  타깃 인스턴스 사양이 현재 사양의 ENI 또는 수량을 지원하지 않을 경우 구성을 변경할 수 없습니다. 자세한 내용은 [엘라스틱 네트워크 사용 제한](https://intl.cloud.tencent.com/document/product/576/18527)을 참조 바랍니다.
 -  타깃 인스턴스 사양이 현재 사양의 외부 네트워크 대역폭 최댓값을 지원하지 않을 경우 구성을 변경할 수 없습니다. 자세한 내용은 [공용 네트워크 대역폭 최댓값](https://intl.cloud.tencent.com/document/product/213/12523)을 참조 바랍니다.

###  관련 영향

변경 후 극소수의 인스턴스에서 개인 IP가 변경될 수 있습니다. 개인 IP가 바뀌었을 경우 변경 페이지에 안내 문구가 표시되며, 관련 문구가 나타나지 않으면 개인 IP에 변화가 없음을 의미합니다.

## 작업 순서

>
> - 비즈니스에 변동 사항이 생길 경우, 상황에 맞게 구성을 변경할 수 있습니다.
>  - 구성 업그레이드 시, 이에 따라 발생할 수 있는 비용을 지불해 주시기 바랍니다.
>  - 구성 다운그레이드 시, 환불 정책을 확인하시기 바랍니다. 강제 재시작 후에 새로운 구성이 즉시 적용되며, CVM이 새로운 구성에 맞춰 실행됩니다.

### 콘솔에서 변경

#### 단일 변경

1. [CVM 콘솔](https://console.cloud.tencent.com/cvm/index)에 로그인한 뒤 CVM 목록으로 이동합니다.
2. 아래 이미지와 같이, 변경하려는 인스턴스의 오른쪽 작업란에서 [More>[Resource Adjustment]>[Adjust Configuration]을 클릭합니다.
![](https://main.qcloudimg.com/raw/15e142fe43b498ada4ee3f12c1bf9413.png)
3. 아래 이미지와 같이, '타깃 구성 선택'에서 인스턴스 상태 및 작업을 확인한 뒤, **필요한 모델 및 인스턴스 사양을 선택하고 사양 및 성능 매개변수를 자세히 확인합니다.** 확인이 끝나면 [다음]를 클릭합니다.
![](https://main.qcloudimg.com/raw/cab2d0c635bfc1dd07d3aa6c763b7c21.png)
4. 인스턴스의 과금 방식에 따라 요금을 확인한 뒤 [다음]를 클릭합니다.
	- 종량제 인스턴스: 새로운 사양 적용 시 동결되는 금액을 확인합니다. 구성 변경 후, 종량제 과금이 첫 단계부터 청구되므로 관련 규칙을 자세히 확인한 뒤 작업을 진행하시기 바랍니다.
	![](https://main.qcloudimg.com/raw/378fe411ba222bbdc473b69919d957c6.png)
5. 인스턴스 실행 상태에 따라 '종료 알림' 안내 문구가 다르므로, 주의해서 읽으시기 바랍니다.
 - 인스턴스가 실행 중일 경우, 아래 이미지와 같이 안내 문구를 확인한 뒤 '강제 종료에 동의'를 선택합니다.
![](https://main.qcloudimg.com/raw/ee1406aa0ad4a28a3f190b93f18ed6fe.png)
 - 인스턴스가 종료 상태일 경우, 아래 이미지와 같이 현재 상태를 다시 한 번 확인하는 안내 문구가 뜹니다.
![](https://main.qcloudimg.com/raw/2b7696edf214708e21bf0099c680a063.png)
6. [Adjust Now]을 클릭하여 주문 페이지로 이동한 뒤, 요금을 지불합니다. 

### API를 통한 변경 

ResetInstancesType 인터페이스를 사용하여 인스턴스 구성을 변경할 수 있습니다. 자세한 내용은 [인스턴스 구성 변경](https://intl.cloud.tencent.com/document/product/213/33239)의 API 문서를 참조 바랍니다.
