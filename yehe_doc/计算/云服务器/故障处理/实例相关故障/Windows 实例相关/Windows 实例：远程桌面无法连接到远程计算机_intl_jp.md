## ユースケース
 Windows リモートで Window インスタンスに接続する時に以下のプロンプトが表示されます：
![](https://main.qcloudimg.com/raw/8c79cadc3e14c9c4e0cbb5303b79f74a.png)

リモートデスクトップは以下のいずれかの理由によりリモートコンピューターに接続できません：
1）サーバーへのリモートアクセスが有効になってない
2）リモートコンピューターの電源がオフになっている
3）ネットワーク上でリモートコンピューターが利用できない

確実にリモートコンピューターの電源がオンにし、ネットワークに接続してリモートアクセスが有効であることを確保します。


## 考えられる原因

以上のプロンプトが表示される理由は以下の原因が含まれます（以下の状況に限らず、実際の状況に応じて分析してください）：
- インスタンスが異常な実行状態になっている
- パブリックIPがなし或はパブリックネットワーク帯域幅が0になっている
- インスタンスにバインドされたセキュリティグループはリモートログインポートを開放していない（デフォルトは3389）
- リモートデスクトップサービスが起動してない
- リモートデスクトップの設定問題
- Windows ファイアウォール設定の問題

## トラブルシューティング手順

### インスタンスが実行状態になっているかを確認する
1. [CVMコンソール](https://console.cloud.tencent.com/cvm/index)にログインします。
2. 「CVM」ページのインスタンスリストで、インスタンスが【実行中】であるかを確認します。下図に示すように：
![CVMリストページ](https://main.qcloudimg.com/raw/03cf75228bc468d2e436f876f229ebc9.png)
 - はい、 [サーバーはパブリックIPに設定してるかを確認](#step01)してください。
 - いいえ、 Windows インスタンスを起動してください。

<span id="step01"></span>
### サーバーはパブリックIPに設定してるかを確認する
CVMコンソールでサーバーはパブリックIPに設定してるかを確認します。下図に示すように：
![パブリックIPなし](https://main.qcloudimg.com/raw/58c75d68372069652ec09ab93cfdbdc0.png)
 - はい、[パブリックネットワークの帯域幅を購入するかを確認](#step02)してください。
 - いいえ、[Elastic IPを申請してバインド](https://intl.cloud.tencent.com/document/product/213/16586)してください。
 
<span id="step02"></span>
###  パブリックネットワーク帯域幅を購入するかを確認する
パブリックネットワーク帯域幅は0Mbになっているかを確認します（最低1Mbps）。
 - はい、 [ネットワークの調整](https://intl.cloud.tencent.com/document/product/213/15517)で 帯域幅を1Mbps或は1Mbps以上に調整してください。
![](https://main.qcloudimg.com/raw/29b771d9de5d1ecdadb872c0378a31c7.png)
 - いいえ、[インスタンスのリモートログインポート（3389）を開放しているかを確認](#step03)してください。

<span id="step03"></span>
### インスタンスのリモートログインポート（3389）が開放してるかを確認する
1. コンソールでログイン対象のインスタンスをクリックして、インスタンス情報ページに入ります。
2. 「セキュリティグループ」タブで、インスタンスのセキュリティグループがリモートログインインターフェースを開放しているかを確認します（デフォルトのリモートデスクトップポート：3389）。下図に示すように：
![セキュリティグループ](https://main.qcloudimg.com/raw/28b5f0a038dd354346745bd97f724350.png)
 - はい、 [ Windows インスタンスのシステム設定を確認](#step04)してください。
 - いいえ、対応するセキュリティグループルールを編集して、開放します。操作方法については[セキュリティグループ操作ガイド](https://intl.cloud.tencent.com/document/product/213/18197)をご参照ください。

<span id="step04"></span>
###  Windows インスタンスのシステム設定を確認する
1.  VNC を利用してインスタンスにログインして、 Windows インスタンスのシステム設定をトラブルシューティングします。
>? VNC ログインの詳細については、 [ Windows インスタンスにログイン](https://intl.cloud.tencent.com/document/product/213/5435) の「VNCでのログイン」をご参照ください。
2. ログインしたインスタンスシステムで、【スタート】をクリックして、【実行】で **services.msc**を入力し、 **Enter**を押して、「サービス」ウィンドウを開きます。
3.  「Remote Desktop Services」 のプロパティをダブルクリックして、リモートデスクトップサービスが起動されているかを確認します。下図に示すように：
![Remote Desktop Service](https://main.qcloudimg.com/raw/6c781636c69eacae76a08b88f9e32b99.png)
 -はい、 [ステップ4](#step04_4)を実行してください。
 - いいえ、「スタートアップの種類」を「自動」に設定し、「サービスの状態」を「開始」に設定してください（つまり【開始】をクリックして、サービスを起動する）。
4. <span id="step04_4">【スタート】をクリックして、【実行】で **sysdm.cpl**を入力し、 **Enter**を押して、「システムのプロパティ」ウィンドウを開きます。</span>
5. 「リモート」タブで、リモートデスクトップが「このコンピューターへのリモート接続を許可する(L)」に設定されているかを確認します。下図に示すように：
![リモートの設定](https://main.qcloudimg.com/raw/cbf2b2797bd48777008753f389574674.png)
 - はい、 [ステップ6](#step04_6)を実行してください。
 - いいえ、リモートデスクトップを「このコンピューターへのリモート接続を許可する(L)」に設定してください。
6. <span id="step04_6">【スタート】をクリックし、【コントロールパネル】を選択して、コントロールパネルを開きます。</span>
7. 「コントロールパネル」で、【システムとセキュリティ】>【Windows ファイアウォール】を選択して、 「Windows ファイアウォール」を開きます。
8. 「Windows ファイアウォール」で、 Windows ファイアウォールの状態を確認します。下図に示すように：
![Windows ファイアウォールの状態](https://main.qcloudimg.com/raw/e74937594a03c141e6e5ac753f025d91.png)
 - 「オン」状態になっている場合は、 [ステップ9](#step04_9)を実行してください。
 - 「オフ」状態になっている場合は [チケットを発行](https://console.cloud.tencent.com/workorder/category?level1_id=6&level2_id=7&source=0&data_title=%E4%BA%91%E6%9C%8D%E5%8A%A1%E5%99%A8CVM&level3_id=142&radio_title=%E4%BA%91%E6%9C%8D%E5%8A%A1%E5%99%A8%E7%99%BB%E5%BD%95%E4%B8%8D%E4%B8%8A&queue=15&scene_code=12686&step=2) してフィードバックしてください。
9. <span id="step04_9">は 「Windows ファイアウォール」で、【Windows ファイアウォールを介したアプリまたは機能を許可する】をクリックして、「許可されたアプリ」ウィンドウを開きます。</span>
10. 「許可されたアプリ」ウィンドウで、「許可されたアプリケーションと機能(A)」が「リモートデスクトップ」でチェックされているかを確認します。下図に示すように：
![リモートデスクトップをチェックする](https://main.qcloudimg.com/raw/dfed8792ce1dd8b32bee4aaa02ef6bbf.png)
 - はい、 [ステップ11](#step04_11)を実行してください。
 - いいえ、「リモートデスクトップ」をチェックし、「リモートデスクトップ」を開放してください。
11. <span id="step04_11"> 「Windows ファイアウォール」で、【 Windows ファイアウォールの有効化または無効化】をクリックして、「設定のカスタマイズ」ウィンドウを開きます。。</span>
12. 「設定のカスタマイズ」ウィンドウで、「プライベートネットワークの設定」と「パブリックネットワークの設定」を「Windows ファイアウォールを無効にする(推奨されません)」に設定します。下図に示すように：
![閉じる](https://main.qcloudimg.com/raw/5d5f8c4d7b783bc1018e03368ae400bc.png)

以上の操作を実行した後でもリモートデスクトップで Windows インスタンスに接続できない場合は、[チケットを発行](https://console.cloud.tencent.com/workorder/category?level1_id=6&level2_id=7&source=0&data_title=%E4%BA%91%E6%9C%8D%E5%8A%A1%E5%99%A8CVM&level3_id=142&radio_title=%E4%BA%91%E6%9C%8D%E5%8A%A1%E5%99%A8%E7%99%BB%E5%BD%95%E4%B8%8D%E4%B8%8A&queue=15&scene_code=12686&step=2) してフィードバックしてください。
