본 문서는 보안 그룹 생성을 예로, 인스턴스를 사용자 정의로 구성할 때 Tencent Cloud가 제공하는 보안 그룹 규칙으로 보안 그룹을 최초 설정하는 방법에 대해 안내합니다. 보안 그룹과 관련된 더 다양한 작업은 CVM 콘솔의 **보안 그룹** 페이지에서 진행할 수 있습니다. 자세한 내용은 [보안 그룹 개요](https://intl.cloud.tencent.com/document/product/213/12452)를 참조 바랍니다.


## 보안 그룹 설정
1. 아래 이미지와 같이 보안 그룹을 설정할 때 실제 수요에 따라 [**보안 그룹 생성**]을 선택합니다.
> 사용 중인 보안 그룹이 있다면, [**기존 보안 그룹**]을 선택합니다.
>
![](https://main.qcloudimg.com/raw/c08ca9a0262f4911fdac90925762e4a6.png)
2. 실제 수요에 따라 개방할 IP/포트를 선택합니다.
보안 그룹 생성 시 다음과 같은 규칙이 있습니다. <ul>
<li><b>ICMP</b>: ICMP 프로토콜을 개방하여, 공용 네트워크에서 서버 Ping을 허용.</li>
<li><b>TCP:80</b>: 80포트를 개방하여, HTTP를 통해 웹서비스에 액세스 허용.</li></li>
<li><b>TCP:22</b>: 22포트를 개방하여, SSH를 통해 Linux CVM에 원격 연결 허용.</li>
<li><b>TCP:443</b>: 443포트를 개방하여, HTTPS를 통해 웹서비스에 액세스 허용.</li>
<li><b>TCP:3389</b>: 3389포트를 개방하여, RDP를 통해 Windows CVM에 원격 연결 허용.</li>
<li><b>내부 네트워크 개방</b>：내부 네트워크를 개방하여, 서로 다른 클라우드 리소스 간의 내부 네트워크 통신 허용（IPv4）.</li></ul>
<blockquote class="d-mod-explain">

<i class="d-icon-explain"></i>설명:
</div>
<ul><li>개방하려는 IP/포트를 선택하면 [보안 그룹 규칙]에 자세한 인바운드/아웃바운드 규칙이 표시됩니다. </li><li>비즈니스에 기타 포트를 개방해야 한다면, <a href="https://intl.cloud.tencent.com/document/product/213/32369">보안 그룹 응용 사례</a>를 참조하여 <a href="https://intl.cloud.tencent.com/document/product/213/34271">보안 그룹을 생성</a>할 수 있습니다. 3. 페이지 안내에 따라 기타 정보를 설정합니다.

## 보안 그룹 규칙

**인바운드 규칙**: 보안 그룹에 연결된 CVM으로 오는 인바운드 트래픽을 허용함
**아웃바운드 규칙**: CVM에서 나가는 아웃바운드 트래픽

- 보안 그룹 내 규칙 우선 순위: **위쪽에 있을수록 우선 순위가 높습니다.**
- 하나의 CVM에 규칙이 없는 보안 그룹이 바인딩된 경우, 모든 인바운드, 아웃바운드 트래픽을 거부하도록 기본 설정되어 있습니다(규칙이 있다면 우선 순위에 따라 적용됩니다).
- 하나의 CVM에 여러 개의 보안 그룹이 바인딩된 경우, 보안 그룹의 **숫자가 작을수록, 우선 순위가 높습니다.**
- 하나의 CVM에 여러 개의 보안 그룹이 바인딩된 경우, 우선 순위가 가장 낮은 보안 그룹의 기본 거부 규칙이 적용됩니다.
