### 과금 매칭 규칙

과금형 예약 인스턴스(Reserved Instance, RI)를 구매하면, 예약 인스턴스가 유효기한 동안 종량제 인스턴스와 자동으로 매칭되며, RI는 현재 Linux 시스템의 종량제 인스턴스에만 매칭될 수 있습니다. 나의 계정에 매칭 가능한 종량제 인스턴스가 없다면, RI는 유휴 상태가 되고 계속해서 요금이 발생하게 됩니다. 이때, 조건에 맞는 종량제 인스턴스를 구매한 후에 즉시 자동 매칭하도록 설정하고, 매칭에 성공하면 종량제 인스턴스 청구서에서 요금 차감이 시작됩니다 

- RI와 종량제 인스턴스의 매칭 상태를 수동으로 관리할 수 없습니다.
- RI 청구 혜택은 1시간 동안 최대 3,600초(한 시간)까지의 인스턴스 사용료에만 적용됩니다. 사용자는 동시에 여러 인스턴스를 실행할 수 있지만, RI 차감 서비스 혜택은 시간당 총 3,600초만 누릴 수 있으며, 인스턴스 사용 시간이 3,600초를 초과한 부분에 대해선 종량제 과금 모드를 기준으로 요금을 부과합니다. 

예를 들어, 실리콘밸리 1존에서 S3.16xlarge256 RI 하나를 구매하고, 계정에서 동일 가용존 및 동일 속성의 S3.16xlarge256 종량제 인스턴스 3대를 1시간 동안 실행했다면, 하나의 종량제 인스턴스에만 1시간의 RI 사용 요금이 부과되고, 나머지 2개의 인스턴스는 종량제를 적용해 3시간 동안의 사용 요금이 부과됩니다. 

하지만, 만약에 실리콘밸리 1존에서 S3.16xlarge256 RI 하나를 구매하고, 같은 시간에 동일 가용존 및 동일 속성의 종량제 인스턴스(A, B, C, D)를 각각 20분 동안 실행했다면, 인스턴스의 실행 시간이 총 1시간이 되어 결과적으로 RI 사용 시간은 1시간, 종량제 사용 시간은 0시간이 됩니다. 따라서, 아래 이미지와 같이 3대의 인스턴스에서 각각 20분씩 RI와 매칭됩니다.

매칭 조건에 부합한 3개의 종량제 인스턴스를 동시에 실행하고 있다면, RI 청구 혜택은 1시간 동안(최대 3,600초) 모든 인스턴스에 동시 적용되며, 해당 시간이 지나면 계속해서 종량제 과금 모드로 요금이 부과됩니다.

#### 적용 시간

구매한 예약 인스턴스의 적용 시간은 정각을 기준으로 합니다. 예약 인스턴스는 생성 시간을 기준으로 이전 정각부터 적용되며, 다음 정각부터 365일 이후의 정각까지가 1년 주기입니다(정각에 생성한 경우 제외, 정각에 생성하면 생성한 시각에 정상 적용되어 라이프사이클이 계산됩니다)

예: 사용자가 13:25에 예약 인스턴스 RI 패키지를 구매 및 활성화했다면 13:00부터 14:00까지 RI 할인 가격으로 계산하고, 유효기한은 14:00 + 365*24입니다.

예: 사용자가 13:00에 예약 인스턴스를 구매했다면 13:00부터 RI 패키지 혜택을 받을 수 있으며, 유효기한은 13:00 + 365*24입니다.
