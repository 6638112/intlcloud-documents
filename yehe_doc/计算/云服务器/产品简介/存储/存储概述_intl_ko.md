## 블록 스토리지 디바이스 매핑

기본적인 데이터 실행을 보장하기 위해 인스턴스마다 하나의 시스템 디스크가 적용되며, 인스턴스에 더 많은 데이터 디스크를 마운트할 수 있습니다. 인스턴스는 블록 스토리지 디바이스 매핑(device-mapping)을 사용하여 해당 스토리지 디바이스를 스스로 인식 가능한 위치에 매핑합니다.
블록 스토리지는 파티션을 바이트 단위로 분할하는 스토리지 디바이스로, 랜덤 액세스를 지원합니다. Tencent Cloud는 로컬 디스크와 클라우드 디스크 등 두 가지 유형의 블록 스토리지 디바이스를 지원합니다.
![](https://main.qcloudimg.com/raw/3815bb250f6178d67b8fe2be11a50bf8.svg)
해당 이미지는 CBS가 블록 스토리지 디바이스를 CVM에 매핑하는 방법을 나타냅니다. `/dev/vda` 를 시스템 디스크에 매핑하고, 두 데이터 디스크를 각각 `/dev/vdb`와 `/dev/vdc`에 매핑합니다.
CVM 인스턴스는 마운트된 로컬 디스크와 클라우드 디스크에 블록 스토리지 디바이스 매핑을 자동으로 생성합니다.
