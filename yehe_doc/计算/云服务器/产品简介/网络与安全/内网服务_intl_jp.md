プライベートネットワークサービスすなわちローカルエリアネットワーク（LAN）サービスであり、クラウドサービスがその内部リンクを介して、相互的にアクセスできます。Tencent Cloudのクラウド製品は、[Internet アクセス](doc/product/213/5224)、あるいはTencent Cloudプライベートネットワークを経由して相互的にアクセスすることができます。あらゆるTencent Cloudデータセンターは万兆/千兆の基盤で相互接続され、高帯域幅と低遅延（レイテンシー）のプライベートネットワーク通信サービスを提供しています。また同一リージョンのプライベートネットワーク通信は全て無料であり、柔軟的にネットワークアーキテクチャを構築できます。

## プライベートIPアドレス
### 概要
プライベートIPアドレスはInternetアクセスできないIPであり、Tencent Cloudのプライベートネットワークサービスの実現形式です。各インスタンスは、プライベートIPをアサインするデフォルトのネットワークインターフェース（eth0）があります。プライベートIPアドレスはTencent Cloudによる自動的なアサインか、ユーザーによるカスタマイズ設定か（[Virtual Private Cloud](/doc/product/215/4927)環境のみ)で得られます。
>**注意事項：**
> OSの内部でプライベートIPを自分で変更すると、プライベートネットワーク通信が中断される可能性があります。

### 属性
 - プライベートネットワークサービスはユーザー属性があり、異なるユーザーは互いに隔離されています。つまり、デフォルトではプライベートネットワークを経由して他のユーザーのクラウドサービスにアクセスすることはできません。
 - プライベートネットワークサービスはリージョン属性があり、異なるリージョン間で隔離されています。つまり、デフォルトではプライベートネットワークを経由して同じアカウントの異なるリージョンのクラウドサービスにアクセスすることはできません。

### ユースケース
プライベートIPが、CLBとCVMインスタンス間、CVMインスタンスとその他のクラウドサービス間（CDN、CDBなど）相互アクセスに利用できます。

### アドレスのアサイン
各CVMインスタンスは、起動する時にデフォルトのプライベートIPアドレスがアサインされます。異なる[ネットワーク環境](/doc/product/213/5227)によってプライベートIPも異なります：
 - 基幹ネットワーク：プライベートIPアドレスはTencent Cloudに自動的にアサインされて、変更不可です。
 - Virtual Private Cloud：初期のプライベートIPは、Tencent CloudによってVPCネットワークセグメントの任意の一つのアドレスから自動的に割り当てされます。ユーザーは`10.[0 - 255].0.0/8`、`172.[0 - 31].0.0/16` と `192.168.0.0/16`といった３つのネットワークセグメントからCVMインスタンスのプライベートIPアドレスをカスタマイズでき、具体的な設定値の範囲がそのインスタンスのVPCによって決定されます。詳しくは[VPCとサブネット](/doc/product/215/4927)をご参照ください。

### プライベートネットワークDNSを設定する
ネットワーク解析にエラーが発生する場合に、ユーザー側からプライベートネットワークDNSを手動で設定することができます。設定方法は下記の通りです：

- **Linuxユーザーに対し**。まずは、CVMにおいて、`/etc/resolv.conf`ファイルを編集することでCVMのDNSを変更します。
コマンド`vi /etc/resolv.conf`を実行し、上記の表に各リージョンに対応したDNS IPの修正を行います。
![](//mc.qcloudimg.com/static/img/9c46100760f1049454b076a3c83c7f8a/image.png)

- **Windowsユーザーに対し**。まず、CVMに【コントロールパネル】-【ネットワークと共有センター】-【アダプターの設定の変更】の順に開き、イーサネットの【プロパティ】を右クリックし、さらに【インターネットプロトコルバージョン4】をダブルクリックして、DNSサーバーのIPを変更します。
![](//mc.qcloudimg.com/static/img/93b7bda1075530ff6e7ba5ece4ab71f4/image.png)

##インスタンスのプライベートIPアドレスを取得する

### コンソールで取得する

 1. [CVMコンソール](https://console.cloud.tencent.com/cvm/)にログインします。

 2. あなたのインスタンスがCVMリストに表示され、マウスをCVMのプライベートIPに移動すると、コピーボタンが表示されます。クリックしてプライベートIPをコピーできます。

![](//mc.qcloudimg.com/static/img/2663aabcbe44c2ad372b5b8ba2bb6a1f/image.png)

###　APIで取得する
&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;&nbsp;[DescribeInstancesインターフェース](/doc/product/213/9388)をご参照ください。

### インスタンスのメタデータで取得する

 1. CVMインスタンスにログインします。具体的なログイン方法は[Linuxインスタンスのログイン](/doc/product/213/5436)と[Windowsインスタンスのログイン](/doc/product/213/5435) をご参照ください。

 2.コマンドを入力します：
```
curl http://metadata.tencentyun.com/meta-data/local-ipv4
```
戻り値は次のように表示されると、プライベートIPアドレスを照会できます。
![](//mc.qcloudimg.com/static/img/14a13eccebc7eee6f83bc026adb30902/image.png)
より詳しい情報は[インスタンスメタデータ](/doc/product/213/4934)をご参照ください。
