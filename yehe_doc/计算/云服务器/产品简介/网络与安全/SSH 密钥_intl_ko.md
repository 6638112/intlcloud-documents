인스턴스의 안전성을 보장하기 위해 Tencent Cloud는 [비밀번호 로그인](/doc/product/213/6093) 및 SSH 키 쌍 로그인 두 가지 암호화 로그인 방식을 제공합니다. 본 문서는 SSH 키 쌍 로그인 설정에 관해 설명합니다.
[Linux CVM 사용자 정의로 설정](/doc/product/213/10517#.E8.AE.BE.E7.BD.AE.E4.BF.A1.E6.81.AF)할 때, CVM 암호화 로그인 방식으로 SSH 키를 선택할 수 있습니다.

## SSH 키 개요
Tencent Cloud는 퍼블릭 키 암호 기술을 사용해 Linux 인스턴스의 로그인 정보를 암호화 및 복호화할 수 있습니다. 퍼블릭 키 암호 기술은 퍼블릭 키를 사용해 임의의 데이터에 대해 암호화하면(일종의 비밀번호) 수신자는 프라이빗 키로 데이터를 복호화할 수 있습니다. 퍼블릭 및 프라이빗 키를 키 쌍이라고 하며, 사용자는 키 쌍을 통해 CVM에 안전하게 연결할 수 있습니다. 이는 일반 비밀번호로 CVM에 로그인하는 것보다 더 안전한 방법입니다.

Tencent Cloud는 퍼블릭 키만 보관하므로 프라이빗 키는 사용자 본인이 보관해야 합니다. 사용자의 프라이빗 키가 있다면 누구든 로그인 정보를 복호화할 수 있으니, 프라이빗 키의 보관에 각별히 주의해야 합니다.


## 기능 및 장점
기존의 사용자 이름 및 비밀번호를 이용한 인증 방식과 비교해 SSH 키를 사용하면 다음과 같은 장점이 있습니다.
- SSH 키 로그인 인증 시 더 안전하고 신뢰할 수 있으며, 무차별 대입 공격을 막을 수 있습니다.
- SSH 키 로그인 방식이 더 간편합니다. 콘솔과 로컬 클라이언트에서 간단한 설정을 통해 인스턴스에 원격 로그인할 수 있으며, 다시 로그인할 때 비밀번호를 재입력할 필요가 없습니다.

## 사용 제한
- Linux 인스턴스만 지원합니다.
- Tencent Cloud는 프라이빗 키 정보를 저장하지 않으므로, SSH 키를 생성한 후 10분 이내에 "다운로드"를 클릭해 프라이빗 키를 획득하여 안전하게 보관하시기 바랍니다.
- 데이터 보안을 위해 키 로딩은 종료 상태에서 실행해야 합니다.
