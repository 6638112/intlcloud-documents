## 1.Histórico

Este módulo se aplica se você usa o recurso de Tencent Cloud Conference (“**Recurso**”). Este módulo é incorporado ao contrato de segurança e privacidade de dados localizado no “[**DPSA**](https://intl.cloud.tencent.com/document/product/301/17347)”. Os termos utilizados, mas não definidos neste Módulo, devem ter o significado dado a eles no DPSA. No caso de qualquer conflito entre o DPSA e este Módulo, este Módulo deve ser aplicado conforme a inconsistência.


## 2.PROCESSAMENTO

Processaremos os seguintes dados em conexão com o Recurso (na medida em que você nos instruiu a integrar a funcionalidade relevante necessária para o processamento desses dados como parte do Recurso):


| **Informações pessoais**                                     | **Uso**                                                      |
| ------------------------------------------------------------ | ------------------------------------------------------------ |
| **Dados de configuração** Gerenciamento de conteúdo: Informações da conferência, informações da programação, Informações do palestrante (imagem, nome e introdução), informações da exibição, informações do produto, informações sobre oportunidades de negócios (nome, e-mail, número de contato, nome do produto, memorando) Gerenciamento de acesso: Gerenciamento de modelos de funções/direitos de acesso (público, expositor): configuração da página, Tema, Questionário de Modelo: Título, programa de Mini perguntas/configuração de página da Web: definições de appID, appSecret, PC e H5 | Isso permite que você projete e configure e gerencie o recurso. |
| **Dados de gerenciamento de usuários** Campos de dados especificados por você dentro do console de gerenciamento (que podem incluir número de telefone, nome, nome da empresa, endereço de e-mail, endereço, país/região, interesses, idade, setor, ocupação e gênero) | Isso permite que você gerencie usuários do Recurso.          |
| **Dados do painel** Visualização de página, classificação de maior número de visitas, dados de visitantes (distribuição/distribuição regional, interesses, setor, sexo, idade, ocupação), volume de vídeo ao vivo, duração do relógio de vídeo ao vivo ou quaisquer outros campos de dados especificados por você (incluindo quaisquer conjuntos de dados designados por você nos Dados de Gerenciamento de Usuários acima) | Isso facilita uma visão geral da interação dos usuários e informações estatísticas relacionadas à conferência. |
| **Dados OCR** Dados do Cartão de Visita (conforme especificado nos cartões de visita digitalizados) | Isso permite que você carregue dados do usuário para o Recurso. |
| **Dados de reunião** Estatística: nome da empresa, nome do administrador, nome da empresa indicada, nome do administrador indicado, horário de reunião agendada, status da reunião de Gerenciamento de Informações Corporativas: usuário, função, nome padrão da empresa, nome do representante, observação e operações | Isso permite que você gerencie o agendamento de reuniões para o Recurso. |
| **Dados de push** Números de telefone dos usuários, endereços de e-mail | Isso permite que você envie mensagens para os usuários dentro do Recurso. |
| **Dados do evento interativo** Configurações básicas, configurações de prêmios, definições do participante (Nome, Número de Telefone e Gênero são sincronizados com os dados de gerenciamento do usuário acima ou coletados diretamente por você) | Isso permite gerenciar eventos interativos dentro do Recurso. |
| **Dados de gerenciamento de tíquetes** Tipo de tíquete, preço, restrições totais de volume de tíquetes, requisitos de aprovação, configurações padrão de tíquetes, configurações de exibição, informações de ordem de tíquete, incluem número do pedido, hora do pedido, nome, número de telefone, tipo de tíquete, preço único, volume, valor total | Isso permite que você gerencie os tíquetes dentro do Recurso. |
| **Dados LVB** Gerenciamento LVB: Tipo LVB, nome LVB e introdução, hora da transmissão, nome do fluxo, pushStreamURL, reprodução ou não, configuração de comentários, configurações de moderação (se aplicável) Revisão de comentários: Nome LVB, hora do problema, nome de usuário, comentário e operação (passar/excluir) | Isso permite que você gerencie transmissões ao vivo dentro do Recurso. |
| **Dados de gerenciamento de Expositores** Dados de configuração de função, informações de sub-conta (nome da conta do expositor, senha, proprietário da conta, número de telefone, função e modelo de exposição) | Isso permite que você gerencie expositor contas dentro do Recurso. |

## 3.região do serviço

Conforme especificado no DPSA.



## 4.SUB-Processadores

Como especificado no DPSA, inclusive Tencent Cloud Computing (Pequim) Co., Ltd., First App Holdings Limited, e Aceville Pte Ltd.

 

## 5.retenção de dados

Os dados são retidos de acordo com as instruções do cliente (e em qualquer caso excluído dentro de 30 dias após o término do serviço), exceto os Dados do Painel que são processados para sua visualização apenas em tempo real.

Você pode solicitar a exclusão desses dados pessoais de acordo com o DPSA.


## 6.condições especiais

Você deve garantir que este Recurso seja usado apenas por usuários finais que têm pelo menos a idade mínima para que um indivíduo possa consentir com o processamento de seus dados pessoais ou com o consentimento dos pais obtido para pessoas com idade inferior à mínima exigida. Aqui pode haver diferença dependendo da jurisdição em que o usuário final está localizado.
