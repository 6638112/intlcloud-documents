## 課金方式
インスタントメッセージングIMでは現在、以下の課金方式が採用されています。
- 月末締め後払い方式：月次決済で、前月に発生した料金を毎月1～3日にアカウント残高から差し引きます。


##  基本サービスの詳細
インスタントメッセージングIMパッケージには、体験版、Professional版、Ultimate版があります。アプリケーション作成後はデフォルトで体験版（無料）になっています。実際のサービス要件に応じてさまざまなパッケージを選択できます。
パッケージの機能比較の詳細を下表に示します。

| 機能特徴         | 体験版      | Professional版         | Ultimate版             |
| ----------------- |---------- |----------------- | ------------------- |
| グローバルマルチポイントカバレッジ     | サポート        | サポート           | サポート                    |
| ユーザー数上限      | 100人       | 無制限                | 無制限                                 |
| 1ユーザーあたりの友達数上限  | 20人   | 3,000人          | 3,000人            |
| 1ユーザーあたりの参加可能グループ数  | 50グループ/人   | 500グループ/人                | 1,000グループ/人            |
| 1グループあたりのメンバー数上限（非公開グループ、公開グループ、チャットルーム） | 20人/グループ  | 200人/グループ  | 非公開グループ：200人/グループ<br />公開グループ、チャットルーム：2,000人/グループ      |
| グループ数上限(解散後カウントなし)      | 100個       | 無制限                 | 無制限                                |
| 作成可能な音声・ビデオチャットルーム数    | 10個        | 50個          | 無制限             |
| 1日あたり純増グループ数 | 10,000個  | 10,000個 | 10,000個    |
| メッセージ履歴の無料保存期間    | 7日間         | 7日間             | 30日間          |
| 無料日のアクティブユーザー数（DAU）の利用可能枠  | 100/月         | 1万/月       | 1万/月           |
| 無料で利用可能なピークグループ数| 10万/月 | 10万/月  |10万/月   |

- **音声・ビデオチャットルームの作成可能数**：1つのSDKAppIDに属する全ユーザーが作成できるチャットルームの合計数を指します。音声・ビデオチャットルームで使用できるのは、テキスト、画像、短い音声、その他のカスタムメッセージを送信する機能のみです。[Live Video Broadcasting](https://intl.cloud.tencent.com/document/product/267)、[Tencent Real-Time Communication](https://intl.cloud.tencent.com/document/product/647)などのサービスを使用する場合は、これらのサービスを有効化する必要があります。
- **1日あたり純増グループ数**：1日あたりに1つのSDKAppIDに属する全ユーザーによって作成されたすべてのタイプのグループ数純増合計を指します。1日あたり純増グループ数は最大で10,000グループがサポートされ、そのうちオンラインメンバーブロードキャストグループを最大で5グループまで含めることができます。作成されたグループ数が1日あたり純増グループ数のしきい値に達してから、さらにグループを作成したい場合は、新しいグループを作成する前に、当日に作成し不要になったグループを解消して削除できます。詳細については、[グループシステム](https://intl.cloud.tencent.com/document/product/1047/33529)をご覧ください。
- **DAU**：同じ日にインスタントメッセージングIMにログインした1人のユーザーを1DAUとしてカウントします。同一ユーザーが繰り返してログインした場合は、1DAUとして累積されません。
- **ピークグループ数**：1つのアプリケーション（SDKAppID）に属する全ユーザーが作成して参加したグループ数の合計。月間のピーク数に基づいて決済されます。
