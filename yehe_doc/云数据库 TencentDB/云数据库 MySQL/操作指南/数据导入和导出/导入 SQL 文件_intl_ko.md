## 작업 시나리오
TencentDB for MySQL은 콘솔을 통한 SQL 파일 가져오기를 지원합니다. 본 기능은 선택한 데이터베이스에서 SQL 명령을 실행할 수 있습니다. 해당 기능을 이용해 데이터베이스/테이블을 배치 및 생성하고, 테이블 아키텍처를 변경해 인스턴스의 초기화 또는 변경을 완료할 수 있습니다.
>? TencentDB for MySQL은 고가용성 버전과 파이낸스 버전 인스턴스만 SQL 파일 가져오기 기능을 지원합니다.

## 작업 순서
1. [MySQL 콘솔](https://console.cloud.tencent.com/cdb)에 로그인한 뒤, 인스턴스 리스트에서 인스턴스 명칭 혹은 '작업'열의 [관리]를 클릭하여 인스턴스 관리 페이지에 접속합니다.
2. [데이터베이스 관리]>[데이터베이스 리스트]에서 [데이터 가져오기]를 클릭합니다.
![](https://main.qcloudimg.com/raw/a8854e74caebb9c69d831dc1583c10c0.png)
3. 팝업 페이지에서 [파일 새로 추가]를 클릭해서 가져올 파일을 클릭한 후, [다음 단계]를 클릭합니다.
>?
>- 시스템 테이블 손상으로 데이터베이스를 사용할 수 없는 상황이 발생하는 것을 방지하기 위해서는 시스템 테이블의 데이터를 가져와서는 안됩니다. 예: mysql.user 테이블 
>- 증량 가져오기 데이터만 지원합니다. 예를 들어 데이터베이스 내 폐기된 데이터가 있는 경우, 데이터 비운 후 가져오기를 실행하십시오.
>- 단일 파일은 2GB를 초과할 수 없으며 파일 이름에는 영어, 숫자, 언더바만 사용할 수 있습니다.
>- 업로드한 파일의 유효기간은 14일이며, 만료 후 자동으로 삭제됩니다.
>
<img src="https://main.qcloudimg.com/raw/fd6c70707a8722ca64a87f71978e2a2a.png">
4. 타깃 데이터베이스 선택 페이지에서 타깃 데이터베이스를 선택하고 [다음 단계]를 클릭하십시오.
5. 확인 페이지에서 가져올 데이터를 확인한 이후에 계정 비밀번호를 입력하고 [가져오기]를 클릭하십시오.
>!
>- 가져오기 조작은 롤백할 수 없습니다. 가져오기 정보를 잘 확인하십시오.
>- 계정 비밀번호를 잊어버렸을 경우 [비밀번호 재설정](https://intl.cloud.tencent.com/document/product/236/31901)을 통해 비밀번호를 변경할 수 있습니다.
