
## 작업 시나리오
본 문서는 TencentDB for MySQL 엔진 버전 업그레이드 시 콘솔에서 버전을 업그레이드하는 작업에 대해 소개합니다.
TencentDB for MySQL은 다음 버전의 데이터베이스 엔진 업그레이드를 지원합니다.
- MySQL 5.5에서 MySQL 5.6으로
- MySQL 5.6에서 MySQL 5.7로

>?
>- 데이터베이스 엔진 버전 업그레이드를 지원하지 않습니다.
>- 메이저 버전 간 업그레이드를 지원하지 않습니다. 예를 들어 MySQL 5.5 인스턴스를 MySQL 5.7 혹은 상위 버전으로 업그레이드 할 경우, 먼저 MySQL 5.6 으로 업그레이드해야 합니다.
>- MySQL 5.7에서 MySQL 8.0로 업그레이드를 지원하지 않습니다.

## 버전 업그레이드 규칙
- `create table …  as select …`구문을 지원하지 않습니다.
- TencentDB for MySQL 5.6, 5.7 마스터/슬레이브 동기화는 GTID에 기반해 실행됩니다. 기본적으로 InnoDB 엔진만 지원합니다.
- MySQL 5.5 버전을 MySQL 5.6 버전으로 업그레이드 시 MyISAM 엔진의 테이블을 InnoDB로 변경합니다. **업그레이드 전에 MyISAM을 InnoDB로 변경해주시기 바랍니다.
- 업그레이드 기간에 TencentDB for MySQL이 slow\_log 테이블을 삭제합니다. 로그 메시지를 보관하려면 메인 버전으로 업그레이드하기 전에 로그 내용을 저장해주시기 바랍니다.
- **업그레이드하려는 버전의 인스턴스에 기타 인스턴스(마스터 인스턴스, 읽기 전용 인스턴스 등)를 연결할 경우, 복제된 데이터의 일치성을 위해 버전 업그레이드가 함께 진행됩니다**.
- TencentDB for MySQL 버전 업그레이드 시 데이터를 이전해야 할 수 있습니다. 오랜 시간이 소모되므로 잠시만 기다려주시기 바랍니다. 사용자의 비즈니스에 영향을 미치지 않으며, 액세스할 수 있습니다.
- 버전 업그레이드를 완료하면 인스턴스를 변경(MySQL 데이터베이스 연결이 몇 초간 끊김)해야 할 수 있으니, 프로그램에 자동 재연결 기능을 적용하고, 인스턴스 점검 시간을 지정하여 변경하시기를 권장합니다. 자세한 내용은 [인스턴스 점검 시간 설정](https://intl.cloud.tencent.com/document/product/236/10929)을 참조하십시오.
>- 단일 인스턴스의 테이블 수량이 100만을 초과하면 업그레이드에 실패할 수 있습니다. 데이터베이스 모니터링에도 영향을 미칠 수 있으므로 테이블 수량을 합리적인 수준으로 적용해 주시고, 단일 인스턴스 테이블 수량은 100만 이상을 넘지 않도록 제어하십시오.

## 작업 순서
1. [MySQL 콘솔](https://console.cloud.tencent.com/cdb/)에 로그인한 뒤 인스턴스 리스트에서 변경할 인스턴스를 선택한 다음 '작업' 열에서 [더보기]>[업그레이드 버전]을 클릭합니다.
>?MySQL 8.0 상위 버전으로 업그레이드 할 수 없습니다.
>
![](https://main.qcloudimg.com/raw/ec14ff513f80db6e87338b5a531f6126.png)
2. 팝업 대화 상자에서 데이터베이스 버전을 선택한 뒤, [업그레이드]를 클릭합니다.
데이터베이스 버전 업그레이드는 데이터 이전과 연관되므로 업그레이드 완료 시 몇 초간 끊김이 발생할 수 있습니다. 하지만 업그레이드 실행 시 전환 시간을 [During maintenance window]로 선택하면 인스턴스 업그레이드 완료 후 다음 [During maintenance window]에 전환이 실행됩니다.
>!전환 시간을 [During maintenance window]로 선택하면 데이터베이스 사양 업그레이드 완료 시 즉시 전환되지 않고, 인스턴스의 [During maintenance window]에 전환이 실행되며, 그전까지 동기화가 유지됩니다. 이 때문에 전체 인스턴스 업그레이드 시간이 길어질 수 있습니다.
>

## 핵심 질문
#### 업그레이드 전에 TencentDB for MySQL이 자동으로 데이터를 백업하나요?
CDB 매일 실시간 이중 시스템 핫 백업은 데이터 백업 + 로그 백업(binlog)을 기반으로 7일~732일간 손실 없이 복구할 수 있습니다.

#### CDB에서 MySQL 5.7을 MySQL 5.6으로 다운그레이드할 수 있나요?
다운그레이드는 지원하지 않습니다. MySQL 5.6 버전이 필요할 경우 MySQL 5.7 인스턴스를 폐기/반품한 뒤, MySQL 5.6 인스턴스를 다시 구매하시기 바랍니다. 불편을 끼쳐드려 죄송합니다.

#### 업그레이드 기간에 마스터/슬레이브가 지연될 수 있나요?
마스터 인스턴스 업그레이드 시 데이터를 대조하기 때문에 마스터/슬레이브가 지연될 수 있습니다.

#### 버전 업그레이드를 완료하면 인스턴스를 변경해야 할 수 있는데, TencentDB for MySQL 인스턴스를 정상적으로 사용할 수 있습니까?
인스턴스의 정상적인 업그레이드 과정은 사용자 이용에 영향을 주지 않습니다. 하지만 인스턴스 연결이 몇 초간 끊길 수 있으니, 프로그램에 자동 재연결 기능을 적용하고, 인스턴스를 선택해 점검 시간 내에 변경하시기를 권장합니다.

### TencentDB for MySQL 버전 업그레이드 시 시간이 얼마나 소요되나요? 어디에서 업그레이드 진행도를 조회할 수 있습니까?
데이터베이스의 업그레이드 시간과 사용자의 현재 데이터베이스 데이터양의 크기 및 데이터 복사 등과 관계가 있어 구체적인 시간은 알 수 없습니다.
TencentDB for MySQL 버전 업그레이드 시 데이터를 이전해야 할 수 있습니다. 오랜 시간이 소모되므로, 잠시만 기다려주시기 바랍니다. 사용자의 비즈니스에 영향을 미치지 않으며, 액세스할 수 있습니다.

#### 인스턴스가 왜 계속 '업그레이드 전환 대기' 상태입니까?
사용자가 전환 시간을 '점검 시간 내 전환'으로 선택했을 수 있습니다. 인스턴스 업그레이드 완료 후 점검 시간 내에 전환 작업이 진행됩니다.
즉시 전환하려면 인스턴스 리스트의 '작업' 열에서 [즉시 전환]을 클릭하십시오. 연결이 몇 초간 끊길 수 있으니 비즈니스의 재연결 메커니즘을 확보하시기 바랍니다.
