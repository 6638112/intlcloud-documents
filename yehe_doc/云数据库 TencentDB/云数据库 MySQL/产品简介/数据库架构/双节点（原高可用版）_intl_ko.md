TencentDB for MySQL은 단일 노드, 이중 노드, 3중 노드의 세 가지 구성을 지원합니다. 본 문서에서는 이중 노드 구성에 대해 소개합니다.

- 이중 노드는 1 마스터 1 슬레이브의 고가용성 모드를 사용하며, 실시간 핫 백업으로 컴퓨터 다운 자동 점검과 장애 자동 마이그레이션을 지원합니다.
- 이중 노드 구성은 서로 다른 격리 정책에 따라 범용형과 전용형으로 나뉩니다. 자세한 내용은 [격리 정책](https://cloud.tencent.com/document/product/236/53253)을 참조하십시오.

## 적용 시나리오
게임, 인터넷, 사물인터넷, 전자상거래 판매, 물류, 보험, 증권 분야 등의 애플리케이션을 커버합니다.

## 구성 특징
- 마스터/슬레이브 복제는 비동기화(기본)와 반동기화 방식이 있으며, 복제 방식은 [콘솔](https://console.cloud.tencent.com/cdb)의 인스턴스 상세 페이지에서 변경할 수 있습니다.
- 읽기 전용 인스턴스, 재해 복구 인스턴스, 보안 그룹, 데이터 마이그레이션, 다중 가용존 배포 등을 포함한 다양한 특성을 지원합니다. 자세한 내용은 [제품 장점](https://intl.cloud.tencent.com/document/product/236/5148)을 참조하십시오.
- 이중 노드 인스턴스의 가용성은 99.95%에 달합니다. 구체적인 프로토콜은 [서비스 수준 계약(SLA)](https://intl.cloud.tencent.com/document/product/301/30977)을 참조하십시오.
- 강력한 하드웨어에 배포된 데이터 노드와 로컬 NVMe SSD 디스크를 사용하는 기본 스토리지로 뛰어난 IO 성능을 제공하여 IOPS가 최대 240000에 달합니다. 실제 IOPS 속도는 설정, 페이지 크기, 비즈니스 부하에 따라 달라지며, 해당 값은 MySQL의 기본 페이징 크기인 16KB를 기준으로 테스트한 것으로써 참조용으로만 제공됩니다.

## 업그레이드 관련 작업
- TencentDB for MySQL은 데이터베이스 엔진 업그레이드를 지원합니다. 자세한 내용은 [데이터베이스 엔진 버전 업그레이드](https://intl.cloud.tencent.com/document/product/236/8126)를 참조하십시오.
