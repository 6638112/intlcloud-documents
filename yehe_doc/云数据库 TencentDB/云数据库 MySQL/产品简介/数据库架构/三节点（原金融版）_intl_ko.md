TencentDB for MySQL은 단일 노드, 이중 노드, 3중 노드의 세 가지 아키텍처를 지원합니다. 본 문서는 3중 노드 아키텍처에 대해 설명합니다.

- 3중 노드는 1 마스터 2 슬레이브 3중 노드 모드를 사용합니다. 강제 동기화 복사 방식을 지원하며, 실시간 핫 백업을 통한 데이터의 일관성 유지로 파이낸스급 신뢰성과 고가용성을 제공합니다.
- 3중 노드 아키텍처는 서로 다른 격리 정책에 따라 범용형과 전용형으로 나뉩니다. 자세한 내용은 [격리 정책](https://intl.cloud.tencent.com/document/product/236/39794)을 참조하십시오.

## 적용 시나리오
게임, 인터넷, 사물인터넷, 전자상거래 판매, 물류, 보험, 증권 등 분야의 응용 시나리오

## 아키텍처 특징
- 마스터/슬레이브 복사 방식: 비동기(기본), 강제 동기화, 반동기화
- 읽기 전용 인스턴스, 보안 그룹, 데이터 마이그레이션, 다중 가용존 배포 등 다양한 특성을 지원합니다. 자세한 내용은 [제품 장점](https://intl.cloud.tencent.com/document/product/236/5148)을 참조하십시오.
- 3중 노드 인스턴스의 가용성은 99.99%에 달합니다. 자세한 프로토콜은 [서비스 수준 계약(SLA)](https://intl.cloud.tencent.com/document/product/236/30978)을 참조하십시오.
- 3중 노드 인스턴스는 멀티 사본 방식을 제공해 데이터 저장의 지속성을 보장합니다. 마스터 인스턴스에서 생성되는 데이터는 다른 슬레이브 인스턴스로 동기화되어 데이터를 보다 안전하게 저장할 수 있으며, 데이터 저장의 지속성이 99.99999% 이상 보장됩니다.
- 강력한 하드웨어에 배포된 데이터 노드와 로컬 NVMe SSD 디스크를 사용하는 기본 스토리지로 뛰어난 IO 성능을 제공합니다. IOPS는 최대 240000에 달하며, 실제 IOPS 속도는 설정, 페이지 크기, 비즈니스 부하에 따라 달라집니다. 해당 값은 MySQL의 기본 페이징 사이즈인 16KB를 기준으로 테스트한 것이며 참조용으로만 제공됩니다.
- 3중 노드 아키텍처는 2개의 슬레이브 인스턴스를 동일 가용존에 배포하거나(예: 베이징 5존의 경우 시스템 기본 분산 정책으로 인스턴스가 동일 디바이스에 배포되지 않도록 보장), 서로 다른 가용존에 배포하도록 선택할 수 있습니다(예: 한 슬레이브 인스턴스는 베이징 5존에, 다른 슬레이브 인스턴스는 베이징 7존에 배포).
