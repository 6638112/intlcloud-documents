## 개요
백업 공간(용량)은 한 리전에 위치한 전체 TencentDB for MySQL 인스턴스의 백업 파일을 저장합니다. 백업 파일은 자동 데이터 백업, 매뉴얼 데이터 백업 및 로그 백업을 포함합니다.

TencentDB for MySQL은 리전에 따라 일정 금액의 무료 백업 용량을 제공합니다. 무료 백업 용량의 크기는 해당 리전에 있는 모든 고가용성 및 파이낸스 버전의 인스턴스(마스터 인스턴스 및 재해 복구 인스턴스 포함) 스토리지 용량의 합계입니다.
>?
>- 읽기 전용 인스턴스 RO 구매 시에는 백업 용량을 증정하지 않습니다. 마스터 인스턴스와 장애 복구 인스턴스를 구매 시에만 증정합니다.
>- 백업 용량은 [MySQL 콘솔](https://console.cloud.tencent.com/cdb/backup)의 데이터베이스 백업 페이지에서 확인할 수 있습니다.

## 백업 가격
백업 용량이 무료 할당액을 초과할 경우, 중국대륙 지역은 0.000127달러/GB/시간을 기준으로 요금을 부과합니다. 기타 리전의 가격은 [TencentDB for MySQL 가격 계산기](https://buy.cloud.tencent.com/price/cdb/calculator)를 참조하십시오.
요금 부과 공간이 1GB 미만일 경우 차감하는 요금이 없습니다. 1시간 미만은 1시간으로 집계합니다. 유연한 TencentDB for MySQL 증정 정책을 적용하므로 대부분의 인스턴스는 백업 용량에 별도로 요금을 지불하지 않습니다.

## 계산 공식

**무료 백업 용량(단일 리전)=해당 지역에서 구매한 MySQL 고가용성 버전과 파이낸스 버전 인스턴스의 스토리지 공간 합계**

**요금 사항(단일 리전)=데이터 백업량(해당 리전)+로그 백업량(해당 리전)-무료 백업 용량(해당 리전)**

>?휴지통의 TencentDB for MySQL 인스턴스 백업은 여전히 백업 용량으로 간주되며, 전체 용량 크기에 포함됩니다.

**계산 예시**
예를 들면 광저우3존에 작동 중인 MySQL 고가용 인스턴스(구매한 데이터베이스 스토리지 공간은 월 500GB)와 광저우4존에서 작동 중인 MySQL 고가용 인스턴스(구매한 데이터베이스 스토리지 공간은 월 200GB)를 소유할 경우, 광저우 리전은 월별 700GB 공간을 무료로 사용할 수 있습니다.

광저우 리전의 총 백업 용량이 700GB를 초과할 경우, 초과부분에 대해서 별도로 과금됩니다. 예를 들어, 데이터 백업이 800GB에 이르고 로그 백업이 100GB인 경우, 과금 용량은 800 + 100 - 700 = 200GB로, 초과한 200GB에 대한 별도 요금이 발생합니다. 초과분의 과금은 이런 방식으로 계속 이어집니다.

## 백업 라이프사이클

<span id = "anliang_zhouqi"></span>
### 종량제 인스턴스
- 백업은 인스턴스의 라이프사이클 기반으로 변화합니다.
- 인스턴스 만료 24시간 미만 시, 정상적으로 백업합니다.
- 인스턴스 만료 24시간 이후, 인스턴스는 휴지통으로 이동하고, 자동 백업이 중지됩니다. 롤백과 수동 백업은 불가능하지만 백업 다운로드는 가능합니다([백업 리스트](https://console.cloud.tencent.com/cdb/backup) 페이지 다운로드). 인스턴스 백업의 무료 증정 용량 초과분은 과금이 중지되며, 콘솔 휴지통에서 연장할 시 인스턴스와 백업을 복구할 수 있습니다.
- 휴지통에 3일 이상 보관된 인스턴스는 비활성화되어 폐기 처리됩니다. 관련 데이터 역시 함께 폐기되니 중요한 백업은 미리 저장하십시오.

## 계정 연체 설명

### 종량제 인스턴스
계정이 연체되면 백업은 사용자의 인스턴스 라이프사이클 기준으로 변경됩니다. 앞서 소개한 종량제 백업 라이프사이클 내용을 참조하십시오.

## 백업 지출 절감 제안
- 사용하지 않는 수동 백업 데이터를 삭제하십시오. 수동 백업은 [MySQL 콘솔](https://console.cloud.tencent.com/cdb)의 인스턴스 관리 페이지>Backup and Restore 화면에서 삭제 가능합니다. 자동 백업은 만료 후 자동 삭제되며, 콘솔에서 임의 삭제가 불가능합니다. 
- 비핵심 비즈니스 데이터의 자동 백업 빈도수를 낮추십시오. 콘솔에서 백업 주기 및 백업 보관 기간을 조정할 수 있으며, 1주에 최소 2회 백업합니다.
>?[롤백 기능](https://intl.cloud.tencent.com/document/product/236/7276)은 백업 주기와 백업 보관일 내의 백업 파일 + 로그 백업(binlog)을 기반으로 자동 백업의 빈도수와 보관일을 단축하여 인스턴스 데이터의 롤백 기능에 영향을 주므로, 백업 설정의 밸런스를 적절히 조절하십시오.
>
- 비핵심 서비스의 데이터 백업과 로그 백업 보관 기간을 줄이십시오. 7일간 백업을 유지할 경우 대부분의 시나리오 요구사항을 지원합니다.

| 비핵심, 비데이터류 서비스 | 7일 권장                                                      |
| 분류 서비스             | 데이터 백업 보관 기간 7일 설정 권장. 실제 비즈니스 요구사항에 맞춰 데이터 수동 백업 및 사용 완료 시 삭제 |
| 테스트 서비스             | 데이터 백업 보관 기간 7일 설정 권장. 실제 비즈니스 요구사항에 맞춰 데이터 수동 백업 및 사용 완료 시 삭제 |
