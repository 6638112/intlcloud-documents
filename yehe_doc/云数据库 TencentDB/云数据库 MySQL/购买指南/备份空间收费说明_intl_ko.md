## 개요
백업 용량은 한 리전 안에 있는 모든 TencentDB for MySQL 인스턴스의 백업 파일을 저장하는 데 사용합니다. 백업 파일은 자동 데이터 백업, 수동 데이터 백업 및 로그 백업을 포함합니다.

TencentDB for MySQL은 리전에 따라 일정 금액의 무료 백업 용량을 제공합니다. 무료 백업 용량의 크기는 상응하는 리전에 있는 모든 고가용성 및 파이낸스 버전의 인스턴스(마스터 인스턴스 및 재해 복구 인스턴스 포함) 스토리지 용량의 합계입니다.
>?
>- 백업 용량은 읽기 전용 인스턴스 RO를 구매할 때는 증정하지 않으며, 마스터 인스턴스와 재해 복구 인스턴스를 구매할 때만 증정합니다.
>- 백업 용량 사이즈에 대한 정보는 [MySQL 콘솔](https://console.cloud.tencent.com/cdb/backup)의 데이터베이스 백업 페이지에서 확인할 수 있습니다.

## 백업 가격
백업 용량이 무료 할당액을 초과할 경우, 중국대륙 지역은 0.000127달러/GB/시간을 기준으로 요금을 부과합니다. 기타 리전의 가격은 [TencentDB for MySQL 가격 계산기](https://buy.cloud.tencent.com/price/cdb/calculator)를 참조 바랍니다.
유료 용량이 1GB 미만일 경우 요금 차감이 발생하지 않으며, 1시간 미만은 1시간으로 집계합니다. TencentDB for MySQL은 효율적인 증정 정책을 사용하므로, 대다수의 인스턴스는 백업 용량 요금을 부과하지 않습니다.

## 계산 공식

**무료 백업 용량(단일 리전)= 해당 리전에서 구매한 MySQL 고가용성 버전 및 파이낸스 버전 인스턴스의 스토리지 용량의 합계**

**유료 부분(단일 리전)= 데이터 백업양(해당 리전)+ 로그 백업양(해당 리전)- 무료 백업 용량(해당 리전)**

>?휴지통 내의 TencentDB for MySQL 인스턴스 백업 역시 백업 용량으로 집계되며, 전체 용량 사이즈 계산 시 포함됩니다.

**계산 예시**
예를 들어, 광저우 3존에서 실행 중인 MySQL 고가용성 버전 인스턴스(구매한 데이터베이스 스토리지 용량은 월 500GB)와 광저우 4존에서 실행 중인 MySQL 고가용성 버전 인스턴스(구매한 데이터베이스 스토리지 용량은 월 200GB)를 각각 하나씩 보유하고 있다면, 광저우 리전에서는 매월 700GB의 무료 백업 용량이 제공됩니다.

광저우 리전의 전체 백업 스토리지 용량이 700GB를 초과할 경우, 예를 들어 데이터 백업이 800GB이고, 로그 백업이 100GB에 달했다면 용량 요금은 800 + 100 - 700 = 200GB입니다. 사용자는 별도로 이 200GB에 해당하는 백업 용량 요금을 내야 하며, 후속적으로 초과분의 백업 용량도 이처럼 계산합니다.

## 백업 라이프사이클

<span id = "anliang_zhouqi"></span>
### 종량제 인스턴스
- 백업은 인스턴스의 라이프사이클에 따라 변동됩니다
- 인스턴스 만료 24시간 이내에는 정상적으로 백업할 수 있습니다
- 인스턴스 만료 24시간 후부터는 인스턴스가 격리되어 휴지통으로 이동됩니다. 이때에는 자동 백업이 중지되고 롤백, 수동 백업이 차단되지만, 백업 다운로드([백업 리스트](https://console.cloud.tencent.com/cdb/backup) 페이지에서 다운로드)는 허용되며, 해당 인스턴스의 무료 용량을 초과한 부분에 대한 과금도 중지됩니다. 사용자는 콘솔 휴지통에서 구독 연장을 통해 인스턴스 및 백업을 복구할 수 있습니다.
- 인스턴스가 휴지통에 격리된 지 3일이 지나면 공식적으로 운영 정지됩니다. 이때 인스턴스가 완전히 폐기되며, 관련된 데이터 백업도 함께 폐기되니, 필요에 따라 제때 백업을 저장하시기 바랍니다.

## 계정 연체 설명

계정이 연체되면 백업은 사용자의 인스턴스 라이프사이클 기준으로 변동됩니다. 위 문서에서 종량제 백업 라이프사이클 소개를 참조 바랍니다.

## 백업 지출 절감을 위한 제안
- 사용하지 않는 수동 백업 데이터(수동 백업은 [MySQL 콘솔](https://console.cloud.tencent.com/cdb)의 인스턴스 관리 페이지>백업 및 복구 페이지에서 삭제할 수 있습니다. 자동 백업이 만료된 후에는 자동으로 삭제되며, 콘솔에서 수동으로 삭제할 수 없습니다)를 삭제합니다. 
- 핵심 서비스 데이터의 자동 백업 빈도(콘솔에서 백업 주기 및 백업 보관시간을 변경할 수 있으며, 일주일에 최소 2회 백업을 권장합니다)를 하향 조정합니다.
>?[롤백 기능](https://intl.cloud.tencent.com/document/product/236/7276)은 백업 주기와 백업 보관 일자 내의 데이터 백업 + 로그 백업(binlog)을 기반으로, 자동 백업의 빈도수와 보관일을 단축하여 인스턴스 데이터의 롤백 시간 및 범위에 영향을 주므로, 백업 설정의 밸런스를 적절히 조절하시기 바랍니다.
>
- 핵심 서비스의 데이터 백업과 로그 백업 저장시간을 단축합니다(백업 보관시간 7일이면 대부분의 시나리오에 충분합니다).
