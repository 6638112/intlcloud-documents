? CDN은 301/302 구성 기능을 따르는 소스를 제공합니다. 노드가 301/302 상태 코드를 리턴하기 위해 소스 요청할 때, CDN 노드는 사용자에게 301/302를 리디렉션하지 않고 점프한 주소에서 소스를 직접 요청합니다.

## 구성 안내
1. [CDN 콘솔](https://console.cloud.tencent.com/cdn)에 로그인하여 왼쪽 디렉터리 [도메인 관리]를 클릭합니다. 관리 페이지로 이동하여 목록에서 편집해야 하는 도메인 이름을 찾아 작업표시줄에서 [관리]를 클릭하십시오.
2. [원본 가져오기 구성]을 클릭하면 **"301/302 구성 따르기"** 모듈이 표시되며, 기본적으로 비활성화 상태입니다.

## 구성 사례
- 도메인 이름은 'www.test.com'이며, 다음과 같이 301/302 구성을 따릅니다.
사용자A가 `http://www.test.com/1.jpg`구성을 요청한 경우: 노드가 캐시 되지 않은 경우, 노드는 오리진 서버에서 필요한 리소스를 가져오도록 요청합니다. 오리진 서버가 리디렉션한 HTTP Resopnse 상태 코드가 301/302이고, 리디렉션된 도메인이 `http://www.abc.com/1.jpg `인 경우
1. 노드는 HTTP Resopnse를 사용자에게 직접 리턴합니다.
  2. 사용자는 `http : // www.abc.com / 1.jpg`에 대한 요청을 시작하며, 도메인 이름이 CDN에 연결되어 있지 않으면 가속 효과가 없습니다.
  3. 사용자B가 'http://www.abc.com/1.jpg'에 요청을 시작하면 위의 과정이 반복됩니다.
- 도메인 이름은 'www.test.com'이며, 다음과 같이 301/302 구성을 따릅니다.
사용자A가 'http://www.test.com/1.jpg'구성을 요청한 경우: 노드가 캐시 되지 않으면 노드는 오리진 서버에서 얻은 리소스를 요청합니다. 오리진 서버가 리디렉션한 HTTP Resopnse 상태 코드가 302이고, 리디렉션 주소가 'http://www.abc.com/1.jpg'인 경우
1. 소스 다음에 301/302 설정이 활성화된 후, 상태 코드가 301/302 인 HTTP Response를 수신한 후 노드는 리디렉션한 주소에 직접 요청을 시작합니다.
  2. 필요한 리소스를 얻은 후에는 노드에 캐시 되어 사용자에게 리턴됩니다.
  3. 이때, 사용자 B도 'http://www.abc.com/1.jpg'에 대한 요청을 시작하는데, 이 요청은 노드에 직접 도달하여 사용자에게 리턴됩니다.
  4. 301/302 설정으로 다시 활성화된 후, 최대 3번의 리디렉션만 수행되며, 한계를 초과하면 301/302를 직접 고객에게 리턴합니다.
